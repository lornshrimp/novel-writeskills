# 降 AI 味审校清单

系统化地降低 AI 检测率至 30% 以下的完整操作清单。

---

## 使用方法

### 三遍审校顺序
1. **第一遍**: 内容审校（逻辑、事实、结构）
2. **第二遍**: 风格审校（AI 味降重）⭐ 核心
3. **第三遍**: 细节打磨（标点、排版、节奏）

### 每一遍都要
- 逐句检查
- 标记问题
- 说明修改理由
- 验证修改效果

---

## 第一遍：内容审校

### 事实准确性

**检查项**:
- [ ] 所有数据都有来源（真实的，不是编造的）
- [ ] 时间正确（年份、月份、顺序）
- [ ] 产品名称正确（大小写、版本号）
- [ ] 人名和公司名正确
- [ ] 技术术语使用准确
- [ ] 链接有效可访问

**常见错误**:
- ❌ 编造数据：「使用量提升了 85.7%」（没有来源）
- ❌ 时间错误：说 2024 年的事件发生在 2023 年
- ❌ 名称错误：「Claude AI」→「Claude」
- ❌ 术语错误：混淆概念

**修改标准**:
- 所有数据必须有真实来源
- 不确定的事实必须删除或标注
- 宁可少写，不要编造

---

### 逻辑清晰性

**检查项**:
- [ ] 前后无矛盾（前面说 A，后面不能说非 A）
- [ ] 因果关系合理（因为 X 所以 Y 说得通）
- [ ] 论据支持论点（例子能证明观点）
- [ ] 结构完整（有引入、展开、总结）

**常见问题**:
- ❌ 自相矛盾：前面说「很简单」，后面说「需要深厚功底」
- ❌ 因果不当：「因为用了 AI，所以成功了」（太简化）
- ❌ 论据不足：只有一个例子就下普遍性结论

**修改标准**:
- 逻辑链条完整
- 论据充分
- 避免绝对化表述

---

### 结构合理性

**检查项**:
- [ ] 小标题概括段落内容
- [ ] 各部分长度均衡（不要某部分特别长）
- [ ] 没有跑题（所有内容围绕主题）
- [ ] 重要信息前置（不要把核心内容放在最后）

**常见问题**:
- ❌ 小标题和内容不符
- ❌ 某部分占了文章一半篇幅
- ❌ 开头铺垫太长，核心内容太少

**修改标准**:
- 结构清晰，一目了然
- 重点突出
- 详略得当

---

## 第二遍：风格审校 ⭐ 核心

这是降 AI 味的关键一遍！

### 1. 删除 AI 套话

**必删清单**:

```
❌ 在当今时代
❌ 随着科技的发展
❌ 在...的背景下
❌ 值得注意的是
❌ 需要指出的是
❌ 综上所述
❌ 总而言之
❌ 让我们来看看
❌ 首先让我们
❌ 接下来让我们
❌ 在实际应用中
❌ 从某种意义上说
❌ 毋庸置疑
❌ 不言而喻
❌ 这里有个细节很关键
❌ 这是个本质区别
❌ 更好的策略是
```

**检查方法**:
1. 全文搜索这些词
2. 全部删除或替换
3. 不要心软！这些词 100% 是 AI 味

**替换对照**:
| 删除 | 替换为 |
|------|--------|
| 值得注意的是 | 注意 / 重点来了 / [直接说内容] |
| 综上所述 | 总结一下 / 说白了 / 简单来说 |
| 让我们来看看 | 我们来看 / 看看 / 说说 |
| 在实际应用中 | 实际用下来 / 我用的时候 |
| 这是个本质区别 | 这两个完全不是一回事 |
| 更好的策略是 | 换个活法 / 这么干 |
| 解决方法 | 怎么办？ |

---

### 2. 拆解 AI 句式

**过度对比**:
```
❌ 不是 A 而是 B，不是 C 而是 D，不是 E 而是 F
✅ 不是 A，而是 B（只用一次即可）
```

**过度并列**:
```
❌ 既...又...不仅...而且...同时...并且...
✅ 拆成多个短句，每句一个要点
```

**过度排比**:
```
❌ 从 A 到 B，从 B 到 C，从 C 到 D，从 D 到 E
✅ 直接列举：A、B、C、D、E
```

**工整排比**:
```
❌ 要么...要么...要么...
✅ 独立短句 + 问号
例如：「商业化产品？贵死了。学术模型？普通人根本用不了。」
```

**前者后者**:
```
❌ 前者是 XX，后者是 YY
✅ 「第一句话是 XX。第二句话，我一听就知道 YY。」
```

**这些不是...是**:
```
❌ 这些不是 AI 能决定的，是你得想清楚的
✅ 「这些 AI 帮不了你。得你自己想明白。」
```

**检查方法**:
1. 找出长句（超过 30 字）
2. 看是否有重复句式
3. 拆成多个短句

---

### 3. 替换书面词汇

**常见书面词汇对照表**:

| 书面词汇 | 口语化替换 |
|---------|-----------|
| 显著提升 | 提升了 XX% / 涨了很多 / 效果很好 |
| 大幅增长 | 涨了 X 倍 / 增长了 XX% |
| 充分利用 | 用好 / 用上 / 多用 |
| 有效降低 | 降低 / 减少 |
| 深入了解 | 搞清楚 / 弄明白 / 了解 |
| 进行操作 | 操作 / 做 / 执行 |
| 实施方案 | 执行方案 / 按方案做 / 落地 |
| 开展工作 | 开始做 / 动手 / 推进 |
| 积极响应 | 响应 / 行动 / 参与 |
| 持续优化 | 一直在优化 / 不断改进 |
| 切实可行 | 可行 / 靠谱 / 能做 |
| 行之有效 | 有效 / 管用 / 好用 |
| 很难突围 | 进去就是送 |
| 关键是找到 | 往死里挖 |
| 不要贪大 | 别贪大 |
| 信息密度极高 | 有多专业？ |
| 这点我深有体会 | 我自己也踩过这坑 |

**检查方法**:
1. 搜索「显著」「充分」「有效」等词
2. 逐个替换
3. 读一遍，感受是否自然

---

### 4. 改成口语化

**被动语态 → 主动语态**:
```
❌ 这个问题被我们解决了
✅ 我们解决了这个问题
```

**名词化 → 动词化**:
```
❌ 进行了优化
✅ 优化了
```

**长定语 → 拆成短句**:
```
❌ 基于多年产品经验并结合最新 AI 技术研发的这款产品
✅ 这款产品基于多年产品经验。我们结合了最新 AI 技术。
```

**检查方法**:
1. 找出「进行」「实施」「开展」等词
2. 改成直接动词
3. 大声朗读，听起来是否自然

---

### 5. 加入真实细节

**抽象 → 具体**:

```
❌ 效果很显著
✅ 用户量从 1000 涨到 10000，涨了 10 倍

❌ 用户反馈很好
✅ 100 个用户中，95 个说好用

❌ 开发很快
✅ 两个小时就做完了

❌ 功能很强大
✅ 原来要 1 小时的任务，现在 20 分钟搞定
```

**检查方法**:
1. 找出形容词（「很好」「很强」「显著」等）
2. 替换为具体数字或场景
3. 确保数字是真实的，不是编造的

---

### 6. 加入个人态度

**中立 → 有态度**:

```
❌ 这个产品有优点也有缺点
✅ 我觉得这个产品挺好用，但 XX 这点让我很不爽

❌ 这个方法可能有效
✅ 我试过，确实有效

❌ 可以考虑使用
✅ 我推荐用这个

❌ 成本较高
✅ 贵死了

❌ 门槛较高
✅ 门槛高到离谱

❌ 难以使用
✅ 普通人根本用不了

❌ 维护成本增加
✅ 维护起来要命

❌ 可能存在问题
✅ 八成是错的

❌ 效果不佳
✅ 有点惨不忍睹
```

**态度表达词**:
- 「我觉得...」
- 「在我看来...」
- 「说实话...」
- 「老实说...」
- 「我用下来...」
- 「让我很不爽」
- 「挺好用」
- 「很喜欢」

**检查方法**:
1. 找出客观中立的表述
2. 加入个人判断和感受
3. 确保态度明确

---

### 7. 删除过度修饰

**过度形容词**:
```
❌ 非常显著的大幅度提升
✅ 提升很大 / 提升了 10 倍

❌ 极其重要的关键因素
✅ 关键因素 / 重要的是
```

**过度副词**:
```
❌ 充分地、有效地、显著地
✅ 删除副词，保留动词
```

**检查方法**:
1. 搜索「非常」「极其」「十分」
2. 删除或替换为具体描述
3. 少用形容词，多用数据和事实

## 第三遍：细节打磨

### 1. 句子长度

**检查标准**:
- [ ] 主流句子 15-25 字
- [ ] 最长不超过 30 字
- [ ] 超过 30 字的长句全部拆短

**拆句方法**:
```
❌ 这个功能不仅提升了效率，而且降低了成本，同时还改善了用户体验，让用户满意度大幅提升。(36字)

✅ 这个功能提升了效率，降低了成本。用户体验也改善了。满意度涨了很多。(3句，分别14/9/8字)
```

**工具**:
- 复制全文到文档
- 标记超过 30 字的句子
- 逐个拆短

---

### 2. 段落长度

**检查标准**:
- [ ] 手机屏幕 3-5 行
- [ ] 约 100-150 字
- [ ] 长段落拆分

**拆段方法**:
- 一个段落只说一件事
- 换话题就分段
- 重要观点单独成段

**例子**:
```
❌ [200字的长段落，密密麻麻]

✅ [第一段：80字]

[第二段：70字]

[第三段：50字]
```

---

### 3. 标点使用

**句号**:
- 多用句号
- 不要用逗号连接多个独立句子
- 一句话一个完整意思就用句号

**逗号**:
- 只在句内停顿时用
- 不要连接 3 个以上的短句
- 连接 2 个短句就够了

**问号**:
- 适当提问，引起思考
- 不要连续多个问句
- 问完要给答案

**感叹号**:
- 只在真正需要强调时用
- 不要滥用
- 一段最多 1-2 个

**引号规范**:
- 使用中文直角引号「」
- 双重引用使用『』
- 不使用英文引号 "" 或 ''

**检查方法**:
1. 找出逗号连接的长句
2. 改成句号分隔
3. 大声朗读，感受节奏
4. 用 grep 验证引号：`grep -n '"' 文章路径`

---

### 4. 节奏变化

**长短句交替**:
```
短句。短句。长一点的句子，有些补充说明。又是短句。
```

**快慢结合**:
- 重要信息：快节奏，短句
- 解释说明：慢节奏，适当加长
- 高潮部分：短句连发

**检查方法**:
- 大声朗读全文
- 感受节奏是否单调
- 调整长短句比例

---

### 5. 排版美观

**留白**:
- [ ] 段落之间空一行
- [ ] 重要观点前后留白
- [ ] 不要密密麻麻

**小标题**:
- [ ] 层级清晰（## → ###）
- [ ] 概括内容
- [ ] 长度适中（不超过 15 字）

**列表**:
- [ ] 对齐
- [ ] 格式统一（- 或 1.）
- [ ] 每项长度相近

---

## AI 味检测终极清单

完成三遍审校后，最后检查：

### 语言层面
- [ ] 删除了所有 AI 套话（「在当今」「综上」等）
- [ ] 没有过度对比和排比
- [ ] 替换了所有书面词汇（「显著」「充分」等）
- [ ] 改成了口语化表达
- [ ] 句子长度合适（15-25字）

### 内容层面
- [ ] 有真实数据（不是编造的）
- [ ] 有具体细节（不是抽象描述）
- [ ] 有个人经历和观点
- [ ] 态度明确（不是中立客观）

### 风格层面
- [ ] 像和朋友聊天（不像论文）
- [ ] 有个人特色（不是通用文案）
- [ ] 节奏有变化（不单调）
- [ ] 读起来舒服（不拗口）

### 整体检验
- [ ] 大声朗读全文，听起来自然
- [ ] 给朋友看，问是否像 AI 写的
- [ ] 用 AI 检测工具测试（目标 <30%）

## 审校技巧

### 1. 逐句检查
不要跳读，每句话都要过一遍

### 2. 大声朗读
读出来能发现很多问题

### 3. 换位思考
想象你在和朋友说话，会怎么表达

### 4. 对比原文
看看 AI 原始输出，提醒自己哪里容易有 AI 味

### 5. 多次审校
第一遍肯定改不完，多过几遍
