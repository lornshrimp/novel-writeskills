# 写作风格指南

高质量中文内容的写作特点和语言风格参考。

---

## 核心调性

### 总体定位
- **实践 + 落地**：不空谈理论，有真实案例和数据
- **口语化**：像和朋友聊天，不是写论文
- **有态度**：明确的观点，不中立客观地假装全面
- **真实感**：有个人经历，有成功也有失败

### 目标读者
- AI 工具使用者
- 编程学习者（尤其是非科班）
- 产品经理
- 创业者和独立开发者

---

## 语言特征

### 1. 口语化表达

**✅ 推荐用法**:
- 「我用下来发现...」
- 「说实话...」
- 「挺好用」「很不错」
- 「其实...」「但是...」
- 「我觉得...」「在我看来...」

**❌ 避免用法**:
- 「在实际使用中」
- 「值得注意的是」
- 「综上所述」
- 「显著提升」
- 「充分利用」

### 2. 简短有力

**句子长度**:
- 主流：15-25 字
- 最长不超过 30 字
- 多用短句，节奏快

**例子**:
- ✅ 「效果很好。用户量涨了 10 倍。」
- ❌ 「在实际应用中取得了显著成效，用户使用量实现了数量级的增长，达到了原来的十倍。」

### 3. 真实细节

**具体 > 抽象**:
- ✅ 「两个小时做完，拿了 5000 美金」
- ❌ 「快速完成开发，获得了可观的奖金」

**真实经历**:
- ✅ 「在公司做产品经理时...」
- ✅ 「我之前做过一个 Chrome 插件...」
- ❌ 不编造没有的经历

### 4. 个人态度

**明确观点**:
- ✅ 「我觉得这个功能挺好用，但 XX 这点让我很不爽」
- ❌ 「这个功能有优点也有缺点，需要根据具体情况判断」

**不怕暴露问题**:
- ✅ 「获得了流量，但也在一夜之间失去了」
- ✅ 「踩了不少坑」
- ❌ 不假装完美

## 文章结构

### 典型开头方式

**方式1：真实场景引入**
```
两个小时做的网站，拿了 5000 美金特等奖。

听起来很夸张？但这就是 AI 编程的威力。
```

**方式2：个人经历引入**
```
我之前在公司做产品经理的时候，主导了一个 AI 功能的接入。

零代码实现，使用量涨了 10 倍。
```

**方式3：直接提问引入**
```
产品经理能写代码吗？

在 AI 时代，答案是：能。
```

**❌ 避免的开头**:
- 「在当今时代...」
- 「随着科技的发展...」
- 「让我们来看看...」

---

### 中间部分结构

**分层清晰**:
1. 用小标题划分
2. 每部分聚焦一个主题
3. 有具体案例支撑

**常用结构**:
```
## 问题/现状
[描述遇到的问题或当前情况]

## 尝试/方案
[说明尝试的方法或解决方案]

## 结果/效果
[真实数据和效果]

## 思考/总结
[个人思考和经验总结]
```

---

### 结尾方式

**方式1：个人思考**
```
AI 编程工具不是替代程序员，而是让更多人能做出东西。

这很重要。
```

**方式2：行动建议**
```
如果你也想学编程，不要等。

先做起来，边做边学。
```

**方式3：开放式结尾**
```
这是我的经历。你的呢？
```

**❌ 避免的结尾**:
- 「综上所述...」
- 「总而言之...」
- 「让我们一起...」

---

## 风格要素

### 1. 标点使用

**多用句号**:
- ✅ 「效果很好。用户量涨了 10 倍。」
- ❌ 「效果很好，用户量涨了 10 倍，而且用户满意度也提升了。」（逗号连接过长）

**适当用问号**:
- 引起思考：「产品经理能写代码吗？」
- 但不过度：不要连续多个问句

**少用感叹号**:
- 只在真正需要强调时用
- 不要每句都用感叹号

---

### 2. Emoji 使用

**适度使用**:
- 标题中：⭐ 🚀 💡（标记重点）
- 列表项：✅ ❌（标记对错）
- 不过度：不是每句话都要 emoji

**例子**:
```
## 核心要点 ⭐

✅ 这样做
❌ 不要那样做
```

---

### 3. 段落设计

**适合手机阅读**:
- 段落不超过 3-5 行（手机屏幕）
- 约 100-150 字
- 长段落拆分成多个短段

**留白**:
- 段落之间空一行
- 重要观点前后留白
- 不要密密麻麻

---

### 4. 中英文混排规范 ⭐

**核心规则**:
1. 中英文之间、中文与数字之间需要添加空格
2. 引号使用「」，双重引号为『「引用的内容」双重引用』

**✅ 正确示例**:
- 「我用 Cursor 做了一个项目」
- 「AI 功能使用量涨了 10 倍」
- 「这是我的第 3 个 Chrome 插件」
- 「在 GitHub 上获得了 100 个 star」

**❌ 错误示例**:
- 「我用Cursor做了一个项目」（缺少空格）
- 「AI功能使用量涨了10倍」（缺少空格）

**特殊情况：标点符号前后无需空格**

当英文或数字前后有标点符号时，标点符号与中文之间不加空格：

- ✅ 「我用 Cursor、Claude Code 和 Windsurf」
- ✅ 「获得了 5000 美金（约 3.5 万人民币）」
- ✅ 「在 Google 开发者大会上，演示了我的插件」
- ✅ 『这是「AI 友好」的文档』

- ❌ 「我用 Cursor 、 Claude Code 和 Windsurf」（标点前多余空格，引号使用了英文引号）
- ❌ 「获得了 5000 美金,约 3.5 万人民币」（括号前多余空格，使用了英文逗号）

**完整规则**:
1. 中文 + 空格 + 英文/数字
2. 英文/数字 + 空格 + 中文
3. 英文/数字 + 标点符号（无空格）+ 中文
4. 中文 + 标点符号（无空格）+ 英文/数字

**为什么这样做？**
- 提升可读性，视觉上更舒适
- 符合中文排版规范
- 让文字显得更专业

## 典型表达模式

### 开场白
- 「说实话...」
- 「我用下来发现...」
- 「之前在做 XX 的时候...」
- 「有意思的是...」

### 转折
- 「但是...」
- 「问题是...」
- 「重点来了」
- 「更有意思的是...」

### 强调
- 「这很重要」
- 「注意这点」
- 「核心是...」
- 「关键在于...」

### 总结
- 「说白了...」
- 「总结一下...」
- 「简单来说...」
- 「换句话说...」

## AI 味警告

### 常见 AI 套话（必须删除）

| AI 味表达 | 替换为 |
|---------|--------|
| 在当今时代 | [删除] |
| 随着科技的发展 | [删除] |
| 值得注意的是 | 注意 / 重点来了 |
| 综上所述 | 总结一下 / 说白了 |
| 让我们深入了解 | 我们来看看 / 说说 |
| 首先让我们 | 先说 / 首先 |
| 显著提升 | 提升了 XX% / 涨了很多 |
| 充分利用 | 用好 / 用上 |
| 进行操作 | 操作 / 做 |
| 不仅...而且... | 而且（用一个就够） |
| 既...又... | （拆成两句） |

### 常见 AI 句式（必须拆解）

**过度对比**:
- ❌ 「不是 A 而是 B，不是 C 而是 D，不是 E 而是 F...」
- ✅ 最多用一次对比

**过度排比**:
- ❌ 「从 A 到 B，从 B 到 C，从 C 到 D...」
- ✅ 直接列举

**过度并列**:
- ❌ 「既...又...不仅...而且...同时...并且...」
- ✅ 拆成多个短句

## 参考来源标注规范

### 何时需要标注来源 ⭐

**必须标注（外部来源）**:
- 行业报告数据
- 第三方研究结果
- 新闻媒体报道
- 其他人的文章或观点
- 学术论文
- 官方统计数据

**无需标注（个人经验）**:
- 自己的项目经历
- 自己的工作经验
- 自己的观察和思考
- 自己做的产品数据
- 自己的使用体验

**示例对比**:

✅ 需要标注:
```
根据某机构 2025 年报告，AI 驱动的视频编辑工具让专业剪辑师每年节省 200 小时。

**数据来源**: XX 2025 AI Video Editing Trends
```

✅ 无需标注:
```
我在公司做产品经理时，主导了一个功能的接入。使用量涨了 10 倍。

（这是个人经历，无需标注来源）
```

### 标注格式

**文末统一标注**:
```markdown
---

**相关数据来源**:
- 行业报告: XX 2025 AI Report
- 媒体报道: TechCrunch "AI in Creative Industries" (2025-03-15)
- 官方数据: GitHub 2025 开发者报告

**备注**:
- 个人经历和项目数据来自真实工作经验，未编造
```

**文中引用格式**:
- 首次提到：「根据 XX 报告...」
- 后续无需重复标注

### 原则

1. **透明原则**：外部数据必须标注来源，让读者可以验证
2. **真实原则**：个人经历必须真实，但无需证明来源
3. **简洁原则**：不要每句话都标注，文末统一标注即可

## 自检清单

写完文章后，检查：

### 语言风格
- [ ] 删除了所有 AI 套话
- [ ] 改成了口语化表达
- [ ] 句子长度适中（15-25 字）
- [ ] 段落适合手机阅读（3-5 行）

### 排版规范 ⭐
- [ ] 中英文之间添加了空格
- [ ] 中文与数字之间添加了空格
- [ ] 标点符号前后没有多余空格
- [ ] 使用中文标点符号（逗号、句号、引号等）

### 内容质量
- [ ] 有真实案例和数据
- [ ] 有个人经历和观点
- [ ] 具体细节（不抽象）
- [ ] 明确态度（不中立）

### 文章结构
- [ ] 开头吸引人（真实场景/经历）
- [ ] 中间逻辑清晰（小标题分层）
- [ ] 结尾有思考或行动建议

### 参考来源 ⭐
- [ ] 外部数据已标注来源（行业报告、媒体报道等）
- [ ] 个人经历无需标注，但确保真实
- [ ] 文末统一列出数据来源

### AI 味检测
- [ ] 没有「在当今时代」等套话
- [ ] 没有过度对比和排比
- [ ] 没有书面词汇（「显著」「充分」等）
- [ ] 读起来像人说话，不像 AI 生成
