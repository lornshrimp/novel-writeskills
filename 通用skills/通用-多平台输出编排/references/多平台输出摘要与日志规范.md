# 多平台输出摘要与日志规范

## 日志更新时机

至少在以下时点写回同一日志文件：

1. 输入解析与路径推断完成后
2. 原章读取完成后
3. 每个平台推演完成后
4. 每个平台落盘成功后
5. 每次回炉后
6. 平台门禁汇总完成后
7. 文本质量扫描完成后
8. 相似度终检完成后
9. 输出摘要前，将 `status=completed`

## 日志字段结构（硬性最小口径）

日志必须可直接支撑断点恢复，至少应覆盖以下层级：

- 顶层：`sop`、`executionId`、`logKey`、`status`、`startedAt`、`updatedAt`、`inputs`、`derived`、`checkpoint`、`errors[]`
- 单章模式：可直接在顶层承载 `outputs`、`platformValidate`、`textQuality`、`similarityCheck`
- 多章模式：必须支持 `chapters[]`，且每个章节条目能独立承载：
  - `derived.chapterNo / chapterIndex / 分部目录 / 分卷目录`
  - `checkpoint.phase / nextAction`
  - `status`
  - `outputs`
  - `platformValidate`
  - `textQuality`
  - `similarityCheck`

## 平台输出条目最小字段

每个 `章节 × 平台` 输出条目至少应包含：

- `title`
- `outputPath`
- `written`
- `notes`
- `attempt`
- `gates`

## gates 字段口径（必须可恢复）

`gates` 至少应支持以下子对象；未命中的平台子项可为空，但字段口径必须统一：

- `countChapter`: `{ reportPath, meetsAll, len, cjk, minLen, minCJK, maxCJK }`
- `countAfterword`: `{ reportPath, hasAfterword, afterwordCJK, minAfterwordCJK, maxAfterwordCJK }`
- `countAfterwordWords`: `{ reportPath, hasAfterword, wordCount, minWords, maxWords }`
- `noCjkExceptMarker`: `{ reportPath, pass }`
- `padLike`: `{ reportPath, pad_like_sentences_before, pad_like_sentences_after, pad_like_lines_before, pad_like_lines_after, forced_kept_due_to_min, removed_pad_sentences }`
- `textQuality`: `{ reportPath, flagsCount, filesFlagged }`
- `styleConformance`: `{ pass, checks, notes }`

其中：

- 中文平台必须重点记录 `countChapter`、`countAfterword`、`padLike`
- 英文平台必须重点记录 `countChapter`、`countAfterwordWords`、`noCjkExceptMarker`
- 所有平台都应支持 `textQuality` 与 `styleConformance`

## checkpoint 写法建议

- 批次级：`batch_manifest_ready`、`chapter:1.2.4:queued`、`chapter:1.2.4:similarity_check:pass`
- 平台级：`chapter:1.2.4:platform:知乎:written`、`chapter:1.2.4:platform:GoodNovel:gate_failed`
- `nextAction` 必须写清下一步是继续某平台、回炉某平台、补参还是结束摘要

## 多章模式的日志要求

- 多章模式可以额外维护批次总表，但不得用一份混合字段掩盖单章细粒度状态。
- 每个章节条目都要能单独回答四个问题：
  1. 当前停在哪一步；
  2. 哪些平台已经真正通过门禁；
  3. 哪些平台失败或待回炉；
  4. 下一步应做什么。

## 并行写回约束

- 并行平台分支写日志前必须先合并最新日志，不得盲写旧副本。
- 任一分支不得覆盖其他分支已写入的 `attempt`、`gates`、`outputPath`、`errors`、`checkpoint`。

## 输出路径映射（目录根）

- 豆瓣：`豆瓣/`
- 出版社：`出版社/`
- 微信订阅号：`微信订阅号/`
- 百度百家号：`百度百家号/`
- 小红书：`小红书/`

## 对话输出规则

- 只输出摘要，不输出正文
- 摘要必须按章节分组（批量模式）
- 每章都要独立列出写入结果、门禁摘要与分语言相似度结果

## 单章摘要最小结构

1. 已生成并写入的平台路径与标题
2. 约束自检
3. 相似度质检（中文组 / 英文组 / N/A）

### 约束自检最小检查项

- 字数门禁是否通过
- 作者有话说门禁是否通过
- 文本质量扫描是否通过
- 风格一致性是否通过
- 信息不丢失是否成立
- ≥80% 新表述是否成立
- 新标题是否同步且不撞题
- 输出结构是否仅含“正文 + `## 作者有话说`”

## 多章节摘要最小结构

1. 逐章结果块
2. 每章独立门禁与相似度结果
3. 批次总览

### 批次总览至少包含

- 已完成章节数
- 失败 / 阻塞章节数
- 若存在失败：列出 `chapterNo + platform + 原因 + 报告路径`

## 成功标准

- 每个平台都遵守各自平台输出 Skill 的硬约束
- 所有已生成平台文件均通过门禁
- 分语言相似度终检全部通过
- 聊天未泄露正文，只回摘要
- 多章节模式下，顺序与输入一致
