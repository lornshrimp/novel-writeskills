# 多平台输出摘要与日志规范

## 日志更新时机

至少在以下时点写回同一日志文件：

1. 输入解析与路径推断完成后
2. 原章读取完成后
3. 每个平台推演完成后
4. 每个平台落盘成功后
5. 每次回炉后
6. 平台门禁汇总完成后
7. 文本质量扫描完成后
8. 相似度终检完成后
9. 输出摘要前，将 `status=completed`

## 对话输出规则

- 只输出摘要，不输出正文
- 摘要必须按章节分组（批量模式）
- 每章都要独立列出写入结果、门禁摘要与分语言相似度结果

## 单章摘要最小结构

1. 已生成并写入的平台路径与标题
2. 约束自检
3. 相似度质检（中文组 / 英文组 / N/A）

## 多章节摘要最小结构

1. 逐章结果块
2. 每章独立门禁与相似度结果
3. 批次总览

## 成功标准

- 每个平台都遵守各自平台输出 Skill 的硬约束
- 所有已生成平台文件均通过门禁
- 分语言相似度终检全部通过
- 聊天未泄露正文，只回摘要
- 多章节模式下，顺序与输入一致
