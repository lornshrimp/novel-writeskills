# 通用-输出微信订阅号版

这是平台共性本体，负责承接“微信订阅号版输出”的跨题材共性规则。

## 常见触发词 / 用户说法速查

- 输出微信订阅号版
- 改成公众号能发的版本
- 做微信订阅号平台改写
- 帮我把这章改得更顺滑更适合连载传播

## 何时使用

- 把任意题材章节改写成更适合微信订阅号阅读与转发的版本。
- 需要强化手机阅读顺滑度、微钩子、轻互动与作者口吻粘性。
- 需要为题材包装层提供统一的微信订阅号平台共性骨架。

## 不适用情形

- 只做题材边界补充，不做平台输出本体。
- 只做多平台编排、落盘和终检，而不处理微信订阅号平台风格本身。

## 本层职责

- 维护微信订阅号平台的跨题材共性规则。
- 统一承接订阅号版输出的结构约束、标题倾向、作者有话说口吻与风格门禁。
- 为 `题材名-输出微信订阅号版` 提供稳定的通用平台骨架。
- 只处理平台共性，不替代题材包装层决定题材边界、题材禁行项、题材专项补丁与题材 refs 调度。

## 继续读取的 references

- `references/平台共性执行细则.md`
- `references/分节级补救映射与详细规则回填.md`

## 与题材包装层的协作说明

- 若对应题材已有 `题材名-输出微信订阅号版`，必须先由题材包装层锁定：题材边界、题材禁行项、题材专项补丁、题材 refs 调度与题材输出口径。
- 本 Skill 只负责微信订阅号平台的跨题材共性：滑读感、微钩子、轻互动、作者有话说口吻、平台门禁、落盘与终检骨架。
- 若题材包装层与本 Skill 出现冲突：
	- 题材边界、题材禁行项、题材专项补丁 → 以题材包装层为准；
	- 平台共性节奏、滑读结构、平台终检门禁 → 以本 Skill 为准。
- 若用户直接命中本 Skill 且仓库内存在对应题材包装层，不应绕过题材包装层直接把平台共性套到正文上。
- 若仓库内暂时不存在对应题材包装层，本 Skill 只能保守执行平台共性，不得臆造题材规则。

## 历史迁移参考（非默认调度）

- `references/跨题材共性与迁移来源.md`
- `references/平台研究补充_微信订阅号都市悬疑.md`

这些文件仅用于追溯迁移来源、核对历史承接关系或处理遗留映射缺口；默认执行时，不把它们当作当前跨题材平台规则主承载。

## 默认执行顺序

1. 先读取对应题材的 `题材名-输出微信订阅号版` Skill。
2. 由题材包装层继续读取其题材 references，先锁定题材边界、题材禁行项、题材专项补丁与题材输出口径。
3. 再读取本 Skill 的 `references/平台共性执行细则.md` 与 `references/分节级补救映射与详细规则回填.md`，执行订阅号平台的滑读感、微钩子、作者口吻与连载粘性强化。
4. 若需追溯历史迁移来源或核对旧条款，再回看“历史迁移参考”，不得本末倒置。
5. 最后才处理降相似度，不得为了降重破坏连载顺滑度。

## 默认输出口径

- 默认输出一版可直接继续落盘或进入平台终检的订阅号派生正文。
- 默认保留原章核心信息、微钩子与连载顺滑度，不新增关键事实。

## 最低交付

- 有明确的手机滑读感、轻互动与持续关注驱动。
- 保住原章核心事件链、章内微钩子与作者口吻黏性。
- 读起来不像公众号杂谈，也不像泛平台轻改稿。

## 硬规则

- 命中本技能时，必须同时加载对应题材的 `题材名-输出微信订阅号版` Skill。
- 题材特有规则不得回写到本文件中平行维护。
- 修改微信订阅号平台共性规则时，应优先修改本 Skill，而不是多个题材入口。
- 不得把历史迁移参考里的 `都市悬疑` 口径直接当成跨题材默认规则继续扩散。
- 平台派生正文默认落在本 Skill 的工作目录 `微信订阅号/` 下。
- 章节级派生正文目录统一按小说结构决定：若作品有分部，则位于 `微信订阅号/第X部/第Y卷/`；若作品无分部，则位于 `微信订阅号/第X卷/`。
- 平台派生正文文件名默认不带日期，沿用既有章节号与平台标题同步规则；日期只用于配套审阅报告、书评等派生产物。
- 章节派生正文必须以 `.md` 文件形式落盘：若对应章节目录不存在，执行前必须先创建完整目录路径，再将改写后的章节内容写入该目录下的 `.md` 文件，不得只输出聊天稿而不写入文件。
- 落盘路径的完整推断优先级：① 用户显式指定 chapterPath → ② 从源文件名与分部/卷信息自动推断 → ③ 若无法确定分部/卷，暂停并向用户确认，不得乱推断后静默落盘。
- 平台稿落盘后必须显式运行 `scripts/count-chapter.ps1 -MinCJK 3500 -MaxCJK 6500` 核正文，并运行 `scripts/count-afterword.ps1 -MinCJK 200 -MaxCJK 300` 核 `## 作者有话说`。
- 生成阶段必须先达到字数保底目标再允许落盘：正文目标 `4200–5800` CJK（硬下限 `3500`），`## 作者有话说` 目标 `220–280` CJK（硬下限 `200`）；未达保底目标时必须先回炉扩写，禁止“先落盘再补字数”。
- 在无损迁移完成前，不得删除源 `输出微信订阅号版.prompt.md` 中尚未完成映射的内容。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只负责微信订阅号平台的跨题材共性骨架。
- 题材边界、题材禁行项与题材特有口径，继续由对应 `题材名-输出微信订阅号版` 承接。
- 题材 Prompt 只应路由到 `题材名-输出微信订阅号版`；不应直接把本 Skill 与题材 refs 并列成双入口。
