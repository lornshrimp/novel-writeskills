# 通用-输出微信订阅号版｜分节级补救映射与详细规则回填

## 来源与恢复等级

- 恢复等级：B（补救性回填）
- 主要来源：两题材薄 Prompt、微信订阅号研究文件。

## 0. 平台任务定义与基准继承

- 任务目标：把章节改写成更适合微信订阅号阅读与分发的版本。
- 必须继承创建正文、正文润色、去AI味的硬约束。
- 平台重点：滑读感、碎片阅读适配、微钩子、连载陪伴感、社交转发友好。

## 0.2 零新增事实

- 不得为转发感、新鲜感而新增事件与冲突。
- 允许调整段落、前置抓点、压缩解释、强化微钩子。

## 0.3 沉浸式小说体裁

- 不得写成公众号运营文、热点总结、方法文或情绪鸡汤。
- 保持章标题、正文、`## 作者有话说` 结构。

## 1. 平台理解

### 1.1 平台机制与读者画像

- 微信订阅号以移动端碎片阅读为主，更重视分段顺滑、轻钩子与持续关注感。
- 长文阅读耐性有限，需要更清楚的分段节奏。
- 研究显示超长篇原生承载弱，应强化“每篇可读 + 连载可追”。

### 1.2 平台风格门槛

- 建议 1–3 句一段，减少长段压读。
- 每个小波段尽量保留一个微钩子。
- 适合轻陪伴式作者有话说，但不得出戏。
- 需要兼顾一次阅读完整感与下一次打开动机。

## 2. 输入与输出

- 输出目录根：`微信订阅号/`
- 路径映射保留 `{分部目录}/{分卷目录}`。
- 文件名：`{chapterNo} {平台新标题}.md`

## 3. 输出内容结构

- 顶部标题：`# 第{chapterIndex}章：{平台新标题}`
- 正文：短段、顺滑、持续微钩子
- `## 作者有话说`：中文，200–300 CJK，可轻陪伴，不得运营腔过重

## 4. 关键硬约束

- 中文平台正文 CJK 必须在 3500–6500。
- 不得写成“公众号爆文壳 + 小说内容”。
- 滑读感不能以弱化事件链为代价。

## 5. 改写前推演问题

1. 哪些段落需要拆短？
2. 本章连续阅读的微钩子布在哪些位置？
3. 作者有话说如何既陪伴又不出戏？

## 6. 平台执行要点

- 段落短、推进清。
- 关系变化、风险变化与下一步行动要更明确。
- 适度强化“读完这一篇还想点下一篇”的感觉。

## 7. 输出格式与检查

- 检查段落是否适合手机阅读。
- 检查是否有稳定微钩子。
- 检查作者有话说是否克制。
