# 通用-输出百度百家号版

这是平台共性本体，负责承接“百度百家号版输出”的跨题材共性规则。

## 常见触发词 / 用户说法速查

- 输出百家号版
- 改成百家号能发的版本
- 做百家号平台改写
- 帮我把这章改得更适合信息平台

## 何时使用

- 把任意题材章节改写成更适合百家号分发语境的版本。
- 需要强化阅读顺滑度、信息抓力、传播友好度与轻话题性。
- 需要为 `题材名-输出百度百家号版` 提供统一的平台骨架。

## 不适用情形

- 只做题材边界补充，不做平台输出本体。
- 只做多平台编排、落盘和终检，而不处理百家号平台风格本身。

## 本层职责

- 维护百度百家号平台的跨题材共性规则。
- 统一承接百家号版输出的结构约束、标题倾向、传播侧重点与风格门禁。
- 为后续新增题材保留直接复用入口。

## 继续读取的 references

- `references/跨题材共性与迁移来源.md`
- `references/平台共性执行细则.md`
- `references/分节级补救映射与详细规则回填.md`

## 默认执行顺序

1. 先读取对应题材的 `题材名-输出百度百家号版` Skill。
2. 先锁定零新增事实、体裁结构、工作目录、章节层级、字数门禁与标题同步规则。
3. 再执行百家号平台的信息抓力、传播友好与顺滑阅读强化。
4. 最后才处理降相似度，不得为了降重写成资讯腔。

## 默认输出口径

- 默认输出一版可直接继续落盘或进入平台终检的百家号派生正文。
- 默认保留原章关键信息、事件链、证据链与钩子驱动，不新增关键事实。

## 最低交付

- 有明确的百家号信息抓点与传播友好度。
- 保住原章核心事件链与读者可跟随的推进线。
- 读起来不像资讯稿、也不像泛平台轻改稿。

## 硬规则

- 命中本技能时，必须同时加载对应题材的 `题材名-输出百度百家号版` Skill。
- 题材特有规则不得回写到本文件中平行维护。
- 平台派生正文默认落在本 Skill 的工作目录 `百家号/` 下。
- 章节级派生正文目录统一按小说结构决定：若作品有分部，则位于 `百家号/第X部/第Y卷/`；若作品无分部，则位于 `百家号/第X卷/`。
- 平台派生正文文件名默认不带日期，沿用既有章节号与平台标题同步规则；日期只用于配套审阅报告、书评等派生产物。
- 平台稿落盘后必须显式运行 `scripts/count-chapter.ps1 -MinCJK 3500 -MaxCJK 6500` 核正文，并运行 `scripts/count-afterword.ps1 -MinCJK 200 -MaxCJK 300` 核 `## 作者有话说`。
- 在无损迁移完成前，不得删除源 `输出百度百家号版.prompt.md` 中尚未完成映射的内容。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只负责百度百家号平台的跨题材共性骨架。
- 题材边界、题材禁行项与题材特有口径，继续由对应 `题材名-输出百度百家号版` 承接。
