# 通用-输出百度百家号版｜平台共性执行细则

## 适用目标

将章节改写成更适合百度百家号语境的版本，同时保持：主要信息不丢失、信息抓力更强、传播友好、输出仍为纯小说正文。

## 平台共性任务定义

- 强调开头抓取效率、信息密度、阅读顺滑与轻话题切口。
- 保留关键事件链、人物冲突、风险推进与章末钩子。
- 不允许新增事件、证据、人物、地点、时间点或因果。

## 共同硬约束

- 正文必须是小说，不得写成资讯稿、总结、清单、点评稿。
- 输出文件只允许：章标题、正文、`## 作者有话说`。
- 正文字数以 CJK 计，必须在 3500–6500 之间。

## 百家号平台节奏共性

- 开头要更快进入可传播的异常点、冲突点或问题点。
- 章内需保持顺滑、可扫读、少空转。
- 允许轻话题感，但不得破坏沉浸式小说体裁。
- 优先让异常点同时具备“可传播 + 可搜索 + 可复述”三项中的至少两项。
- 地域、职业、禁忌名词等高价值信息，尽量在前 1000 字自然显形，便于搜索承接。
- 优先使用冷静白描、镜头推进与日常物件线索化，避免夸张抒情和空泛氛围词。
- 现实职业、生活化场景与公平伏笔要更清楚，让读者能顺着异常点一路跟下去。
- 若章节已经有强异常，百家号化优先提升抓力与顺滑度，而不是把文本硬压成资讯摘要。
- 百家号的关键是“抓得住第一眼，也能继续读下去”：前面要有抓点，后面要有小说里的后果。
- 话题感要藏在剧情里，不要写成平台热词拼盘。
- 标题、开头、前千字是抓取核心，但正文仍然必须是小说，不得变成新闻通报口吻。

## 百度百家号的表达原则

- 用清楚的异常点、职业点、地域点带起文章，但别把叙述写成资讯稿。
- 句子尽量利落，信息尽量前置，但动作和人物反应不能丢。
- 如果一段只是在交代背景，优先补一个可见事件或动作。
- 作者有话说要极轻，不要二次总结正文。

## 执行顺序

1. 先加载对应题材 Skill。
2. 先锁定零新增事实、体裁结构、路径与字数门禁。
3. 再处理百家号平台的信息抓力、传播友好与顺滑阅读。
4. 最后才处理降相似度，不得为了降重写成资讯腔。
