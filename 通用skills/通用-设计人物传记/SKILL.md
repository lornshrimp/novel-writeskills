# 通用-设计人物传记

人物传记应写成作者可长期调用的裁判源，而不是漂亮但难落地的人设简介。

## 继续读取的 references

- `references/人物传记最小结构模板.md`
- `references/心理弧线与索引规则.md`
- `references/质量控制与验证流程.md`
- `references/人物文件落盘与三级索引规则.md`
- `references/缺陷执念秘密与反派动机.md`

## 默认适用场景

- 新建某个角色的人物传记
- 把旧传记重写为可写、可审、可对齐大纲的版本
- 给人物补充写作抓手、关系网、心理弧线与索引映射

## 默认执行顺序

1. 先确认目标人物文件与写入策略。
2. 判断人物层级：核心 / 重要 / 次要。
3. 先按模板搭建身份、动机、边界、关系、抓手、秘密、弧线、索引。
4. 再补心理弧线与事件触发链。
5. 最后走质量控制与验证流程。

## 通用硬规则

- 传记不是宣传文案，禁止只写抽象褒贬。
- 人物必须有动机，也必须有代价。
- 能力不能万能，关系不能只是“认识谁”。
- 写作抓手必须可直接贴回正文使用。
- 除非用户明确要求只脑暴，不得只在聊天里给传记长文而不写入对应文件。
- 主角至少要写清执念、缺陷、秘密三者中的两项；反派必须写清可落地动机来源。
