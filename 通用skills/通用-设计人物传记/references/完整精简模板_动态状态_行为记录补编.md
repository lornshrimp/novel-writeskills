# 完整精简模板_动态状态_行为记录补编

> 吸收外部 `character-template` 与 `archive-maintenance` 的人物档案结构，用于把人物传记补强成“稳定字段 + 动态状态 + 行为记录 + 语言指纹”的长期裁判源。

## 一、人物模板分层

### 完整模板（主角 / 反派 / 核心重要角色）

应尽量覆盖：

- 基本信息
- 性格核心
- 语言指纹
- 背景
- 角色弧线
- 当前状态
- 章节行为记录

### 精简模板（普通配角）

至少覆盖：

- 基本信息
- 性格与功能
- 语言指纹
- 当前状态
- 章节行为记录

### 升级规则

若配角戏份上升、结构作用明显扩大，可升级为完整模板；升级时补字段，不抹旧记录。

## 一点五、来源层映射（新增）

- `模板层级层`：完整模板 / 精简模板 / 升级规则是否判断正确。
- `动态状态层`：位置、情绪、已知信息、持有物、身体状态、最后更新章节是否真实维护。
- `行为记录层`：逐章行为记录、关系实质变化、语言指纹情绪变体是否可持续追加。

## 二、完整模板建议字段

### 基本信息

- 全名
- 年龄
- 性别
- 职业 / 身份
- 外貌特征（突出辨识点）
- MBTI（若项目确实在用）

### 性格核心

- 核心特质
- 核心价值观
- 内心渴望
- 最大恐惧
- 致命缺陷
- 行为倾向（在压力下会怎样）

### 语言指纹

- 说话风格
- 口头习惯
- 情绪变化时的语言变化

### 背景

- 背景故事
- 核心事件
- 秘密

### 角色弧线

- 起点状态
- 核心冲突
- 预期变化方向

## 三、动态状态字段

章节推进时优先更新：

- 位置
- 情绪状态
- 已知信息
- 持有物品
- 身体状态
- 最后更新章节

这些字段是连续性与正文校准时最常用的第一抓手。

## 四、章节行为记录

- 每章一句话
- 追加，不回改
- 只写关键行为和结果
- 不写长段心理复盘

作用：

- 快速回忆人物在最近几章做了什么
- 给连续性、审阅、正文创作提供硬事实

## 五、稳定字段 vs 动态字段

### 稳定字段

- 性格核心
- 价值观底色
- 语言指纹主轴
- 背景与成长经历
- 角色弧线大方向

### 动态字段

- 当前位置
- 当前情绪
- 当前已知信息
- 当前持有物
- 最新身体状态
- 最新关系位差

默认不要把单章情绪波动误写成性格底色改变。

## 六、关系总览建议

人物传记最好能与关系总览互相回指，至少能快速定位：

- 与谁是什么关系
- 当前状态
- 关键事件
- 最后更新章节

## 七、语言指纹补充要求

除了“平时怎么说”，还应尽量写：

- 愤怒时怎么说
- 恐惧时怎么说
- 伪装时怎么说
- 亲密时怎么说
- 对上位者 / 下位者 / 熟人 / 陌生人是否换声口

## 八、红线

- 不得只有人物优点没有缺陷
- 不得只有静态人设没有动态状态
- 不得只有背景故事没有行为记录
- 不得所有角色都用同一种说话模板
- 不得把“当前状态”空着长期不维护

## 九、最小动态工件（新增）

```markdown
【模板层级层】完整 / 精简 / 是否应升级
【动态状态层】位置 / 情绪 / 已知信息 / 持有物 / 身体状态 / 最后更新
【行为记录层】本章关键行为 / 关系变化 / 语言变体
【当前判定】Green / Yellow / Red
```

## 十、Green / Yellow / Red 动态判定（新增）

- `Green`：模板层级判断正确，动态状态真实可追，行为记录可续写。
- `Yellow`：基本字段在，但动态维护不稳定或行为记录偏虚，后续易断档。
- `Red`：模板选错、动态状态长期空置、行为记录缺失，人物档案只剩静态外壳。
