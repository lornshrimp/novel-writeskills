# 质量控制与验证流程

## 成功标准

- 标题区已标注人物类型，核心 / 重要人物默认还标出心理弧线步数、传记版本与最后更新
- 人物传记在原有细分结构之外，能被清楚归并到背景、性格、外貌、其它四大基础栏位，且四类均未缺失
- 写作抓手交付区块完整，至少能落到“口头禅 / 句式习惯、习惯动作与微反应、触发器、底线 / 开关、留痕偏好、反制动作模板”这六类中的核心字段
- 与大纲零矛盾
- 人物动机清晰且带代价
- 成长时间线能解释人物当前的选择、误判与伤口来源
- 外部目标 / 内部需求 / 价值观 / 误信彼此不打架
- 关系网能产出事件
- 公众形象与自我认知存在可写差值，而不是两套重复表述
- 写作抓手能直接贴回正文
- 核心 / 重要人物的基本信息里已补足职业 / 身份轨迹、家族关系 / 原生关系、可复盘细节库与结构功能位
- 至少有一个可回指的物件 / 称呼 / 地点锚点
- 心理弧线具备触发事件、后果与回指点，核心 / 重要人物最好显式标注总步数
- 索引能作为交叉核查入口
- 人物在大纲中的行动、时间线与关系描述能与传记逐项对上
- 主要人物具备外貌特征、标志性动作、语言特色三重识别抓手
- 重要人物成长轨迹包含“初始缺陷 → 触发事件 → 认知转变 → 成长体现”
- 若本次是“补充 / 扩展”而非彻底重写，文件中已保留 `本轮修订（YYYY-MM-DD）` 作为补丁入口
- 核心人物若承担长线供血或规则压力，默认还应具备“身份认同危机 / 内在冲突体系 / 情感变化链条 / 价值观演进”中的关键模块

## 验证流程

1. 完成传记主体
2. 检查标题区与版本信息：人物类型、心理弧线步数、传记版本、最后更新是否按人物层级写全
3. 先检查背景、性格、外貌、其它四大基础栏位是否都已落位，且不是靠零散句子硬拼出来
4. 检查 `写作抓手交付` 是否已成组落位：至少能直接调取口头禅、微反应、触发器、底线 / 开关、留痕偏好、反制动作模板
5. 对照总纲 / 分卷检查一致性
6. 对照其他人物传记检查关系
7. 检查职业 / 身份轨迹、家族关系 / 原生关系、题材或时代压力锚点、可复盘细节库是否具备作者侧调用价值
8. 检查写作抓手、压力反应、安慰机制与锚点物是否可直接用
9. 检查外部目标 / 内部需求 / 价值观 / 误信是否能互相解释，不是四套平行口号
10. 检查秘密是否至少有一个可验证或可落剧情
11. 检查心理弧线是否有回摆、失败与误判，以及阶段性失去 / 获得 / 误信击穿点；核心 / 重要人物要抽查关键步骤的可回指载体
12. 补全索引，确保可回查
13. 对照总纲 / 分卷 / 章清单，检查人物事件是否与传记中的年龄、经历、关系网、社会位置与资源来源一致
14. 抽查人物是否具备可直接落到正文的记忆点、公众形象差值、成长弧阶段证据与必要的补丁入口（如 `本轮修订`）

## 大纲联动检查

### 人物传记匹配检查

- 大纲中涉及的每个人物事件都必须与对应人物传记一致
- 人物在大纲中的行为必须符合传记中描述的性格特征
- 人物年龄、经历时间与关系变化要与传记中的时间线统一
- 人物之间的关系描述要与各自传记记录一致
- 大纲中的社会位置、资源权限、公众形象与人物传记中的设定要能互相印证

### 人物塑造强化要求

- **外貌特征**：1–2 个鲜明外在特征
- **标志性动作**：1–2 个稳定行为习惯
- **口头禅 / 语言特色**：明确其表达方式、常用词或说话节奏
- **压力反应 / 安慰机制**：至少各 1 个可拍出来的动作或行为反应
- **象征物 / 记忆锚点**：至少 1 个可被后文反复调用的物件、地点或称呼

### 动态成长弧设计

- **初始缺陷**：人物开始时的性格缺陷、认知局限或能力不足
- **外部目标 vs 内部需求**：人物表面追求什么，真正缺的又是什么
- **触发事件**：促使人物开始改变的关键事件或转折点
- **认知转变**：人物价值观、世界观、误信或行为模式的具体变化过程
- **成长体现**：新的认知在后续情节中的具体体现和应用
- **可回指载体**：最好至少给关键阶段绑定物件、称呼、地点、编号、时间戳或关系节点

### 矛盾检测与解决

- 发现与人物传记冲突的地方，用 `⚠️ 人物矛盾：[具体描述]` 标记
- 提供“大纲修改 / 传记更新 / 双向校准”的具体方案
- 大纲修改后，相关人物传记也应同步更新，避免形成双口径

## 常见返工原因

- 标签很多，正文抓手太少
- 抓手有标题没内容，或只有一两条公版动作，无法直接回贴正文
- 外层识别很多，公众形象与自我认知没有差值
- 心理成长像总结，不像事件轨迹
- 心理弧线标了步数，却没有关键触发、后果与可回指载体
- 关系网没有权力结构
- 能力边界没写，人物像万能工具
- 只有人物设定，没有职业 / 身份轨迹、可复盘细节库和结构功能位，导致后续难以维护
