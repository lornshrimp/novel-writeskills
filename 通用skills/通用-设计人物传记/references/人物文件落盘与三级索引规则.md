# 人物文件落盘与三级索引规则

## 作者侧定位

人物传记默认是作者侧的：

- 人物裁判源
- 调用手册
- 交叉核查入口

不是读者向小传，也不是正文段落。

## 默认写入策略

### 修改现有文件

- 用户说“重写 / 推翻重写” → 完整覆盖
- 用户说“补充 / 扩展” → 在相应小节补写，或文末新增“补充记录（YYYY-MM-DD）”
- 用户未说明 → 默认保守追加“本轮修订（YYYY-MM-DD）”并把详细内容嵌入对应结构

### 新建人物

- 默认新建到：`人物传记/人物名.md`
- 必须写入完整骨架，而不是只写几段描述

## 必须显式写入的作者抓手

- 口头禅 / 句式习惯
- 习惯动作 / 微反应
- 触发器
- 底线 / 开关
- 留痕偏好
- 反制动作模板

这些内容必须落在文件正文里，不能只留在聊天里。

## 三级索引落盘规则

人物传记末尾必须保留“三级大纲情节对应索引”，最少包含：

- 来源层级：总纲 / 分部 / 分卷 / 章节
- 编号
- 时间节点
- 事件描述
- 重要程度
- 心理阶段 / 步骤号
- 本事件对人物的后果

## 矛盾标记

若发现人物传记与三级大纲体系冲突，必须显式标记：

- `⚠️ 潜在矛盾：[具体描述矛盾内容及所在大纲层级]`

并给出：

- 大纲修改
- 传记更新
- 双向校准

三选一或组合方案。
