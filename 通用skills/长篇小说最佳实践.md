# 最佳实践 V2：从灵感到多平台分发（题材 Prompt 入口 + 门禁驱动版）

> 适用范围：本仓库所有题材目录（如 `女频爱情/`、`都市悬疑/`、`异能志怪/` 等）
>
> 核心原则：**用户直接调用的是题材目录下 `.github/prompts/*.prompt.md`，不是通用 Skill。**

---

## 1. 先统一调用架构（避免入口错位）

统一按这条链路工作：

`用户任务 -> 题材目录 Prompt -> 题材 Skill 路由 -> 通用 Skill 能力本体`

意味着：

- 用户在对话里应使用题材 Prompt 能力名（如“竞对分析”“设计大纲”“多平台输出SOP”）。
- 不要让用户直接说“调用通用-xxx skill”。
- `通用skills/` 是能力底盘，不是用户侧入口。

一句话记忆：**对用户说 Prompt，对系统走 Skill。**

---

## 2. 先回答一个关键争议：总纲没成型，先做人设/设定/台账是否合适？

**结论：可以做，但只做“轻量版”，不做“重定稿版”。**

### 2.1 为什么不能过早重投入

- 大纲未稳时，主线目标、卷级推进、终局约束都可能变化。
- 若先重写人物/设定/台账，后续会出现高返工、口径冲突、线索失配。

### 2.2 最优做法：两段式建模

1. **总纲前（轻量底盘）**：只保留最小可行信息
   - 人物：核心欲望 + 关键短板 + 主要关系冲突
   - 设定：3~5 条硬规则（边界/代价/失效）
   - 台账：只登记“主线必须兑现”的关键伏笔
2. **总纲后（重度定稿）**：再扩写完整人物传记、技术设定、线索伏笔台账

---

## 3. 全流程最优形态：5 段 4 门禁（不是一条直线）

相比“线性九步法”，V2 推荐“分段 + 门禁 + 回环”流程：

1. 探索段：灵感、题材定位、竞对
2. 架构段：总纲与分卷骨架
3. 建模段：人物/设定/台账定稿
4. 生产段：正文创作、审阅回炉
5. 发行段：签约评估、组包、多平台分发与数据回流

### 门禁 G1（探索 -> 架构）

- 一句话卖点可复述
- 目标平台竞对差距清晰
- 开篇 3 章承诺可执行

### 门禁 G2（架构 -> 建模）

- 总纲 V1 可拆章、可连载
- 分卷起步节奏可落地
- 关键回报节点明确

### 门禁 G3（建模 -> 生产）

- 人设/设定/台账与总纲一致
- 无明显冲突或断链
- 能直接支撑章节施工

### 门禁 G4（生产 -> 发行）

- 章节质量达标（开头抓力/中段回报/章末钩子）
- 审阅报告闭环完成
- 母稿稳定可适配

## 5. 运营回流：把分发数据反哺创作（很多团队缺这一步）

多平台分发后，不是“完成”，而是进入下轮优化输入：

1. 汇总读者反馈与完读/追更信号（按平台）
2. 识别强反馈场景与弱反馈段落
3. 回写到：下一卷大纲、章节控制卡、人物关系推进策略

这一步决定了你能否形成可持续迭代的连载引擎。

---

## 6. 常见误区与修正

1. **误把通用 Skill 当用户入口**
   - 修正：用户侧永远从题材目录 `prompt` 进入。

2. **总纲未稳就重度建模人物/设定/台账**
   - 修正：先轻量底盘，过 G2 再重度定稿。

3. **审阅只给聊天意见，不落盘**
   - 修正：审阅报告、补丁动作、复审结论都必须落库。

4. **多平台过早分发**
   - 修正：先把母稿做稳，再执行 `多平台输出SOP.prompt.md`。

5. **把编辑顺序做反**
   - 修正：先结构/发展层，再语言层，最后拷贝校对层。

## 7. 一份最短可执行清单（V2）

以女频爱情新书为例：

1. `设计大纲.prompt.md` + `竞对分析.prompt.md`，跑探索段并过 G1。
2. `设计大纲.prompt.md` + `审阅大纲.prompt.md` + `大纲审阅优化SOP.prompt.md`，跑架构段并过 G2。
3. `撰写人物传记.prompt.md` + `撰写技术手法程序证据设定.prompt.md` + 对应审阅 Prompt，跑建模段并过 G3。
4. `创建小说正文.prompt.md`（或 `撰写小说章节SOP.prompt.md`）+ 审阅/润色/去 AI 味，跑生产段并过 G4。
5. `评估起点签约概率.prompt.md` -> `起点内投材料组包.prompt.md` -> `多平台输出SOP.prompt.md`，进入发行段并建立数据回流。

## 8. 外部方法论校准（本次联网复核要点）

本次联网检索到的共识（用于校准本流程）：

- **人物与情节需绑定**：角色设定应服务故事推进，而非孤立堆料。
- **编辑顺序应宏观先行**：先结构发展层，再句级层，再技术校对层。
- **外部读者反馈时机**：建议至少在较扎实二稿后再交 Beta/外部审阅。
- **连载宜分块嵌套**：场景 -> 章节 -> 弧线分层推进，并保留方向校准与迭代空间。

这与本仓库 V2 的“门禁 + 轻重分层 + 回流”方案一致。

---

## 9. 结语

这套 V2 最佳实践的目标，不只是“写完”，而是让你把灵感变成可持续生产线。

只要记住两句话：

1. **入口永远在题材 Prompt，不在通用 Skill。**
2. **总纲前轻建模，总纲后重定稿，分发后要回流。**

那么从第一章到多平台分发，流程会更稳、返工更少、迭代更快。

## 10. 开始创作前先做环境准备（小说工作目录）

> 这里的“工作目录”不是本项目根目录，而是你这本书自己的目录，例如：`她不再回头/`。

### 10.1 第一步：创建小说工作目录

示例目录：`D:\小说项目\她不再回头\`

### 10.2 第二步：在小说工作目录下创建 junction（Windows）

进入 `她不再回头/` 后，执行以下 4 条：

- `mklink /j .github 本项目中对应题材下的.github目录路径`
- `mklink /j 通用skills 本项目中的通用skills目录路径`
- `mklink /j 写作研究 本项目中对应题材下的写作研究目录路径`
- `mklink /j scripts 本项目中的scripts目录路径`

例如当前题材是女频爱情，则 `.github` 与 `写作研究` 应指向 `女频爱情/` 下对应目录。

### 10.3 第三步：在 VS Code 中打开小说工作目录再调用

后续执行应使用：

- **斜杠命令**：如 `/竞对分析`、`/设计大纲`
- **上下文引用**：如 `#小说总大纲.md`、`#审阅意见/第06章审阅.md`

不是写 `女频爱情/.github/prompts/xxx.prompt.md` 这种路径命令。

## 11. 调用时怎么写（/命令 + #引用 模板）

### 11.1 通用调用模板（可直接复制）

> /【命令名】
>
> 目标：一句话说清本轮要完成什么。
>
> 请读取以下上下文：
>
> - #文件A
> - #文件B
>
> 本轮约束：
>
> - 必须保留什么（例如成长线、关系线、关键设定）
> - 不能做什么（例如不新增事实、不改世界观硬规则）
>
> 输出要求：
>
> - 写回到哪个文件或目录
> - 需要几个版本（如主版 + 短版）
> - 聊天侧只回摘要，不贴全文

### 11.2 最小输入清单（防止“喂不饱”）

每次至少给这四类信息中的三类：

1. **对象**：处理哪本书、哪一卷、哪一章。
2. **依据**：#大纲、#正文、#人物、#设定、#审阅报告。
3. **目标**：本轮要提升什么（抓力、追更、签约概率、平台适配等）。
4. **约束**：哪些不能动（事实、时间线、角色定位、字数门槛）。

## 13. 快速迁移到其他题材（只改四项）

把上述示例迁移到 `都市悬疑`、`赛博庞克`、`太空科幻` 时，只改：

1. `/命令名`（保持命令语义一致）
2. `#引用文件`（改为该题材下真实文件）
3. 题材约束（风格、边界、禁行项）
4. 输出目录（正文/审阅/分发目录）

其余结构（目标、依据、约束、输出）保持不变。
