# 通用-输出七猫版｜平台共性执行细则

## 适用目标

将章节改写成更适合七猫平台的版本，同时保持：主要信息不丢失、免费阅读平台节奏更强、输出仍为纯小说正文。

## 平台共性任务定义

- 强调直给反馈、强情节推进、高密度抓点与追读动机。
- 保留关键事件链、关系变化、风险升级与章末钩子。
- 不允许新增事件、证据、人物、地点、时间点或因果。
- 平台推荐更偏向“追读率、完读率、书架占比、评论催更率”这一整套留存信号，尤其不接受开头慢、回报晚、钩子弱的文本。

## 共同硬约束

- 正文必须是小说，不得写成清单、总结、教程或讲解稿。
- 输出文件只允许：章标题、正文、`## 作者有话说`。
- 正文字数以 CJK 计，必须在 3500–6500 之间。

## 七猫平台节奏共性

- 七猫属于免费阅读 + 算法加权平台，核心不是文学性，而是传播效率与留存效率。
- 核心用户多为 20–40 岁移动端碎片阅读读者，更偏好“接地气、不累脑、有共鸣、够上头”的内容。
- 开头更快进入冲突或异常。
- 章内抓点与阶段性回报应更密，尽量避免连续空铺垫。
- 章末必须给出清晰的下一章阅读动机。
- 底层共情、现实困境与可控异能应同时显形，避免只剩空爽感。
- 生理反应优先于抽象抒情，适配下沉读者的直觉式阅读。
- 优先使用短句、短段、动作推进与生活化对白，避免大段设定说明或复杂故布疑阵。
- 线索、误导、反转、新疑点的闭环要更直观，让读者能“边看边懂、边懂边想追”。
- 免费阅读平台不等于可以牺牲推理公平性；合理回看与证据可追仍是基础门槛。
- 七猫读感的核心是“每隔一小段就让读者觉得有收获”：发现一点、缓一口气、又来一点新的压力或新的答案。
- 强推进不等于乱冲突；阶段性回报要建立在前因和选择上，不能靠临时硬塞爽点。

## 七猫的表达原则

- 段落拆短，句子偏直，动作与结果优先。
- 每段最好能看出一件事变了：情绪、关系、风险、线索或目标。
- 章末尽量让读者清楚知道“下一章会更麻烦”或“下一章会知道更多”。
- 若一段只在解释，立刻补回现场动作或人物反应。

### 七猫平台的抓力与留存标准

- 开篇前三章必须快速立冲突、立悬念、立现实代价，否则很难进入更高流量池。
- 原生七猫创作常见单章在 1500–2500 字，但本仓库平台派生稿仍按 `3500–6500` CJK 门禁执行；因此应通过**更多分段、更多阶段回报、更多局部小闭环**去模拟七猫原生读感，而不是机械缩短字数。
- 对免费平台读者来说，“看得懂”优先于“埋得深”，但“直给”不等于“无逻辑”；所有抓点都应能回指到事件链与人物选择。

### 七猫平台的语言与段落规则

- 单段尽量不超过 3 行，1–3 句构成一个读屏单元。
- 多用客观白描、动作短句、生活化对白，少用复杂复句、修辞堆叠与大段内心抒情。
- 场景描写必须同时承担至少一种功能：推进剧情、埋伏线索、加重风险、强化情绪。
- “环境好看”不是目标，“环境立刻服务悬念和冲突”才是目标。

### 七猫平台的回报与钩子密度

- 至少保证每章一次清晰阶段回报，不能通章只铺不收。
- 若题材允许更强爽点表达，可参考“每章数个微回报、每数章一次明显中回报、每 5–10 章一个大回报”的免费平台节奏，但不得为了回报强行新增事实。
- 章末钩子应优先落在：更急的现实代价、刚到手线索被打断、误导翻面、关系破裂、旧问题刚缓解新问题立刻顶上来。
- 回报最好是“读者看懂了什么”，而不只是“人物赢了什么”。

## 执行顺序

1. 先加载对应题材 Skill。
2. 先锁定零新增事实、体裁结构、路径与字数门禁。
3. 再处理七猫平台的直给反馈、强推进与追读压力。
4. 最后才处理降相似度，不得为了降重牺牲平台抓力。
