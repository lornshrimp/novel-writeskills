# 总纲审阅完整模板（通用骨架）

适用于：总大纲、全书总纲、战略层文件组、故事面总盘、主线谜团总盘。

## 通用层与题材层的拆分原则（强制）

- 本文件只承载跨题材复用的**总纲审阅通用骨架**。
- 各题材 Skill 必须在此骨架上补写自己的“题材吸引力与差异化核查”“题材专项评分项”“题材结论重点”。
- 题材 Skill **不得删减**本模板的通用主骨架，只能补写、替换题材专项段落中的占位项。
- 若对象不是单一文件而是总纲包，报告必须显式写明已覆盖的文件组与未覆盖对象。
- 审阅总纲包时，默认将 `小说总大纲.md` 视为主线故事基准，将 `故事面/` 视为以支线为主的完整故事文件组；二者交织联动不等于重复维护。

## 落盘位置与文件名格式（强制）

- 总纲审阅报告直接位于 `审阅意见/` 根目录，不另建总纲专用子目录。
- 文件名示例：

`审阅意见/总纲审阅报告_[名称或编号]_[审阅日期].md`

以下章节在任一题材下都必须保留：

6. `## 🧩 题材吸引力与差异化核查（由题材 Skill 补写｜强制）`
7. `## 🧭 超长篇工程化核查`
8. `## 🪜 长篇耐力与中段失温预警（强制）`

## 🧰 可写性与工程化可执行性评估（作者向｜强制｜P0）
- [ ] **可写性（P0）**：是否能把“每一卷/每一场”的目标、动作、阻力、回报、代价写成可执行清单，而不是抽象口号。
- [ ] **冲突落点可见（P0）**：关键冲突是否都能落到可拍镜头、可留痕工件、可验证后果上；若只有概念没有载体，判为不可写风险。
- [ ] **证据链/程序链/因果链可跑通（P0）**：主角或核心推进者的破局抓手是否前置铺垫、可留痕、可回收；关键反转是否有记录、编号、第三方节点或规则后果。
- [ ] **对抗升级可调度（P0）**：是否给出阶段递进的升级按钮，并标注触发条件与后果。
- [ ] **写作导航性（P0）**：是否存在作者可直接复用的结构化交付物（如：场景清单、伏笔-回收表、卷群推进表、人物职责表、章节控制接口）。

## 🎨 文学性与新颖性（加分项｜P2/P3）
- [ ] 文学表达的张力：是否有可记忆的场景意象/一句话钉子，但不过度代替结构施工。
- [ ] 结构与机制新意：是否避免同质化套路，能一句话讲清“本书独有机制卖点”。

---

## 🔥 网文长线吸附与签约潜力复核（强制）
- [ ] **编辑 1 分钟记忆点**：是否能用一句话讲清“这书为什么不是旧套路换皮”，编辑/读者是否能快速记住其题材稀缺度。
- [ ] **首卷承诺清晰**：总纲是否已写清前 `3/5` 章抓手、首卷阶段胜利、第一轮主线拼图与第一轮代价回响。
- [ ] **平台主口径明确**：是否已经决定更偏起点长线、番茄快反馈、豆瓣深读、盐选短反馈或海外平台表达友好；若想多平台兼容，主次口径是否清楚。
- [ ] **卖点一致性成立**：总纲主卖点与简介、题材定位、书评话术、平台承诺是否同向，而不是各说各话。
- [ ] **开篇承诺到终局兑现同向**：开篇 promised 的题材体验、谜面方向、情绪承诺与终局闭环是否一条线，不得中后段换题。

## 🧭 超长篇工程化核查
- [ ] 四层大纲存在性：思想纲领/总大纲/分部大纲/分卷大纲（或同等层级）是否齐备；若缺层，是否有替代口径。
- [ ] 主支线职责边界：`小说总大纲.md` 是否稳定承载主线故事，`故事面/` 是否稳定承载以支线为主的完整故事；两者是否互相供血而非彼此打架。
- [ ] 主线/支线比例：主线推进是否被支线长期稀释；支线是否服务主线机制递进或人物弧，而不是单纯充字数。
- [ ] 冲突金字塔：个人困境→关系网阻力→制度/机制压迫是否逐层加码；是否存在对抗“越写越轻”的战力崩坏。
- [ ] 悬念链与回收：伏笔是否留痕可追溯；回收是否靠证据链/程序链/因果链完成；是否存在坑填不上或无声蒸发。
- [ ] 弹性节点：是否预留可替换/可合并的缓冲剧情节点以应对连载反馈。
- [ ] 阶段递进：试水→扩盘→收束→终局（或同等分期）的边界是否清晰；每阶段的新冲突/新代价/新升级是否递进。

## 🪜 长篇耐力与中段失温预警（强制）
- [ ] **主卖点冷热周期**：主卖点是否出现连续 `2` 卷以上基本失踪的危险窗口；若有，是否已有补回计划。
- [ ] **主线供血与配菜比例**：卷群推进中，主线推进卷与配菜卷的比例是否失衡，是否存在“案很多、主线不动”的中段断供风险。
- [ ] **冷线回归窗口**：人物线、旧债线、伏笔线是否标了合理回归窗口，而不是放着忘掉。
- [ ] **AI 总纲病预警**：是否出现“概念很多、动作很空、人物像功能按钮、阶段目标全是抽象口号”的 AI 式总纲病。

## 使用说明（强制）

- 若当前任务是题材 Skill 执行，必须把题材专属检查项、题材专项评分项与题材化结论重点补进上面的占位章节。
- 若题材 Skill 只输出摘要版或删掉其中任一主章节，视为未完成保真迁移。
- 若总纲对象其实是“总纲包”，不得只写单文件结论，必须在报告开头点名覆盖范围。
- 若总纲对象是“总纲包”，必须先判断 `小说总大纲.md` 与 `故事面/` 的职责边界，不能把有意设计的主支线双轨结构直接判成重复维护。
- 若未单独判断平台主口径、首卷承诺与长篇耐力，视为未完成网文战略层审阅。
- 若未写明“借鉴哪本竞对的什么点、弱于哪本竞对的什么点、本轮先改什么”，视为竞对接入未完成。
