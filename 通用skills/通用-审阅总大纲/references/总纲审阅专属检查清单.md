# 总纲审阅专属检查清单

## 一、对象与范围

- 是否明确本次审的是总纲包，而不是单一主文件
- 是否明确 `小说总大纲.md` 承载主线故事、`故事面/` 承载以支线为主的完整故事文件组
- 是否检查主线与支线的交织、联系与相互推动，而不是误判为重复维护
- 只有在职责重叠且口径冲突 / 无新增职责时，才判为冗余或总纲层不一致
- 若范围受限，是否显式写明未覆盖对象

## 二、四阶段机制递进

- 试水期是否有异常启动与第一抓手
- 扩盘期是否通过多个单元持续供血
- 收束期是否具象化对抗与机制升级
- 终局期是否提前布设闭环条件

## 三、单元案矩阵供血能力

- 每个单元案是否至少推进一块主线拼图
- 是否存在漂亮但脱主线的冗余单元案
- 是否存在主线只在后期突然推进的断供风险

## 四、总纲级连贯性

- `M0 / X / C` 是否清楚，不互相篡位；主线总纲与故事面是否各司其职
- 是否有稳定的系列锚点
- 中段是否存在结构塌陷风险

## 五、最小补丁思路

- 补一块主线拼图
- 补一个阶段闭环
- 补一个系列锚点
- 补一条代价链
- 补一个开篇承诺块

## 六、网文长线与签约口径

- 是否存在一句话记忆点，能让编辑快速记住这套书为什么值得签
- 首卷承诺是否清楚：前 3/5 章抓手、首卷阶段胜利、第一轮主线拼图是否已写明
- 平台主口径是否明确：起点 / 番茄 / 豆瓣 / 盐选 / 海外 等主次关系是否清楚
- 开篇承诺与终局兑现是否同向，是否存在中后段换题风险

## 七、长篇耐力与 AI 总纲病预警

- 主卖点是否出现连续 2 卷以上基本失踪的危险窗口
- 主线 / 配菜比例是否失衡，是否存在“案很多、主线不动”的断供风险
- 冷线、旧债、人物线是否标明回归窗口，而不是长期蒸发
- 是否出现“概念很多、动作很空、人物像功能按钮”的 AI 式总纲病

## 八、规划归档与模板组合镜像

- 是否写明主模板 / 辅助模板 / 模板组合声明，而不是只说“参考了某模板”
- 是否存在章节规划表、TODO 状态、章节摘要入口与节奏地图
- 节奏地图是否显式检查连续 3 章以上重章的疲劳风险
- 是否存在活跃悬念 / 伏笔追踪、决策日志、更新日志或最小进度快照入口
- 主题是否从人物经历、性格底色与选择后果中自然长出，而不是先立主题口号再逼人物表演

## 九、问题来源层与战略入库判定

- 问题是否区分为：总纲结构层 / 卷群分配层 / 执行接口层
- 是否给出 Green / Yellow / Red 战略入库判定，而不是只有“能用 / 不能用”的模糊结论
- 是否指出哪一层最该优先回写，以及为什么这一步能最快抬升签约 / 追读潜力
