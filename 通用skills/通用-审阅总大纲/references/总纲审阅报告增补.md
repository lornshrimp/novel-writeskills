# 总纲审阅报告增补

## 报告里必须写重的部分

### 1. 范围说明

- 覆盖了哪些总纲层文件
- 哪些未覆盖
- 由此带来的判断风险
- 是否明确区分 `小说总大纲.md` 的主线职责与 `故事面/` 的支线职责

### 2. 主支线协同、机制递进与供血诊断

- `小说总大纲.md` 与 `故事面/` 是否分别承载主线 / 支线的完整故事，而不是彼此摘要或副本
- 主线与支线如何相互交织、相互联系、相互推动
- 哪些重合是正常联动，哪些才是真正口径冲突或职责打架
- 四阶段机制递进卡在哪里
- 当前最强的主线里程碑锚点是什么
- 单元案矩阵有没有持续供血
- 终局闭环条件是否提前埋好

### 3. 平台主口径、签约潜力与首卷承诺诊断

- 这份总纲最像哪类平台成品，为什么
- 编辑 1 分钟内最容易记住的题材稀缺点是什么
- 首卷承诺是否已经在总纲层写死，而不是交给下游临场发挥
- 开篇 promised 的题材体验与终局闭环是否同向

### 4. 长篇耐力与中段失温预警

- 主卖点是否会在中段多卷断供
- 哪条人物线 / 旧债线 / 伏笔线最容易被写丢
- 是否存在 AI 式总纲病：概念很满、动作很空、人物像功能按钮

### 5. 具体修改建议

建议以“插入位置 + 结构补丁 + 预期收益”格式书写，例如：

- 在【某故事面 / 某矩阵位置】补写【机制或拼图】，让【某阶段】不再空转。
- 将【某单元案】明确绑定到【某块主线拼图】，避免主线断供。
- 在【某阶段结尾】增加【阶段胜利 / 反噬 / 代价】，形成卷群级闭环。
- 在【首卷承诺块】补一句【前 3/5 章具体抓手】，让下游分卷与章节有统一入口。
- 在【平台定位段】明确【主平台 / 兼容平台】与【放弃项】，避免后续拆卷节奏错配。
- 若【小说总大纲.md】与【某故事面】是联动关系但职责写得发虚，优先补写【主线职责 / 支线职责 / 交接接口】，不要误按“重复维护”粗暴删并。

### 6. 竞对差距量化卡（必须写重）

报告里不得只写“整体弱于竞对”这类空话；默认按统一母版顺序写：

1. `Top 10 候选池收束`
2. `3–5` 本主压制样本四层深拆速览
3. `统一量化卡`
4. `汇总判断 / 最危险落后项 / 优先反超位 / 盲区说明`

其中“统一量化卡”固定列必须保留：

- 我方当前分（0–5）
- 竞对均值分（0–5）
- 分差（我方-竞对）
- 关键证据
- 72 小时补丁
- 不应照搬点

总纲层的核心维度默认覆盖：

- 一句话卖点 / 平台归类效率
- 首卷承诺与前门抓手
- 主线供血与中段耐力
- 单元案矩阵供血
- 终局闭环与回收压力
- 差异化与不可替代性

每项至少都要写出：

- 我方当前分（0–5）
- 竞对均值分（0–5）
- 分差
- 关键证据
- 72 小时补丁
- 不应照搬点

并在量化卡后补齐：

- 竞对总差距判断：领先 / 持平 / 落后
- 当前最危险的 `2` 个落后项
- 最需要优先反超的 `2` 个维度
- 阻断点 / 判断盲区

### 7. 高分封顶规则（必须写明）

- 若竞对量化卡未完成：综合评分默认不得高于 `7.0/10`
- 若竞对项低于 `7.0/10`：综合评分默认不得高于 `8.4/10`
- 若竞对项低于 `6.0/10`：综合评分默认不得高于 `7.4/10`
- 若缺少 `Top 10 候选池`、`3–5` 本主压制样本四层深拆速览或统一量化卡任一固定列，视为竞对模块未完成，综合评分默认不得高于 `7.0/10`
- 若当前总纲在竞对验证过的关键强点上仍明显落后 `3` 项及以上，不得写成“可以直接放行”
