# 总纲审阅报告增补

## 报告里必须写重的部分

### 1. 范围说明

- 覆盖了哪些总纲层文件
- 哪些未覆盖
- 由此带来的判断风险
- 是否明确区分 `小说总大纲.md` 的主线职责与 `故事面/` 的支线职责

### 2. 主支线协同、机制递进与供血诊断

- `小说总大纲.md` 与 `故事面/` 是否分别承载主线 / 支线的完整故事，而不是彼此摘要或副本
- 主线与支线如何相互交织、相互联系、相互推动
- 哪些重合是正常联动，哪些才是真正口径冲突或职责打架
- 四阶段机制递进卡在哪里
- 当前最强的主线里程碑锚点是什么
- 单元案矩阵有没有持续供血
- 终局闭环条件是否提前埋好

### 3. 平台主口径、签约潜力与首卷承诺诊断

- 这份总纲最像哪类平台成品，为什么
- 编辑 1 分钟内最容易记住的题材稀缺点是什么
- 首卷承诺是否已经在总纲层写死，而不是交给下游临场发挥
- 开篇 promised 的题材体验与终局闭环是否同向

### 4. 长篇耐力与中段失温预警

- 主卖点是否会在中段多卷断供
- 哪条人物线 / 旧债线 / 伏笔线最容易被写丢
- 是否存在 AI 式总纲病：概念很满、动作很空、人物像功能按钮

### 5. 具体修改建议

建议以“插入位置 + 结构补丁 + 预期收益”格式书写，例如：

- 在【某故事面 / 某矩阵位置】补写【机制或拼图】，让【某阶段】不再空转。
- 将【某单元案】明确绑定到【某块主线拼图】，避免主线断供。
- 在【某阶段结尾】增加【阶段胜利 / 反噬 / 代价】，形成卷群级闭环。
- 在【首卷承诺块】补一句【前 3/5 章具体抓手】，让下游分卷与章节有统一入口。
- 在【平台定位段】明确【主平台 / 兼容平台】与【放弃项】，避免后续拆卷节奏错配。
- 若【小说总大纲.md】与【某故事面】是联动关系但职责写得发虚，优先补写【主线职责 / 支线职责 / 交接接口】，不要误按“重复维护”粗暴删并。
