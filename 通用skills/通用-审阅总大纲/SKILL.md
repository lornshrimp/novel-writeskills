# 通用-审阅总大纲

总纲审阅看的是这组文件能不能作为全书战略蓝图，而不是文案好不好看。

## 总纲包对象边界（强制）

- `小说总大纲.md` 默认承载小说的主线故事，是主线推进、阶段里程碑与终局收束的基准文档。
- `故事面/` 下各文件默认主要承载支线故事；它们不是主线总纲的注释页或拆分副本，而是各自能成立、能持续供血的完整故事文件。
- 审阅时必须同时判断：主线与支线是否相互交织、相互联系、相互推动，并共同服务全书长期连载。
- 不得因为 `小说总大纲.md` 与 `故事面/` 共享人物、事件、线索、阶段节点，就直接判成“重复维护”；这通常是主支线联动的正常设计结果。
- 只有当多份文件在没有新增职责差异的前提下机械重复同一信息，或彼此口径冲突、互相打架时，才可判为冗余维护或总纲层不一致。

## 保真迁移要求（强制）

命中总纲层审阅任务时，除本文件外，**必须继续读取**：

- `references/总纲审阅专属检查清单.md`
- `references/总纲审阅报告增补.md`
- `references/总纲审阅完整模板.md`

## 完整模板执行要求（强制）

- 产出总纲审阅报告时，必须以 `references/总纲审阅完整模板.md` 作为主骨架，不得退化成“几段摘要 + 几条建议”的简版报告。
- 报告至少要完整覆盖：阅读链路确认、概览复核、P0 工程化核查、题材专项核查、超长篇工程化、系列连贯性、机制递进、时间线、主题承诺、问题分级、评分、具体回写建议、要求对照、执行顺序、总体结论。
- 若当前任务还加载了题材 Skill，则必须在通用完整模板的占位章节中继续补写题材专项段、题材专项评分项与题材化结论重点。
- 若缺少题材专项段，仍必须保留该章节标题并明确说明“待题材 Skill 补写”，不得直接删章。

## 卖点保真要求（强制）

- 审阅时必须单独判断主卖点是否突出鲜明，能否被编辑和读者快速识别。
- 必须检查主卖点是否已经进入 `M0 / X / C`、卷群推进与终局闭环，而不是只停在定位口号。
- 若中段多卷无法说明卖点如何持续供血，应判为结构性风险，而不是轻描淡写带过。

## 网文长线与签约预判（强制）

- 审总纲时，必须把它视为“是否值得编辑签、是否撑得住长期付费连载”的战略蓝图，而不是单纯结构草图。
- 必须显式判断：`一句话记忆点/题材稀缺度`、`首卷承诺`、`平台主口径`、`中段多卷耐力`、`开篇承诺到终局兑现的一致性` 是否已经在总纲层定型。
- 若总纲出现“首卷很亮、中段断供、终局另起炉灶”或“概念很多、动作很空、人物像功能按钮”的 AI 式总纲病，必须判为战略层风险。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，审阅前必须先读取并纳入对照。
- 审阅报告不得只给内部自洽结论；必须显式判断当前总纲相对竞对在哪些维度已领先、持平或落后，以及最先该补哪条超越路径。
- 若当前总纲在竞对已验证的关键强点上明显更弱，必须直判为竞争风险；总纲审阅的默认目标是推动当前作品在关键维度上**全方位超越竞争对手**。

## 常见触发词 / 用户说法速查

- 帮我审总纲
- 看看这套全书总纲稳不稳
- M0 / X / C 有没有问题
- 这总纲能不能撑长篇
- 给我总纲审阅报告
- 这份总纲卖点是不是写丢了

## 何时使用

- 审阅总大纲、故事面、伏笔总纲、故事多面体或相关总表级文件。
- 检查主线谜团 `M0`、重要支线 `X`、单元案矩阵 `C` 是否稳定供血。
- 产出可直接落库、可直接回写的总纲审阅报告。

## 报告落盘位置与命名规则（强制）

- 总纲审阅报告必须直接落在项目根下的 `审阅意见/` 目录中，不再额外下沉到总纲专用子目录。
- 总纲审阅报告文件名必须带日期，日期格式固定为 `YYYY-MM-DD`。
- 推荐完整相对路径写法为 `审阅意见/总纲审阅报告_[对象名]_2026-04-15.md`。
- 总纲包本体文件（总纲、故事面、伏笔总纲、故事多面体及相关总表）默认不带日期；日期只用于对应审阅报告。

## 默认执行顺序

1. 先确认审阅范围是否真的是总纲包，并区分 `小说总大纲.md` 的主线职责与 `故事面/` 的支线职责。
2. 再检查四阶段机制递进与 `M0 / X / C` 关系，以及主支线是否相互交织供血。
3. 再判断单元案矩阵供血能力、跨卷推进与终局闭环。
4. 最后按报告增补输出最小补丁方案。

## 默认输出口径

- 默认输出一份带对象判断、结构风险、卖点风险与最小回写建议的总纲审阅报告。
- 默认不直接重写总纲本体，除非用户明确要求边审边回写。

## 最低交付

- 能指出总纲包对象是否完整、`M0 / X / C` 是否成立。
- 能指出 `小说总大纲.md` 与 `故事面/` 的职责边界是否清楚，主支线是否互相推动而不是互相抢位。
- 能指出卖点供血、跨卷推进与终局闭环的关键风险位。
- 能给出最小可回写补丁，而不是只给“总体可用 / 不可用”的结论。

## 复审独立性规则（强制）

- 每一轮总纲审阅都必须把当前版本当作新的审阅对象，重新独立、从严、完整地审一遍，不得把“上一轮问题修没修”当作本轮高分的主要依据。
- 核查上一轮提出的问题是否修复是必要步骤，但这只是一项基础核验，不足以替代本轮对结构、卖点、递进、供血、闭环与可回写性的全面复审。
- 若本轮发现新的结构性问题、遗漏点、弱化段或新的卖点失踪风险，必须如实写入本轮报告，不得因为上一轮修复较多就淡化、跳过或自动抬分。
- 本轮评分只能依据当前版本在本轮完整审阅下的实际表现给出，不得按“修复比例”“改动辛苦程度”或“上一轮分数惯性”顺推加分。

## 审阅重点

- 对象定义是否清楚
- `小说总大纲.md` 与 `故事面/` 的职责边界是否清楚
- 主线与支线是否形成交织推进，而不是各写各的或互相抢位
- 主卖点是否清晰可感、可被快速归类
- 首卷承诺、平台主口径与签约窗口是否明确
- 四阶段机制递进是否完整
- 单元案矩阵是否持续供血主线
- 长篇耐力与中段失温风险是否可预判
- 卖点是否持续供血并被阶段性回收
- 总纲级连贯性与可回写性是否成立

## 硬规则

- 禁止只评价“设定很多 / 格局很大 / 很有野心”这类抽象印象。
- 禁止把 `小说总大纲.md` 与 `故事面/` 默认判成重复维护；先判断它们是否分别承载主线 / 支线的完整故事职责。
- 必须落到对象、结构、供血、闭环与回写动作。
- 必须明确说明这份总纲更偏哪类平台、编辑会先记住什么、首卷承诺与终局是否同向。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责总纲层审阅与回写建议，不直接代替总纲重写。
- 若用户要直接搭建或重写总纲，应优先使用 `通用-设计总大纲`。
