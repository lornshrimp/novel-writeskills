# 拼章检查卡_开头中段章末与钩子链复核

> 用于把单场成立的章节，进一步复检成“整章也成立”。适合解决：场景各自能看，但拼起来章首不抓、中段没回报、章末不推、钩子链断、情感弧发平的问题。

## 何时启用

- 章节已完成骨架稿或分场景扩写，准备进入拼合阶段。
- 单场都不差，但整章读完像工作纪要。
- 章首、中段、章末各自有内容，但职责和波峰不够清晰。
- 需要在落稿前快速判断这章能否进润色 / 去味 / 审阅。

## 五项硬复检

1. **章首有效性**：前 `20%` 是否真正拖住读者。
2. **中段值回票价**：至少一次明确回报是否成立。
3. **章末翻页驱动**：最后是否留下必须承接的新问题 / 新危险 / 未完成动作。
4. **钩子链连续性**：是否回应了上章出口钩，同时把本章出口钩交给下章。
5. **章级弧线**：人物、关系、资源、判断、规则理解中，至少一种状态发生可感知变化。

## 常见拼章事故

- 第一场很抓，后面一路放气。
- 场景很多，但读者说不出中段究竟拿到了什么。
- 最后一场有气氛，没有动作对象。
- 上章钩子没回应，本章结尾也只是在继续吊空问号。
- 每场都推进一点点，但整章没有真正变化。

## 处理顺序

若拼章失败，默认优先回写：

1. 场景职责分配
2. 中段回报落点
3. 最后一场职责
4. 钩子链响应关系
5. 情感弧入口 / 出口

## 一句话收口

**拼章检查卡解决的不是“单场有没有东西”，而是“这些场能不能合成一章值得追的章”。**
