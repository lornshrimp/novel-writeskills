# 场景串章与章级回报分配卡

适用于把若干场景单元真正并成“可读的一章”，避免场景各自成立，但章节整体没有波峰、没有回报、没有章末驱动。

## 一、先明确：有场景，不等于有章节

章节正文不是把 3–5 个场景顺次贴起来。

真正的一章还必须成立：

- 章首吸附
- 章中兑现
- 章末驱动
- 卖点连续可感

所以在正文落稿前，需要先做一次串章分配。

## 二、场景并章时至少要分配四类职责

### 1. 章首抓力由哪一场承担

- 第一场通常负责，但不一定只能第一场负责。
- 必须明确：哪一场先把异常 / 风险 / 情绪缺口抛给读者。

### 2. 中段回报由哪一场兑现

- 一章至少一次明确回报。
- 这个回报不能被写成“继续推进”，而要能被读者说出来。

### 3. 情绪 / 关系 / 规则变化由哪一场放大

- 并非所有变化都等于回报。
- 需要指定：哪一场负责把局面拧紧、把人物关系顶歪、把规则试出来。

### 4. 章末钩子由哪一场收束

- 最后一场通常承担，但要写清是：
  - 危机
  - 揭露
  - 转折
  - 未完成动作

## 三、场景串章分配表

| 场景单元 | 本场职责 | 本场给读者什么 | 与卖点关系 | 结束后交给谁 |
| --- | --- | --- | --- | --- |
| 场景 1 | 抓眼 / 启动 | [异常 / 风险 / 情绪锚] | [重认卖点 / 亮卖点] | 场景 2 |
| 场景 2 | 推进 / 试探 | [信息 / 关系 / 压力变化] | [保温 / 预热] | 场景 3 |
| 场景 3 | 中段回报 | [明确收益] | [兑现 / 转化] | 场景 4 |
| 场景 4 | 升级 / 失手 | [代价 / 误判 / 反噬] | [加压] | 场景 5 |
| 场景 5 | 章末钩子 | [新问题 / 新危险 / 新行动] | [推下一章] | 下一章 |

不是每章都必须 5 场，但这五类职责不能长时间缺位。

## 四、串章时的三个强制检查

### 1. 是否有“中段值回票价”

如果一章读完后只能说“发生了不少事”，却说不清中段拿到了什么，说明章节仍然发塌。

### 2. 是否有“卖点连续可感”

- 第一场让读者认出这本书
- 中段让读者再次确认这本书的价值
- 章末让读者愿意继续追下去

若卖点只出现在第一场，后面全靠流程推进，章节容易失温。

### 3. 是否有“章级弧线”

哪怕只是小弧线，也应看见：

- 人物判断变了
- 关系位置变了
- 资源状态变了
- 规则理解变了

如果五场下来什么都没真正变，只是信息累积，这更像工作纪要，不像章节。

## 五、什么时候场景需要合并或压缩

出现以下任两项，建议压缩场景：

- 两场承担同一种职责
- 两场都在解释，没有新动作
- 两场之间只是换地点，没有新后果
- 一场删掉后，章级回报与钩子不受影响

## 六、正文落稿前的最小串章问答

1. 哪一场最先让读者入戏？
2. 哪一场最明确给出本章中段回报？
3. 哪一场负责把局面推坏或推深？
4. 最后一场靠什么让读者翻页？
5. 本章卖点在哪两处以上被重新感知？

## 六点五、拼章检查卡接口（新增）

若以上五问里有两问及以上答不清，默认不要继续硬拼正文，而是继续读取 `拼章检查卡_开头中段章末与钩子链复核.md`，至少补做：

- 章首有效性复检
- 中段值回票价复检
- 章末翻页驱动复检
- 钩子链响应复检
- 章级弧线复检

## 七、与作者有话说 / 后记的边界

- 场景没有承担完的回报，不能指望 `## 作者有话说` 来补。
- 正文里没显影的东西，后记最多登记，不负责救正文。
- 所以串章分配必须在落稿前完成，而不是写完之后靠附属构件擦屁股。

## 八、一句话收口

**场景单元解决“每一场怎么成立”，串章卡解决“这些场怎么合起来像一章，而且是一章值得追的章”。**
