# 三阶段骨架扩写拼合_体裁与扩字补编

> 吸收外部 `writer-skill`、`chapter-guide`、`content-expansion`、`cultural-symbols` 与 `genre-standards` 的共性工法，用于把“直接落正文”从一次性闷写，升级成可复检、可扩写、可拼合的稳定流程。

## 总原则

- 正文创作默认拆成：**骨架稿 → 分场景扩写 → 拼合检查**。
- 若本章结构简单、场景单一，可以把三阶段压缩执行；但凡涉及多场景、承重 beat、复杂关系翻面、长链悬念或卷级功能位，默认不得跳过骨架阶段。
- 外部技能中建议的章级字数为 `3000–5000`；落到本仓库时，仍以本仓库门禁为准：正文优先满足 `BodyCJK 3500–6500`。

## 第一阶段：骨架稿

### 输入最小包

- 本章控制卡 / 章节规划
- 相关人物传记当前状态字段
- 上一章结尾的实际出口状态
- 本章涉及的关键设定、规则、证据、关系约束
- 若规划中已拆 beat，则先按 beat 序列组织骨架

### 骨架稿只写什么

每个场景优先只写：

- 核心动作：谁做了什么，顺序如何
- 推动情节的关键台词：只留承重台词，不展开文气
- 场景进入点与离开点
- 承重 beat 的明确对应动作 / 台词
- 下一场的触发条件

### 骨架稿绝对不写什么

- 感官描写
- 环境铺陈
- 大段心理分析
- 比喻、抒情、修辞花活
- 规划语言残留，如“这里将展示 / 本场景旨在 / 接下来读者会看到”

### 骨架稿强制自检

每个场景至少检查：

- 进入状态是否与规划一致
- 离开状态是否与规划一致
- 承重 beat 是否全部在场
- 承重 beat 顺序是否保持
- 人物行为是否有动机支撑
- 与下一场的因果 / 对比 / 并行 / 递进关系是否成立
- 是否残留大纲语言或说明腔

若骨架稿已经像“详细说明版提纲”，说明骨架写厚了；回退，继续压干。

## 第二阶段：分场景扩写

### 扩写前按需继续读取

- 对话密集：读取对话冲突与对话写法相关裁判源
- 感官 / 氛围 / 场景首次亮相：读取描写与场景相关裁判源
- 涉及钩子、悬念、回收：读取章首 / 章末 / 线索伏笔类裁判源
- 体裁差异明显：读取体裁标准
- 存在文化 / 神话 / 历史母题：读取文化符号补充规则
- 去模板腔 / 去 AI 味：读取去 AI 味规则
- 字数不足：优先读取内容充实规则，不靠空解释补字

### 扩写时应补进去的层

在不破坏骨架顺序的前提下，逐场景补入：

- 感官描写
- 人物内心活动（短、具体、贴动作）
- 环境氛围
- 段落长短与呼吸节奏
- 情感转变的可见执行
- 关系位移、信息差、动作后果的现场感

### 场景完成确认

每个场景扩写完后至少复核：

- 实际转变是否发生
- 情感转变是否达成
- 扩写内容是否破坏骨架逻辑
- 人物是否仍与传记一致
- 是否引入新的规划语言残留

## 第三阶段：拼合检查

### 拼合时重点检查

- 场景衔接是否自然
- 本章开头前 `20%` 是否有效切入
- 章末钩子是否已成立
- 本章是否回应了上一章出口钩
- 本章整体情感弧是否达成
- 体裁风格是否匹配当前项目要求
- 是否仍残留骨架语言 / 教学口吻 / 总结腔

### 拼合后若字数不足，扩写优先级

优先从高到低：

1. 展开被跳过的关键过程
2. 加厚关键对话层次
3. 深入关键决策或情感转折前后的内心波动
4. 放慢决定性瞬间
5. 激活与当前情绪最相关的 1–2 个感官维度
6. 编织与主线直接相关的次要线索或配角反应
7. 补充环境烘托

禁止反着来：

- 禁止先堆背景说明
- 禁止先灌世界观讲解
- 禁止用无功能闲聊或均匀句群补字

## 文化符号吸收口径

若本章涉及文化 / 神话 / 民俗符号，默认区分三层：

- **原始记载**：符号的根
- **后世演变**：符号的流变版本
- **合理推演**：在不冒充考据的前提下，为当前故事做新生改写

执行时优先把符号落在：

- 物件
- 场景
- 结构原型
- 角色行为

而不是让角色或叙述者开口解释“这象征着什么”。

## 体裁标准吸收口径

### 武侠

- 动作清晰度优先于招式堆砌
- 侠义通过选择体现，不靠长篇说教
- 句式可略带文白夹杂，但不要生硬掉书袋

### 仙侠

- 体系一致性与代价先于华丽名词
- 突破、天劫、感悟都要有具体代价与感官表现
- “天道 / 因果 / 道心”必须能落到规则和后果

### 悬疑

- 信息控制与误导公平性优先
- 每章只扛 `1–2` 个核心线索更稳
- 异常细节、光影、时间差、不协调感要服务拼图满足感

### 言情

- 关系递进必须可信
- 潜台词、微表情、空间距离比直白表白更重要
- 情绪推动以细节和错位为主，不要空喊深情

### 科幻

- 概念清晰，但不写成 info-dump
- 技术必须服务人性 stakes
- 未来感靠具体日常界面与社会细节，不靠名词暴晒

## 红线

- 不能把骨架稿直接当成正文交付
- 不能在扩写阶段随意改掉承重 beat 的功能顺序
- 不能把“扩字”做成“加废话”
- 不能让文化符号、体裁标签或世界观解释压过人物处境与场景冲突
- 不能在拼合后跳过“开头有效性 / 章末钩子 / 情感弧 / 上章响应 / 正文字数”这五项复检
