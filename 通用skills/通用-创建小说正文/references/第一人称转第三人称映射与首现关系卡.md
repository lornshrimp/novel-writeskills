# 第一人称转第三人称映射与首现关系卡

> 来源口径：`经验总结.md`（第一人称转第三人称的系统化）。

## 适用场景

- 把知乎体 / 口述体初稿（第一人称）转为小说体（第三人称有限）。
- 做旧稿重写时，保信息不失真地完成叙事视角改造。

## 基础映射表（示例）

| 原文指代 | 转换后（建议） |
| --- | --- |
| 我 | 主角名 / 他（按句位选择） |
| 我老婆 | 人名（首次）/ 她（后续） |
| 我女儿 | 人名（首次）/ 他女儿（次首）/ 她（后续） |
| 我爷爷 | 人名（首次）/ 他爷爷（后续） |
| 我二叔 | 二叔（首现加关系） |
| 我发小阿彪 | 他的发小阿彪（首现）/ 阿彪（后续） |
| 括号吐槽 | 融入叙述或删除 |

## 首次出现硬规则

- 人物首次出现必须“**关系 + 名称**”同框一次。
- 第二次出现可简化为“名称”或“代词”，但不得造成代词歧义。
- 多人物同场时，优先用名字锚定，再用代词跟进。

## 转换流程（四步）

1. 先做“人称替换”，不改情节顺序。
2. 再做“关系首现补全”，补足首次关系锚。
3. 再做“口语吐槽拆洗”，把括号吐槽改为动作/情绪反应。
4. 最后做“代词歧义复检”，确保每个“他/她”都可回指。

## 常见失误与修复

- **失误**：全章“他”泛滥，读者不知道是谁。  
  **修复**：关键段落回补名字锚点。
- **失误**：把“我觉得”硬改成“他觉得”，仍是解释腔。  
  **修复**：改写为可见动作或体感。
- **失误**：把口语吐槽原样保留。  
  **修复**：拆成对话冲突、动作反应或场景反常。

## 完成判定

- 视角稳定为第三人称有限。
- 人物首现关系清楚，后续指代不混乱。
- 信息零损失，且不残留明显自述腔。
