# 章节附属构件与后记规范

## 作者有话说（读者向小剧场）

### 目的

这是给读者看的“小剧场”，**不是章节点评或创作总结**。只做三件事：

1. 用一两个具体的小瞬间，跟读者交换情绪。
2. 提一个贴着剧情的小问题，让读者自然想象“如果是我会怎么选”。
3. 轻轻勾一下后面的不确定性，但不说明“后面会发生什么”。

### 字数与位置

- 中文平台：200–300 字
- 英文平台：100–150 words
- 不计入正文字数
- 置于正文与 `## 章节后记` 之间
- 使用独立二级标题：`## 作者有话说`

### 建议写法（遵循，而不是范文）

- 视角：尽量用“你”来对话读者，而不是“我来分析这一章”。
- 切入口：抓一个**很小的细节**开头（比如某一句系统话术、某个权限弹窗跳出来又被关闭、某条会议邀请被反复撤回重发），问读者“你有没有类似的感觉 / 时刻”。
- 共鸣：允许用一句现实体验做对照（排队、加班、亲人就医、被系统拒绝等），但要落在“心情”和“选择”上，不讲政策，不讲结构。
- 互动：结尾用 1 个简短问题或二选一，邀请读者留言，比如“如果是你，会假装没看到，还是顺着那一下，把线拽出来看看？”

### 明确禁止（本节高频问题，必须避免）

- 把“作者有话说”写成章节书评或结构拆解：
  - 禁止使用“这一章是……的第一个收束镜头”“本章主要完成了……功能点”一类创作术语。
  - 禁止概括整章结构和所有线索；那是给作者自己的后记和大纲用的，不是给读者用的。
- 自夸式总结：
  - 避免“我很在意这几句”“这一章我特别满意哪一段”这类自我评价。
  - 如果要提创作意图，只能用一句很轻的“这一段写的时候一直在想某个画面”，点到为止。
- 章节自指与连载导览腔：
  - 禁止出现“这几章”“前几章”（含“前面几章”“前面这几章”）“后几章”（含“后面几章”“后面这几章”）“整章”（含“这整章”“这一整章”）“上一章”“下一章”“本章 / 这章主要”这类把读者拉回作者工作台的表达。
  - 这类话术只允许留在 `## 章节后记`、控制卡、日志、审阅报告等作者侧区块，不得混入正文或 `## 作者有话说`。
- 直白预告后文：
  - 禁止出现“下一章我们会看到……”“后面某人会如何如何”这类明确剧透。
  - 只能用情绪和风险感来暗示，例如“接下来，规则会越来越实心”“有人要被迫做选择”，不点剧情细节。
- 设定讲解与参数堆砌：
  - 不在这里解释世界观设定、制度细节、技术原理或系统运作，这些要在正文自己长出来。
  - 不罗列 ID、时间轴、机制名称或作者后台编号；读者只需要感觉，不需要看到设定文档。

### 悬念与思想表达

- 不直接揭示本章或系列的暗线与思想结论，保持悬念与思考空间。
- 可以用“如果有一天规则逼你在 A 和 B 之间选，你会怎么选？”这类问题，把读者轻轻推向主题边缘，但不要自己给出答案。
- 真正的“观点”和“总结”，留在 `## 章节后记` 中以作者工作笔记形式记录，而不是在“作者有话说”里讲道理。

### 强提醒（硬性约束）

- 任何关于“所有制”“阶级”“分配权”等核心思想与暗线，不得在“作者有话说”中直接明示或下结论；仅可通过问题和生活小场景，引导读者自行联想和讨论。
- 这里只允许出现**一丢丢创作花絮 + 一点点共鸣 + 一个小问题**，不要扩展成小论文或大纲复述。
- 若项目已有单独的“作者有话说”精修链路，精修阶段可再调用 `通用-润色作者有话说`。

## 章引语（Chapter Epigraph）规范

### 定义

每一章正文开头、章标题下方的一段或两段短引语，用于在进入情节前，先用一句“制度话术 / 文档摘句 / 人物只言片语”预埋本章的母题与情绪基调。

### 结构与格式

- 独立引用块，置于章标题正下方。
- 建议采用 1 行主引语 + 1 行来源说明。
- 主引语控制在 1–2 句内，来源说明控制在 1 行内，整体不超过 3 行引用。

示例：

- `> “如果你没看见，就等于它没发生。”`
- `> —— 《物业客服应答话术（培训版·节选）》`

### 来源类型优先级

1. 伪官方 / 机构文本：制度手册、培训材料、公告与标语等。
2. 制度话术：合同条款、回执小字、界面提示语、宣传口号中的完整句子。
3. 人物只言片语：重要人物曾说过但不在本章直接出现的台词。
4. 社会俗语：被行业 / 平台 / 社区语境改写后的“新鸡汤”“新格言”。

### 内容功能要求

- 与本章分卷大纲中的**核心母题 / 冲突**强绑定。
- 与正文至少一个关键镜头或证据要素形成前后照应，避免纯抒情或空泛哲思。
- 可以形成反差：章引语语气轻飘、冷静或粉饰太平，正文用具体人的疼痛与损失去“反证”它。
- 禁止直接剧透本章关键情节或结局，不点名具体事件，只提示规则 / 话术 / 价值观。

### 写作操作规范

- 每一章**必须**包含一则章引语。
- 创作前，在输入参数或控制卡里先粗选本章拟引用的制度话术或伪文档来源。
- 如章引语引用的文本在设定文件或既有章节中有原文出处，应尽量保持原句或轻微改写，并在来源说明中写明出处。
- 如为新造伪文本，须符合对应机构 / 系统的语气与口径，并在 `## 章节后记` 的【证据要素】或【设定 / 程序呈现】中注明其拟归属的文件 / 文类，便于后续沉淀到设定文档。
- 严禁在章引语中直接给出作者立场的评价句，应保持“制度自说自话”的口吻，把评判权交给正文与读者。

## 章节后记规范

### 位置与标题

正文结尾后，先添加独立二级标题 `## 作者有话说`（中文平台 200–300 字；英文平台 100–150 words），随后添加 `## 章节后记`。

### 内容结构（执行时按以下模板完整填写）

```markdown
## 章节后记
【本章定位】承接XX章 / 推进XX支线 / 引爆XX冲突（卷 / 部中功能）
【功能点落地核对】（强制）
- 功能点1：[内容] → 落地镜头：第X段“[原文片段]”
- 功能点2：[内容] → 落地镜头：第X段“[原文片段]”
【冲突与悬念】显性冲突：XX；隐性信息缺口：XX
【爽点登记】（强制｜工程化验收）
- 爽点主类型：权力 / 资源碾压｜智力碾压｜身份反转｜复仇 / 逆袭｜知识装X｜科技 / 工具碾压（选1）
- 爽点辅类型：（可留空或选1）
- 微爽点（1–2条，写成事件）：1.XX；2.XX
- 回报形式（可复选）：信息｜反击｜反转｜程序推进｜智性满足
- 证据载体（至少1条）：界面 / 日志 / 编号 / 回执 / 口供 / 物证 → “[具体字段 / ID / 时间戳]”
- 代价 / 后果（至少1条）：关系 / 职业 / 名誉 / 法律风险 / 心理 → XX
【伏笔与回收】（强制）
- 新埋伏笔F-XX：[内容] → 预计回收卷 / 章：XX
- 回收伏笔F-XX：[内容] → 落地镜头：第X段
【设定/程序呈现】（强制）
- 设定1：[名称] → 体验化镜头：第X段“[原文片段]”（避免说明书式讲解）
【人物发展】（强制）
- 角色A：成长弧阶段从[前]→[后] / 代价：[具体内容]
- 标签落地：[外貌 / 动作 / 口头禅] → 第X段“[原文片段]”
【证据要素】（强制）
- 证据1：[类型，如受理回执 / 工单编号 / 绩效或考勤记录 / 鉴定结论] → 内容：“[具体字段 / ID / 时间戳]” → 叙事功能：XX
- 证据2：[类型，如通话记录 / 门禁记录 / 病历单] → 内容：“[具体片段]” → 叙事功能：XX
【钩子类型】（强制）剧情转折 / 秘密揭露 / 新危机（可复选）｜钩子原文：“[具体内容]”
【数字纪律检查】无违规精确值 / 违规项：[内容]｜例外理由：XX
【偏离记录】（强制）无偏离 / 偏离项：[人物 / 设定偏离内容]｜待修正：是 / 否
【后续铺垫】下一章需承接：1.XX；2.XX
【读者吸引点】2–3个核心驱动力（含一句话卖点 / 差异化点）：1.XX；2.XX
【竞对对照】（强制｜章节优化落地）
- 本章主对标竞对章节（1–2章）：[作品名-章号]
- 可借鉴点（至少1条）：[竞对章节强点] → 我方拟落地位置：[第X段/第X场]
- 当前落后点（至少1条）：我方在[维度]弱于[竞对章节]，表现为：[具体描述]
- 下一章优先优化动作（至少1条）：[动作]｜目标提升：[章首抓力/中段回报/章末钩子/段评触发]
【优化建议】后续迭代需强化：情感细节 / 节奏配比 / 象征运用
```

### 使用提醒

- 证据、回报、钩子、偏离与后续铺垫，必须与正文内容一一对应，不可后记里凭空补剧情。
- 若某个功能点、证据、人物推进或钩子在正文里并未真实落地，不得以后记硬补；应如实标记为偏离、遗漏或待反查。
- 后记是工作记录，不是再写一遍剧情摘要。
- 竞对字段必须写“可借鉴点 + 落后点 + 下一章动作”三件事，不能只写“已参考竞对”。
- 若精确日期 / 时间 / 编号暂无裁判源支撑，只能写最小合理口径，并在对应字段明确标记“待反查”。
- 若题材层对年份、编号、口径有更严格限制，应在后记的对应字段中同步复检。
