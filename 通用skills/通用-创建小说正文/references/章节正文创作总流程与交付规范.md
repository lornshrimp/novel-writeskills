# 章节正文创作总流程与交付规范

## 目标定位（读者向文本｜通用默认）

- 最终产物是：**给读者阅读的章节正文**，以及读者可见的 `## 作者有话说`。
- 首要评价轴是：追读动机、章内回报、沉浸可读性、章末钩子强度。
- 一切设定、程序、证据、机制与信息细节，都必须先服务“入戏、可复盘、不给说明书感”。
- 但同时：正文必须严格受作者侧裁判源约束（人物档案 / 大纲 / 设定文档 / 时间轴 / 台账）；一旦冲突，以裁判源为准，正文必须改。

## 使用范围

适用于长篇连载任一章节正文 + `## 作者有话说` + `## 章节后记` 的创作全流程。

默认正文目录为 `小说正文/`。若用户未明确指定其它目录或其它平台版本，则本 Skill 生成 / 覆盖的章节文件默认应视为 `小说正文/` 下的源 Markdown 正文，而不是某个平台派生稿。

若题材层已定义更细的默认平台、标题风格、时间口径、编号口径或世界观限制，继续服从题材层。

## 总体目标

将分卷大纲的功能点 / 冲突 / 伏笔 / 关键节点，转化为**正文 CJK 3500–6500（含边界，不含“作者有话说”与后记）**的完整章节文本，并同时生成：

- `## 作者有话说`
- 标准化 `## 章节后记`

## 成功判据（通用层）

1. 正文 CJK 字符数 3500–6500（含边界，不含“作者有话说”与后记）。
2. 至少 1 个章末钩子落地并驱动下一章。
3. 分卷大纲列示的“功能点 / 冲突 / 伏笔 / 关键节点”全部镜头化（正文内呈现，非后记说明）。
4. 至少 2 条可审计“证据要素”自然嵌入剧情（界面 / 日志 / 编号 / 回执等）。
5. 至少 1 位主要人物成长弧推进 1 个阶段或兑现代价，且人物标签落地。
6. `## 章节后记` 按规范完整登记全部强制字段。
7. 含 `## 作者有话说`，且置于正文与后记之间，不计入正文字数。
8. 人物传记一致性零偏差：正文中出现的所有人物，其身份 / 阵营 / 职业 / 能力边界 / 社会关系 / 时间线与同名人物传记一致。
9. 章节标题必须与正文开头展示的标题一致；是否需要平台化升级，服从题材层或项目层规则。
10. 正文中段必须至少完成 1 次可被读者明确感知的回报：新证据落袋、误导被证伪、规则边界被试出、人物关系实质变化、一次有效反击成立，其一即可。
11. 心理活动尽量采用近距离限知与短促表达；连续内心独白原则上不超过 3 行，优先用动作、呼吸、停顿、错误操作、微小感官刺点替代抽象抒情。
12. 若本章位于卷内前 3 章或承担新链路启动职责，须尽量满足：前 300 字冲突显形、前 1000 字异常机制或关键规则显形、前 3000 字完成首轮回报。

## 起点默认开篇节拍补充

- 前 300 字：尽快把主角当前困境与异常信号咬在一起。
- 前 500 字：必须让读者知道“继续看下去会得到什么”。
- 前 1000 字：主角身份、欲望、困境、异常机制四项中至少清楚两项，且异常机制不宜继续隐身。
- 前 3000 字：至少完成 1 次具体回报，形式可为首次破局、规则试错成功、证据落袋、误导被推翻或小规模反击成立。

## 现实职业与信息差补充

- 若章节天然带职业性、程序链或行规，优先把专业操作写成可读者感知的压制感，而不是科普段。
- 别人看见表象，主角看懂规则，是正文抓力的重要来源；但主角必须通过动作和试错兑现，而不是空口判定。
- 职业专业感越强，越要同步落代价、漏洞和露馅风险，避免写成无代价万能解题器。

## 创作前必做校验

### 必读文件顺序（未读完尽量不要动笔）

1. 总大纲：作品总纲、全书基线与世界规则。
2. 分卷 / 章清单：本章所在卷的分卷大纲或章清单文件。
3. 调研与现实素材：与本章案件、地点、职业、环境、情绪纹理相关的材料。
4. 人物档案：本章实际出场人物的传记、人物设定、关系网、口头禅与阶段状态。
5. 设定与程序：与本章地点、职业、程序、规则边界、关键物件直接相关的设定条目。
6. 编号、术语与时间线规范：对照总大纲 / 分卷时间轴、术语表、编号口径进行核对。
7. 样章标杆（如存在）：项目内节奏、钩子、追读表现更稳定的章节，只学技法，不复刻语言。

### 阅读完成后，开写前必须归档的引用资源清单

【引用资源清单（开写前）】

- 分卷映射：功能点 / 冲突 / 伏笔 / 关键节点 / 钩子目标（对应到具体编号或页内锚点）
- 关联人物档案：文件名与段落编号
- 人物传记一致性核对：逐人列出“本章用到的关键约束”（关系 / 归属 / 能力边界 / 阶段 / 口头禅等）与对应传记段落
- 关联设定与程序：文件名与口径
- 证据要素来源：界面 / 日志 / 编号 / 回执的出处、时间戳与 ID 编码规则
- 口径交叉校验：界面用语 / 术语 / 时间线已对齐（勾选）
- 待反查项：缺失信息的最小合理假设与定位路径

信息缺失时，仅可做最小合理假设，并在后记中标记“待反查”。

## 输出结构（刚性要求）

1. 正文（正文 CJK 3500–6500，文学性优先；正文开头必须展示本章章节标题，可带可选副标题）
2. 章末钩子（≥1 个，类型可为剧情转折 / 秘密揭露 / 新危机）
3. `## 作者有话说`（置于正文与后记之间；中文平台 200–300 字，英文平台 100–150 words）
4. `## 章节后记`（按规范填写，含强制字段）
5. 证据要素（≥2 条，嵌入正文，不可只放后记）
6. 反向引用表（若需结构化登记，只能放在后记）

## 正文隐式骨架与输出约束（仅供内化，不得外显）

### 写作前“深入思考”（强制｜不外显）

在开始输出正文前，必须先在内心完成一轮深入推演：

1. 本章对读者的承诺：本章要满足的“爽点 / 智性满足 / 情绪回报 / 反转”是什么，并能用一句话说清“本章卖点 / 差异化点”。
   - 爽点工程化补充：明确“爽点主类型×1 + 辅类型×0–1”；写出本章计划投放的 1–2 个微爽点（写成**事件**），并说明每个微爽点的**回报形式**与**代价 / 后果**；至少 1 个微爽点应绑定可复盘的“证据载体”（界面 / 编号 / 回执 / 口供 / 物证）。
2. 节拍：入场（开头 2–3 段钩子）→ 推进（中段至少 1 次实质进展 / 回报）→ 变局（一次升级 / 代价 / 反转）→ 章末强钩子。
3. 互动点：至少 1 处可触发“本章说 / 段评”的句子或选择（道德困境 / 规则漏洞 / 反常细节 / 一句制度话术）。
4. 设定与人物一致性：本章新增信息是否与人物传记与技术设定冲突。
5. 悬念链：本章至少推进或回收 1 条既有“信息缺口 / 疑问”，同时新增 1 条更尖锐的新缺口，用于章末钩子。

### 输出约束（硬性）

- 正文部分必须是连续的文学叙事文本。
- 在 `## 作者有话说` 之前，禁止出现方括号字段、清单项、表格、模板提示、元注释、提纲式编号段落或任何“指南口吻”的指令语。
- 暗线与思想表达不得直接外显结论或口号；应通过人物命运与事件显影，引导读者自行推理与思考。
- 现实共鸣可以有，但避免口号化；尽量用具象细节与人物处境承载情感。
- 证据要素至少 2 条，只能以剧情内的具象物与界面自然呈现。
- 精确值尽量只出现在凭证、回执、文书、界面或记录类载体中。
- 章末钩子应落在具体事件或未完成动作上，避免把“钩子”写成解释性总结。
- 反向引用与线索回收应先在叙事中完成；若需结构化登记，只能进入后记。
- 若题材层对年份、编号、世界内真实编号与作者后台编号有额外限制，继续服从题材层。

## 创作内部清单（内化执行，不外显）

在写作时于心中自检以下要素是否“已在文本中被读者自然读到”：

- 开场前 2–3 段即冲突触发与五感支点（温 / 光 / 声 / 味 / 触）
- 本章卖点 / 差异化点：可用一句话复述，并在文本中有对应“可指认的镜头 / 证据 / 对话”
- 若为分卷前几章：卷级抓人链条是否成立（开头抓人 + 中段不塌且推进有回报 + 章末钩子推进）
- 时间与地点自然露出（对话 / 界面 / 环境物件带出具体时段与微场景）
- 至少一类核心对立（人-人 / 人-制度 / 人-自我）形成稳定拉扯
- 镜头组：建立 → 推进 → 转折 / 揭示（3–6 个镜头的节奏递进，避免说明）
- 关键对话（3–8 句），身份位差通过语体 / 语气 / 沉默与动作体现
- 至少 1 处用回避 / 打断 / 省略 / 借题发挥 + 动作 / 沉默承载潜台词
- 动作因果链 ≥3 条：起因 → 动作 → 结果 → 新张力
- 中段至少一次“信息 / 情绪 / 资源 / 关系”的实质性兑现
- 至少 1 处自然可评的细节 / 选择 / 制度话术，能引发读者争论或共鸣
- 信息配比：新增情报与保留信息交错，保留空白用于悬念
- 感官优先级：需要增强代入时，优先补触觉 / 嗅觉 / 听觉，其次再上视觉总述
- 中段防塌检查：若章首已经很炸，中段就更要给“证据、回报、关系变化、规则试错结果”中的至少一项
- 反虚浮过滤器：凡出现“他很害怕 / 她很绝望 / 气氛很压抑 / 这地方很怪”一类抽象判断，优先改写为身体反应、动作卡顿、界面细节、物件异常或空间变化
- 证据嵌入 ≥2 条
- 人物状态变化至少 1 类：心理 / 资源 / 关系 / 位置的前后对比
- 章末钩子：从“问题句 / 触发条件 / 新变量入场”三类中至少取一，落到具体事物与动作

## 章内抓力与代入感补强细则（默认启用｜不外显）

- **前 300 字规则**：优先使用“日常场景 + 微反常 + 角色情绪代价”组合切入；反常越小越好，但必须具体，且能在后文进入证据链或规则链。
- **小钩子密度**：每 500–800 字尽量安排一个小钩子，形式可为一句不对劲的话、一个物件位置变化、一次动作失误、一个时间点对不上、一个人物短暂露馅。
- **现实生活针脚**：不断缝入让读者觉得“这事就发生在我楼下 / 我手机里 / 我单位里”的生活针脚。
- **认知升级优先**：比起单纯把能力写得更猛，更优先让主角“看懂一层、试出一层、利用一层规则”。
- **章中小释放**：哪怕是压抑章，中段也必须给一个小释放：答案碎片、局部反杀、确认一件事、抓住一个把柄，其一即可。

## 执行五步流（创作全流程闭环）

1. 对齐输入：核对分卷大纲任务，确保功能点等全量列出。
2. 镜头拆解：为每个功能点设计至少 1 个“可拍镜头”。
3. 分段写作：快 / 中 / 慢节奏交错，先人物体验再技术显影，避免技术文档化。
4. 钩子与伏笔：结尾落地钩子，推进或回收前置伏笔。
5. 后记登记：逐项填写强制字段，标记“待反查”与“偏离记录”。

## 文件写入硬性要求（对执行者的附加指令）

- 本工作流用于在 VS Code 工作区内由 AI 助手直接**生成并写入章节文件**，而不是只在对话中展示结果。
- 当用户在编辑器中定位到某个目标章节文件时，应识别该文件作为当前目标章节文件。
- 应将生成的**章节正文 + 章末钩子 + `## 作者有话说` + `## 章节后记`** 按本规范直接写入该文件。
- 如文件原本已有草稿内容，由用户指令决定是覆盖重写还是在末尾追加；如用户未说明，保持保守策略：在原文后追加新版本，并在末尾用 `## Changelog` 简要说明本次生成。
- 如用户明确指定“请为第 X 章新建文件”，应在对应卷目录下按命名规范创建新文件，并写入完整内容。
- 生成完成后，如无特殊指令，不要在聊天里重复完整章节正文，只需简要说明：
  - 写入了哪个文件（相对路径 + 文件名）
  - 是否覆盖 / 追加
  - 是否已包含 `## 作者有话说` 与 `## 章节后记` 及必需字段

## 正面创作原则（必须遵守）

1. 文学性优先：高于技术说明与理念宣讲，避免沦为设定说明书。
2. 程序可信：职业 / 程序 / 证据链有逻辑链，明确限制、成本与副作用。
3. 意象统一：环境 / 道具符合卷内常态卡，象征体系前后呼应。
4. 主题潜在化：通过人物遭遇显影，不用生硬口号直讲结论。
5. 人物一致性：年龄、动机、语言习惯完全对齐人物传记。
6. 成长弧规范：按“初始缺陷 → 触发事件 → 认知转变 → 代价”推进。
7. 公平推理底线：关键线索必须在叙事中出现，误导必须可证伪。
8. 中文规范：职业与机构称谓清晰一致。
9. 代入优先表达：少用抽象判断，优先把情绪、风险与压迫落到身体、物件、光线、声音、气味、界面与手续摩擦上。
10. 真实日常承压：任何能力显影、规则压迫或怪异降临，都尽量借助日常载体发生，而不是凭空降临。
11. 中段必须交付：本章中段至少要让读者拿到一次明确收益。

## 负面禁忌清单（绝对禁止）

### 技术文档化五陷阱

- 过度解释机制 / 流程：改为呈现“程序如何压迫人 / 如何被人利用”的人物体验。
- 平铺直叙开场：需用反常细节 / 紧急事件 / 现场异样切入。
- 冷静说明语调：加入具象比喻与人物情绪波动。
- 单一路径冲击：叠加关系破裂、资源剥夺等多重压力。
- 感官缺失：补充声 / 光 / 触 / 味 / 温等细节。

### 反 AI 味创作规范

- 禁课堂式句首：如“在本章中”“综上所述”“不难发现”。
- 禁万能抽象词：如“显著”“协同”“赋能”“范式”“闭环”。
- 句长交错：短句 + 复合句搭配，避免均匀句式。
- 具象支撑：每 1 个抽象判断配 ≥3 个可拍镜头细节（动作 / 微表情 / 物件）。
- 数字纪律：叙事用近似词，精确值优先放在凭证 / 回执 / 记录类载体中。
- 若题材层对时间、年份、编号外显另有禁令，继续服从题材层。
- 系统声部：冷静克制，仅给最小信息，解释性内容移至后记。

### 其他刚性禁忌

- 证据要素不得后置：必须嵌入正文场景，不可只用后记补充。
- 分卷任务不遗漏：功能点 / 冲突等必须镜头化落地，未落地需在后记标记。
- 钩子类型不连续：同一类型钩子连续不超过 3 章。
