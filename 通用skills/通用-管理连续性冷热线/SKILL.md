# 通用-管理连续性冷热线

连续性控盘不是后台表格管理，而是为了保证读者的追读体验不断电。

## 保真迁移要求（强制）

命中连续性控盘任务时，除本文件外，**必须继续读取**：

- `references/连续性与冷热线保真规则.md`
- `references/动态状态模板.md`
- `references/冷线复查与回归决策表.md`
- `references/合理休眠说明模板.md`
- `references/章节链路热度与回写层级表.md`
- `references/归档更新_决策日志_进度追踪补编.md`

## 常见触发词 / 用户说法速查

- 哪些线快忘了
- 谁该回来了
- 这几章有没有失温
- 伏笔是不是超期了
- 关系桥接断了没有
- 写完这章后更新动态状态
- 哪些旧债必须碰一下

## 何时使用

- 写章前，需要判断哪些人物 / 关系 / 主支线 / 伏笔应该回归。
- 写章后，需要把最新变化写回动态状态，避免后续忘线。
- 连续 3–10 章后，需要检查剧情有没有中段失温、角色蒸发、伏笔断档。

## 本 Skill 的核心任务

1. 识别**热线、温线、冷线、归档线**。
2. 判断哪些冷是正常休眠，哪些是危险失温。
3. 检查人物、关系、伏笔、规则边界、未回收问题是否越过应回归窗口。
4. 在写后更新动态状态，而不是只靠记忆。

## 执行顺序

固定优先级：先扫描，再回写，最后处理失温决策。

1. 先核对输入的卷、章节或动态状态是否有效；若无效，先提示重新输入有效值，并停在参数校正。
2. 再读取最近章节、相关卷纲与线索 / 伏笔 / 关系台账。
3. 再判断当前哪些元素是热、温、冷、归档。
4. 再识别需要本章 / 近期回归的对象，并说明原因。
5. 再识别允许休眠的对象。
6. 再为允许休眠的对象写明合理休眠说明。
7. 然后扫描主卖点显影链、中段回报链、章末接棒链、章首抓眼链等章节链路对象的热度。
8. 写后先只按模板更新动态状态。
9. 再只更新回归压力。
10. 再只更新未回收钩子与情绪债务。
11. 最后只更新章节链路热度。
12. 若某条线已经连续多章失温，再按决策表决定是推进、提醒、转移、休眠还是归档，并明确优先回写层级。

## 归档更新、决策日志与进度快照补强（新增）

- 写后更新不只改“哪条线升温了”；默认还要同步判断人物档案哪些动态字段变了：位置、情绪、已知信息、持有物、身体状态，以及是否需要追加章节行为记录。
- 人物关系只在发生**实质变化**时更新；不要为了显得勤快，每章都把关系表重新搅一遍。
- 若本章新引入地点、规则、势力、时间节点，或改变旧设定状态，默认同步刷新对应世界观条目。
- 情节记录层默认同步更新：章节摘要、TODO 状态、悬念 / 伏笔状态、超期提醒与回收余波。
- 章节定稿若产生结构性偏移、节奏权重调整或钩子方向改动，默认追加决策记录；若只是微调措辞，不必强记成结构决策。
- 连续性扫描结束时，默认给出一个最小进度快照：当前章节、当前步骤、是否被阻塞、最近完成章节、待处理问题、下步 `1–3` 个动作。
- 一切归档默认只记已发生事实，不预测未来章节；若定稿章节与旧资料冲突，以定稿章节为准，并在日志中写明修正原因。

## 默认输出口径

- 若用户未指定扫描对象，默认逐项扫描人物、关系、主支线、伏笔、旧债和未兑现情绪债务这 6 类对象；若用户已指定范围，则不额外扩张扫描对象。
- 默认不仅判断“提没提到”，而是判断是否发生了**有效触碰**。
- 默认在需要时输出合理休眠说明，而不是只说“这条线先放着”。

## 最低交付

- 明确指出哪些线是热线、温线、冷线、归档线。
- 明确指出本章 / 下章必须回来的对象。
- 明确指出哪些对象可以继续休眠，以及为什么。
- 写后能回答：推进了哪条线、回收了哪条线、升温 / 降温 / 转移了哪条线。

## 与其他 Skill / Prompt 的边界

- 本 Skill 不代替章节正文审阅，只负责“有没有忘”和“该不该回”。
- 本 Skill 不直接生成控制卡，但通常是 `通用-生成章节控制卡` 的上游检查器与下游写回器。
- 本 Skill 不负责重写总纲，但会指出哪条线已经冷到需要回到上游修纲。
