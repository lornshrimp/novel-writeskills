# 归档更新_决策日志_进度追踪补编

> 吸收外部 `archive-maintenance`、`decision-log-template` 与 `progress-tracker-template` 的归档共性规则，把连续性管理从“记得哪些线热”补强为“章节定稿后如何更新事实层、决策层与进度层”。

## 一、基本原则

- 只记录**已经发生的事实**，不预测未来章节。
- 只更新**有变化的字段**，不无意义地全表重抄。
- 若定稿章节与旧资料冲突，以**定稿章节**为准修正事实层，并在日志中写明原因。
- 稳定字段与动态字段分开维护，避免“一章一改性格底色”。

## 一点五、三层归档映射（新增）

- `事实更新层`：人物动态状态、关系变化、世界观条目、悬念 / 伏笔状态。
- `决策记录层`：结构决策、节奏偏离、钩子类型选择、偏离原规划的原因。
- `进度追踪层`：当前章节、当前步骤、阻塞状态、最近完成章节、下步动作。

默认顺序：先更 `事实更新层`，再补 `决策记录层`，最后刷 `进度追踪层`；不要只写“在推进”，却不更新已经发生的事实。

## 二、人物档案更新

### 稳定字段（通常不随单章改）

- 基本信息
- 性格核心
- 语言指纹
- 背景
- 角色弧线主轴

### 动态字段（章节定稿后更新）

- 位置
- 情绪状态
- 已知信息
- 持有物品
- 身体状态
- 最后更新章节编号

### 章节行为记录

- 采用**追加**，不回改旧记录
- 每章一句话
- 只写关键行为，不写文学感想

### 升格规则

- 配角戏份明显上升时，可从精简模板升级到完整模板
- 升级时补齐缺失字段，不要抹掉既有动态记录

## 三、人物关系总览更新

关系表只在**发生实质变化**时更新：

- 关系类型
- 当前状态
- 关键事件（追加章节编号）
- 最后更新

禁止写成“复杂”“很微妙”这类模糊话，尽量落到具体：

- 表面合作但互不信任
- 已结盟，但利益冲突开始显影
- 关系缓和，但旧债未清

## 四、世界观文件更新

根据章节定稿结果，按需更新：

- `地理 / 地点`
- `规则 / 设定`
- `组织 / 势力`
- `时间线`

### 只在这些情况更新

- 新地点首次真正进入故事
- 新规则被建立或揭示
- 新组织 / 势力正式上场
- 已有条目发生了状态改变

不要为未来可能会出现的内容提前建空条。

## 五、情节记录更新

章节定稿后至少补：

- 大纲中的 TODO 状态
- 本章章节摘要（`2–3` 句话）
- 悬念追踪：新增 / 部分回应 / 已回收 / 超期
- 伏笔追踪：埋设 / 回收 / 余波

### 悬念 / 伏笔状态建议

- 活跃
- 部分回应
- 已回收
- 超期
- 转移到其他线

## 六、决策日志

在以下情况追加决策日志：

- 场景拆分 / 合并的关键结构决策
- 节奏权重偏离常规
- 钩子类型选择有特别理由
- 章节定稿偏离原规划但有正当理由
- 新增了原计划外但必须记住的结构性安排

### 每条决策最小字段

- 日期
- 状态：`Active / Superseded / Under Review`
- 影响范围：全书 / 卷 / 章 / 场景
- 决策内容
- 理由
- 备选方案及弃用原因
- 置信度
- 依赖条件
- 触发重新审视的条件
- 关联决策

已失效决策保留，不物理删除。

## 七、进度跟踪器

连续性管理不仅是“谁该回来了”，还应知道项目当前走到哪里。

### 快速状态字段

- 章节范围
- 当前状态：`已完成 / 规划中 / 写作中 / 审查中 / 润色中 / 未开始`
- 当前步骤

### 进行中详情

- 当前处理章节
- 当前步骤
- 是否被阻塞
- 最近完成章节
- 待处理问题

### 决策暂存

- 本会话做出的关键决策摘要
- 理由
- 置信度

### 下步计划

- 下次会话的 `1–3` 条清晰动作

### 会话历史

- 日期
- 完成工作
- 遗留问题

## 七点五、最小归档工件（新增）

```markdown
【事实更新层】本轮变了什么
【决策记录层】本轮做了什么结构决定
【进度追踪层】当前章节 / 当前步骤 / 是否阻塞 / 下步动作
【当前判定】Green / Yellow / Red
```

## 八、热 / 温 / 冷 / 归档 与事实更新的关系

- **热线**：近期必须继续推进；若已写后，优先更新动态状态与悬念状态
- **温线**：可短暂休眠，但必须在进度或回归压力中留痕
- **冷线**：允许阶段性休眠，但要写明合理休眠说明
- **归档线**：已完成或已无继续供血需求，转入历史记录，不再强行回温

## 九、一致性检查清单

连续性层复核至少要问：

- 人物当前状态是否与最近章节一致
- 世界规则是否被新章节改写了
- 活跃悬念是否超期无推进
- 活跃伏笔是否被遗忘
- 大纲 TODO 状态是否与实际流程一致
- 是否存在“正文已定稿，归档仍停在旧版本”的错位

## 十、落盘习惯

- 字段无内容时写 `暂无`
- 章节编号统一 `chNN`
- 人名保持一致，不用昵称替代正式名
- 描述尽量客观、短、可检索

## 红线

- 不得把未来计划写进事实层
- 不得回改旧章节行为记录
- 不得每章都重写关系表而不判断是否发生实质变化
- 不得因为“看起来没冲突”就跳过日志、摘要、悬念 / 伏笔状态更新

## Green / Yellow / Red 归档判定（新增）

- `Green`：事实已更新、关键决策已留痕、进度状态真实可续跑。
- `Yellow`：事实层已更，但决策或进度追踪仍缺一块，后续续跑会发虚。
- `Red`：只有“做了什么”的感觉，没有事实更新、没有决策留痕、没有真实进度快照。
