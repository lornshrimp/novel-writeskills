# 连续性与冷热线保真规则

## 一、看“有效触碰”，不看“提没提到”

一个人物、关系、伏笔、线索是否真正回归，要看它是否带来了：

- 直接推进
- 压力提醒
- 关系变化
- 后果显影
- 合理休眠说明

只提一句名字、不改变任何压力，不算回归。

## 二、热度口径

- **热线**：当前驱动主要冲突或读者期待。
- **温线**：暂时不是核心，但最近有实质存在。
- **冷线**：再不碰就会被读者忘记或显得作者忘了。
- **归档线**：已完成或已说明休眠。

## 三、默认预警窗口

- 核心 recurring 角色：约 3–5 章无有效触碰，应复查。
- 主要支线：约 4–6 章无实质推进，应复查。
- 临近回收窗口的伏笔：若连续错过 2 次自然回收机会，应预警。
- 剧烈变化后的重要关系：2 次出场内没有桥接，视为高风险。

## 四、有效响应类型

某个元素变冷后，可选响应只有这几种：

- 直接推进
- 间接提醒，但必须改变预期
- 压力转移到另一条线
- 在动态状态中写“合理休眠说明”
- 明确归档 / 收束

禁止：

- 象征性 cameo
- 机械提醒
- 为了打卡而硬塞一句旧线索

## 五、连续追读口径

- 每章都应完成“推进或回收一个既有问题 + 新增一个更危险的问题”。
- 若一条线长期没有承担“被回答 / 被提醒 / 被加压 / 被转移”的任一动作，读者就会觉得作者忘了。
- 若一段关系、一个重要人物、一个高压伏笔在上次剧烈变化后迟迟没有桥接，读者会把这种空档感知为失温而非留白。

## 六、章章有推进 / 章章有回报口径

- 每章至少推进一条线索、一次成长或一次关系变化。
- 每章中段必须有一次明确回报；若当前章节计划无法指出该回报对应的是哪条线，那么线热判断大概率也会失真。
- 每章结尾至少留下一个明确可追问点；这通常意味着至少有一条线被升温、转移或改写。

## 七、连续性控盘对象清单

默认需要同时检查以下对象：

- 主线谜团 / 主冲突
- 重要支线
- 单元案 / 事件余波
- 关键线索
- 关键对象
- 伏笔与呼应总纲中的回收窗口
- 时间轴中的硬约束节点
- 重要人物关系债务
- 已立起但尚未兑现的规则边界 / 代价

## 八、卷中段失温的典型症状

以下任一症状出现两次以上，通常说明卷中段已经在失温：

- 章节能概括剧情，但说不出哪条线获得了实质推进。
- 主要人物仍在出场，但关系没有新变化，像机械维持存在感。
- 伏笔还在“记得提”，但读者已经无法判断它与当前危机有什么关系。
- 新事件不断发生，却没有把旧压力转接进来。
- 章末仍有钩子，但钩子与现有冷热线没有明确挂钩。

## 九、写后更新时必须写入的字段

每次章节完成后，更新动态状态时至少记录：

- 本章真正推进了哪条线
- 本章真正回收了哪条线
- 本章让哪条线升温 / 降温 / 转移
- 哪个人物关系债务被新建或加重
- 哪个伏笔进入回收倒计时
- 哪个对象暂时休眠，以及为什么可以休眠

## 十、最终验收口径

连续性管理做到位时，读者应当感受到：

- 没有哪条重要线突然蒸发
- 没有哪次重大变化被作者假装没发生
- 旧债会回来，而且回来时会改变局势
- 新线不是平地起楼，而是从旧线裂缝里长出来
