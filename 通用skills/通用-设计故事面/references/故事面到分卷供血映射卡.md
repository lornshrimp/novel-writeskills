# 故事面到分卷供血映射卡

适用于把故事面真正翻译成分卷层可执行任务，避免“故事面很强，卷纲接不住”。

## 一、先明确：故事面不是直接拿来写章，而是先降一层到卷

故事面负责：

- 决定这本书长期在打什么仗
- 提供可持续的冲突链 / 事件链 / 回报链
- 给分卷层持续供血

分卷层负责：

- 把其中一段供血切成阶段性故事
- 变成卷内主问题、卷内回报、卷尾交接缺口

## 二、故事面向分卷降维时，至少要翻译五项

### 1. 谜面翻译成卷内主问题

- 故事面的一句话谜面，在本卷必须变成一个卷内可追的问题。
- 若故事面只能保持全书级抽象压迫，不能转成某卷的主问题，说明这一面还没降维成功。

### 2. 硬抓手翻译成卷内抓手

- A/B/C/D 抓手里，本卷至少要指定 1–3 个主抓手。
- 不要把所有抓手平均摊进每一卷，否则会发散。
- 每卷优先问：本卷最值钱的可落纸抓手是什么？

### 3. 交接硬点翻译成卷尾缺口

- 故事面的跨卷硬抓手线，必须在卷尾变成下一卷要追的钉子。
- 如果卷尾只有氛围悬念，没有交接硬点，说明故事面没有真正供到下一卷。

### 4. 代价回声翻译成卷内成本

- 每卷都要写：本卷主角付了什么成本、反派消耗了什么资源、规则暴露了什么边界。
- 故事面里的“代价回声”不应停留在总纲层口号。

### 5. 供血锚点翻译成章群落点

- 故事面里的供血锚点，不一定一章一个，但必须落成章群：
  - 前段重认卖点章群
  - 中段兑现 / 反噬章群
  - 卷尾交接章群

## 二点五、映射来源层（新增）

- `故事面母题层`：一句话谜面、A/B/C/D 抓手、代价回声、交接硬点。
- `卷级转译层`：本卷主问题、本卷主抓手、本卷最值钱的回报、卷尾交接钉子。
- `章群落点层`：前段重认卖点章群、中段兑现 / 反噬章群、卷尾交接章群。

## 三、故事面 → 分卷的最小映射表

| 故事面元素 | 本卷承担什么 | 卷内位置 | 可见回报 | 卷尾交接钉子 |
| --- | --- | --- | --- | --- |
| 一句话谜面 | 本卷主问题 | 卷内布局 / 第一情节点 | 读者知道这卷在打什么 | 下一卷更大的问题 |
| A/B/C/D 抓手 | 本卷主抓手 1–3 个 | 中段 / 关键章 | 新证据 / 新破绽 / 新规则 | 留给下卷继续追 |
| 节点责任 | 本卷具体追责对象 | 反应段 / 进攻段 | 谁签 / 谁放行 / 谁退回被看见 | 更高层节点露头 |
| 代价回声 | 本卷成本与损耗 | 全卷 | 主角 / 对手都付代价 | 下一卷难度升高 |
| 交接硬点 | 卷尾必须留下的钉子 | 卷尾 | 本卷闭环后仍有余压 | 下一卷直接接棒 |

## 四、给故事面作者的三句硬问

1. 这一面在本卷到底让主角追什么？
2. 这一面在本卷到底让读者拿到什么？
3. 这一面在本卷结束时到底把哪颗钉子交给下一卷？

答不出来，说明故事面还停留在概念层。

## 五、卷前供血分配法

每个故事面建议先标：

- **主供血卷**：这一面最完整、最能打的一卷
- **副供血卷**：补充、反咬、回收的一卷
- **影子卷**：只露影、不主打的一卷

不要把同一故事面平均铺满所有卷。

## 六、常见失误

### 1. 故事面太强，卷纲只能抄概念

结果：卷纲写成概念转述，落不到事件。

### 2. 故事面抓手太多，本卷没有主抓手

结果：什么都写，什么都不尖。

### 3. 故事面代价没往卷里压

结果：主角和反派都像无限资源。

### 4. 故事面交接硬点没落成卷尾钉子

结果：下一卷像重新开书。

## 七、最低验收

一个故事面如果真的供到了分卷层，至少能做到：

- 指出本卷主问题
- 指出本卷主抓手
- 指出本卷最值钱的一次回报
- 指出卷尾交接钉子
- 指出下一卷接棒方向

## 七点五、最小映射工件（新增）

```markdown
【故事面母题层】一句话谜面 / 抓手 / 代价 / 交接硬点
【卷级转译层】本卷主问题 / 主抓手 / 最值钱回报 / 卷尾钉子
【章群落点层】前段 / 中段 / 卷尾章群
【当前判定】Green / Yellow / Red
```

## 八、一句话收口

**故事面不是让分卷“参考一下”，而是要把全书级冲突压成某一卷真正可打的一场仗。**

## 九、Green / Yellow / Red 映射判定（新增）

- `Green`：母题、卷级转译、章群落点三层都清楚，故事面能稳定供到分卷层。
- `Yellow`：卷级问题已立，但章群落点仍偏粗或交接钉子不够硬。
- `Red`：故事面还停在概念层，既转不成卷，也落不进章群。
