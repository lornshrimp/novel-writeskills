# 章首补丁回写控制卡与场景职责表

适用于章首强化后，判断问题到底已经在局部修好，还是应该反向回写到章节控制卡与场景职责分配。

## 一、先明确：章首强化不总是句子问题

有些章首不抓，不是因为首句不够漂亮，而是因为上游没有钉死：

- 哪一场负责抓眼
- 哪个主抓手应在前段显影
- 本章卖点要在前段怎样被看见

如果这些上游职责没定，单独修开头往往只能止血，不能治本。

## 二、什么时候只修章首即可

满足以下大多数情况时，可视为局部补丁：

- 本章其实已有明确抓眼职责，只是落地偏平
- 主抓手、本章任务、首场职责本来就清楚
- 章首问题主要是解释先行、动作太晚、异常不够尖
- 修完前 150–300 字后，后续场景仍能自然承接

## 三、什么时候必须回写上游

出现以下任两项，建议回写控制卡或场景职责：

### 1. 不知道第一场为什么存在

- 只能说“这章得开一下场”
- 却说不清第一场是抓眼、试探，还是回报前置

### 2. 主抓手在章首该不该出现，答不清

- 不知道本章卖点是该前段亮、前段保温，还是故意延迟

### 3. 章首修完后，第二场接不上

- 说明问题不只在首段，而在场景职责排序

### 4. 开头必须靠偷跑后文回报才能抓人

- 一旦这么做，往往意味着本章中段回报位置设计有问题

### 5. 开头“能抓了”，但和本章主推进线脱节

- 读者被拉进来了，却不是被拉进本章真正要解决的麻烦

## 四、回写优先级

### 第一优先：回写场景职责

先问：

- 第一场是否明确承担“抓眼启动”
- 第二场是否明确接“推进 / 试探”
- 主抓手是否安排在前两场的合理位置显影

### 第二优先：回写章节控制卡

若场景职责也答不清，再回写：

- 本章主抓手
- 本章对主卖点的关系
- 本章为什么非写不可
- 本章前段是否必须承担重认卖点职责

## 四点五、来源层映射（新增）

- `控制卡层`：本章主抓手、本章与主卖点关系、本章为什么非写不可。
- `场景职责层`：第一场抓眼启动、场景 `1→2` 交接、主抓手显影位。
- `章首补丁层`：句子切入方式、异常前置、动作前置、首段压缩。

默认回写顺序：先 `场景职责层`，再 `控制卡层`，最后才继续局部磨 `章首补丁层`。

## 五、最小回写表

| 章首暴露的问题 | 优先回写层级 | 该补哪项 |
| --- | --- | --- |
| 第一场只有背景，没有事 | 场景职责 | 第一场职责改为抓眼启动 |
| 章首异常与本章主线脱节 | 控制卡 | 主推进线 / 主抓手 |
| 章首必须硬偷跑中段回报才有力 | 控制卡 + 场景职责 | 中段回报位置 / 第一场职责 |
| 修完首段，第二场接不住 | 场景职责 | 场景 1→2 交接关系 |
| 开头抓人但卖点没被认出 | 控制卡 | 本章与主卖点关系 |

## 六、章首强化后的三问复检

1. 第一场现在能否一句话说清自己负责什么？
2. 第一场抓住读者的东西，是否就是本章真正要推进的麻烦？
3. 如果不改后文，只改章首，后续场景还能自然接住吗？

若第 3 题为否，优先回写上游层级，而不是继续雕首句。

## 六点五、Green / Yellow / Red 回写判定（新增）

- `Green`：问题主要在章首补丁层，局部修就够。
- `Yellow`：章首补丁层之外，还牵出轻度场景职责问题，但不一定要大回写。
- `Red`：控制卡层与场景职责层至少一层明显失效，继续只雕章首只会把问题抹顺。

## 六点六、最小输出模板（新增）

```markdown
【章首判定】Green / Yellow / Red
【主要失效层】控制卡层 / 场景职责层 / 章首补丁层
【先回写哪一层】
【若继续只修首段会发生什么】
```

## 七、一句话收口

**章首强化只负责“把门踹开”，但如果门后根本没有安排好谁先上场、谁接第一棒，就必须回写控制卡和场景职责。**
