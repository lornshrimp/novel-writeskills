## 章首 P0 强规则

1. 主角必须尽早在场；默认不接受“前几段全是背景人物或旁白”。
2. 禁止开头堆设定、堆名词、堆术语；设定要融入动作，不要脱离事件单独讲。
3. 开头默认要满足“动作 + 反应”的最小单元，否则只是日记，不是故事。
4. 开头冲突优先“生活化真实冲突”，避免无现实根基的硬冲突与老套毒点。
5. 前三章默认按职责推进：
   - 第 1 章：立主角抓手与当前困局；
   - 第 2 章：主角必须对困局做出有效动作；
   - 第 3 章：给可感知回报或更强追读钩子。

## 章首 P1 优化项

1. 一句话一功能：首段不要同时承担世界观讲解、人物关系百科和价值观阐释。
2. 读者端信息优先级：先让读者知道“现在出什么事”，再补“为什么会这样”。
3. 复杂金手指开局应简化呈现：第一屏只交付“能干什么 + 会付什么代价 + 为什么马上用”。
4. 章首应避免“只点爽不落地”：必须让主角动作引发可见反应，才算成立。

## 自检清单（开头回写前）

- [ ] 主角是否在首屏明确在场？
- [ ] 是否出现了可见动作与外部/他人反应？
- [ ] 是否有真实、可代入的即时困局？
- [ ] 是否删掉了可后置的大段设定？
- [ ] 是否对齐黄金三章职责而非只修句子？
