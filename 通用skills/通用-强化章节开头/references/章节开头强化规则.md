# 章节开头强化规则

## 一、处理范围

本 Skill 的处理重心是：

- 正文第一句
- 前 2–3 段
- 前 150–300 字左右的入场带宽

不是整章重写。

## 二、开头的最低达标线

一个合格开头，至少命中以下四项中的两项：

- 异常
- 冲突
- 情绪锚点
- 悬念缺口

## 二点五、来源层速判（新增）

- `场景职责层`：第一场为什么存在、是否明确承担抓眼启动、是否贴着本章主推进线。
- `压力显影层`：异常、冲突、风险、损失窗口、问题引擎是否已在前 `150–300` 字真正落地。
- `表达压缩层`：解释先行、句群平均、弱修饰词过多、动作晚于解释、开头不够硬的问题是否存在。

默认顺序：先判 `场景职责层`，再判 `压力显影层`，最后修 `表达压缩层`；不要先把句子修漂亮，再发现第一场根本没职责。

## 三、开头最常见的死法

### 1. 解释先行

先交代背景、规则、设定、时代，再让故事开始。

### 2. 气氛空转

只有压抑、诡异、不安，但没有一个具体动作或反常细节。

### 3. 节奏太平均

句子长度差不多，信息密度差不多，像匀速滑行。

### 4. 正确但不危险

什么都顺，但读者没有“非读不可”的感觉。

### 5. 人在场，事不在场

- 主角有情绪，但没有眼下必须做的动作
- 读者知道人物很烦，却不知道这章为什么现在就要看

### 6. 氛围有了，问题没有

- 画面挺好，但没有具体问题把读者勾住
- 看完只有“有感觉”，没有“想继续”

### 7. 冲得太猛，但没有落点

- 满屏惊叹、惊悚、形容词
- 实际没有证据、没有动作、没有人被逼到墙角

## 四、优先使用的强化动作

- 让异常更早出现
- 让动作先于解释
- 让一个反常细节站出来
- 让人物当下的困境更可感
- 尽早抛出一个读者想追的问题
- 用具体物件、动作、声音、界面、时间窗代替虚张声势

## 五、推荐公式

1. 一处异常或冲突先出现。
2. 主角立刻做出一个反应动作。
3. 情绪落到身体或物件上。
4. 留一个具体问题，把读者往下拖。

## 五点二、章首补丁最小工件（新增）

```markdown
【本章第一场职责】
【前150-300字主压力】
【本章主抓手是否前置】是 / 否 / 延迟
【读者追问】
【最晚第几句要出现动作】
【最晚第几句要出现风险 / 异常】
```

## 五点五、直接陈述优先，不要一开场就忙着解释

章首最怕刚开口就进入解释姿态。

### 易犯问题

- 句子总在说“因为……所以……”
- 章首大量使用“非常”“真的”“似乎”“有点”这类弱修饰词
- 作者像在替读者预先总结，而不是把现场摆出来

### 优先改法

- 先陈述发生了什么，再考虑是否补原因
- 先让人物做动作，再决定要不要解释动机
- 先让一个具体物或异常细节站出来，不要让结论抢跑

章首如果能写成：

- 一个事实
- 一个动作
- 一个异样

通常就比：

- 一个解释
- 一个总结
- 一个情绪形容词

更有抓力。

## 五点六、章首弱修饰词清理

以下词在章首要特别警惕：

- 非常
- 真的
- 有点
- 似乎
- 好像
- 仿佛

不是绝对禁用，而是要先问：

- 删掉后，句子是不是更硬？
- 我是不是能用更具体的动作、声音、物件代替它？

章首越靠前，越要靠事实和动作站住，而不是靠语气词扶着走。

## 六、常用入场类型

- 异常冷开场
- 对话冲突开场
- 行动受阻开场
- 倒计时 / 窗口期开场
- 困境再压一层开场

## 六点五、对话型开场专门要求

若章首选择“对话冲突开场”，前 3–6 句至少应交出以下两项：

- **位差**：谁更强势，谁更被动，谁在压谁
- **风险**：这句说错、说晚、说穿会出什么事
- **声音差**：去掉提示语，也能大致听出不是一个口气
- **问题感**：读者会立刻追问“他们为什么这样说话”

不要把对话型开场写成：

- 日常寒暄
- 背景互相提醒
- 只有声音，没有利益和压力

更有效的做法是：

- 让一句台词先把关系顶歪
- 让主角立刻给出反应动作
- 让这波对话直接贴住本章主卖点，而不是只做通用型热身

## 七、禁止事项

- 不要把答案直接提前说穿
- 不要用背景大段说明换掉悬念
- 不要用华丽句子代替事件
- 不要顺手改中后段结构
- 不要先讲设定、背景、主题或人物档案

## 八、验收口径

开头强化完成后，应满足：

- 更抓人
- 更快入戏
- 更具体
- 更有问题感
- 但不剧透、不抢光后文回报

## 八点五、Green / Yellow / Red 判定门禁（新增）

- `Green`：第一场职责清楚，前 `150–300` 字至少两项抓力信号成立，且抓住的就是本章真正要推进的麻烦。
- `Yellow`：开头已有抓力，但某一层仍发虚，例如压力有了但主抓手没认出来，或句面压缩后仍偏慢。
- `Red`：第一场职责不明、压力和问题引擎说不清、必须偷跑后文回报才有劲，或修完开头后第二场明显接不住。

## 九、起点主渠道开头审核补丁

> 非“输出*版”任务默认按起点读者与编辑首轮审读口径执行。

### 1) 三条起点评估红线

- 前 300 字必须出现可识别的核心冲突或现实困境，不能只写背景与气氛。
- 前 1000 字要立住主角抓手（能力 / 职业 / 资源 / 信息差至少一项）。
- 第 1 章结尾要给明确追读动机，避免“还没开始”式温吞收束。

### 2) 审稿友好的开场动作

- 用“身份 + 危机 + 代价”三件套替代抽象自述。
- 尽量在首屏出现可拍镜头（物件、界面、动作、话术、回执、点位）。
- 若使用对话开场，前三句内必须出现位差或风险，不做寒暄热身。

### 3) 常见拒稿型开头（默认规避）

- 设定讲义型：先讲世界观再讲故事。
- 情绪空转型：只有痛苦、压抑、迷茫，没有可执行动作。
- 反派脸谱型：只给“坏”，不给动机与现实支点。

### 4) 起点审稿节奏提示（执行向）

- 审稿首轮通常高度集中在前三章；默认把“第一章首屏 + 第一章结尾 + 第三章结尾”作为三处关键钩点。
- 开头若未给“角色当前要解决的问题”，即使语言好看，也按弱开场处理并回炉。
