# 通用-强化章节开头

章节开头不是门口寒暄，而是把读者一把拽进屋里，再顺手把门反锁。

## 保真迁移要求（强制）

命中“章首强化 / 开头抓人 / 前 150–300 字重做”任务时，除本文件外，**必须继续读取**：

- `references/章节开头强化规则.md`
- `references/章节开头检查清单.md`
- `references/黄金三章与五类开篇钩子.md`

## 何时使用

- 章首信息没错，但不够抓人。
- 前 2–3 段铺垫太多，异常太晚，读者容易滑走。
- 开头太稳、太慢、太解释、太平均，像在热身而不像故事已经开始。
- 用户只想抢救开头，不打算整章重修。

## 不适用情形

- 本章真正问题在中段空转、回报缺失或结尾无力。
- 你想通过新增设定、重改剧情结构来救开头。
- 你要做整章精修，而不是只修开头。

## 本 Skill 的核心任务

1. 强化前 **150–300 字**的入场抓力。
2. 让开头至少命中以下四项中的两项：
   - 异常
   - 冲突
   - 情绪锚点
   - 悬念缺口
3. 让开头更像“已经开始的故事”，而不是“准备开始的故事”。
4. 禁止把开头写成设定说明、背景介绍或慢热散文。

## 执行顺序

1. 先圈定正文第一句、前 2–3 段、前 150–300 字左右的入场带宽。
2. 识别当前最大问题属于哪类：
   - 解释先行
   - 气氛空转
   - 节奏太平均
   - 正确但不危险
   - 人在场、事不在场
   - 氛围有了、问题没有
   - 冲得太猛但没有落点
3. 保住原章已经存在的：关键信息、第一波冲突、视角与人物状态。
4. 优先做的动作：
   - 让异常更早出现
   - 让动作先于解释
   - 让一个反常细节站出来
   - 让人物当下困境更可感
   - 尽早抛出一个读者想追的问题
5. 只强化章首，不顺手重做中后段。
6. 复检：是否更抓人、更具体、更快入戏，但没有剧透、没有偷加事实、没有误伤后文回报。

## 默认输出口径

- 默认只修改前 150–300 字或前 2–3 段。
- 若用户指定“只动首句 / 只动前 150 字”，则缩小范围。
- 默认不改标题，除非用户明确要求。
- 默认直接回写章首对应段落。

## 推荐公式

1. 一处异常或冲突先出现。
2. 主角立刻做出一个反应动作。
3. 情绪落到身体或物件上。
4. 留一个具体问题，把读者往下拖。

## 开篇补充原则

- 若任务命中第一章或前三章开头，默认同时检查它在黄金三章里的职责，而不是只看这一段好不好看。

## 最低交付

- 在极短篇幅内完成至少两项：异常 / 冲突 / 情绪锚点 / 悬念缺口。
- 让主角出现一个眼下必须处理的动作、窗口或风险。
- 让开头最后一句把读者推向下一段，而不是停在背景说明上。

## 硬规则

- 不得用大段背景说明替代开场冲击。
- 不得把答案直接提前说穿。
- 不得用华丽句子代替事件与动作。
- 不得新增原稿没有的关键事件与关键信息。
- 不得顺手改中后段结构。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只管“进门那一下”，不负责整章文风统筹。
- 若用户要求全章精修，应交给对应的正文润色类 skill。
- 若用户要求强化结尾，不由本 Skill 处理。
