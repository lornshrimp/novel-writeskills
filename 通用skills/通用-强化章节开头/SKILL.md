# 通用-强化章节开头

章节开头不是门口寒暄，而是把读者一把拽进屋里，再顺手把门反锁。

## 保真迁移要求（强制）

命中“章首强化 / 开头抓人 / 前 150–300 字重做”任务时，除本文件外，**必须继续读取**：
- `references/钩子类型分类与样本库.md`（**必读** — 了解六大钩子类型与实战样本）
- `references/章节开头强化规则.md`
- `references/章节开头检查清单.md`
- `references/黄金三章与五类开篇钩子.md`
- `references/反常开场十机制速查.md`
- `references/滑梯效应与首段安抚.md`
- `references/章首补丁回写控制卡与场景职责表.md`
- `references/腾讯专栏章首抓力增补规则.md`
- `references/腾讯专栏黄金三章多维共识增补.md`
- `references/腾讯专栏爆款文切入点与框架增补.md`
- `references/腾讯专栏开篇以情节为核心增补.md`

## 常见触发词 / 用户说法速查

- 帮我救一下开头
- 这章章首不抓人
- 前两段太慢了
- 只修前 150 到 300 字
- 首屏留存不行
- 这开头信息对，但没第一钩

## 何时使用

- 章首信息没错，但不够抓人。
- 前 2–3 段铺垫太多，异常太晚，读者容易滑走。
- 开头太稳、太慢、太解释、太平均，像在热身而不像故事已经开始。
- 用户只想抢救开头，不打算整章重修。

## 不适用情形

- 本章真正问题在中段空转、回报缺失或结尾无力。
- 你想通过新增设定、重改剧情结构来救开头。
- 你要做整章精修，而不是只修开头。

## 章首补丁前的裁判源复核（强制）

- 动手改前 `150–300` 字前，必须认真仔细阅读与本章实际出场人物对应的人物传记，以及与本章背景、技术、规则、证据、程序链、世界观有关的技术设定文档。
- 不得只凭开头片段、上一章印象或题材惯性硬修章首；凡涉及人物当前处境、能力边界、关系状态、设定代价、证据机制的开场信息，都必须回到作者侧裁判源核对。
- 若同一人物传记或技术设定存在多个版本，必须先确认当前可执行的最新版本；若关键裁判源缺失，必须先补齐、定位或明确阻断，不能边猜边补第一屏。
- 章首强化后若改变了人物当下状态、开局风险、规则显影方式或线索入口，必须回看一次相关裁判源，确认没有把后文承接修歪。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，强化章首前必须先读取并纳入第一屏判断。
- 若目标平台为起点中文网，或未注明平台但默认按起点口径处理，且当前题材目录下尚无可用竞对主报告，且本章属于**第一章 / 黄金三章 / 卷首章 / 首个收费节点 / 关键转折章**之一，强化章首前应优先调用 `通用-分析竞对作品`，至少补齐 **Top10 候选池、`3–5` 本主样本与四层深拆速览**；若任务限制导致本轮无法补齐，必须显眼记录盲区，并降低章首优化强度判断。
- 不得把这些报告只当成背景资料；章首补丁必须显式说明本次准备在哪些维度追平、反超竞对，以及哪些旧坑必须避开。
- 章首强化的默认目标不是“更顺更抓人”而已，而是让当前作品至少在**前 `300` 字抓力、题材归类效率、人物抓手、问题引擎、首屏讨论点**这 `5` 个默认关键维度里形成可举证的追平路径，并尽量在其中 `2–3` 项建立明确领先方案。

## 起点化竞对吸收口径（新增）

当章首服务起点中文网，或未标平台而默认按起点基线处理时，默认把竞对吸收拆成以下四层，并翻译成前 `150–300` 字补丁约束：

- **市场数据层**：用榜单位次、均订 / 首订 / 收订比代理信号、推荐资源位等，反推当前章首在前 `300 / 1000` 字抓力、题材归类效率与继续阅读驱动上至少该达到什么强度。
- **内容创作层**：重点吸收主样本在异常前置、冲突前置、人物抓手、第一动作、问题引擎与首段信息密度上的做法，把它翻成首句、首段和前两三段的具体补丁。
- **运营策略层**：重点吸收主样本如何在第一屏制造本章说讨论点、段评截停点、手机端滑梯效应与首屏留存优势，把它翻成更适合移动端的开场压缩与落针方式。
- **受众反馈层**：重点吸收本章说 / 书友圈 / 书评区里高频被夸的章首抓手、被骂的慢热病灶、老白读者不耐烦的解释段落，提前写进补丁避坑清单。

起点语境下，默认尽量使用以下术语，而不是写成空泛的“开头更抓人”：`均订`、`高订`、`首订`、`追订`、`收订比`、`分类月票榜`、`畅销榜`、`新书榜`、`本章说 / 段评`、`书友圈`、`老白读者`。拿不到精确值时必须写成代理信号，不得伪造具体数字。

## 本 Skill 的核心任务

1. 强化前 **150–300 字**的入场抓力。
2. 优先公式固定如下：默认先抓“异常”与“冲突”；若当前章不适合双压，至少保留“异常 / 冲突”之一，再补“情绪锚点 / 悬念缺口”之一；若自然命中超过两项可以保留，但不得为凑项硬堆：
   - 异常
   - 冲突
   - 情绪锚点
   - 悬念缺口
3. 让开头更像“已经开始的故事”，而不是“准备开始的故事”。
4. **禁止把开头写成设定说明、背景介绍或慢热散文。这是最常见的新手坑。** 开篇千万不要铺垫，千万千万不要铺垫 —— 直接把主角扔进冲突里，让读者跟着一起紧张，自然就追更了。
5. 尽快让读者感知本书主卖点，至少让开头的异常、冲突或困境与主卖点同向。
6. 若按起点口径执行，动手前必须先写清：本次章首补丁要追平 / 反超哪类主样本的哪一项前 `300 / 1000` 字抓力。

## 首屏共性硬要求（新增）

- 前 `300` 字默认至少完成三件事中的两件：人物当下处境、眼前损失风险、一个必须立刻处理的异常或冲突。
- 若命中第一章到第三章，默认还要检查“黄金三章职责”是否成立：第一章立人物抓手与麻烦，第二章让人物真的顶上第一个挑战，第三章必须给一次可感知的小高潮、反击或更强升级。
- 第一章的章首强化，必须额外确认“问题引擎”是否落地：读者看完首段后，心里要留下一个明确追问（不是背景信息，而是等待答案的问题）。
- 章首失效常见信号（命中任一都应回炉）：节奏太平均、正确但不危险、人在场但事不在场、只有设定无现场压力。
- 若修完章首仍发现第一场职责、主抓手前置位置或场景 1→2 交接发虚，只允许用“章首补丁回写控制卡与场景职责表”判断并回写上游职责；不在本 Skill 内直接改中后段正文，也不继续硬修首句。
- 复检：是否更抓人、更具体、更快入戏，但没有剧透、没有偷加事实、没有误伤后文回报。

## 默认输出口径

- 默认只修改前 150–300 字或前 2–3 段。
- 若用户指定“只动首句 / 只动前 150 字”，则缩小范围。
- 默认不改标题，除非用户明确要求。
- 默认直接回写章首对应段落。

## 强力开头句与极端场景（必读）

**开篇 300 字必须让读者"喘不上气"**。知乎经验总结：

- **强力开头句**：直接从冲突、悬念或诡异入场，而不是背景和铺垫。示例：
  - "我保证，打开那扇门是我这辈子最后悔的事。"
  - "我睁开眼，发现自己躺在陌生婚床上，旁边躺着一具尸体。手机显示：我是凶手，通缉令已发布。"
  - "如果知道这趟航班会改变我的一生，我登机前一定会吃顿更好的。"

- **极端场景开局**：不要从平稳状态开始，直接切入人物面临的最大危机、最强烈的欲望冲突或最诡异的处境。让每一句话都在建立"为什么我必须继续读"的压力。

- **对话式开头**：有时纯对话能更快地展现冲突。通过两个人的对话，无旁白描写，直接显露立场碰撞。

## 推荐公式

1. 一处异常或冲突先出现。
2. 主角立刻做出一个反应动作。
3. 情绪落到身体或物件上。
4. 留一个具体问题，把读者往下拖。

## 六类切入与上章响应补强（新增）

- 默认优先从六类有效切入里选：**动作、对话、反常情境、悬念、危机、发现**；若开头不属于这六类之一，需额外证明它的抓力来源。
- 默认把以下开头判为高风险：天气描写起手、起床 / 走路 / 吃饭等日常流程、复述上一章、长段背景说明、无信息量寒暄。
- 开头前 `20%` 至少要让读者看懂：谁、在哪、正在发生什么；若只剩气氛和抽象感觉，说明信息没落地。
- 若当前不是首章，本章开头默认要对上章出口钩作出响应：可直接连续、间接关联或有意延迟；若有意延迟，最好在 `3` 章内回到这条线。
- 若修完开头仍发平，优先尝试从第三段或首个真正异常点重新起切，而不是继续给原首段抛光。

## 第一屏压缩法（默认口径）

- 前 1–3 句尽量先让读者看见一件事：谁眼下要失去什么、什么规则突然出错了，或什么反常物件先跳出来。
- 前 150–300 字优先完成“人物抓手 + 第一压力 + 一个追问”三件事中的至少两件半，而不是平均分配给背景、设定与气氛。
- 若这是第一章到第三章之一，默认同步检查：第一章先立主角抓手与第一麻烦，第二章让主角真的顶上第一个挑战，第三章给一次小高潮或更强升级。
- 开头里最值钱的危险，最好是具体窗口、倒计时、关系反噬、资源失去或位置滑落，而不是抽象地说“他很压抑 / 情况不妙”。
## 极其快的节奏与极其强烈的冲突——章首强化方法（新增）

### 快节奏开头的特征与规则

**1. "开篇 30 秒的冲击力"（前 100-150 字）**
快节奏的章节开头应该在前 100-150 字内就抛出：
- 一个谁在、他在干什么、眼下他失去什么或即将失去什么
- 不是背景介绍，而是"已经在发生的事件"

示例对比：
```
❌ 慢：李明推开公司大门，走进了熟悉的办公室。今天是周一，又是新的一周的开始。他的心里有点忐忑，因为上周五的项目汇报没有得到领导的表扬。

✅ 快：推门的一瞬间，李明就看到了——领导办公室的灯亮着，办公室里站着两个陌生的人。其中一个正在翻他的工作电脑。
```

快版本在前 50 字内就让读者知道：有人在他的电脑上干什么，而且可能是坏事。

**2. "倒计时感"的即刻建立**
快节奏开头应该让读者立刻感到有"时间在流逝、事情在紧急发展"：
```
他有 10 分钟时间 / 信号很快就会被切断 / 她已经开始走进危险地带 / 炸弹在 30 秒后爆炸
```

这种倒计时感能自然地让读者加快阅读速度，跟上节奏。

**3. "对抗方向"的前置明确**
快节奏开头应该让读者立刻理解：这个主角面对的敌手是谁、是什么、在哪里。
```
不是："李明感到很不安。"（敌手模糊）
而是："灯亮的办公室里，一个陌生人正在翻李明的电脑。"（敌手明确）
```

**4. "第一个动作"的即刻出现**
快节奏开头不应该让主角停留在思考或观察；应该立刻看到他的反应动作：
```
他的脚步停了下来。/ 他的手握紧了手机。/ 他转身朝楼梯走去。
```

动作能比心理活动更快地推进节奏。

### 强冲突开头的特征与规则

**1. "两个对立方向"的即刻碰撞**
强冲突的开头应该立刻让读者看到至少两个不同方向的力量在碰撞：
```
李明想要离开，但领导挡住了他的去路。
李明想要隐瞒真相，但陌生人已经找到了证据。
李明想要求救，但没有人接他的电话。
```

这样的对立能立刻制造压力和张力。

**2. "代价的具体显露"**
强冲突的开头应该让读者看到这个冲突的代价是什么——不是"有麻烦"，而是"具体会失去什么"：
```
❌ "李明意识到他可能会失去工作。"
✅ "领导的表情冷了下来。李明知道这意味着什么——裁员名单里已经有他的名字了。"
```

具体的代价（裁员名单）比抽象的威胁（会失去工作）更能推动快速的冲突感。

**3. "信息的翻面或陷阱"**
强冲突的开头可以利用"读者与主角都误解了什么"来制造冲击：
```
李明以为陌生人是审计，但听到第一句话时，他意识到——这不是审计，这是调查。
```

这种信息的突然翻转能制造强冲击。

**4. "权力不对等"的明确表现**
强冲突开头应该让读者感到有人掌握了话语权、主动权或生死权：
```
陌生人靠在李明的办公桌上，指点着屏幕上的数字："这些钱，你来解释一下是怎么回事。"
```

这个姿态（靠在桌子上、指点屏幕）就暗示了权力的不对等。

### 快节奏 + 强冲突的开头模板

```
## 第 X 章开头强化

**当前开头的功能分析**
- 这章要完成什么职责（黄金三章职责 / 章节控制卡职责）
- 上一章留下了什么悬念或压力
- 这章应该立刻推进到什么

**快节奏的三个要素**
- 倒计时（有多少时间）
- 对手或敌手（谁在追、谁在阻）
- 第一个动作（主角怎么反应）

**强冲突的三个要素**
- 两个对立的力量或目标
- 具体的代价（不是"有麻烦"，而是"会失去什么"）
- 权力的不对等（谁掌握主动权）

**改写后的前 300 字应该完成**
- [ ] 让读者在 50 字内知道"谁、在哪、在干什么"
- [ ] 让读者在 100 字内感到"有压力、有倒计时、有敌手"
- [ ] 让读者在 200 字内看到"主角的第一个反应动作"
- [ ] 让读者在 300 字内理解"这章的对立方向和可能的代价"
```

### 执行时的快速检查清单（快冲突开头专项）

[ ] 前 50 字是否已经抛出了一个异常或冲突？
[ ] 前 100 字是否建立了倒计时感（时间、危险、压力）？
[ ] 是否清晰地表现了对手或敌手的存在（不是模糊威胁）？
[ ] 是否立刻出现了主角的第一个反应动作（不是心理活动）？
[ ] 是否有至少两个对立的力量或目标在碰撞（不是单向压力）？
[ ] 代价是否具体而非抽象（不是"可能有问题"，而是"会失去 X"）？
[ ] 权力分布是否清晰（谁掌握主动权）？
[ ] 是否避免了大段背景介绍或设定说明？
[ ] 是否与本书的主卖点对齐（冲突方向是否指向题材核心）？
[ ] 开头是否为下一段做了推动，而不是停在信息说明上？
## 开篇补充原则

- 若任务命中第一章或前三章开头，默认同时检查它在黄金三章里的职责，而不是只看这一段好不好看。
- 若当前开头最大问题是“有信息但没有第一钩”，优先从两难、矛盾、突兀、悲剧、切身、留白等反常机制里挑一类落地，而不是只把句子写怪。

## 最低交付

- 在极短篇幅内完成至少两项：异常 / 冲突 / 情绪锚点 / 悬念缺口。
- 让主角出现一个眼下必须处理的动作、窗口或风险。
- 让开头的抓力与本书主卖点对齐，而不是只做通用型抓眼球。
- 让开头最后一句把读者推向下一段，而不是停在背景说明上。

## 硬规则

执行时只按以下三组顺序检查：先信息与事实边界，再章首表达边界，最后范围与卖点边界；前一组未过，不进入后一组。

- 先守信息与事实边界：
  - 未认真仔细阅读本章相关人物传记与技术设定 / 背景设定文档前，不得硬修开头的人物处境、关系状态、规则显影与风险来源。
  - 不得新增原稿没有的关键事件与关键信息。
  - 不得把答案直接提前说穿。
- 再守章首表达边界：
  - 不得用大段背景说明替代开场冲击。
  - 不得用“前几章 / 前面几章 / 前面这几章 / 后几章 / 后面几章 / 后面这几章 / 上一章 / 前文”式 recap 开头代替入场；若要承接旧事，应用伤痕、残局、冷场、未处理消息、残留物件或人物状态把承接写在现场里。
  - 不得用“整章 / 这整章 / 这一整章”式作者复盘口吻开头；如果一句话天然像在给读者总结整章功能，说明它不该出现在章首。
  - 不得用华丽句子代替事件与动作。
- 最后守范围与卖点边界：
  - 不得顺手改中后段结构。
  - 不得把开头修得更抓人，却和全书主卖点脱节。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只管“进门那一下”，不负责整章文风统筹。
- 若用户要求全章精修，应交给对应的正文润色类 skill。
- 若用户要求强化结尾，不由本 Skill 处理。
