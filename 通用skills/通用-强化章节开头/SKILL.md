# 通用-强化章节开头

章节开头不是门口寒暄，而是把读者一把拽进屋里，再顺手把门反锁。

## 保真迁移要求（强制）

命中“章首强化 / 开头抓人 / 前 150–300 字重做”任务时，除本文件外，**必须继续读取**：

- `references/章节开头检查清单.md`
- `references/黄金三章与五类开篇钩子.md`
- `references/反常开场十机制速查.md`
- `references/滑梯效应与首段安抚.md`
- `references/章首补丁回写控制卡与场景职责表.md`
- `references/腾讯专栏章首抓力增补规则.md`
- `references/腾讯专栏黄金三章多维共识增补.md`
- `references/腾讯专栏爆款文切入点与框架增补.md`
- `references/腾讯专栏开篇以情节为核心增补.md`

## 常见触发词 / 用户说法速查

- 帮我救一下开头
- 这章章首不抓人
- 前两段太慢了
- 只修前 150 到 300 字
- 首屏留存不行
- 这开头信息对，但没第一钩

## 何时使用

- 章首信息没错，但不够抓人。
- 前 2–3 段铺垫太多，异常太晚，读者容易滑走。
- 开头太稳、太慢、太解释、太平均，像在热身而不像故事已经开始。
- 用户只想抢救开头，不打算整章重修。

## 不适用情形

- 本章真正问题在中段空转、回报缺失或结尾无力。
- 你想通过新增设定、重改剧情结构来救开头。
- 你要做整章精修，而不是只修开头。

## 本 Skill 的核心任务

1. 强化前 **150–300 字**的入场抓力。
2. 让开头至少命中以下四项中的两项：
   - 异常
   - 冲突
   - 情绪锚点
   - 悬念缺口
3. 让开头更像“已经开始的故事”，而不是“准备开始的故事”。
4. 禁止把开头写成设定说明、背景介绍或慢热散文。
5. 尽快让读者感知本书主卖点，至少让开头的异常、冲突或困境与主卖点同向。

## 首屏共性硬要求（新增）

- 前 `300` 字默认至少完成三件事中的两件：人物当下处境、眼前损失风险、一个必须立刻处理的异常或冲突。
- 若命中第一章到第三章，默认还要检查“黄金三章职责”是否成立：第一章立人物抓手与麻烦，第二章让人物真的顶上第一个挑战，第三章必须给一次可感知的小高潮、反击或更强升级。

 - `references/腾讯专栏开篇设定引入禁忌与主角视角增补.md`
    - `references/腾讯专栏开篇设定引入禁忌与主角视角增补.md`
    - `references/腾讯专栏多作家开头设计步骤与低阅读难度共识增补.md`
    - `references/腾讯专栏多作家开头设计步骤与低阅读难度共识增补.md`
    - `references/腾讯专栏畅销书开篇四类型框架增补.md`
   - 节奏太平均
   - 正确但不危险
   - 人在场、事不在场
   - 让一个反常细节站出来
   - 让人物当下困境更可感
   - 尽早抛出一个读者想追的问题
5. 若修完章首仍发现第一场职责、主抓手前置位置或场景 1→2 交接发虚，先用“章首补丁回写控制卡与场景职责表”判断是否回写上游。
6. 只强化章首，不顺手重做中后段；若确认是上游职责问题，再回写控制卡或场景职责，而不是继续硬修首句。
7. 复检：是否更抓人、更具体、更快入戏，但没有剧透、没有偷加事实、没有误伤后文回报。

## 默认输出口径

- 默认只修改前 150–300 字或前 2–3 段。
- 若用户指定“只动首句 / 只动前 150 字”，则缩小范围。
- 默认不改标题，除非用户明确要求。
- 默认直接回写章首对应段落。

## 推荐公式

1. 一处异常或冲突先出现。
2. 主角立刻做出一个反应动作。
3. 情绪落到身体或物件上。
4. 留一个具体问题，把读者往下拖。

## 第一屏压缩法（默认口径）

- 前 1–3 句尽量先让读者看见一件事：谁眼下要失去什么、什么规则突然出错了，或什么反常物件先跳出来。
- 前 150–300 字优先完成“人物抓手 + 第一压力 + 一个追问”三件事中的至少两件半，而不是平均分配给背景、设定与气氛。
- 若这是第一章到第三章之一，默认同步检查：第一章先立主角抓手与第一麻烦，第二章让主角真的顶上第一个挑战，第三章给一次小高潮或更强升级。
- 开头里最值钱的危险，最好是具体窗口、倒计时、关系反噬、资源失去或位置滑落，而不是抽象地说“他很压抑 / 情况不妙”。

## 开篇补充原则

- 若任务命中第一章或前三章开头，默认同时检查它在黄金三章里的职责，而不是只看这一段好不好看。
- 若当前开头最大问题是“有信息但没有第一钩”，优先从两难、矛盾、突兀、悲剧、切身、留白等反常机制里挑一类落地，而不是只把句子写怪。

## 最低交付

- 在极短篇幅内完成至少两项：异常 / 冲突 / 情绪锚点 / 悬念缺口。
- 让主角出现一个眼下必须处理的动作、窗口或风险。
- 让开头的抓力与本书主卖点对齐，而不是只做通用型抓眼球。
- 让开头最后一句把读者推向下一段，而不是停在背景说明上。

## 硬规则

- 不得用大段背景说明替代开场冲击。
- 不得用“前几章 / 上一章 / 前文”式 recap 开头代替入场；若要承接旧事，应用伤痕、残局、冷场、未处理消息、残留物件或人物状态把承接写在现场里。
- 不得把答案直接提前说穿。
- 不得用华丽句子代替事件与动作。
- 不得新增原稿没有的关键事件与关键信息。
- 不得顺手改中后段结构。
- 不得把开头修得更抓人，却和全书主卖点脱节。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只管“进门那一下”，不负责整章文风统筹。
- 若用户要求全章精修，应交给对应的正文润色类 skill。
- 若用户要求强化结尾，不由本 Skill 处理。
