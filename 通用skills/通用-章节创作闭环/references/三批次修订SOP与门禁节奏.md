# 三批次修订 SOP 与门禁节奏

> 来源口径：`修订体系.md` / `经验总结.md`。

## 批次顺序（不可逆）

```text
A（人物/钩子/对话）→ B（伏笔/金手指/精简）→ C（网文化/道具/风格）→ 终审
```

> 禁止跳批次：A 未完成不得进 B；B 未完成不得进 C。

## 批次来源层速判（新增）

- `步骤执行层`：当前到底处在 A / B / C 哪一批，是否真的完成该批基础动作。
- `门禁判定层`：本批硬门槛是否过线，哪些字段仍未通过。
- `回炉调度层`：若未过线，下一轮先补哪一项、回到哪一步、是否允许继续推进。

默认顺序：先判 `步骤执行层`，再判 `门禁判定层`，最后给 `回炉调度层`；不要先写“进入下一批”，再发现上一批门禁没过。

## A 批次门禁

- 前200字钩子覆盖率：目标 100%。
- 后200字留钩覆盖率：目标 90%+。
- 对话占比：25%-55%。
- 配角动机：每个常驻配角需有独立动机信号。

## B 批次门禁

- 伏笔清单完整（埋设/状态/回收窗口）。
- 金手指弧线存在“升级 + 代价 + 触发器”。
- 连续 200 字纯解释段应打散。
- 高频冗余词与拟声词需控频。

## C 批次门禁

- 章末必须有悬疑钩 / 选择钩 / 危机钩之一。
- 重要道具需形成跨章呼应弧线。
- 清理元叙述与 AI 万能句式。
- 同概念用词统一。

## 执行节拍建议

- 每 3-5 章做一次快审（字数 / 对话比 / 钩子覆盖）。
- 连续多章回炉时，先修“结构门禁”再修“语言质感”。
- 修订日志必须记录：本轮批次、未过门禁项、下一步补丁动作。

## 批次最小工件（新增）

```markdown
【当前批次】A / B / C
【步骤执行层】已完成 / 未完成
【门禁判定层】未过项：
【回炉调度层】下一步补丁动作：
【当前判定】Green / Yellow / Red
```

## Green / Yellow / Red 批次判定（新增）

- `Green`：当前批次动作真实执行，门禁过线，可以推进下一批。
- `Yellow`：当前批次主体已做，但仍有 `1–2` 个门禁项待补，推进需谨慎。
- `Red`：批次步骤未真跑完、关键门禁未过、或回炉动作未写清，不得宣布进入下一批。
