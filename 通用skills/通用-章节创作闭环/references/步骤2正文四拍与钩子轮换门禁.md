# 步骤 2 正文四拍与钩子轮换门禁

> 适用位置：`通用-章节创作闭环` 的步骤 2（`通用-创建小说正文`）执行后立即复核。

## 目的

把“写完一章”升级为“写完且过节拍门禁”。

## 四拍门禁（单章）

```text
钩子（前200）→ 蓄力（中段加压）→ 回报（中段兑现）→ 留钩（后200）
```

### 通过标准

- 前 200 字：至少 1 个有效钩子（冲突 / 悬念 / 异常 / 动作 / 对话）。
- 中段：至少 1 次可感知回报（关系、处境、信息、行动、身份其一）。
- 后 200 字：存在未闭合压力点，不得用预告腔替代。

## 钩子轮换门禁（跨章）

- 不得连续 3 章使用同类钩子。
- 若连续 2 章同类钩子，第 3 章必须强制换型。

推荐轮换：

```text
悬念钩 → 冲突钩 → 异常钩 → 动作钩 → 对话钩 → 悬念钩...
```

## 日志记录字段（强制）

每章至少记录：

- `HookType`：本章钩子类型
- `MidPayoff`：本章中段回报句（可回指）
- `TailDriver`：章末接棒问题
- `RotationCheck`：是否与前两章过载重复（Yes/No）
- `GatePass`：四拍门禁是否通过（Pass/Fail）

## 失败处置

- `GatePass=Fail`：不得进入终审；必须先回写步骤 2 或专项补丁（开头 / 章末 / 中段回报）。
- `RotationCheck=Yes`：必须在下一章控制卡中预设换型钩子，并标记为高优先级。
