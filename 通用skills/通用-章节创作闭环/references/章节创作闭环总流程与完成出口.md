# 章节创作闭环总流程与完成出口

## 支持的输入

- `OUTLINE_FILE`
- `CHAPTER_FILE`
- `RELATED_CONTEXT_FILES[]`（可选）
- `CHAPTER_FILES[]` / `OUTLINE_FILES[]` / `CHAPTER_BATCH[]`（批量模式）
- `BATCH_ID`（可选）

## 执行器身份

- 这是执行型闭环，不是规划型闭环。
- 命中后必须真实推进步骤 0→12，而不是只给计划、阶段汇报、初稿清单或“接下来做什么”。
- 除非出现硬性缺参导致下一步无法继续，否则不得中途暂停等待用户说“继续”。

## 标准执行顺序

0. 初始化与环境检查
1. 根据大纲创作章节初稿
2. 字数检测与循环扩写（直至 ≥ 3500）
3. 章节精修润色
4. 去 AI 味处理
5. 第一次审阅并生成审阅报告
6. 根据审阅报告进行章节修订
7. 第二次审阅并更新审阅报告
8. 根据第二版审阅报告进行第二次修订
9. 第三次审阅与问题清单终审
9.5. 质量门槛检查与回炉循环（直至连续两轮独立综合评分均严格大于 9.2）
10. 读者视角生成阅读笔记
11. 生成分章书评
12. 输出任务完成摘要

## 步骤 9.5 前后的链路闭环复核（默认启用）

在步骤 9、9.5 之间，默认增加一轮“章节链路闭环复核”：

- 这章承接的故事面 / 卖点是否真的进章
- 控制卡中的主抓手 / 中段回报 / 章末钩子是否真的落地
- 场景职责是否真实成立
- 首尾补丁、润色、去AI味是否在解决问题，而不是遮问题

若任一关键项发虚，应优先判断回写层级，而不是直接因“分数过线”宣称完成。

## 阶段推进闸门

- 步骤 1 / 2 只代表“正文初稿阶段完成”，绝不代表整个 SOP 完成。
- 每完成一个步骤，都必须立即判定 `nextStep` 并继续推进；只要 `nextStep <= 11`，就不得输出完成收尾语。
- 批量模式下，整批步骤 1→2 完成后必须立刻进入步骤 3→4；步骤 3→4 完成后必须进入步骤 5→9.5；全部章节通过 9.5 后才允许进入步骤 10、11、12。
- 如果发现自己刚完成的是“本相位最后一章”而不是“全 SOP 最后一步”，必须自动切到下一相位，而不是结束对话。

## 初稿阶段原则

- 步骤 1 的目标是把章节真正写出来，不是第一次就写到最优。
- 初稿首先要解决“离开空白页”和“把核心场景落地”，而不是同时完成所有美学要求。
- 如果一直卡在开头，允许先写：
  - 中段关键回报
  - 章末危险动作
  - 一段高压对话
  - 一个最清楚的场景热点
- 再回头把章首接出来。

## 废料场与回收原则

- 步骤 3 之后删掉的材料，默认不应直接彻底丢弃。
- 可转存为废料场，用于：
  - 下一版修订回收
  - 下一章伏笔
  - 作者有话说 / 后记补充
  - 平台派生稿差异化素材

若早期稿有价值但不适合当前版本，应视为“待回收素材”，不是“失败内容”。

## 唯一完成出口

只有当以下条件同时满足时，才允许宣称完成：

1. 步骤 0–11 已真实执行
2. 最新 `CHAPTER_FILE` 已重跑字数
3. 最新 `REVIEW_REPORT_FILE` 与最新正文一致
4. `READER_NOTE_FILE` 与 `BOOK_REVIEW_FILE` 已生成
5. 日志状态为 `completed`
6. 当前聊天输出的是步骤 12 的摘要模板，而不是阶段汇报

## 完成定义

### 单章

只有同时满足以下条件，才允许宣告完成：

1. 步骤 0–11 已在本次执行中实际跑完，而不是依赖历史文件推断。
2. `CHAPTER_FILE` 已完成创作 / 补字 / 润色 / 去AI味 / 多轮回炉修订。
3. `REVIEW_REPORT_FILE` 已被本次最新终审覆盖更新，且包含 `综合评分：X.X/10`。
4. `连续两轮独立终审综合评分均严格大于 9.2`（首次过线后须再跑一轮回炉修订 + 独立复审，第二轮也过线方可收口）。
5. 最新字数脚本结果已在正文最终版本上重新核对。
6. `READER_NOTE_FILE` 与 `BOOK_REVIEW_FILE` 已在本次执行中生成或覆盖更新。
7. 日志中的 `status` 与 `checkpoint` 已与真实状态一致。

### 批量

只有同时满足以下条件，批次才允许宣告完成：

1. 批次内每一章都满足单章完成定义。
2. 不存在任何一章仍停留在 `drafted`、`needs_polish`、`needs_review`、`needs_rework`、`score_below_threshold` 或等价中间态。
3. 批次主日志与逐章子日志均显示 `completed`。
4. 批量摘要已逐章列出终稿、终审报告、阅读笔记、书评、字数与评分。
5. 批次内每章若存在“链路未闭 / 待回写上游”状态，也不得视为真正完成。

## 禁止误判完成的情形

- 只有初稿落盘
- 只有正文达到 3500+
- 只有审阅报告生成
- 只有批量创作相位完成
- 任一章节仍缺 `READER_NOTE_FILE` 或 `BOOK_REVIEW_FILE`
- 任一章节 `status != completed`
- 任一章节 `checkpoint.nextStep < 12`
- 任一章节尚未连续两轮独立审阅评分均严格大于 9.2（包括首次刚过线尚未追加复审的情形）
- 任一章节仍处于“surface_fixed_only / needs_upstream_rewrite”之类链路未闭状态

## 强制补跑原则

- 文件存在不等于步骤已执行
- 正文一旦在步骤 2 之后被改写，下游报告、字数与摘要都视为过期
- 发现缺步、顺序错乱、结果过期或彼此不一致，必须自动回到最近可信节点补跑
- 若发现此前漏跑了润色、去AI味、回炉、读者产物或最终摘要，必须自动补跑，不得争辩式收尾
