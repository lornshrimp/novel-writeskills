# 章节终审门槛与摘要规范

## `9.2+` 的唯一判定条件

- 只有最新 `REVIEW_REPORT_FILE` 中明确写出 `综合评分：X.X/10`
- 且 `X.X > 9.2`（严格大于，`9.2` 本身不算通过）
- `8.8 / 8.9 / 9.0 / 9.1 / 9.2` 一律视为未达标
- **单次过线不等于可以收口**；必须在首次过线后，再追加一轮回炉修订，再做一次独立严格复审，若该复审评分仍严格大于 9.2，才允许宣告达标退出循环（即连续两轮独立审阅均严格大于 9.2）

## 首轮审阅链不得跳步

- 步骤 5 / 6 / 7 / 8 / 9 必须先完整执行一遍。
- 步骤 5、6、7、8、9 的首轮执行，不得由任何历史评分、预估评分或“看起来已经够好”来决定跳过。
- 只有在首次完成步骤 9 之后，才允许进入步骤 9.5 做评分分流。

## 回炉规则

- 步骤 5 / 6 / 7 / 8 / 9 至少完整执行一遍，之后才允许根据 9.2 门槛决定是否追加子循环
- 门槛判断只在步骤 9 完成后生效
- 若未达标，则继续执行 `8 → 9 → 门槛判断`
- **首次步骤 9 评分严格大于 9.2，仍不得直接进入步骤 10**；须继续执行一轮 `8（回炉修订）→ 9（独立严格复审）`，若复审评分仍严格大于 9.2（即连续两轮均过线），才允许退出回炉循环
- 连续两轮均过线前不得进入步骤 10、11、12

## 字数与新鲜度规则

- 每次正文在步骤 2 之后发生修改，都必须重跑字数脚本。
- 最终是否达到字数门槛，以最新 `Len / CJK / BodyCJK / Meets3500 / WithinRange` 为准。
- 3500–6500 的正文门禁，只能由 `BodyCJK` 判定；`Len` 与其它字符数只作参考。
- 若报告、日志或摘要写成“去空白统计”“NoWhitespaceLen”或其它自定义字符数，并拿它们判断是否达标，视为门禁失败，必须回炉重跑脚本并刷新相关工件。
- 最新正文、最新审阅报告、最新字数、最新日志、最终摘要必须彼此一致；任一过期都不得收尾。

## 摘要输出规则

- 只输出摘要，不输出正文
- 摘要必须与最新正文、最新报告、最新字数、最新日志一致
- 批量摘要必须显式输出聚合门禁字段：`gatePassRate`、`rotationOverloadCount`、`midPayoffCoverage`、`tailDriverCoverage`
- 若聚合门禁未达标（任一项不满足阈值），摘要中不得写“已完成”，只能写“部分完成/待回炉”

## 摘要与终审三层映射（新增）

- `鲜度一致层`：正文、报告、字数、日志、摘要是否为同一最新版本。
- `门槛判定层`：单章 `9.2+`、批次聚合门禁、脚本字数字段是否都真过线。
- `收口输出层`：摘要是否老实输出“已完成 / 不可放行 / 待回炉”，而不是抢跑庆功。

```text
- 阅读笔记：{READER_NOTE_FILE}
- 分章书评：{BOOK_REVIEW_FILE}

终稿约束自检：
- 字数：{是否满足=是/否}（Len={Len}；BodyCJK={BodyCJK}；Meets3500={Meets3500}；WithinRange={WithinRange}；统计来源=`scripts/count-chapter.ps1`）

- 信息不丢失：{是/否，覆盖概况：事件链/目标代价/关系变化/设定/伏笔/章末钩子}
```


## 批次放行判定（摘要层）

仅当以下条件同时满足，批次摘要才可输出“可放行/已完成”：

- gatePassRate >= 0.90
- rotationOverloadCount < 2
- midPayoffCoverage = 1.00
- tailDriverCoverage = 1.00

任一条件不满足时，摘要必须输出“不可放行/待回炉”，并在“需要进一步操作”中列出优先补丁动作。

## 一致性字段

日志至少应跟踪：

- `chapterFresh`
- `reportFresh`
- `wordCountFresh`
- `summaryFresh`

任一为 `false`，都不得输出最终摘要。

## 终审最小工件（新增）

```markdown
【鲜度一致层】chapterFresh / reportFresh / wordCountFresh / summaryFresh
【门槛判定层】单章9.2+ / 批次聚合门禁 / BodyCJK门禁
【收口输出层】已完成 / 不可放行 / 待回炉
【当前判定】Green / Yellow / Red
```

## Green / Yellow / Red 终审判定（新增）

- `Green`：鲜度一致、门槛全过、摘要口径诚实，可收口。
- `Yellow`：分数或字数基本到位，但鲜度或聚合门禁仍有待补项，不宜写“已完成”。
- `Red`：过期工件、门槛未过或摘要抢跑，必须回炉，不能以摘要掩盖未闭环事实。
