# 九部件与进度节奏续跑卡

适用于章节创作闭环中的卡顿、断点恢复、返工过大或"知道材料很多但不知道下一步写什么"的情况。

## 一个判断

当创作开始靠蛮力硬推，而不是靠结构顺推时，优先不要继续盲写，先回到进度节奏层。

## 一点五、续跑来源层（新增）

- `结构定位层`：当前章节处在四部分与五个里程碑中的哪个位置。
- `节拍续跑层`：本章 `3–7` 个节拍现在是不是各有主任务、彼此能顺推。
- `回写恢复层`：若当前继续写只会返工，应先回哪层恢复秩序（故事面 / 控制卡 / 场景节拍）。

## 九部件最小骨架

故事推进至少要重新确认以下九部件：

### 四个连续部分

1. 布局
2. 反应
3. 进攻
4. 解决

### 五个里程碑转折点

1. 引发性起点 / 危机起点
2. 第一情节点
3. 中间点
4. 第二关键点
5. 第二情节点

## 在闭环中的用途

### 1. 判断当前卡在哪

先问：

- 我现在写的是四部分里的哪一段？
- 这一段的任务是布局、反应、进攻还是解决？
- 眼前这一章是在承接哪个转折点，还是在把故事送向下一个转折点？

如果答不出来，说明当前稿件只是在堆内容，没有站稳结构位置。

### 2. 用进度节奏表恢复秩序

不要一上来重写整章，先用 1–2 行写出每个场景节拍：

- 这一场发生什么
- 它把故事往哪里推
- 它为下一个场景留下什么压力

一章通常先压成 3–7 个节拍就够用。

### 3. 识别盲写信号

出现以下任两项，建议暂停正文、回建微型进度表：

- 写了很多，却说不清每场的任务
- 场景之间有事，但方向不明
- 中段不断横移，没有明确回报
- 明知道要往后写，却不知道下一个场景为什么非写不可
- 返工越来越重，越改越散

## 闭环续跑步骤

1. 标出当前章节在九部件中的位置。
2. 用节拍表重写当前章的场景顺序，而不是先改正文措辞。
3. 确认每个节拍只承担一个主任务。
4. 确认本章怎样把读者送向下一处转折压力。
5. 再回正文层落稿、润色、去 AI 味与审阅。

## 最低可执行格式

### 单章节拍微表

- 节拍 1：开场异常 / 当前压力
- 节拍 2：第一次试探 / 反应 / 获取线索
- 节拍 3：局势加码 / 误判 / 新阻碍
- 节拍 4：中段回报 / 新认知 / 新代价
- 节拍 5：向章末钩子推进

### 多章节续跑表

- 第 N 章：承接什么转折点余波
- 第 N+1 章：给出什么中段回报
- 第 N+2 章：怎样把故事推向下一个里程碑

## 续跑最小工件（新增）

```markdown
【结构定位层】当前位于：布局 / 反应 / 进攻 / 解决
【节拍续跑层】当前章 3-7 个节拍主任务：
【回写恢复层】若继续盲写最该先回哪层：
【当前判定】Green / Yellow / Red
```

## 注意

- 进度节奏表不是为了把创作写死，而是为了减少盲写返工。
- 先在节拍层看清，再回正文层写，通常比在 4000 字废稿里挖出口更快。
- 真正的闭环不是"我硬写完了"，而是"我知道这章为什么这样排、下一章为什么要接着写"。

## Green / Yellow / Red 续跑判定（新增）

- `Green`：结构定位清楚、节拍主任务清楚、可继续顺推正文。
- `Yellow`：还知道大方向，但节拍局部发虚，需要先补微表再写。
- `Red`：不知道自己正写哪段、节拍互相撞车、再写只会扩大返工，应先停笔回上游恢复秩序。
