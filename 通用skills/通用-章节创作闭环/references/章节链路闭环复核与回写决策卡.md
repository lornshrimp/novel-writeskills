# 章节链路闭环复核与回写决策卡

适用于章节创作闭环在步骤 5–9.5 之间，判断这一章到底是“已完成有效闭环”，还是只是正文产物齐了、但链路仍有断点。

## 一、先明确：文件齐全不等于闭环成立

一章即使已经有：

- 正文
- 审阅报告
- 阅读笔记
- 分章书评

也不自动等于闭环成立。

真正的闭环，还要确认这条链是否咬合：

- 故事面供血
- 卷纲承接
- 章节控制卡
- 场景职责
- 开头补丁
- 章末补丁
- 润色 / 去AI味后的保真
- 终审结论

## 二、闭环层最该问的不是“做没做”，而是“有没有把上游承诺真写出来”

## 三、八项闭环复核

### 1. 供血线是否真实进章

- 本章说自己承接的故事面 / 主卖点，是否真在正文里显影？
- 还是只存在于卷纲与控制卡文字里？

### 2. 控制卡是否真正落地

- 主抓手是否落地
- 中段回报是否落地
- 章末钩子是否落地
- 下章接棒对象是否清楚

### 3. 场景职责是否站住

- 第一场真在抓眼
- 中段那场真在回报
- 最后一场真在钩子

### 4. 首尾补丁是否解决了问题，而不是遮住问题

- 章首强化后，第一场是否更稳
- 章末强化后，最后一场是否更尖
- 若只是更顺更好看，但职责仍虚，视为未闭环

### 5. 润色是否保住结构职责

- 是否更顺后仍保留主抓手、回报、钩子和卖点显影位

### 6. 去 AI 味是否保住供血线

- 是否更自然后仍保留抓力、回报、钩子、声音差

### 7. 审阅是否已定位到真正问题层

- 如果最新审阅只说“文笔可再增强”，但实际存在链路失配，视为审阅未闭环

### 8. 回写层级是否已做出判断

- 若当前问题仍在，是否已经决定优先回写层级

## 三点五、来源层映射（新增）

- `上游承诺层`：故事面供血、卷纲承接、控制卡承诺。
- `执行落地层`：场景职责、首尾补丁、润色与去AI味后的保真。
- `校验判定层`：审阅定位、回写决策、是否真的可以宣称完成。

闭环复核默认先判 `上游承诺层`，再判 `执行落地层`，最后判 `校验判定层`。

## 四、闭环判断表

| 复核项 | 通过标准 | 未通过时优先回写层级 |
| --- | --- | --- |
| 供血线 | 主卖点 / 故事面已进正文 | 故事面 / 卷纲 |
| 控制卡落地 | 主抓手 / 回报 / 钩子可指认 | 控制卡 |
| 场景职责 | 抓眼 / 回报 / 钩子场明确 | 场景职责 |
| 首尾补丁 | 增强了职责，不是遮羞 | 开头 / 章末补丁 |
| 润色保真 | 更顺但不更虚 | 场景职责 / 控制卡 |
| 去AI味保真 | 更自然但不断供 | 场景职责 / 控制卡 |
| 审阅定位 | 已指出真实问题层 | 审阅报告重判 |
| 回写决策 | 已明确下一步该改哪层 | 闭环不得宣称完成 |

## 五、何时即使评分过线，也不能视为完成

出现以下任两项时，即使分数 ≥ 9.2，也建议视为“形式过线、链路未闭”：

- 章首 / 中段 / 章末至少一处职责发虚
- 主卖点只能在摘要里说清，正文里感不强
- 结尾驱动存在，但下一章接棒对象不清
- 去 AI 味 / 润色后更顺，却明显更平
- 审阅报告没有指出真正的失配层

## 六、最小回写决策

若未闭环，必须至少写清：

1. 当前最主要的问题层
2. 优先回写层级
3. 为什么不是继续在当前层补丁
4. 回写后需重跑哪些步骤

## 六点五、最小闭环决策工件（新增）

```markdown
【上游承诺层】最虚项：
【执行落地层】最虚项：
【校验判定层】最虚项：
【优先回写层级】
【回写后需重跑步骤】
【当前判定】Green / Yellow / Red
```

## 七、一句话收口

**章节创作闭环的真正出口，不是“所有文件都生成了”，而是“这章从上游承诺到读者体验已经真正咬合，且知道一旦失配该回写哪一层”。**

## 八、Green / Yellow / Red 闭环复核判定（新增）

- `Green`：上游承诺、执行落地、校验判定三层都成立，可视为真实闭环。
- `Yellow`：文件基本齐，但至少一层仍有轻度失配，需要补跑或回写。
- `Red`：闭环仍停留在“文件齐了”的假象，继续往下走只会扩大错位。
