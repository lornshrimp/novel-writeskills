# 通用-设计封面生图提示词

封面提示词不是堆素材，而是把“这是什么书、卖给谁、视觉第一眼抓什么”说清楚。

## 常见触发词 / 用户说法速查

- 帮我做封面提示词
- 这本书封面该怎么出图
- 给我两套封面生图方案
- 这套提示词只会堆素材
- 帮我做平台展示友好的封面方案

## 何时使用

- 新书需要首版封面提示词。
- 现有提示词只会堆素材，不会控制构图、主体与平台展示。
- 需要对多个风格方向做 A/B 方案比较。

## 默认执行顺序

1. 先确定平台缩略图环境、目标读者第一眼感受与允许出现的文字元素。
2. 再读取作品相关材料，锁视觉母核、情绪母核与题材识别核。
3. 再锁主体：人物、物件、地标、光源或危险符号，并判断走人物 / 场景 / 物件 / 关系哪条主线。
4. 再按平台矩阵写构图、色调、风格限制、文字安全约束与禁行元素。
5. 最后产出至少 A/B 两套主提示词、负面提示词、排版说明与默认落盘方案。

## 首屏缩略图补充（新增）

- 封面方案默认必须回答：缩略图环境下一眼先看到谁 / 什么、先感到哪种情绪、先认出哪类题材；只剩“好看”但不识别，视为失败。
- 若作品卖点强依赖人物关系、情绪温差或身份对峙，至少一套方案要把这种关系信息压进人物距离、视线、姿态或光影结构里，而不是只堆背景素材。
- 若封面缩略图下主体、标题区与高亮意象互相争抢，默认优先牺牲装饰层，而不是牺牲题材识别与平台点击效率。

## 默认输出口径

- 默认输出至少 A/B 两套可直接交付的封面提示词方案。
- 默认同时给出主提示词、负面提示词、构图说明与平台展示提醒。

## 最低交付

- 至少 2 套有明确差异的主提示词方案。
- 每套都包含负面 / 避坑限制。
- 每套都说明主体、构图、色调、风格、文字层级与缩略图可读性。
- 默认包含可直接落盘的文件命名与交付结构。

## 硬规则

- 禁止封面只好看、不识题材。
- 禁止缩略图环境下主体淹没在花里胡哨背景里。
- 禁止人物脸、手、文字区与标题区相互打架。
- 禁止主提示词把题材词、卖点词、情绪词误写成会被模型抄进画面的额外文字。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责封面提示词本体与交付结构。
- 不负责书名设计、简介撰写或平台投稿材料组包。
