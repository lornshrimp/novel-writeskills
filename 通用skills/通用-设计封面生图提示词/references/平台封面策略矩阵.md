# 平台封面策略矩阵

## 使用说明

本矩阵负责封面提示词的**平台共性方法**：

- 判断缩略图环境下的第一识别点
- 判断平台读者更吃人物、场景、物件还是关系张力
- 约束标题区、主体区与背景区的竞争关系

题材化视觉元素、题材化场景菜单与题材化禁行项，继续交由对应题材 Skill 补充。

## 平台策略

### 起点中文网

- 核心目标：让读者一眼看出“这是强类型长篇，不是泛文艺封面”
- 视觉重点：体系感、职业感、规则感、升级空间
- 构图偏好：场景 + 核心职业物件 / 人物剪影
- 标题要求：稳、硬、清晰，不能花哨
- 避免：纯恋爱脸、短视频惊悚海报感、纯情绪图

### 番茄小说网

- 核心目标：让读者 1 秒内感知“有事发生了，而且不对劲”
- 视觉重点：异常、冲突、单一高识别焦点
- 构图偏好：强反差场景 / 单一危险物件
- 标题要求：大、狠、醒目，小图也能打出来
- 避免：信息过多、人物太小、文艺留白过重

### 知乎盐选

- 核心目标：让人觉得“像真实异闻 / 纪实缝隙 / 人性悬疑”
- 视觉重点：现实感、口述感、案件感、可信度
- 构图偏好：近现实场景、日常空间微异化
- 标题要求：避免过度玄幻化，更像都市异闻故事封面
- 避免：重特效、强游戏 UI、二次元感

### 豆瓣阅读

- 核心目标：让人觉得“有质感、有象征、有余味”
- 视觉重点：文学感、留白感、象征性、人性深度
- 构图偏好：物件主导 / 极简场景隐喻
- 标题要求：讲究留白与秩序，不能炸屏
- 避免：廉价恐怖、低俗血腥、强网文廉价感

### 微信订阅号

- 核心目标：碎片阅读里也能迅速理解题材和异常
- 视觉重点：现实共鸣、社交传播、日常微异常
- 构图偏好：都市微空间 + 单一反常细节
- 标题要求：手机端优先，清楚直接
- 避免：世界观堆砌、元素过杂、文字太小

### 百度百家号

- 核心目标：让读者第一眼明白“谁 / 在哪 / 出了什么邪门事”
- 视觉重点：搜索入口清楚、职业感直给、地域感可识别
- 构图偏好：职业人物 / 地域场景 / 核心禁忌物件的直给组合
- 标题要求：大、清楚、直接，避免文艺绕弯
- 避免：纯抽象象征、职业入口不明、空留白

### 小红书

- 核心目标：让读者觉得“这事可能就发生在我住的地方”
- 视觉重点：情绪抓手、生活空间异化、年轻代入
- 构图偏好：微空间场景 + 一个异常物件 / 异常人影
- 标题要求：清楚、轻锐、能在信息流里停住人
- 避免：宏大世界观海报感、硬核设定砸脸

### B站

- 核心目标：一眼有画面感，有讨论欲，也有二创潜力
- 视觉重点：名场面、反差感、轮廓清晰
- 构图偏好：人物剪影 + 反常场景 / 高识别瞬间
- 标题要求：缩略图里仍清晰，不要被画面吞掉
- 避免：轻小说恋爱海报感、纯梗图感、细节碎裂

### 新浪微博

- 核心目标：像撞见一张“出事现场”的截图
- 视觉重点：瞬时危机、主观视角、现场切片
- 构图偏好：危险动作瞬间 / 第一视角切片
- 标题要求：短平快，可转发讨论
- 避免：太像出版封面、太工整、缺少危机瞬时性

### WebNovel

- 核心目标：全球读者快速看懂“是什么书、危险点在哪”
- 视觉重点：电影感、边缘生存、超自然悬疑
- 构图偏好：人物剪影 + 环境危机
- 标题要求：干净、cimematic、readable
- 避免：文化专属符号堆积、不可译意象过满

### GoodNovel

- 核心目标：情绪张力、危险人物、关系拉力一眼可见
- 视觉重点：人物神秘感、凝视、距离、命定感
- 构图偏好：人物主导型 / 关系主导型
- 标题要求：有戏剧性，但不能盖过人物危险感
- 避免：过冷、过技术、纯纪实感过强

### MyFiction

- 核心目标：熟悉生活里藏着超自然秘密
- 视觉重点：日常都市奇幻、年轻感、单一钩子
- 构图偏好：生活场景 + 一个反常元素
- 标题要求：简洁、轻电影感、焦点单一
- 避免：元素开会、偶像剧海报味、无主焦点

## 平台差异使用规则

- 同一本书做多平台版本时，必须明确写出“为什么这个平台版本要这么变”
- 差异不应只停留在换颜色；至少应落到：主视觉类型、构图、色板、标题强度、缩略图策略中的两项
- 题材识别度必须保留，不能平台化后洗成通用惊悚海报
