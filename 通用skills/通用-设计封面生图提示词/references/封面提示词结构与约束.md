# 封面提示词结构与约束

## 推荐结构

1. 主体：谁 / 什么是第一焦点。
2. 场景：空间、天气、时间或危险氛围。
3. 构图：近景 / 半身 / 对角线 / 中央聚焦等。
4. 色调：主色、对比色、光源方向。
5. 风格：写实、插画、电影感、国潮、海外商业等。
6. 文字区预留：别把标题区堵死。

## 输入规则

### 必要输入

- `platform`：目标平台
- `title`：封面上要展示的书名；不分部小说统一使用小说的目标投放名，分部小说统一使用对应部的目标投放名

### 可选输入

- `genre`
- `hook`
- `theme`
- `protagonist`
- `mustInclude`
- `mustAvoid`
- `plannedText`
- `styleBias`
- `variants`（默认 `2`）

## 书名取值规则（强制）

- `title` 不等于任意上下文里出现的总书名、内部暂定名或系列母名，而是**封面实际入画主标题**。
- 不分部小说：`title` 统一使用小说的目标投放名。
- 分部小说：`title` 统一使用对应部的目标投放名。
- 若同一本书同时存在总书名、分部名、平台投放名多个口径，优先取本次目标平台实际投放所用名称；不得把总书名误放到分部封面上。

### 平台别名归一化

- `起点` / `起点中文网` / `Qidian`
- `番茄` / `番茄小说` / `番茄小说网`
- `知乎` / `知乎盐选` / `盐选`
- `豆瓣` / `豆瓣阅读`
- `微信` / `微信订阅号` / `公众号`
- `百度百家号` / `百家号` / `百度号`
- `小红书` / `rednote` / `RED`
- `B站` / `哔哩哔哩` / `bilibili`
- `新浪微博` / `微博` / `weibo`
- `WebNovel` / `webnovel`
- `GoodNovel` / `Goodnovel`
- `My Fiction` / `MyFiction` / `my fiction`

## 必须先读的上下文

优先从以下位置提取作品信息：

1. `小说简介/`
2. `小说大纲/`
3. `小说正文/`
4. `技术设定/`
5. `人物传记/`

若工作区无法判断具体故事，可基于**目标投放名 + 平台读者画像**做保守型封面方向，但必须明示推断依据。

## 核心思考步骤

1. 判断该平台读者更吃：世界观硬设定 / 强冲突异常感 / 真实纪实感 / 文学留白 / 情感关系与危险人物
2. 提炼三核：
   - **视觉母核**
   - **情绪母核**
   - **题材识别核**
3. 判断封面主导路径：人物主导型 / 场景主导型 / 物件主导型 / 关系张力型
4. 先确认封面主标题使用的投放名口径：不分部小说取小说目标投放名，分部小说取对应部目标投放名。
5. 决定标题排版：主标题位置、作者名位置、字体气质

## 输出结构

每套方案都建议按以下顺序给：

1. 平台判断
2. 封面方向摘要
3. 主提示词
4. Negative Prompt
5. 排版落地说明

## 常见约束

- 主体优先大于背景。
- 缩略图可辨识度优先大于细节堆叠。
- 风格词要服务题材，不要互相打架。
