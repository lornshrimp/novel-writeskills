# 文字安全与落盘规范

## 固定硬约束

- 封面默认是**竖版小说封面**
- 默认纵向比例建议：`3:4`
- 封面主标题必须使用按投放口径确定后的书名，并原样保留，不得擅自改写、缩写或重命名
- 主标题必须是画面中最大、最醒目的文字
- 作者名必须明显小于主标题，作为副级信息出现
- 不得加入平台 logo、APP logo、水印、二维码、角标广告语
- 不得做成影视海报风宣传图而忽略书封可读性

## 封面书名口径（强制）

- 不分部小说：封面主标题统一使用小说的目标投放名。
- 分部小说：封面主标题统一使用对应部的目标投放名。
- 若上下文同时出现总书名、分部名、平台投放名、内部暂定名，以本次目标平台实际投放名为准。
- 同一套方案中的主提示词、排版说明、文字安全限制里，`书名` 均指上述封面主标题口径，不得前后切换。

## 作者名平台规则

- 中文平台作者名默认使用：`产品经理独孤虾`
- 英文平台作者名默认使用：`lornshrimp`
- 除非用户明确要求，否则不得擅自改写上述作者名

## 文字安全规则（强制）

- 画面中只允许出现**已规划的文字元素**
- 默认情况下，已规划文字通常只有：封面主标题（按目标投放名口径确定） + 作者名
- 若用户明确给出 `plannedText`，则只允许这些额外文字出现
- 题材词、情绪词、卖点词、视觉目标词，必须改写成：场景、物件、人物状态、光影、构图要求，不能写成可能被模型照抄进画面的额外字
- 除已规划文字外，禁止把描述性短语放进引号、书名号、强调符号或独立成行后塞进主提示词

## 易误入的危险写法

- `突出“危险”“禁忌”“职业”`
- `画面写上“规则感”`
- `背景出现题材关键词`
- `传达“谁要先死”的感觉`

以上都应改写为画面表达要求，而不是文字要求。

## 安全改写模板

- `整体气质偏冷、规则感强、职业化、异常压迫感明确，但这些信息只通过场景、物件和人物状态体现，不以额外文字出现`
- `除已规划文字外，不要出现任何其他汉字、英文单词、标语、标签、UI字样`
- `用场景与物件传达题材识别度，而不是增加额外文字元素`

## 主提示词落笔模板（强制）

### 中文平台

- 无额外计划入画文字时：
  - `除主标题《目标投放名》与作者名外，画面中不要出现任何其他可读文字，包括汉字、英文、数字、标语、标签、界面字、背景字、漂浮字。`
- 有额外计划入画文字时：
  - `除主标题《目标投放名》、作者名以及以下计划入画文字外，画面中不要出现任何其他可读文字：{plannedText 列表}。`

### 英文平台

- 无额外计划入画文字时：
  - `No readable text in the image except the exact target release title and the author name.`
- 有额外计划入画文字时：
  - `No readable text in the image except the exact target release title, the author name, and these explicitly planned text elements: {plannedText list}.`

### 抽象总结句补丁

若主提示词中出现抽象卖点总结句，必须补一句：

- `这些信息只通过画面呈现，不以额外文字出现。`

## 输出格式最低要求

每套方案至少包含：

1. 平台判断
2. 封面方向摘要
3. 主提示词
4. Negative Prompt
5. 排版落地说明

默认输出 2 套方案：

- `方案A`：平台主流爆款取向
- `方案B`：差异化 / 质感取向

## Negative Prompt 最低项

每套方案的 Negative Prompt 至少覆盖：

- 低清晰度 / blurry / low detail
- 文字乱码
- 除已规划文字外的额外文字
- 漂浮字 / 背景字 / 标语 / UI字样 / 标签字
- 多余人物
- 多手多指
- 五官崩坏
- 过度血腥
- logo / watermark / QR code
- busy composition / cluttered background

## 默认落盘要求

- 结果默认应整理成 Markdown 文件落盘，而不是只停留在聊天展示
- 默认落盘目录：`小说简介/`
- 默认文件名建议：
  - 单平台：`《目标投放名》{平台名}封面生图提示词_vYYYY-MM-DD-r1.md`
  - 多平台：`《目标投放名》{平台A}与{平台B}封面生图提示词_vYYYY-MM-DD-r1.md`
- 若同日已有同名文件，递增 `rN` 版本号，不覆盖旧版
- 若本次只是修订既有方案，也优先新建修订版，不覆盖旧稿

## 最小追问原则

只有以下信息完全无法从上下文或工作区判断时，才追问：

1. 目标平台
2. 目标投放名（若作品分部，则为对应部的目标投放名）
3. 必须出现 / 必须避免的视觉元素

除此之外，优先直接开始。

## 成功标准

- 平台调性判断正确
- 缩略图下仍保有识别度
- 主标题与作者名层级清楚
- 除已规划文字外，没有额外误入文字
- 主提示词可直接复制到生图模型
- 默认已具备可落盘交付结构
