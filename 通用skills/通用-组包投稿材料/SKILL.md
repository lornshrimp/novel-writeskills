# 通用-组包投稿材料

投稿材料包不是把文件塞进一个文件夹，而是替编辑快速回答“这书能不能签、卖点在哪、作者会不会写”。

## 何时使用

- 为平台内投、外投或出版投稿准备材料包。
- 审计现有材料是否缺关键件、顺序混乱、抓手不清。
- 需要把多个写作工件压缩成编辑快速可读的一套包。
- 需要把“6000 字正文 + 2000 字大纲 + 简介 + 自检”之类的固定投稿四件套组出来。

## 开始前必须确认

1. 目标平台 / 投稿渠道。
2. 作品名 / 目标投放名。
3. 目标正文来源或样章范围。
4. 大纲材料层级：总纲 / 分卷 / 前十章节拍 / 其它。
5. 主角、核心机制、差异化卖点。
6. 当前简介、评估结论与其它既有材料是否存在。

若缺关键材料：

- 必须先列缺口。
- 必须下调完成度表述。
- 缺正文时不得伪造正文；缺总纲时不得硬造长线承诺。

## 默认执行顺序

1. 先判目标平台 / 渠道与材料标准。
2. 再盘点现有材料、缺口与是否已有可复用评估结论。
3. 再按“先抓眼、再说服、再证明能写”的顺序组装。
4. 若目标是固定规格内投包，按对应平台四件套标准压缩与重排。
5. 按默认落盘规则写回文件，并给出聊天侧最小回执。
6. 最后做缺件提示、补件顺序、风险说明与最弱一环判断。

## 最低交付

- 明确必备材料与可选增强材料。
- 给出建议排序，而不是杂乱罗列。
- 对每一件材料说明它解决编辑哪一个判断问题。
- 若目标是起点内投，必须至少交付：可提交正文、内投版大纲、简介、自检。

## 与其他 Skill 的协作边界

- 若平台适配或签约判断仍不清楚，应先吸收对应平台评估结果，再组包。
- 若简介需要大幅重写，应联动对应的简介类 Skill，而不是在组包时草草带过。
- 本 Skill 负责“把材料组织成可投包”；不替代上游的题材定位、正文精修或平台签约评估本体。

## 通用硬规则

- 禁止只有大纲，没有样章与卖点承诺。
- 禁止只有情怀描述，没有平台感与市场定位。
- 禁止材料之间互相打架：书名、简介、大纲、样章像四本书。
- 禁止缺关键材料却伪装成完整投稿版。
- 禁止把正文写成批注稿、摘要稿或节选拼贴稿。
