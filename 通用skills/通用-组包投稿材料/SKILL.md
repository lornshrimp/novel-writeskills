# 通用-组包投稿材料

投稿材料包不是把文件塞进一个文件夹，而是替编辑快速回答“这书能不能签、卖点在哪、作者会不会写”。

## 完整组包执行要求（强制）

- 投稿包不是把材料塞进一个文件里，而是替编辑快速回答至少 `5` 个问题：
	1. 这是什么书，能不能快速归类；
	2. 开篇会不会让我继续看；
	3. 简介承诺的卖点，正文和大纲会不会兑现；
	4. 这书有没有 `50` 万字以上的长线供血与首卷承诺；
	5. 作者是不是具备稳定把它写下去的执行能力。
- 若目标是起点或其它强连载平台，组包前必须先过内部快检：`30 秒归类`、`3 分钟继续看`、`3 万字 / 首卷值得继续评估`。
- 若目标是起点内投，必须显式区分**形式最低提交线**（约 `6000` 字正文 + `300` 字大纲 + 核心人设 / 人设表）与**默认交付的成熟内投包**；只够前者时，必须降级为“可投包预备版”或“形式最低提交版”。
- 若材料里存在平台签约评估报告、章节 / 人物 / 设定 / 分卷 / 总纲审阅报告，必须先吸收，再决定能不能正式组包；不得无视上游结论，直接把风险稿硬压成“可投版”。
- 若书名、简介、样章、大纲、人物、设定在承诺上互相打架，必须先统一题材承诺；统一前只允许输出“可投包预备版”，不得伪装成正式投稿版。

## 审核信号吸收要求（强制）

- 只要相关材料存在，组包时默认优先吸收以下能力的结论：
	- `通用-平台签约评估框架`
	- `通用-审阅章节正文`
	- `通用-审阅人物传记`
	- `通用-审阅技术设定`
	- `通用-审阅分卷大纲`
	- `通用-审阅总大纲`
- 若已有这些结论，必须把它们翻译成组包动作，而不是只在聊天中“参考一下”：
	- **停止正式组包**：存在致命问题时，先回炉；
	- **降级为预备版**：材料还没过线，但值得继续补；
	- **允许正式组包**：前门、人物、机制、长线、执行都已达到可投阈值。
- 若没有现成审阅报告，但材料明显暴露出与这些对象对应的硬伤，也必须按同等严口径处理，不得借“还没正式审过”逃避判断。

## 何时使用

- 为平台内投、外投或出版投稿准备材料包。
- 审计现有材料是否缺关键件、顺序混乱、抓手不清。
- 需要把多个写作工件压缩成编辑快速可读的一套包。
- 需要把“6000 字正文 + 2000 字大纲 + 简介 + 自检”之类的固定投稿四件套组出来。

## 开始前必须确认

1. 目标平台 / 投稿渠道。
2. 作品名 / 目标投放名。
3. 目标正文来源或样章范围。
4. 大纲材料层级：总纲 / 分卷 / 前十章节拍 / 其它。
5. 主角、核心机制、差异化卖点。
6. 当前简介、评估结论与其它既有材料是否存在。
7. 是否已有平台签约评估、章节、人物、设定、分卷、总纲审阅结论；若有，必须先吸收。

若缺关键材料：

- 必须先列缺口。
- 必须下调完成度表述。
- 缺正文时不得伪造正文；缺总纲时不得硬造长线承诺。

## 默认执行顺序

1. 先判目标平台 / 渠道与材料标准。
2. 再盘点现有材料、缺口，以及是否已有可复用评估 / 审阅结论。
3. 先做组包前质量闸门：停止正式组包 / 仅做预备版 / 可正式组包。
4. 再按“先抓眼、再说服、再证明能写、最后交代风险”的顺序组装。
5. 若目标是固定规格内投包，按对应平台四件套标准压缩与重排，并强制统一书名、简介、样章、大纲承诺。
6. 再做一次包内快检：30 秒归类、3 分钟继续看、3 万字 / 首卷继续评估理由是否成立。
7. 按默认落盘规则写回文件，并给出聊天侧最小回执。
8. 最后输出缺件提示、补件顺序、风险说明、当前最强与最弱一环。

## 默认输出口径

- 默认输出一套可投递、可解释、可快速评估的投稿材料包。
- 默认同时给出缺件说明、补件顺序与最弱一环判断。
- 默认还要显式说明：当前是正式投稿包还是可投包预备版，以及为什么。

## 最低交付

- 明确必备材料与可选增强材料。
- 给出建议排序，而不是杂乱罗列。
- 对每一件材料说明它解决编辑哪一个判断问题。
- 若目标是起点内投，必须至少交付：可提交正文、内投版大纲、简介、自检。
- 说明当前包是否已经吸收平台评估与审阅信号；若没有，必须写明为什么仍只能是预备版。
- 若需要实际落盘，`## 组包自检` 建议直接套用 `references/组包自检模板与风险矩阵模板.md`，不要临时拼凑检查项。

## 与其他 Skill / Prompt 的边界

- 若平台适配或签约判断仍不清楚，应先吸收对应平台评估结果，再组包。
- 若章节、人物、设定、分卷或总纲存在明显未过线风险，应先吸收对应审阅结果，再决定是否正式组包。
- 若简介需要大幅重写，应联动对应的简介类 Skill，而不是在组包时草草带过。
- 本 Skill 负责“把材料组织成可投包”；不替代上游的题材定位、正文精修或平台签约评估本体。

## 硬规则

- 禁止只有大纲，没有样章与卖点承诺。
- 禁止只有情怀描述，没有平台感与市场定位。
- 禁止材料之间互相打架：书名、简介、大纲、样章像四本书。
- 禁止缺关键材料却伪装成完整投稿版。
- 禁止把正文写成批注稿、摘要稿或节选拼贴稿。
- 禁止无视平台签约评估或审阅层的致命问题，硬把稿子包成“可投正式版”。
- 禁止简介承诺的卖点在样章里长期不兑现，或大纲承诺的长线在样章里毫无影子。
- 禁止用“设定很强”“世界观很大”冲淡前门无效、人物发虚、规则无边界、首卷断供等硬伤。
