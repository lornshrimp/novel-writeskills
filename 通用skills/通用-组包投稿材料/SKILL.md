# 通用-组包投稿材料

投稿材料包不是把文件塞进一个文件夹，而是替编辑快速回答“这书能不能签、卖点在哪、作者会不会写”。

## 起点默认平台与总纲模板符合性要求（强制）

- 若用户没有明确指定目标平台，默认按起点中文网口径处理。
- 当目标平台是起点中文网，或用户未标平台而走默认起点口径时，组包前必须先读取总纲中的 `## 模板组合声明` / 模板使用说明。
- 若模板声明缺失，必须回读 `通用skills/写作研究/起点模板/总纲模板/README.md` 反推主模板；若仍找不到更合适模板，默认按 `通用skills/写作研究/起点模板/总纲模板/九线写作总纲.md` 作为最低结构基线。
- 组包时只承认 `1` 个主模板 + `0–2` 个辅助模板。若大纲里并列多个主模板，视为模板声明失格，需要先纠正。
- **起点投稿材料中的大纲必须严格按照所选主模板的结构字段与格式要求撰写**（硬性强制，不是"尽量"）；声明了模板但没按模板格式写大纲，该大纲视为缺失，不得进入正式组包流程；辅助模板只作结构补丁，不得把大纲压成与模板无关的通用摘要。
- 若模板声明缺失但材料总体自洽，可继续组包，但默认只能输出“可投包预备版 / 需回炉版”。
- 若模板声明缺失且结构混乱到无法证明首卷承诺、长线供血与主卖点落点，则停止正式组包，先回到 `通用-设计总大纲` 修补大纲。

## 模板符合性前置闸门（强制执行顺序）

- 走起点口径时，**模板符合性检查必须发生在任何内投版总大纲 / 第 1 卷卷纲撰写之前**，不能写完大纲再回头补一句“主模板是什么”。
- 模板符合性至少要先得出 `4` 个判断：
	1. 当前主模板是什么；
	2. 辅助模板是什么、是否越位；
	3. 现有总纲 / 分卷是否已经能支撑按主模板字段重写；
	4. 当前包能否进入“正式投稿版”，还是必须降级为“可投包预备版 / 需回炉版”。
- 若上述 `4` 项判断里任一项无法成立，必须先停在“模板符合性结论”阶段，**不得直接产出正式版内投大纲**。
- 起点大纲撰写时，必须沿用主模板的字段层级与命名习惯；可以做题材化翻译，但不得把主模板改写成“高概念 / 卖点 / 长线潜力”式通用摘要体。
- 若主模板要求的是“主角人设 + 副本 / 单元案矩阵”，就必须逐项交付这些字段；若主模板要求的是“九线 / X线 / C线 / 单元案矩阵”，就必须逐项交付对应字段；**不能用自拟结构替代主模板字段**。
- 聊天回复与落盘文件中，必须显式写出：`主模板 / 辅助模板 / 是否九线回退 / 模板符合性结论`；缺少该结论视为流程未完成。

### 模板字段对齐证明（新增硬门）

- 仅仅“知道主模板叫什么”不算过闸；**必须读取主模板原文件，并抽取该模板的一级 / 二级必交字段清单**。
- 在起点内投大纲开写前，必须先形成一份 `模板字段覆盖表`，至少包含三列：`主模板原始字段`、`当前内投大纲对应位置`、`是否已覆盖（是 / 否）`。
- `模板字段覆盖表` 可以作为聊天中的中间检查，也可以写入内投文件的大纲区；但无论是否落盘，**都必须先完成这张表，才能进入正式版大纲撰写**。
- 若主模板字段在内投版总大纲 / 第 1 卷卷纲里仅被“概念性提到”，但没有以该字段要求的结构内容实质展开，仍按“未覆盖”处理。例如：主模板要求 `主角人设 -> 性格 / 外貌 / 背景 / 能力 / 萌点 / 金手指`，却只写了一个“主角与核心机制”摘要段，依然算缺项。
- 若 `模板字段覆盖表` 中存在任一必交字段为“否”，则：
	- 不得输出“正式投稿版”；
	- 不得声称“内投版总大纲 / 第 1 卷卷纲已完成”；
	- 必须先回炉重写大纲，再重新做字段覆盖检查。
- 禁止在大纲写完后凭主观判断说“应该差不多覆盖了”；**必须逐字段回看原模板并逐字段对表。**

## 完整组包执行要求（强制）

- 投稿包不是把材料硬塞进一个 Markdown 文件里，而是替编辑快速回答至少 `5` 个问题：
	1. 这是什么书，能不能快速归类；
	2. 开篇会不会让我继续看；
	3. 简介承诺的卖点，正文和大纲会不会兑现；
	4. 这书有没有 `50` 万字以上的长线供血与首卷承诺；
	5. 作者是不是具备稳定把它写下去的执行能力。
- 若目标是起点或其它强连载平台，组包前必须先过内部快检：`30 秒归类`、`3 分钟继续看`、`3 万字 / 首卷值得继续评估`。
- 若目标是起点内投，本仓库不再沿用旧的“`6000` 字正文 + `300` 字大纲”最低提交线，也不再把“单份 `2000` 字大纲”视为默认大纲件；统一按**版本化多文件投稿包**执行。
- 起点内投的四份对外交付件至少包含：`内投版简介`、`内投版正文（至少 30000 字连续正文）`、`内投版总大纲（最低 5000 字）`、`内投版第 1 卷卷纲（最低 5000 字）`；`组包自检` 作为作者侧同版本必备配套文件，必须与这四份主件保存在同一版本目录内。
- 缺任一对外交付件，都不得视为已达提交线；即使文件齐备，若模板符合性、前门效率、长线供血或执行准备仍不过线，也只能降级为“可投包预备版”，不得直接挂“正式投稿版”。
- **内投版总大纲必须覆盖全书骨架，第 1 卷卷纲必须单独成文件**：阅文官方作家专区收录的起点责编明确把“只写第一卷、后续卷次待定”列为主动失分项。起点编辑签约本质上是在赌一个 `50` 万字以上的长线连载；总大纲不覆盖全书，编辑无法判断“后续卷次是不是真的活的”，签约意愿会直接下降。后续卷次不必逐章细写，但每卷必须写出核心冲突、推进目标与阶段性卖点，不得留空或写“待定”；第 `1` 卷卷纲则负责把首卷承诺、前门节奏与卷末接棒细化到可执行层。
- 若材料里存在平台签约评估报告、章节 / 人物 / 设定 / 分卷 / 总纲审阅报告，必须先吸收，再决定能不能正式组包；不得无视上游结论，直接把风险稿硬压成“可投版”。
- 若书名、简介、样章、大纲、人物、设定在承诺上互相打架，必须先统一题材承诺；统一前只允许输出“可投包预备版”，不得伪装成正式投稿版。

## 开篇材料快检补充（新增）

- 组包时默认检查样章前 `300` 字是否立起人物抓手与第一压力，前三章是否形成至少一轮完整的钩子 / 蓄力 / 回报或升级。
- 若样章主要靠设定说明、抽象判断或作者说理推进，而不是靠动作、感官、选择与后果推进，默认归入前门无效风险。
- 若主角没有真实短板、务实动机与可见成长空间，即使设定、题材或概念不差，也不得把人物项包装成高成熟度卖点。
- 若正文明显不适配手机端阅读（大段厚墙、句群均匀、首屏迟钝），默认在样章可投性与前门效率项降级处理。

## 交易层统一补充（新增）

- 若作品核心卖点依赖情绪价值、关系拉扯、成长兑现、身份翻转或人物处境，组包时必须让其中至少一项在书名、简介、样章开头中形成连续显影；不能三份材料各讲各的。
- 若编辑在 `30` 秒内还看不清题材归类、主角当下困局与继续看的理由，默认只能降级为预备版或先修版。
- 组包时不得只把“概念成立”包装成“可投成熟度高”；必须同步回答人物可追、前门有效、长线可供血这三个问题。

## 审核信号吸收要求（强制）

- 只要相关材料存在，组包时默认优先吸收以下能力的结论：
	- `通用-平台签约评估框架`
	- `通用-审阅章节正文`
	- `通用-审阅人物传记`
	- `通用-审阅技术设定`
	- `通用-审阅分卷大纲`
	- `通用-审阅总大纲`
- 若已有这些结论，必须把它们翻译成组包动作，而不是只在聊天中“参考一下”：
	- **停止正式组包**：存在致命问题时，先回炉；
	- **降级为预备版**：材料还没过线，但值得继续补；
	- **允许正式组包**：前门、人物、机制、长线、执行都已达到可投阈值。
- 若没有现成审阅报告，但材料明显暴露出与这些对象对应的硬伤，也必须按同等严口径处理，不得借“还没正式审过”逃避判断。

## 何时使用

- 为平台内投、外投或出版投稿准备材料包。
- 审计现有材料是否缺关键件、顺序混乱、抓手不清。
- 需要把多个写作工件压缩成编辑快速可读的一套包。
- 需要把“版本化投稿目录 + 四份对外交付件 + 同版自检”这类固定投稿包组出来。

## 开始前必须确认

1. 目标平台 / 投稿渠道。
2. 作品名 / 目标投放名。
3. 目标正文来源或样章范围。
4. 大纲材料层级：总纲 / 分卷 / 前十章节拍 / 其它。
5. 若目标平台为起点中文网，或用户未指明平台，必须确认总纲中的 `模板组合声明` 是否存在：主模板、辅助模板、是否默认回退、各模板职责分别是什么。
6. 若模板声明缺失，是否已按 `总纲模板/README.md` 反推主模板；仍无法判断时是否已默认回退 `九线写作总纲.md`。
7. 主角、核心机制、差异化卖点。
8. 当前简介、评估结论与其它既有材料是否存在。
9. 是否已有平台签约评估、章节、人物、设定、分卷、总纲审阅结论；若有，必须先吸收。

若缺关键材料：

- 必须先列缺口。
- 必须下调完成度表述。
- 缺正文时不得伪造正文；缺总纲时不得硬造长线承诺。

## 默认执行顺序

1. 先判目标平台 / 渠道与材料标准；若未明确平台，默认按起点中文网处理。
2. 再盘点现有材料、缺口，以及是否已有可复用评估 / 审阅结论。
3. 若目标是起点中文网（含未标平台的默认起点口径），先读取总纲中的 `模板组合声明`；若缺失，则回读 `总纲模板/README.md` 反推主模板，仍无更合适模板时默认回退 `九线写作总纲.md`。
4. 先做大纲模板符合性初检：主模板是否合理、辅助模板是否越位、关键字段是否已进入总纲与投稿材料。
5. 读取主模板原文件，抽取必交字段，先制作 `模板字段覆盖表`。
6. **先给出模板符合性结论**：可正式进入大纲撰写 / 只能输出预备版 / 必须先回炉总纲。
7. 再做组包前质量闸门：停止正式组包 / 仅做预备版 / 可正式组包。
8. 再按“先抓眼、再说服、再证明能写、最后交代风险”的顺序组装。
9. 若目标是固定规格内投包，按“`内投版简介` → `内投版正文` → `内投版总大纲` → `内投版第 1 卷卷纲` → `组包自检`”的版本目录结构压缩与重排，并强制统一书名、简介、样章、大纲承诺。
10. 大纲写完后，必须用同一张 `模板字段覆盖表` 做一次反向复核；若仍有缺项，立即降级并回炉。
11. 再做一次包内快检：30 秒归类、3 分钟继续看、3 万字 / 首卷继续评估理由是否成立。
12. 按默认版本目录规则写回 `投稿材料/`，同步维护当前版本索引，并给出聊天侧最小回执。
13. 最后输出缺件提示、补件顺序、风险说明、当前最强与最弱一环。

## 默认输出口径

- 默认输出一套可投递、可解释、可快速评估的**版本化投稿材料包**。
- 默认同时给出缺件说明、补件顺序与最弱一环判断。
- 默认还要显式说明：当前是正式投稿包还是可投包预备版，以及为什么。
- 若目标平台为起点中文网，默认还要显式说明：主模板 / 辅助模板 / 是否触发默认九线回退，以及当前模板符合性结论。
- 若目标平台为起点中文网，默认还要显式说明：`模板字段覆盖表` 是否已完成、是否全部过线；若未全部过线，必须显式降级。
- 若目标平台为起点中文网，默认还要显式说明：当前版本目录、版本号、当前版本索引是否已更新。

## 最低交付

- 明确必备材料与可选增强材料。
- 给出建议排序，而不是杂乱罗列。
- 对每一件材料说明它解决编辑哪一个判断问题。
- 若目标是起点内投，必须至少交付：`内投版简介`、`内投版正文（至少 30000 字连续正文）`、`内投版总大纲（最低 5000 字）`、`内投版第 1 卷卷纲`、`组包自检`。
- 若目标平台为起点中文网，必须给出主模板、辅助模板、默认九线回退状态与模板符合性结论。
- 若目标平台为起点中文网，必须完成主模板原始字段抽取与逐字段覆盖检查；未完成时，`内投版总大纲 / 第 1 卷卷纲` 不得计入已完成项。
- 说明当前包是否已经吸收平台评估与审阅信号；若没有，必须写明为什么仍只能是预备版。
- 若需要实际落盘，`## 组包自检` 建议直接套用 `references/组包自检模板与风险矩阵模板.md`，不要临时拼凑检查项。
- 若需要实际落盘，必须建立清晰的作品根目录、版本子目录与当前版本索引，不得把不同轮次投稿材料混放在同一层。

## 与其他 Skill / Prompt 的边界

- 若平台适配或签约判断仍不清楚，应先吸收对应平台评估结果，再组包。
- 若章节、人物、设定、分卷或总纲存在明显未过线风险，应先吸收对应审阅结果，再决定是否正式组包。
- 若简介需要大幅重写，应联动对应的简介类 Skill，而不是在组包时草草带过。
- 若模板声明缺失或主模板明显不符，应先回源 `通用-设计总大纲` 修补，而不是在组包阶段偷偷重造立项逻辑。
- 本 Skill 负责“把材料组织成可投包”；不替代上游的题材定位、正文精修或平台签约评估本体。

## 硬规则

- 禁止只有大纲，没有样章与卖点承诺。
- 禁止只有情怀描述，没有平台感与市场定位。
- 禁止材料之间互相打架：书名、简介、大纲、样章像四本书。
- 禁止缺关键材料却伪装成完整投稿版。
- 未明确平台时，默认按起点中文网口径执行；禁止跳过这一默认口径直接写模糊平台版规则。
