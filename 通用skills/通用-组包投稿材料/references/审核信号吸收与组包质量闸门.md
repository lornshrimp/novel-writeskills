# 审核信号吸收与组包质量闸门

## 这份规则解决什么问题

投稿组包最容易犯的错，不是“缺文件”，而是**上游明明没过线，却被包装成像能投**。

因此，组包前必须先把“签约评估脑 + 审阅脑”吸进来，再决定：

1. 停止正式组包；
2. 只做可投包预备版；
3. 允许正式组包。

## 必须优先吸收的结论源

只要材料存在，组包时默认优先吸收：

- `通用-平台签约评估框架`
- `通用-审阅章节正文`
- `通用-审阅人物传记`
- `通用-审阅技术设定`
- `通用-审阅分卷大纲`
- `通用-审阅总大纲`

## 起点默认口径下的模板符合性信号（强制）

- 若目标平台是起点中文网，或用户未指明平台而使用默认起点口径，组包前必须检查总纲中的 `## 模板组合声明`。
- 若声明缺失，必须回读 `通用skills/写作研究/起点模板/总纲模板/README.md` 反推主模板；仍无法判断时，默认按 `九线写作总纲.md` 作为最低结构基线。
- 模板符合性主要看三件事：主模板是否合理、辅助模板是否越位、大纲 / 组包材料是否已经规范吸收对应模板的关键结构要求。
- 模板不符不等于内容本身一定差，但必须记入组包质量闸门，至少影响“能不能算正式投稿包”。

### 模板符合性必做动作（新增强制）

1. **先判模板，再写大纲**：模板符合性检查必须发生在任何内投版总大纲 / 第 1 卷卷纲撰写之前。
2. **先出结论，再决定版本**：模板符合性结论必须先回答——当前是可正式组包、只能预备版，还是必须先回炉总纲。
3. **先对字段，再写内容**：确认主模板字段后，内投大纲必须逐项对齐主模板字段；辅助模板只能补结构，不得改写主模板骨架。
4. **先写文件，再看是否过闸**的做法一律视为流程违规；即使内容质量不错，也必须按“模板不符”处理。
5. **先出字段覆盖表，再允许挂正式版名头**：必须从主模板原文件抽出必交字段，形成 `主模板原始字段 / 当前内投大纲对应位置 / 是否覆盖` 的覆盖表；缺任一必交字段时，不得挂正式投稿版。
6. **字段被概念性合并，不算自动通过**：例如主模板要求“主角人设六项”，若内投大纲只写一个“主角与核心机制”摘要段，仍按缺项处理，直到六项都被实质展开。

## 三档质量闸门

### A. 停止正式组包

命中以下任一类，原则上先回炉，不做正式投稿包：

- **章节层**：存在功能断裂、事实重大冲突、关键线索不公平、前门无有效回报。
- **人物层**：主角或重要对手动机 / 代价塌陷，能力万能，首秀失败，读者联结极弱。
- **设定层**：规则互相打架、没有边界、没有痕迹、没有现实锚点。
- **分卷层**：卖点过少 / 过晚 / 过弱，本卷只有推进没有回报，首卷承诺失效。
- **总纲层**：主卖点两卷以上失踪，主支线职责混乱，“首卷亮、中段断供”。
- **模板层**：模板声明缺失且结构混乱到无法证明首卷承诺、长线供血与主卖点落点。
- **模板执行层**：主模板虽已声明，但内投版总大纲 / 第 1 卷卷纲实际未按主模板字段撰写，仍然用了通用摘要体或自拟流水结构。
- **模板验收层**：没有制作字段覆盖表，或覆盖表显示仍有必交字段未被实质展开，却仍然挂“正式投稿版”。

### B. 可投包预备版

适用于：

- 材料还不完整；
- 仍未凑齐 `30000` 字连续正文、`5000` 字总大纲或独立的第 `1` 卷卷纲之一；
- 审阅层没有致命问题，但存在明显未修完的中高风险；
- 书名、简介、样章、大纲还没完全统一；
- 未规范使用对应主模板，但整体仍大体自洽；
- 已完成模板声明，但还没完成字段覆盖表，或覆盖表仍有缺项；
- 长线和执行准备还不足以支撑“正式提交”。

补充：若模板声明清楚，但当前只完成了“主模板判断”，还没把内投大纲按主模板字段重写，也只能算预备版，不能先挂正式版名头。

这时必须显眼标注：**可投包预备版，不是正式提交版。**

### C. 允许正式组包

至少同时满足：

- 题材与平台可快速归类；
- 前门样章能抓住编辑继续看；
- 主角、机制、代价与卖点已同框成立；
- 首卷承诺与长线供血清楚；
- 四份对外交付件齐备，组包自检与它们同版保存；
- 已按版本目录制落盘，`当前版本索引.md` 能明确指向本轮版本；
- 模板声明清楚，且大纲已按所选主模板的字段与层级实质撰写；
- 主模板原始字段已逐项完成覆盖检查，且覆盖表无缺项；
- 更新与投稿材料具备基本可信度。

## 五类审阅信号该如何翻译成组包动作

### 1）章节正文信号 → 样章裁判

- `P0 / P1` 风险未修掉：停止正式组包或降级为预备版。
- 章首抓力、中段回报、章末接棒对象不足：优先改样章，不先改大纲包装。
- 证据公平或规则边界不足：样章不得直接交付。

### 2）人物传记信号 → 卖点与简介裁判

- 主角一句话定义不成立：简介重写前置。
- 动机—代价闭环不足：样章与大纲都要改，不可只修简介。
- 大纲承诺的人物功能与传记不一致：整包降级。

### 3）技术设定信号 → 机制与说明裁判

- 规则互相打架 / 边界缺失：停止正式组包。
- 只有解释、没有动作链：样章和大纲都需回炉。
- 术语、编号、时间规定口径不统一：视为材料内部失分点。

### 4）分卷大纲信号 → 首卷承诺与中段续航裁判

- 前 5 章追读链不足：样章与大纲必须一起修。
- 卖点保真不足：简介与大纲不可继续夸大承诺。
- 单元案不能供血主线：大纲不可包装成“长线很稳”。

### 5）总大纲信号 → 长线潜力裁判

- 主线对象与主支线职责混乱：停止正式组包。
- 主卖点未进入主线系统：大纲必须重写。
- 首卷承诺到终局不一致：只能做预备版或先回炉。

## 组包自检至少要回看什么

1. 书名、简介、样章、大纲是不是同一套承诺。
2. 前 `300 / 500 / 1000 / 3000 / 6000` 字是不是都有人可继续看的理由。
3. `30000` 字提交正文是否连续、同向，能否持续证明首卷承诺不是只在首屏好看。
4. 主角、机制、代价、回报是不是同时出现，而不是各说各话。
5. 总大纲、第 `1` 卷卷纲、简介与正文是不是能互相印证。
6. 起点口径下，总纲是否规范使用了对应主模板，辅助模板是否只是补结构。
7. 当前包最强的一环和最弱的一环分别是什么。

## 组包时最容易自欺的三种情况

- 简介很强，正文很弱。
- 大纲很大，样章很空。
- 设定很亮，人物和回报发虚。

只要命中其中一种，就不能把稿子包装成“风险已清”的正式版。
