# 起点内投材料四件套与裁判标准

## 目标定位

起点内投材料不是资料堆叠，而是要尽快把编辑最关心的 3 类信息摆清楚：

1. 这是什么题材，市场入口清不清？
2. 开篇能不能抓住人，主角和机制亮没亮？
3. 这东西有没有 50 万字以上的连载潜力？

进一步说，真正有效的内投包，还得把第 `4` 和第 `5` 类信息也摆清楚：

- 简介承诺的卖点，正文和大纲会不会兑现？
- 作者有没有把它稳定写下去的执行能力？

换成更直白的快扫顺序，就是：

1. 我会不会先被它抓住看下去？
2. 我会不会信它真能兑现自己说的卖点？
3. 我会不会愿意继续看、继续评估，甚至往下约稿？

## 形式最低提交线 vs 默认交付线

- 本仓库不再沿用旧的“`6000` 字连续正文 + `300` 字大纲”最低提交线，也不再把“单份 `2000` 字大纲”视为默认大纲件。
- 起点内投的**四份对外交付件**统一为：`内投版简介`、`内投版正文（至少 30000 字连续正文）`、`内投版总大纲（最低 5000 字，且覆盖全书骨架）`、`内投版第 1 卷卷纲`。
- `组包自检` 不是对外主件，只能作为作者侧同版本内部核对文件存在，与四份主件放在同一版本目录内，供回炉、复核与下轮评估直接读取；**不得随稿发送给编辑**。
- 缺任一对外交付件，都只能视为**未达统一提交线**；四份主件齐备但模板符合性、质量闸门或执行准备不过线时，也只能标为**可投包预备版**，不得直接挂正式投稿版。

## 编辑包红线（强制）

- 对外交付四件中，禁止写入任何作者自评、自吹自擂、自我背书、最强最弱分析、残留风险点评、概率判断或替编辑做判断的话。
- 禁止出现类似：`当前最强一环`、`当前最弱一环`、`当前结论`、`可以直接进入编辑内投`、`编辑会继续看`、`建议现在投稿 / 内投` 之类句子。
- 若已有平台评估、审阅报告或作者侧核对结论，只能吸收成**实际改稿动作**，不得把这些判断原文搬进 `简介 / 正文 / 总大纲 / 第 1 卷卷纲 / 汇总导出件`。

## 起点口径下的总纲模板对齐（强制）

> **硬性强制要求：内投大纲必须按所选主模板的结构格式撰写，不是参考，不是"尽量"，是强制。** 声明了模板但没按模板写大纲，等于什么都没做；不按模板写的内投大纲视为大纲缺失，不得进入正式组包流程。

- 若目标平台是起点中文网，或用户未标平台而使用默认起点口径，组包前必须先读取总纲里的 `## 模板组合声明`。
- 模板声明至少要写清：主模板、辅助模板、是否触发默认 `九线写作总纲.md` 回退、选择原因与各模板职责。
- 若模板声明缺失，必须回读 `通用skills/写作研究/起点模板/总纲模板/README.md` 反推主模板；仍无法判断时，默认按 `九线写作总纲.md` 作为最低结构基线。
- **内投版总大纲与第 1 卷卷纲都必须严格按照所选主模板的结构字段与格式要求撰写**，不得用通用流水摘要或自拟结构替代；辅助模板只作为补结构约束，不得喧宾夺主。
- 若大纲未按主模板结构字段撰写，该大纲不得被计入"已完成大纲"，必须回炉重写。

## 开始前最小确认

1. 作品名 / 目标投放名。
2. 起点内投目标版本的题材定位（一句话）。
3. 目标正文来源：默认优先读取开篇连续正文，至少能组成 `30000` 字的连续提交正文。
4. 总纲 / 分卷 / 前十章节拍，至少命中其一。
5. 主角、金手指 / 异常机制、核心卖点。
6. 当前简介材料是否已有；若已有，判断是重写还是提炼。
7. 是否已有平台签约评估、章节正文、人物传记、技术设定、分卷大纲、总大纲审阅结论；若有，必须先吸收。
8. 若当前仍凑不齐 `30000` 字连续正文、`5000` 字总大纲或独立的第 `1` 卷卷纲，必须显眼标明本轮只能输出“可投包预备版 / 缺件版”。
9. 若走起点口径，是否已确认总纲中的主模板 / 辅助模板 / 默认九线回退状态；若声明缺失，是否已完成反推。

## 缺材处理规则

- 必须先列出缺口。
- 只能做“可投包预备版”，不得伪装成完整提交版。
- 缺正文时不得伪造正文；缺总纲时不得硬造长线承诺。

## 组包前质量闸门

### A. 先停止正式组包

命中以下任一类，原则上先回炉，再谈正式内投包：

- 样章前门存在明显致命问题：不开局、不立主角、不亮机制、不交回报。
- 主角 / 重要对手动机与代价塌陷，人物根本立不住。
- 规则 / 异常机制无边界、无代价、无现实后果。
- 首卷承诺和长线供血明显失效，大纲像空心大气球。
- 书名、简介、样章、大纲承诺严重打架。
- **大纲未按所声明的主模板结构字段与格式要求撰写**（硬性停止项：不按模板写的大纲不算已有大纲，必须先重写）。

### B. 只允许做可投包预备版

适用于：

- 仍有明显中高风险没修完；
- 材料不完整；
- 平台适配仍需进一步判断；
- 还没有把上游评估和审阅结论吸收到包里。

### C. 允许正式组包

至少同时满足：

- 可快速归类；
- 开篇能继续看；
- 主角与机制已亮；
- 四份对外交付件齐备，总大纲能证明长线，第 `1` 卷卷纲能证明首卷执行；
- 更新与投稿准备具有最低可信度。

## 固定四件套

### 1）内投版简介（建议 200–300 字主版）

- 必须像起点会点开的简介，不像大纲压缩块。
- 结构优先：
  1. 首句给异常 / 冲突；
  2. 次句给主角与处境；
  3. 中段给卖点与代价；
  4. 末句留单一尖锐悬问。
- 若已有简介，优先重写成更适合目标平台的点击版，而不是原文微调。
- 简介里承诺的最核心卖点，必须在样章前门、总大纲前段与第 `1` 卷卷纲里都能找到影子。

### 2）内投版正文（至少 30000 字）

- 必须是可直接提交的**连续正文**，不要夹批注、总结、解释语，也不再默认只交 `6000` 字开篇样章。
- 前 300 字见冲突。
- 前 1000 字见主角有效入场。
- 前 1000 字内让异常 / 金手指 / 核心机制显性。
- 6000 字内至少有一次可感知的小闭环或强钩子递进。
- 30000 字范围内应能让编辑看到：开篇抓力、第一轮有效回报、首卷承诺的早期兑现，以及至少一次中段续航证据。
- 最好能同时证明：30 秒可归类、3 分钟愿继续看、首卷承诺已有影子。
- 若已有章节审阅结论，必须先修掉会直接杀死投稿的前门硬伤，再交付正文。
- 若原始章节字数超标：允许压缩、删冗、重排，但必须保证叙事自然，不得变成节选拼贴感。

### 3）内投版总大纲（至少 5000 字）

**核心前提一：声明模板不等于按模板写。**

模板组合声明只是告知编辑「这本书用了哪套结构骨架」；真正有效的内投大纲，必须按所声明的主模板的结构字段逐项填充，用主模板的裁判逻辑驱动每个模块的内容。如果只写了一句「主模板：九线写作总纲.md」，随后却交出一篇与模板结构毫无关联的通用流水摘要，那么这个声明没有任何意义——等于告诉编辑「我知道规则，但我没按规则写」。

**核心前提二：总大纲必须覆盖全书骨架，不能只写第一卷。**

起点编辑签约一本书，实质是在赌一个 50 万字以上长线连载项目。他要判断的核心问题之一是：**这本书能不能写满 50 万字以上，后续卷次是不是真的活的**。一份只覆盖第一卷、后续卷次写"待定"的大纲，物理上无法回答这个问题。阅文官方作家专区收录的责编建议中，明确把"大纲写到第一卷，第二卷来个待定"列为**主动失分项**，是让编辑直接扣分的写法。

后续卷次不需要像第一卷那样逐章细写，但必须写出每卷的核心冲突、主线推进目标与阶段性卖点，足以让编辑判断"后续是不是活的、有没有 50 万字以上的供血"。

**字数最低线是 5000 字，不是上限。** 提交给编辑的结构证明越充分，呈现的小说优点越多，编辑掌握的信息越完整，签约成功概率越高；在 `5000` 字之外如果还有空间证明长线、供血链与人物代价，应当继续写，不要因为「达线了」就停下来。

#### 固定结构

0. **模板组合声明**（主模板 / 辅助模板 / 是否默认九线回退 / 选择原因与各模板职责分工）

1. **一句话高概念**

2. **题材定位与差异化卖点**

3. **主角与核心机制**

4. **主线目标与阶段推进**（按主模板要求的推进单元与层级填充，不得自由发挥结构层级）

5. **分卷推进方向（强制覆盖全书卷次）**：按主模板的卷 / 单元架构写，有几卷 / 几个单元就写几个，每卷必须写出核心冲突、主线推进目标与阶段性卖点；**严禁"后续卷次待定"**——这是编辑明确点名的主动失分写法，后续卷次无论多简略也必须写出骨架。

6. **长线潜力说明**（主模板要求覆盖的推进层级与阶段供血，逐层写清楚，不能只说「潜力很大」；必须让编辑看完后能判断"50 万字以上是不是真的能撑住"）

7. **若已有评估或审阅结论**：只允许把有效修订吸收进结构与内容本身，**不得在对外大纲里直写评估结论、自评或投稿建议**

#### 执行要求

- **按主模板结构写，不是写成通用大纲**。若主模板是九线写作总纲，就要把九线的关键字段（如主线、X 线、C 线、单元案矩阵等）逐项落到对应模块；若是其他模板，同理。
- **辅助模板只补结构痛点**，不得把整篇大纲都往辅助模板方向写，更不得喧宾夺主。
- **总大纲负责全书骨架，第 `1` 卷卷纲负责首卷细化**：总大纲中的第一卷应至少概括卷目标、核心事件节点与供血职责；更细的章级 / 场景级锚点转入独立的第 `1` 卷卷纲，不必把两份文件写成彼此复制品。
- 每一段都尽量服务起点男频的读法：目标、阻碍、回报、升级。
- 不能只讲世界观，必须讲得出首卷承诺、中段供血与长期升级路径——这些内容要对应到主模板指定的结构位置。
- 不是全书详细流水账，但结构骨架必须可以让编辑快速判断「结构是不是活的、是不是真的按模板跑的、后续卷次有没有实质性内容」。

### 4）内投版第 1 卷卷纲

- 第 `1` 卷卷纲必须独立成文件，负责把首卷承诺、前门节奏与卷末接棒细化到**可直接推进正文**的程度。
- 本文件不设单一硬性字数门槛，但不能薄到无法支持前 `10` 章、第一次有效回报与卷末更贵缺口的判断。
- 至少要写清：
  1. 首卷主题承诺与卷目标；
  2. 卷内主冲突 / 主对手 / 关键代价；
  3. 前 `10` 章节拍或等价节奏锚点；
  4. 第一次有效回报、第一次阶段闭环、卷末更贵缺口；
  5. 与第 `2` 卷接棒的缺口、压力或升级对象。
- 若总大纲已经覆盖到第一卷，也不能拿总大纲原文直接顶替卷纲；卷纲必须承担更细的执行职责。

### 5）作者侧内部核对清单（不对外发送）

至少包含：

- 题材是否可一句话归类
- 开篇四节点是否达标（300 / 1000 / 3000 / 6000）
- 主角与机制是否已亮
- 总大纲是否能看见 `50` 万字以上的推进力
- 第 `1` 卷卷纲是否已经把首卷承诺细化到可执行层
- 当前仍缺哪些文件 / 哪些区块需回炉
- 对外交付件是否误混入自评、风险判断或替编辑做判断的句子

## 版本目录要求

- 起点内投包默认落盘到：`投稿材料/起点内投/《作品名》/vYYYY-MM-DD-rN/`
- 同一版本目录内至少要有：`内投版简介`、`内投版正文`、`内投版总大纲`、`内投版第 1 卷卷纲`；若保留 `组包自检`，它只能是作者侧内部核对文件
- 作品根目录需保留 `当前版本索引.md`，显式指向最新一版，避免评估或后续组包时把不同时间版本混在一起

建议：实际落盘时优先直接套用 `references/组包自检模板与风险矩阵模板.md` 作为作者侧内部核对文件，不要把它并入给编辑的材料。

## 建议阅读链路

1. 目标平台研究资料
2. 既有平台评估报告或相关结论
3. 章节 / 人物 / 设定 / 分卷 / 总纲审阅结论（如有）
4. 作品当前简介
5. 总纲 / 分卷 / 前十章节拍
6. 主角与核心人物资料
7. 技术设定 / 异常规则
8. 开篇正文或候选正文

## 执行流程

1. 先判断当前作品是否适合做该平台的内投包；若明显不适配，先直说，不要硬包。
2. 先吸收已有平台评估与审阅结论，做一次组包前质量闸门判断。
3. 从正文里选出最适合目标平台读法的连续样章材料；必要时压缩、删冗、重排。
4. 从总纲 / 分卷 / 前十章节拍里按主模板结构字段分别撰写`内投版总大纲`与`内投版第 1 卷卷纲`：前者负责全书骨架，后者负责首卷执行，不得压成与模板无关的通用摘要。
5. 把简介改写成目标平台会点开的版本，并与正文 / 总大纲 / 卷纲承诺对齐。
6. 对整套材料做一次作者侧内部核对，确认四份对外交付件之间没有口径冲突，也没有混入自评或替编辑做判断的话。
7. 将结果写入对应版本目录，并同步更新 `当前版本索引.md`；回复中说明写入路径、当前版本号、覆盖 / 追加策略与仍待补写 / 回炉的文件。

## 文风与完成标准

- 直接、清楚、编辑友好。
- 正文不带解释腔，不夹“作者说明”。
- 大纲不写成空话，不写成设定百科。
- 简介不写成书名延长版，也不写成散文。
- 若材料不足，老实标“预备版”，不要虚张声势。
- 如果上游审阅和评估仍显示硬伤，不得用包装腔把问题洗没。
- 对外交付件不得写任何“作者自我评语”“替编辑定结论”或“自证可投”的句子。

## 组包顺序的抓眼-证明-继续看原则

### 先抓眼

- 书名、简介、样章前 300 字必须先把编辑留下来。
- 如果开头完全不抓人，后面的大纲价值也来不及被看见。

### 再证明

- 证明你不是只会包装：
  - 简介承诺的卖点，样章里要看得到影子
  - 大纲里写的长线，不要和样章像两本书
  - 主角、机制、冲突必须互相能对上

### 再做作者侧内部核对

- 风险、缺件、回炉说明只允许留在作者侧内部文件中；**不给编辑的材料里不写这些内容。**
- 对外交付件只负责把事实材料写清，不负责替编辑做判断。

### 再让编辑愿意继续看

- 内投包的目标不是一次讲完全部，而是让编辑判断“值得继续往下看”。
- 所以每一件材料都应回答：
  - 正文：你会不会继续翻？
  - 大纲：你相不相信它能继续写？
  - 简介：你能不能一眼归类卖点？

以下条件全部满足，才算完成：

- 有平台导向简介
- 有至少 `30000` 字的连续正文
- 有至少 `5000` 字的总大纲，且结构符合总纲中声明的主模板字段要求
- 有独立的第 `1` 卷卷纲
- 若生成内部核对清单，它已与对外交付件明确隔离，且不随稿发送
- 已完成组包前质量闸门判断
