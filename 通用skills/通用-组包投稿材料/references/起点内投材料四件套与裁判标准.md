# 起点内投材料四件套与裁判标准

## 目标定位

起点内投材料不是资料堆叠，而是要尽快回答编辑最关心的 3 个问题：

1. 这是什么题材，市场入口清不清？
2. 开篇能不能抓住人，主角和机制亮没亮？
3. 这东西有没有 50 万字以上的连载潜力？

## 开始前最小确认

1. 作品名 / 目标投放名。
2. 起点内投目标版本的题材定位（一句话）。
3. 目标正文来源：默认优先读取前 1–3 章或最适合凑成 6000 字的连续开篇材料。
4. 总纲 / 分卷 / 前十章节拍，至少命中其一。
5. 主角、金手指 / 异常机制、核心卖点。
6. 当前简介材料是否已有；若已有，判断是重写还是提炼。

## 缺材处理规则

- 必须先列出缺口。
- 只能做“可投包预备版”，不得伪装成完整提交版。
- 缺正文时不得伪造正文；缺总纲时不得硬造长线承诺。

## 固定四件套

### 1）内投版正文（约 6000 字）

- 必须是可直接提交的连续正文，不要夹批注、总结、解释语。
- 前 300 字见冲突。
- 前 1000 字见主角有效入场。
- 前 1000 字内让异常 / 金手指 / 核心机制显性。
- 6000 字内至少有一次可感知的小闭环或强钩子递进。
- 若原始章节字数超标：允许压缩、删冗、重排，但必须保证叙事自然，不得变成节选拼贴感。

### 2）内投版大纲（约 2000 字）

必须包含以下模块：

1. 一句话高概念
2. 题材定位与差异化卖点
3. 主角与核心机制
4. 主线目标与阶段推进
5. 前 10 章节奏锚点
6. 分卷或中前期推进方向
7. 长线潜力说明（为什么能继续写下去）

要求：

- 不是全书详细流水账。
- 重点是让编辑快速判断“结构是不是活的”。
- 每一段都尽量服务起点男频的读法：目标、阻碍、回报、升级。

### 3）内投版简介（建议 200–300 字主版）

- 必须像起点会点开的简介，不像大纲压缩块。
- 结构优先：
  1. 首句给异常 / 冲突；
  2. 次句给主角与处境；
  3. 中段给卖点与代价；
  4. 末句留单一尖锐悬问。
- 若已有简介，优先重写成更适合目标平台的点击版，而不是原文微调。

### 4）组包自检（简版）

至少包含：

- 题材是否可一句话归类
- 开篇四节点是否达标（300 / 1000 / 3000 / 6000）
- 主角与机制是否已亮
- 大纲是否能看见 50 万字以上的推进力
- 当前包最弱的一环是什么

## 建议阅读链路

1. 目标平台研究资料
2. 作品当前简介
3. 总纲 / 分卷 / 前十章节拍
4. 主角与核心人物资料
5. 技术设定 / 异常规则
6. 开篇正文或候选正文
7. 既有平台评估报告或相关结论

## 执行流程

1. 先判断当前作品是否适合做该平台的内投包；若明显不适配，先直说，不要硬包。
2. 从正文里选出最适合目标平台读法的连续样章材料；必要时压缩重排。
3. 从总纲 / 分卷 / 前十章节拍里压出平台可读的大纲，突出主线、节奏和长线。
4. 把简介改写成目标平台会点开的版本。
5. 对整套材料做一次简版自检，指出最弱一环。
6. 将结果写入内投材料文件，并在回复中说明写入路径、覆盖 / 追加策略与当前最需要补强的地方。

## 文风与完成标准

- 直接、清楚、编辑友好。
- 正文不带解释腔，不夹“作者说明”。
- 大纲不写成空话，不写成设定百科。
- 简介不写成书名延长版，也不写成散文。
- 若材料不足，老实标“预备版”，不要虚张声势。

以下条件全部满足，才算完成：

- 有可提交的正文部分
- 有约 2000 字的大纲部分
- 有平台导向简介
- 有简版自检
- 已写入仓库中的对应 Markdown 文件
- 已明确告诉用户这包材料当前最强与最弱的地方
