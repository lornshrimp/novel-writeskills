# 投稿材料包清单与排序

## 核心原则

投稿材料包的排序不是为了作者好整理，而是为了让编辑按最省力的顺序完成判断：

1. 这是什么书；
2. 为什么值得继续看；
3. 这书会不会兑现承诺；
4. 作者有没有能力继续写下去。

## 常见必备件（正式投稿包）

1. 书名 / 目标投放名
2. 一句话卖点
3. 内投版简介
4. 内投版正文（至少 `30000` 字连续正文）
5. 内投版总大纲（最低 `5000` 字）
6. 内投版第 `1` 卷卷纲

> 说明：`组包自检`、`当前版本索引`、`版本说明`、`模板字段覆盖表` 属于作者侧内部文件，**不计入给编辑的正式投稿包**。

## 建议排序（起点 / 连载平台默认口径）

1. **书名 + 一句话卖点**：先让编辑知道这是什么书。
2. **内投版简介**：再让编辑判断有没有点击与继续看的理由。
3. **内投版正文**：再证明前门真的抓得住人。
4. **内投版总大纲**：再证明这不是只会开头的稿子，而是真的有长线骨架。
5. **内投版第 1 卷卷纲**：再证明首卷承诺已经细化到可执行层，不是只会空喊“后面会更精彩”。

## 作者侧内部配套件（不随稿发送）

- `组包自检`
- `当前版本索引 / 版本说明`
- `模板字段覆盖表`

## 每件材料各自解决什么问题

- **书名 / 一句话卖点**：解决“这是什么题材、卖点清不清”。
- **简介**：解决“我为什么愿意点开、继续看”。
- **正文**：解决“这套承诺是不是真的落进了开篇”。
- **总大纲**：解决“它能不能支撑全书长线，不是一锤子买卖”。
- **第 1 卷卷纲**：解决“首卷承诺、前 `10` 章节拍和卷末接棒是不是已经落到执行层”。
- **作者侧内部配套件**：只解决作者自己的核对、追踪与回炉需求，不属于编辑阅读件。

## 可选增强件

- 当前版本索引 / 版本说明
- 模板字段覆盖表
- 平台适配说明
- 爽点 / 悬念 / 单元案矩阵摘要
- 作者写作节奏与更新能力说明
- 主角 / 人物补充卡
- 规则 / 设定补充卡
- 已有平台评估结论摘要

## 不推荐排序

- 先扔长大纲，再补简介与样章
- 先放作者说明，再放可读材料
- 先放设定百科，再放正文
- 把人物、设定、总纲全堆前面，让编辑迟迟看不到前门
- 把整套投稿材料塞进一个单一 Markdown 文件，靠文件名版本号硬撑，不做版本目录与当前版本索引
- 把 `组包自检`、`当前最强 / 最弱一环`、`当前结论`、`建议现在内投` 之类作者侧判断混进给编辑的材料里
