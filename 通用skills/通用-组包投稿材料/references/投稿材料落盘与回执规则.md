# 投稿材料落盘与回执规则

## 默认落盘原则

- 除非用户明确要求“只草拟不落文件”，否则必须把结果写入仓库。
- 优先维护可直接查看、可继续迭代的正式文件，不要把完整投稿包只停留在聊天窗口。

## 起点内投包默认路径

- 作品根目录：`投稿材料/起点内投/《作品名》/`
- 当前版本索引：`投稿材料/起点内投/《作品名》/当前版本索引.md`
- 本轮版本目录：`投稿材料/起点内投/《作品名》/vYYYY-MM-DD-rN/`

## 版本目录默认结构

默认不再把起点内投包写成单文件；每一轮投稿包都应落到独立版本目录中，推荐至少包含：

1. `00_版本说明.md`（可选但强烈建议）
2. `01_内投版简介.md`
3. `02_内投版正文_至少30000字.md`
4. `03_内投版总大纲_至少5000字.md`
5. `04_内投版第1卷卷纲.md`
6. `05_组包自检.md`
7. `06_模板字段覆盖表.md`（起点口径下强烈建议；若未单独成文件，至少要在 `00_版本说明.md` 或大纲文件中显眼保留）

其中，以下 `5` 件仍视为固定核心交付：

1. `01_内投版简介.md`
2. `02_内投版正文_至少30000字.md`
3. `03_内投版总大纲_至少5000字.md`
4. `04_内投版第1卷卷纲.md`
5. `05_组包自检.md`

若目标平台为起点中文网，或未标平台而走默认起点口径，则模板符合性结论与字段覆盖表也视为流程关键证据；缺少这部分内容时，视为流程未完成。

落盘建议：

- `00_版本说明.md` 建议写清：当前包定位（正式投稿版 / 可投包预备版）、模板符合性结论、材料来源、当前最强与最弱一环。
- `05_组包自检.md` 默认优先套用 `references/组包自检模板与风险矩阵模板.md`。
- 若对应题材 Skill 已提供题材化 `起点内投材料落盘模板.md`，则在通用结构基础上优先套用题材模板。

## 可选导出形态

若用户明确要求给编辑额外导出“单附件 / 汇总件”，可在当前版本目录下追加：

- `99_汇总导出件.md`

但这只是**额外导出件**；作者侧源文件仍必须保留版本目录制，不得倒退回“单文件组包版”作为默认形态。

## 默认版本策略

- 每一轮投稿包默认新建一个版本子目录，不与历史版本混放。
- `当前版本索引.md` 必须显式写明：当前推荐读取的版本号、版本路径、版本状态（正式投稿版 / 可投包预备版 / 需回炉版）、创建时间与简短变更说明。
- 只有用户明确要求“继续修这一版”，或当前版本目录本来就是统一维护的主目录时，才在同一版本目录内覆盖式更新；否则默认新建下一版。
- 若本轮仍属“可投包预备版”，应在 `当前版本索引.md` 与 `00_版本说明.md` 中显眼写明，不得让历史版本看起来像已经正式过线。

## 对话侧最小回执

完成后至少说明：

- 写入了哪个作品根目录、哪个版本目录
- `当前版本索引.md` 是否已更新
- 本轮是覆盖当前版本还是新建下一版本
- 当前包是正式投稿版还是可投包预备版
- 本轮是否吸收了已有平台评估 / 审阅结论
- 当前包最需要继续补强的地方

## 完成底线

- 没有实际落盘，不算正式完成
- 没有建立清晰的作品根目录、版本目录与当前版本索引，不算正式完成
- 没有说明落盘路径与版本策略，不算回执完整
- 没有指出最弱一环，不算完成收口
- 没有区分正式投稿版 / 可投包预备版，不算状态标识完整
