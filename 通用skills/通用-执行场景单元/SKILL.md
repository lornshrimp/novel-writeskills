# 通用-执行场景单元

场景单元是正文的施工现场，不是抽象梗概的停车场。

这是**通用母入口**：凡是“把一个场景落成可执行正文单元”的任务，都应优先从这里进入；若该场景同时属于微空间 / 受限空间，再继续调用 `通用-执行微空间受限场景` 做专项加压。

## 何时使用

- 设计或重写单章、单场景、章节微型剧本。
- 把大纲条目落成可写正文的场景施工卡。
- 修复正文里“有信息、没场景”“有情绪、没动作”的问题。

## 何时继续调用 `通用-执行微空间受限场景`

当场景同时满足以下任一特征时，不要只停在本 Skill，必须继续调用 `通用-执行微空间受限场景`：

- 空间明确且受限：楼道、电梯、值班室、茶水间、车库、候诊区、出租屋、公区
- 压迫主要来自“熟悉日常空间中的微反常”
- 需要把空间常态、第一处不对劲、角色动作与即时后果咬在一起
- 想写成“贴脸、受限、可复盘、后果立即”的小空间场景

## 默认执行顺序

1. 先写本场目标、冲突对象与失败代价。
2. 再确定抓眼入口：反常细节、危险信号、情绪缺口或行动倒计时。
3. 前 300 字内尽快抛出当前场景的第一处有效刺激，不要把核心冲突压到场景中后段。
4. 再排动作链、对话链、证据链与状态变化。
5. 最后补章末钩子与下一场触发条件。

若命中微空间 / 受限空间，再追加：

6. 调用 `通用-执行微空间受限场景`，补空间常态、微反常升级、动作失手与后果落地。

若需要从零搭场景、校准动作—后果连接、处理失手升级或去掉通用模板腔，再优先读取母 Skill 新增的四份 references，而不是直接把所有问题都压给子 Skill。

## 最低交付

- 有明确的场景目标与阻力。
- 有 3–5 个可拍镜头或动作节点。
- 有至少 1 次状态变化。
- 有明确钩子，不让场景平着收尾。

## 通用硬规则

- 禁止场景只有说明，没有行动。
- 禁止把关键信息全部塞进解释段落。
- 禁止每段都同一节奏，导致阅读无波峰。
- 禁止章末无问题、无代价、无新缺口。
- 禁止开场热机太久，前几百字都在交代背景却没有异常、风险或行动倒计时。
- 禁止职业知识只做炫技说明，必须落到判断、流程、动作与后果上。
- 禁止一整个场景只有启动没有中段回报；读者必须在本场拿到一点值回票价的变化。

## 与 `通用-执行微空间受限场景` 的关系

- 本 Skill 负责所有场景共用的目标 / 阻力 / 动作 / 后果骨架。
- `通用-执行微空间受限场景` 负责微空间场景里的空间常态、微反常、即时后果、现实针脚与专项检查。
- 两者不是平行重复关系，而是“母入口 + 专项增强包”关系。
