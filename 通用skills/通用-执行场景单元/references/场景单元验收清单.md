# 场景单元验收清单

- [ ] 开场是否在短时间内给出当前场景要解决什么问题
- [ ] 是否能清楚指出本场在本章中承担的是抓眼、推进、回报、升级还是钩子职责
- [ ] 是否存在至少 1 个可视化反常细节或风险信号
- [ ] 是否存在 3–5 个“起因 → 动作 → 结果”的微因果链- [ ] 若本场存在打破均势的动作（意外 / 挑衅 / 信息差 / 环境变化），是否有可见的**反应方落点**：至少一个人物产生情绪变化、行动变化或态度转变，而不只停在主角内心独白- [ ] 是否通过动作、对话、感官而不是纯说明传递核心信息
- [ ] 若场景主要靠对话推进，删掉提示语后是否仍能大致分出人物声音差
- [ ] 若场景主要靠对话推进，对话是否暴露了人物在护什么、怕什么或想抢什么
- [ ] 场景里是否至少给出一种读者联结入口：认同 / 同情 / 喜爱 / 好奇
- [ ] 是否出现了人物、关系、资源、位置中的至少 1 项状态变化
- [ ] 若本场承担中段回报或章末钩子职责，是否真的把这一职责落到具体动作 / 证据 / 揭露上
- [ ] 章末是否留下明确的下一步问题、风险或代价
