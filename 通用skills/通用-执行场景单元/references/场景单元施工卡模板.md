# 场景单元施工卡模板

```markdown
# 场景单元施工卡

## 一、场景身份
- 场景类型：
- 对应章节控制卡：
- 本场在本章中的职责：抓眼启动 / 推进试探 / 中段回报 / 升级代价 / 章末钩子
- 对应故事面 / 供血线：
- 日常用途 / 当前处境：
- 当前时段：
- 进入状态：
- 离开状态：

## 二、本场任务
- 场景目标：
- 主要阻力：
- 失败代价：
- 本场最晚必须发生的变化：

## 三、抓眼入口
- 开场抓眼方式（反常 / 风险 / 情绪缺口 / 未完成动作）：
- 第一处可拍细节：
- 为什么它会逼出第一个动作：

## 四、动作链
- 第一个动作：
- 第二个动作：
- 第三个动作（可选）：
- 哪个动作会立刻引发后果：

## 四点二、Beat 序列（1–4 个）
- Beat 1：类型｜承重 / 灵活｜做什么：
- Beat 2：类型｜承重 / 灵活｜做什么：
- Beat 3（可选）：类型｜承重 / 灵活｜做什么：
- Beat 4（可选）：类型｜承重 / 灵活｜做什么：

## 四点五、对话链（若本场有重对话必填）
- 主要说话双方 / 多方：
- 各自眼下在护什么、怕什么、想抢什么：
- 人物声音差依据（身份 / 熟悉度 / 职业惯性 / 情绪处理方式）：
- 哪一句先把关系顶歪：
- 哪一句会暴露人物短板或代价观：
- 读者更容易先跟上谁，为什么：

## 五、证据与信息
- 本场主证据载体：
- 哪一步让信息开始显影：
- 哪一步让判断发生变化：

## 六、后果与状态变化
- 主要后果：
- 状态变化（人物 / 关系 / 资源 / 位置 / 判断）：
- 若动作失手，最坏会怎样升级：

## 七、结尾钩子
- 本场局部回报：
- 场景末尾留下的新问题 / 新代价 / 新缺口：
- 下一场的触发条件：
- 若本场不是最后一场，它向下一场移交什么：
- 与下一场关系：因果 / 对比 / 并行 / 递进
- 过渡方式：时间跳跃 / 空间切换 / 情绪桥接 / 硬切
- 建议字数：

## 八、施工前验收
- 是否能说明本场承接的是本章哪一项任务：
- 目标是否清楚：
- 阻力是否具体：
- 动作是否真实存在：
- 进入状态与离开状态是否确实不同：
- 承重 beat 是否全部在场：
- 若有重对话，声音差是否可听见：
- 若有重对话，对话是否同时承担塑人和联结：
- 后果是否足够立即：
- 是否避免大段解释代替场景：
```
