# 场景单元施工卡模板

```markdown
# 场景单元施工卡

## 一、场景身份
- 场景类型：
- 日常用途 / 当前处境：
- 当前时段：

## 二、本场任务
- 场景目标：
- 主要阻力：
- 失败代价：
- 本场最晚必须发生的变化：

## 三、抓眼入口
- 开场抓眼方式（反常 / 风险 / 情绪缺口 / 未完成动作）：
- 第一处可拍细节：
- 为什么它会逼出第一个动作：

## 四、动作链
- 第一个动作：
- 第二个动作：
- 第三个动作（可选）：
- 哪个动作会立刻引发后果：

## 五、证据与信息
- 本场主证据载体：
- 哪一步让信息开始显影：
- 哪一步让判断发生变化：

## 六、后果与状态变化
- 主要后果：
- 状态变化（人物 / 关系 / 资源 / 位置 / 判断）：
- 若动作失手，最坏会怎样升级：

## 七、结尾钩子
- 本场局部回报：
- 场景末尾留下的新问题 / 新代价 / 新缺口：
- 下一场的触发条件：

## 八、施工前验收
- 目标是否清楚：
- 阻力是否具体：
- 动作是否真实存在：
- 后果是否足够立即：
- 是否避免大段解释代替场景：
```
