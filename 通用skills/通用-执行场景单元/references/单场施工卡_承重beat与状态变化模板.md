# 单场施工卡_承重beat与状态变化模板

> 用于把一个场景压成真正可执行的最小施工单元。适合单场职责发虚、动作链存在但承重 beat 不清、进入状态和离开状态不稳定、写完后很难验收的情况。

## 何时启用

- 一场戏删了以后，自己也说不清会损失什么。
- 场景承担关键职责，但只写得出“会发生一些事”。
- 场景需要明确承重 beat，避免后续正文扩写时越写越偏。
- 场景要承接控制卡里的中段回报、升级代价或章末钩子。

## 最小原则

- 一场戏默认只钉 **一个单一主任务**。
- 一场戏默认必须写清 **进入状态** 与 **离开状态**。
- 承重 beat 默认不可删除或无故换序。
- 若场景主要靠对话推进，继续叠加 `对话场景子模板_声音差_潜台词_升级阶梯.md`。

## 复检问答

1. 这场戏里，谁最想要什么？
2. 谁 / 什么力量正在挡住他？
3. 这一场结束时，哪一种状态被改写了？
4. 如果把本场删掉，章节四拍会缺哪一拍？
5. 哪个 beat 属于承重，不可磨平？

## 常见病灶

- 只有氛围，没有主任务。
- 只有信息，没有阻力。
- 只有动作，没有状态变化。
- 只有场景存在感，没有章级职责。
- 只有承压，没有反应方落点。

## 回写优先级

若本场写完仍发虚，默认按以下顺序回写：

1. 单一主任务
2. 进入 / 离开状态
3. 承重 beat
4. 失败代价
5. 下一场移交条件

## 一句话收口

**单场施工卡解决的是：这一场到底靠什么成立，靠什么承重，靠什么把局面推到下一个位置。**
