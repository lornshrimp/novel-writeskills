# 场景动作后果速配表

## 一、使用目的

动作的价值，不在“角色动了”，而在“局势立刻被改写了”。

一个有效动作应尽快带来以下至少一种：

- 信息显影
- 风险升级
- 误判成立
- 关系张力
- 规则显影
- 下一动作被逼出

## 二、基础原则

### 1. 一个动作优先接一个主后果

### 2. 后果应比动作更具体

### 3. 后果最好改变下一步选择

## 三、常用动作 → 主后果

| 动作 | 最适合接的主后果 | 常见落地方式 |
| --- | --- | --- |
| 走近 | 风险升级 | 距离缩短，暴露更多变量 |
| 伸手 / 触碰 | 规则显影 | 反馈不符合预期 |
| 回头 / 再确认 | 误判 | 方向、对象或时间判断被打乱 |
| 停住 | 信息缺口 | 暂时保命，但失去接触点 |
| 后退 / 退出 | 关系张力 | 被误解、被追问、错过时机 |
| 再试一次 | 风险升级 | 第二次反馈更糟或更明确 |
| 对照 / 重放 | 证据入场 | 新的口径差或细节跳出来 |
| 截图 / 记录 | 新风险 | 留痕的同时暴露自己 |
| 开门 / 推门 | 规则显影 | 门的状态、开法、回弹不对 |
| 叫人 / 发消息 | 关系张力 | 外部反馈和现场不一致 |

## 四、动作失败后的优先去向

- 确认失败 → 信息缺口
- 试探失败 → 风险升级
- 留证失败 → 误判加深
- 求助失败 → 关系张力

动作失败不能白失败，失败本身也要制造推进。

## 五、验收口径

一个合格的“动作—后果”连接，应满足：

- 动作有明确目的
- 后果来得快
- 后果能逼出下一动作
- 场景确实往前，而不是原地抖气氛
