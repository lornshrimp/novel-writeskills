# 职业爽点与信息差破局

## 职业爽点的四步骨架

1. 旁人先慌、先错、先乱做
2. 主角一句话点破职业规矩
3. 主角按流程操作，不乱炫技
4. 用一句行话或职业判断收束场面

若要把职业爽点写得更抓眼，建议在第 2 步前先补一个“误判正在制造代价”的瞬间，让主角的专业判断像救命，而不是像讲课。

## 职业爽点不是炫知识

真正有效的职业爽点，必须同时满足：

- 主角的职业知识能解决当前问题
- 行规、工具、程序或术语能直接推进情节
- 专业感能带来压制感、可信度与回报
- 专业判断必须在本场拿回一点掌控感，但随后最好再引出更深风险，避免爽点一次性打空

## 信息差破局的基本用法

优先写成以下链条：

- 别人看见了现象
- 主角看懂了规则
- 主角先试错 / 先验证
- 主角利用规则漏洞破局
- 代价或后患随后跟上

## 场景中的高价值职业元素

- 行规 / 口诀 / 禁忌
- 工具与其使用次序
- 流程摩擦
- 专业术语
- 旧经验与新判断冲突
- “别人把异常当巧合，主角知道那是流程节点被动过”的职业识别力
- “先用经验压住场，再用新判断修正经验”的专业升级感

## 适合落在场景里的信息差爽点

- 别人把异常当巧合，主角看出危险
- 别人只会蛮干，主角知道不能碰哪一步
- 别人只看表面，主角抓住证据细节
- 别人以为主角在冒险，主角其实在卡规则漏洞

## 高风险误区

- 把职业知识写成说明书
- 只有术语，没有动作
- 主角一上来就全知全能
- 只有职业炫技，没有后果与代价
- 只写“主角懂很多”，却不让旁人误判形成对照
- 只靠黑话立专业感，却没有工具、顺序、验证和失手成本

## 默认检查

- 这场戏里主角到底凭什么压过别人
- 职业流程是否真的影响了胜负
- 信息差是否被具体事件兑现
- 代价有没有跟上，而不是爽完就没事
