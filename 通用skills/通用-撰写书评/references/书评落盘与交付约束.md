# 书评落盘与交付约束

## 输入格式

用户可提供：

- 书评类型：全书 / 分部 / 分卷 / 分章
- 对应范围的标题（例如：分部名、分卷名、章节号与标题）
- 目标平台（可选，默认起点中文网）
- 写作角度（可选，默认综合角度）
- 该范围对应的创作意图、核心冲突、人物线索、主题标签等简要说明（可选）

## 文件存放位置约定

### 全书书评、分部书评、分卷书评

统一存放在仓库根目录下的 `书评/` 目录中。

文件名需包含以下五部分：

1. 范围 / 标题
2. 平台适配
3. 角度聚焦
4. 撰写日期
5. `.md` 后缀

日期格式统一为 `YYYY-MM-DD`。

平台适配与角度聚焦建议使用简短、稳定、可读的命名，例如：

- 平台：`起点`、`番茄`、`百家号`、`七猫`、`知乎盐选`、`豆瓣`、`小红书`、`微博`、`B站`、`MyFiction`、`WebNovel`
- 角度：`综合`、`设定`、`节奏`、`人物`、`氛围`、`议题`、`情绪`、`口碑`

示例路径：

- `书评/全书书评_起点_综合_2025-12-08.md`
- `书评/全书书评_百家号_口碑_2025-12-08.md`
- `书评/第1部·日常裂缝_起点_节奏_2025-12-08.md`
- `书评/1.1 消失的门禁记录_知乎盐选_议题_2025-12-08.md`
- `书评/1.1 消失的门禁记录_小红书_情绪_2025-12-08.md`

### 分章书评

按所属分部、分卷分别存放在 `书评/第X部/第Y卷/` 目录中。

文件名需包含以下五部分：

1. 章节号与标题
2. 平台适配
3. 角度聚焦
4. 撰写日期
5. `.md` 后缀

日期格式统一为 `YYYY-MM-DD`。

示例路径：

- `书评/第1部/第1卷/1.1.1 半夜有人刷了我家门禁_起点_综合_2025-12-11.md`
- `书评/第1部/第1卷/1.1.1 半夜有人刷了我家门禁_B站_情绪_2025-12-11.md`

## 默认交付格式

- 先用 1–2 句话标明“哪一类书评 + 对应范围 + 目标平台 + 写作角度”。
- 再给出以 `#` 标记起头、且标记后保留一个空格的标题。
- 再给正文。
- 再追加 `# 短评` 小节。

## 写回与版本策略

- 默认按独立文件落盘。
- 若用户未说明是否覆盖旧版本，默认保守新增新版本，不直接覆盖旧稿。
- 对话中至少说明：写入了哪个文件、采用什么命名、是覆盖还是新增。
- 只有用户明确说“只草拟不动文件”时，才允许不落盘。

## 最低自检

落盘前至少确认以下四点：

1. 标题已使用 Markdown 一级标题格式。
2. 正文后已追加 `# 短评`。
3. 文件名中已体现平台、角度与日期。
4. 书评没有因为压缩而变成剧情摘要或剧透说明。
