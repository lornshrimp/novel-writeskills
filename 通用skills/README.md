# 通用skills 目录说明

本目录用于承载**跨题材稳定复用的通用 Skill 能力本体**。

这些 Skill 是题材目录下各类题材版 Skill 的被路由目标；题材版 Skill 负责**可发现入口、题材包装层与题材边界补充**，而通用 Skill 负责维护共性能力本体。

## 目录定位

- `通用skills/`：放通用 Skill 能力本体
- `题材目录/.github/skills/`：放题材包装层、题材入口、题材专属补充
- `scripts/`：放不专属于某个 Skill 的通用脚本
- `仅用于参考/`：放外部参考，不作为规范性能力资产

## 使用原则

- 跨题材稳定复用的核心能力，应优先沉淀为通用 Skill。
- 已有通用 Skill 的能力，题材侧应建立对应题材版 Skill 作为默认发现入口。
- 题材版 Skill 必须显式加载、调用或组合调用对应通用 Skill。
- 共性规则只维护在通用 Skill 中，不在题材包装层平行复制。

## 命名说明

本 README 当前以**现有通用 Skill 目录名**作为标准名。

- 总表中的标准能力名应优先与当前目录名保持一致
- 若未来确需迁移命名，应先统一目录名，再同步更新 README 与题材包装层映射
- 在目录名未正式迁移前，不再在 README 中并行维护一套“推荐新名”

## 与题材包装层的关系

本目录只维护通用能力本体；题材目录下对应 Skill 应负责：

- 保留题材名可发现性
- 作为该题材的默认入口
- 显式加载、调用或组合调用对应通用 Skill
- 补充题材边界、题材禁行项、题材文件组和题材模板

不要在题材包装层重新复制通用 Skill 的完整规则库。

建议使用方式：

## 后续维护优先级建议

### 第一优先级

- 复核新补建通用 Skill 与各题材包装层的映射是否全部接通
- 复核新增通用 Skill 的 `references/`、模板与硬规则是否已达到可稳定复用的深度
- 复核多平台输出类 Skill 与各题材 `多平台输出SOP.prompt.md` 的路由一致性

### 第二优先级

- 逐步把输出类 Skill 也纳入“通用 Skill 体系总表”的长期维护范围
- 持续校对 README 与真实目录实况，避免再次出现“目录已落地但 README 仍写待补建”

### 第三优先级

- 视后续实际建设情况，再评估是否需要统一现有目录命名与新增能力命名

## 维护建议

更新本 README 时，优先同步以下信息：

- 新增了哪些通用 Skill
- 哪些标准能力名已有正式落点
- 哪些能力仍停留在 Prompt 层、尚未升级为通用 Skill
- 哪些题材包装层还未跟上通用层建设
