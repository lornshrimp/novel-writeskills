# 道具呼应弧线与出场追踪

> 来源口径：`经验总结.md` §5 道具呼应（Prop Resonance）、`修订体系.md` §C批次·道具呼应。

## 核心原则

**关键道具不是一次性道具。** 它在故事中反复出现，每一次出场都携带之前的叙事重量，形成一条可感知的成长弧线，为读者建立情感锚点。

与"伏笔"的区别：伏笔是为了后来的揭示而埋设；道具呼应是让同一载体在连续出场中不断升级内涵，让读者对它产生情感依赖。

## 弧线设计要求

每件关键道具至少应经历以下阶段：

| 阶段 | 内涵 | 示例 |
| --- | --- | --- |
| 初登场 | 功能性出现，交代外观与用途 | 担子第一次被主角挑上肩 |
| 升华 | 开始承载关系或情感 | 担子在夜行场景中成为主角与世界的连接 |
| 考验 | 在关键节点被"使用"或"威胁" | 担子在最危险的时刻被调整平衡 |
| 符号化 | 从功能性物件变成象征符号 | 担子象征传承与使命 |
| 终局 | 弧线完成：放下、传递或毁灭 | 担子被最终放下，意味着任务完成 |

## 出场追踪方法

C 批次修订时，按以下步骤检查：

1. **列出全文中出现超过 3 次的所有物品/地点**。
2. **逐次标注每一次出场推动了哪条弧线**（关系弧 / 成长弧 / 主题弧）。
3. **删除仅用来"凑字数"的出场**（出场后对任何弧线无推进）。
4. **补足缺口**：若两次出场之间间隔超过 15 章，且无任何暗示，考虑插入一次轻量呼应。

## 弧线示例（来自《挑担》创作实战）

### 石头的弧线

```text
ch-010 捡起：树洞里发光的东西，捡起来看看（功能性出现）
ch-015 发热：放在口袋里自己发热（异常信号）
ch-047 引导：主动引导主角改变方向（有意识）
ch-065 对话：和主角产生信息交换（会回应）
ch-079 有记忆：记住了主角的决定（有情感）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
弧线：死物 → 温热 → 有意识 → 会回应 → 有记忆 → 有情感
```

### 担子的弧线

```text
ch-001 出现：爷爷遗物，主角接手（身份绑定）
ch-010 夜行：主角挑着担子往后山跑（工具化）
ch-043 调整：在关键场景重新调整平衡（象征转折）
ch-070 自动适应：担子开始"配合"主角（关系升级）
ch-082 放下：完成使命后主动放下（传承符号）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
弧线：爷爷遗物 → 主角工具 → 成长伙伴 → 使命象征 → 传承放下
```

## 伏笔清单联动

道具呼应应与伏笔清单台账并行管理：

- 在台账中为关键道具建立独立条目，`类型` 标记为"道具弧线"。
- 每次出场登记到台账，标注该次出场推进了哪条弧线阶段。
- 终稿前复核：每件道具的弧线是否"有始有终"，不存在中断或空洞。

## 常见失误

| 失误 | 表现 | 修复方法 |
| --- | --- | --- |
| 道具遗忘 | 前半段出现，后来消失 | B 批次时建台账追踪 |
| 出场重复无推进 | 每次出现都是同一功能 | 赋予每次新的内涵层 |
| 过度象征 | 每章都用这件道具比喻主题 | 控制出现频率，保留稀缺性 |
| 终局无呼应 | 结尾忘了给道具"收场" | 在最终章安排弧线的终点动作 |
