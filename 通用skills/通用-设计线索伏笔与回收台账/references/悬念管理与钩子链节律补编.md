# 悬念管理与钩子链节律补编

> 吸收外部 `hook-techniques` 的单章钩子、跨章节连接与伏笔回收规则，用于把线索伏笔台账补强成“提问—推进—回收—再开新问”的长期管理系统。

## 一、单章钩子常用类型

- 突然揭示
- 紧急危机
- 未完成的动作
- 身份反转
- 两难选择
- 神秘物品或线索
- 时间限制
- 承诺或威胁
- 离奇消失
- 言外之意

选择时优先问：

- 会改变读者对已知事件的理解吗
- 会改变角色下一步选择吗
- 会让读者想立刻翻下一章吗

## 二、跨章节悬念管理

### 总原则

- 每章至少回应一个旧悬念，再提出一个新悬念
- 小悬念最好 `1–3` 章内回收
- 中悬念最好 `5–8` 章内回收
- 大悬念可跨卷 / 贯穿全书

## 二点五、来源层速判（新增）

- `单章钩子层`：本章出口钩、本章强度等级与钩子类型是否有变化意识。
- `跨章响应层`：上章出口钩、本章入口响应、下章预期响应是否清楚。
- `活跃负载层`：同时活跃悬念线的数量、强度、轮换顺序与失衡风险是否可控。

### 失衡信号

- 连续多章只开新问，不回旧问
- 一章内回收太多悬念，读者来不及消化
- 某条线长期无推进，读者已经忘了它

## 三、多线悬念并行

活跃悬念线建议控制在 `3–4` 条：

- 主线悬念
- 人物悬念
- 关系悬念
- 时间悬念
- 世界悬念

不同线应交替推进，不要让某条线被冷落太久。

## 四、钩子链验证

每章可按以下四项快速检查：

| 项 | 内容 |
| --- | --- |
| 上章出口钩 | 上章留下的问或动作 |
| 本章入口响应 | 直接 / 间接 / 延迟 / 断裂 |
| 本章出口钩 | 本章新留的钩子 |
| 下章预期响应 | 预计如何回应 |

### 响应模式

- 直接连续
- 间接关联
- 有意延迟
- 断裂（通常是失误）

有意延迟一般最好 `3` 章内回到这条线。

## 五、悬念强度等级

- 好奇级
- 关切级
- 迫切级
- 生存级
- 终极级

不同章节位置用不同强度，持续高压会让读者麻木。

## 六、伏笔埋设与回收

### 埋设技巧

- 嵌入日常场景
- 用更抢眼的事件分散注意力
- 给细节一个当前表面功能
- 多次自然出现，降低警觉

### 常见回收方式

- 对称场景
- 重复对话
- 物品回归
- 信息重组

## 七、禁忌

- 虚假悬念：结果只是无关紧要的误会
- 机械降神：解决方式靠此前从未提过的信息 / 能力
- 过度留白：留太多问号不给推进
- 低风险钩子：角色和读者都不真正在乎的结尾

## 八、台账字段建议补充

除既有编号、状态、窗口外，建议显式记录：

- 悬念强度
- 当前响应状态
- 上次推进章节
- 预期回收章节范围
- 回收方式
- 回收后余波
- 是否与其他悬念线交叉

## 红线

- 不得只记“埋了什么”，不记“怎么回”
- 不得只记“回收了”，不记“回收效果”
- 不得让活跃悬念线无限膨胀
- 不得连续多章用同一种钩子类型而无变化意识

## 最小悬念工件（新增）

```markdown
【单章钩子层】本章出口钩 / 类型 / 强度
【跨章响应层】上章出口 / 本章响应 / 下章预期响应
【活跃负载层】当前高关注线数量 / 主次 / 轮换顺序 / 失衡风险
【当前判定】Green / Yellow / Red
```

## Green / Yellow / Red 悬念判定（新增）

- `Green`：单章钩子、跨章响应、活跃负载三层都清楚，悬念链有节律不疲劳。
- `Yellow`：悬念仍能跑，但某一层开始失衡，例如响应偏慢或同类钩子过密。
- `Red`：只会开新问不会回旧问，或活跃线无限膨胀，悬念系统已经进入失控态。
