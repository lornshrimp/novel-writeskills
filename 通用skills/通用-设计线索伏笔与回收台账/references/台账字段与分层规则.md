# 台账字段与分层规则

## 建议字段

| 字段 | 用途 |
| --- | --- |
| 编号 | 唯一索引，建议按 `CL` / `FB` / `MIS` 分层 |
| 类型 | 线索 / 伏笔 / 误导 / 回收点 |
| 首次出现 | 首次提出的文件、卷、章 |
| 当前状态 | 提出 / 强化 / 待回收 / 已回收 / 废弃 |
| 关键载体 | 证据、动作、对话、物件、规则、制度卡口 |
| 回收窗口 | 建议在第几卷 / 第几章前后处理 |
| 回收动作 | 谁通过什么动作完成回收 |
| 爽点兑现 | 回收时给读者的情绪回报 |
| 代价 / 余波 | 回收后失去什么、引出什么新缺口 |

## 推荐管理表格式

```markdown
| 编号 | 伏笔内容 | 挖坑位置 | 填坑位置 | 跨越范围 | 重要程度 | 状态 |
|------|----------|----------|----------|----------|----------|------|
| T1 | 十年前旧案真相的关键拼图 | 第一卷第1章 | 第四卷结尾 | 跨卷 | 核心 | 进行中 |
```

## 分层建议

- `CL`：线索，偏事实拼图与取证路径。
- `FB`：伏笔，偏承诺、回声、前后呼应。
- `MIS`：误导，必须写清可证伪点。

## 跨层级伏笔分类

- **战略级伏笔**：影响全书主线走向的核心秘密（跨阶段 / 跨卷回收）
- **战术级伏笔**：影响某一阶段 / 卷群情节发展的重要线索
- **执行级伏笔**：影响章节阅读体验的具体细节

## 常用编号管理系统

- 总大纲伏笔：`T1`、`T2`、`T3`……
- 分部大纲伏笔：`P1.1`、`P1.2`、`P2.1`……
- 分卷大纲伏笔：`V1.1.1`、`V1.1.2`……

## 层级联动机制

- **向下细化**：总大纲的战略级伏笔必须在分部与分卷中有具体体现
- **向上汇报**：分卷中的执行级伏笔如果影响重大，需要在上级大纲中标注
- **横向协调**：同一层级的不同文件之间，伏笔口径必须一致

## 稳定规则

- 一条强线索最好在首次出现时就绑定一个可感知载体。
- 一条核心伏笔必须至少经历“提出 + 强化 + 回收”三个阶段。
- 误导必须能被更高等级的证据、动作或规则打穿。
- 不允许只写伏笔功能，不写具体埋设方式、发展轨迹、揭示技巧与呼应效果。
