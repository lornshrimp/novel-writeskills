# 台账字段与分层规则

## 建议字段

| 字段 | 用途 |
| --- | --- |
| 编号 | 唯一索引，建议按 `CL` / `FB` / `MIS` 分层 |
| 类型 | 线索 / 伏笔 / 误导 / 回收点 |
| 首次出现 | 首次提出的文件、卷、章 |
| 当前状态 | 提出 / 强化 / 待回收 / 已回收 / 废弃 |
| 悬念强度 | 好奇级 / 关切级 / 迫切级 / 生存级 / 终极级 |
| 关键载体 | 证据、动作、对话、物件、规则、制度卡口 |
| 上次推进 | 最近一次被强化、被回应或被转移的章节 |
| 钩子链状态 | 直接回应 / 间接关联 / 有意延迟 / 断裂风险 |
| 回收窗口 | 建议在第几卷 / 第几章前后处理 |
| 回收动作 | 谁通过什么动作完成回收 |
| 回收方式 | 对称场景 / 重复对白 / 物件回归 / 信息重组 |
| 爽点兑现 | 回收时给读者的情绪回报 |
| 代价 / 余波 | 回收后失去什么、引出什么新缺口 |
| 是否超期 | 否 / 预警 / 已超期 |

## 推荐管理表格式

```markdown
| 编号 | 伏笔内容 | 挖坑位置 | 填坑位置 | 跨越范围 | 重要程度 | 状态 |
|------|----------|----------|----------|----------|----------|------|
| T1 | 十年前旧案真相的关键拼图 | 第1卷第1章 | 第4卷结尾 | 跨卷 | 核心 | 进行中 |
```

## 分层建议

- `CL`：线索，偏事实拼图与取证路径。
- `FB`：伏笔，偏承诺、回声、前后呼应。
- `MIS`：误导，必须写清可证伪点。

## 来源层速判（新增）

- `台账结构层`：编号、类型、载体、首次出现、当前状态与跨层编号口径是否稳定。
- `调度窗口层`：上次推进、回收窗口、回收动作、回收方式与超期预警是否清楚。
- `公平兑现层`：爽点兑现、代价 / 余波、可证伪性与回收后新缺口是否明确。

默认顺序：先稳 `台账结构层`，再看 `调度窗口层`，最后判 `公平兑现层`；不要先写“终局会回收”，却连当前状态和窗口都不稳。

## 跨层级伏笔分类

- **战略级伏笔**：影响全书主线走向的核心秘密（跨阶段 / 跨卷回收）
- **战术级伏笔**：影响某一阶段 / 卷群情节发展的重要线索
- **执行级伏笔**：影响章节阅读体验的具体细节

## 常用编号管理系统

- 总大纲伏笔：`T1`、`T2`、`T3`……
- 分部大纲伏笔：`P1.1`、`P1.2`、`P2.1`……
- 分卷大纲伏笔：`V1.1.1`、`V1.1.2`……

## 层级联动机制

- **向下细化**：总大纲的战略级伏笔必须在分部与分卷中有具体体现
- **向上汇报**：分卷中的执行级伏笔如果影响重大，需要在上级大纲中标注
- **横向协调**：同一层级的不同文件之间，伏笔口径必须一致

## 稳定规则

- 一条强线索最好在首次出现时就绑定一个可感知载体。
- 一条核心伏笔必须至少经历“提出 + 强化 + 回收”三个阶段。
- 误导必须能被更高等级的证据、动作或规则打穿。
- 不允许只写伏笔功能，不写具体埋设方式、发展轨迹、揭示技巧与呼应效果。

## 悬念管理补充规则

- 默认尽量做到“每章回应一个旧问题，再打开一个新问题”；若长期只开新问不回旧问，台账里必须显式标红。
- 同时活跃的高关注悬念线最好控制在 `3–4` 条；超过时应写明主次与轮换顺序。
- 小悬念通常 `1–3` 章内回收，中悬念 `5–8` 章，大悬念可跨卷；超过窗口未推进，状态应改为 `预警 / 已超期`。
- 需特别标红的坏钩子：虚假悬念、机械降神、过度留白、低风险钩子、连续多章同类型钩子疲劳。

## 最小台账工件（新增）

```markdown
【台账结构层】编号 / 类型 / 载体 / 首次出现 / 当前状态
【调度窗口层】上次推进 / 回收窗口 / 回收动作 / 回收方式 / 是否超期
【公平兑现层】爽点兑现 / 代价余波 / 可证伪点 / 回收后新缺口
【当前判定】Green / Yellow / Red
```

## Green / Yellow / Red 台账判定（新增）

- `Green`：三层都清楚，台账既能追踪窗口，也能追踪兑现质量。
- `Yellow`：结构在，但窗口调度或兑现层仍偏虚，后续容易“记得编号，忘了回报”。
- `Red`：只有编号和名字，没有窗口、动作、余波与公平性口径，台账无法真正指导回收。
