# 回收窗口与兑现检查

## 回收窗口建议

- 章级小伏笔：建议在 1–3 章内回收或强化。
- 卷级伏笔：建议在本卷内至少有一次强化或阶段兑现。
- 长线伏笔：必须写清首次兑现节点，而不是只写“终局揭晓”。
- 每一层伏笔都应明确“提出 → 强化 → 回收 → 余波 / 再生”的链条。

## 爽点兑现检查

一条合格回收至少要回答：

1. 读者终于知道了什么？
2. 主角 / 角色因此得到了什么优势或更深困境？
3. 回收之后又撕开了什么新缺口？

## 详细设计要求

### 战略级伏笔详述格式

每个战略级伏笔尽量补齐以下五项：

- **埋设方式**：在哪个场景、通过什么事件、以什么伪装形式植入
- **发展轨迹**：前期 / 中期 / 后期 / 终局如何逐步显露
- **真相本质**：表面现象与真实目的的对比
- **揭示技巧**：如何引导读者从怀疑到确认
- **呼应效果**：与前文埋设的精确对应、重读价值与余波

### 回收失败信号补充

- 回收只说“解释清楚了”，却没有人物动作、证据载体或关系变化
- 伏笔完成解释，但没有带来爽点、代价或更大的问题
- 不同层级文件中的同一伏笔编号状态不一致

## 常见失败信号

- 回收只靠旁白解释，没有动作与后果。
- 回收结果没有改变人物关系、风险等级或认知版图。
- 伏笔被“解释掉”了，但没有形成任何追读奖励。
