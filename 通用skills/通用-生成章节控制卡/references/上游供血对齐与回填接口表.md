# 上游供血对齐与回填接口表

适用于生成章节控制卡前，把本章与故事面、卷纲、章清单的关系钉死，避免控制卡只对当前章负责、不对上游负责。

## 一、控制卡最常见的问题

控制卡常常写得很细，但仍然不稳，因为它只回答了：

- 这一章怎么写

却没有回答：

- 这一章为什么是现在写
- 这一章承接了谁
- 这一章写完后把什么交给下一章

所以在控制卡前，必须先做一次上游对齐。

## 二、本章生成前必答五问

### 1. 本章主要承接哪个故事面？

- 不是泛泛地说“承接主线”
- 而是明确到哪一个故事面 / 哪一条供血线

### 2. 本章位于本卷哪一段？

- 卷内布局
- 卷内反应
- 卷内进攻
- 卷内解决 / 卷尾缺口

同一条线在不同段里的写法不同。

### 3. 本章属于显性回报章，还是保温章？

- 显性回报章：读者这章明确拿到东西
- 保温章：不主打回报，但必须防止这条线熄火

### 4. 本章最值钱的上游抓手是什么？

- 是哪一个 A/B/C/D 抓手
- 是哪个节点责任
- 是哪颗卷尾交接钉子的前置信号

### 5. 本章结束后把什么交给下一章？

- 新行动
- 新误判
- 新证据
- 新代价
- 新位置变化

## 二点五、来源层映射（新增）

- `故事面承接层`：本章承接哪个故事面 / 哪条供血线。
- `卷段职责层`：本章位于本卷哪一段、章级身份为何。
- `接棒移交层`：本章结束后把什么动作 / 误判 / 新证据 / 新代价交给下一章。

若三层里有一层答不清，上游对齐就还没完成，控制卡不宜判成熟。

## 三、章节控制卡前置对齐表

| 项目 | 本章答案 | 若答不出意味着什么 |
| --- | --- | --- |
| 主要承接故事面 | [待填] | 本章脱离主供血线 |
| 本卷结构位置 | [待填] | 本章节拍易漂 |
| 章级身份 | 显性回报 / 保温 / 交接 | 不知道该写多重 |
| 本章主抓手 | [待填] | 这章容易虚 |
| 本章对主卖点的关系 | 强化 / 兑现 / 转化 / 保温 / 铺垫 | 卖点可能断供 |
| 下一章接口 | [待填] | 章末钩子容易假 |

## 四、显性回报章与保温章的不同写法

### 显性回报章

控制卡里必须写清：

- 具体回报类型
- 回报落在哪个场景单元
- 回报后局势如何变化

### 保温章

控制卡里必须写清：

- 本章虽然不主打回报，但如何让这条线保持热度
- 通过哪个小钩子、哪个现实锚点、哪个关系动作续火
- 下一次显性回报由哪章接棒

## 五、最值钱的不是“这章好看”，而是“这章对上游负责”

控制卡真正要避免的是：

- 本章局部成立
- 但写完后，对故事面没贡献
- 对卷级主问题没推进
- 对下章没交接

这种章很容易写得顺，却会偷走整卷节奏。

## 六、建议写进控制卡的新增前置字段

在正式控制卡前，可先写 6 行：

- 对应故事面：
- 对应供血线：
- 卷内结构位置：
- 章级身份：显性回报 / 保温 / 交接
- 本章主抓手：
- 下章接棒对象：

这 6 行答清，后面的场景单元通常就不容易失焦。

## 七、失败警报

出现以下任一项，建议先别生成正文：

- 本章承接故事面说不明白
- 本章是显性回报还是保温说不清
- 本章主抓手只有抽象情绪，没有载体
- 本章写完后下一章为什么必须接着写，说不出来

## 七点五、Green / Yellow / Red 对齐判定（新增）

- `Green`：故事面承接、卷段职责、接棒移交都清楚，本章能对上游和下游同时负责。
- `Yellow`：本章局部能写，但至少一层仍偏抽象，例如下一章接口还只有情绪没有对象。
- `Red`：本章承接谁、位于哪段、写完后交给谁都说不清；此时继续下钻控制卡只会把失焦写细。

## 七点六、最小对齐工件（新增）

```markdown
【故事面承接层】
【卷段职责层】
【接棒移交层】
【当前判定】Green / Yellow / Red
```

## 八、一句话收口

**控制卡不是只对本章负责，它还要对故事面的供血、卷纲的推进和下一章的接棒负责。**
