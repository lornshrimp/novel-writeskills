# 章节控制卡保真规则

## 一、控制卡不是摘要，而是施工命令

控制卡要回答“这一章怎么写”，而不是“这一章大概写什么”。

必须能直接指导：

- 起章位置
- 中段回报
- 场景切换
- 风险规避
- 章末钩子

## 二、每章至少要钉死的六件事

1. **核心改变**：这一章结束后，哪件事与开始前不同。
2. **主推进线**：本章主要服务哪条线。
3. **回归压力**：哪条冷线 / 旧债 / 旧人必须回来碰一下。
4. **中段回报**：读者在本章中段明确拿到什么。
5. **主要风险**：最容易写成空转 / 说明书 / 假高潮的位置。
6. **章末钩子**：危机 / 揭露 / 转折 / 未完成动作中的哪一种。

## 二点五、写前先对齐上游供血

在钉死六件事之前，至少要先答清：

- 本章主要承接哪个故事面
- 本章位于本卷哪一段
- 本章是显性回报章、保温章还是交接章
- 本章最值钱的主抓手是什么

若这些答不清，后面的控制卡容易局部成立、整体失焦。

## 二点六、来源层速判（新增）

- `上游供血层`：故事面、卷段、章级身份、本章对主卖点的关系。
- `章内职责层`：核心改变、主推进线、回归压力、中段回报、章末钩子。
- `场景拆分层`：3–5 个场景单元、各自压力变化、承重 beat 与交接关系。

控制卡如果要稳定服务正文，三层都要回答得出；缺哪一层，后面都会在正文阶段返工。

## 三、场景单元建议

常规章节建议拆为 3–5 个场景单元：

- 单元 1：压力已在场
- 单元 2：误判 / 推进 / 试探
- 单元 3：中段回报
- 单元 4：升级或代价
- 单元 5：章末钩子

不要求每章完全一致，但至少要保证“中段有事发生”。

## 四、写前失败信号

若出现以下任一项，控制卡应判定为未完成：

- 只能说“这一章推进剧情”，却说不出推进哪条线
- 说不出中段回报
- 说不出这章为什么非写不可
- 章末只有“埋个钩子”，但钩子没有类型和对象

## 五、章内强制节拍口径

生成控制卡时，至少要预先钉死：

1. 章首前 150–300 字准备用什么完成以下三项中的至少两项：
   - 异常 / 冲突触发
   - 情绪锚点
   - 悬念缺口
2. 章节中段准备用什么交付一次明确回报。
3. 章末准备用什么更危险的新问题推向下一章。

若控制卡只能写“这一章先铺垫，后面再爆”，则视为未达标。

## 六、中段回报不是可选项

章节中段至少完成 1 次可被读者明确感知的兑现。控制卡里应优先写成以下之一：

- 新证据落袋
- 误导被证伪
- 规则边界被试出
- 人物关系发生变化
- 一次有效反击成立

控制卡里若只有“中段继续调查 / 继续对话 / 继续推进”，没有具体回报类型，说明这一章仍然没有准备好。

## 七、现实锚点和小钩子密度要求

- 每 1000 字附近至少应预埋 1 个现实锚点。
- 每 500–800 字尽量布置一个小钩子，形式可以是：
  - 反常细节
  - 话术破绽
  - 新物证
  - 人物微反常
  - 一个未完成动作

控制卡不需要把这些都写成段落，但至少要能指出本章小钩子主要落在哪两个场景单元里。

## 八、卷首章节的额外要求

若本章属于卷内前 1–5 章，控制卡必须承担卷级吸引力职责：

- 明确本章负责哪一项卷级功能：规则落地、主角代价显影、首个可追线索、制度对抗、主线影子入场
- 明确本章是否承担连续冲击链中的一环
- 明确章末钩子如何把读者推向本卷下一章

## 九、章节标题与章末钩子也要进入控制卡

- 先判断原标题是否适配当前连载风格：短、信息密度高、含冲突 / 悬念 / 收益承诺其一
- 若不适配，控制卡应先提供主标题与可选第二标题候选
- 章末钩子必须明确类型：危机 / 揭露 / 转折 / 未完成动作，并写清对象

## 十、与上游工程文件的对齐要求

控制卡生成前，默认至少应交叉核对：

- 所属卷纲 / 章清单
- 当前单元案或事件骨架
- 线索台账
- 时间轴（如有关）
- 冷线 / 人物债务 / 关系桥接压力

## 十一、最终验收口径

一张可直接服务正文创作的控制卡，应至少满足：

- 能解释这章为什么非写不可
- 能指出本章的中段回报
- 能拆出 3–5 个带压力变化的场景单元
- 能说清本章风险与钩子类型
- 能对齐上游卷纲、事件骨架、冷线和现实锚点

## 十二、最小控制卡工件（新增）

```markdown
【上游供血层】故事面 / 卷段 / 章级身份 / 主卖点关系
【章内职责层】核心改变 / 主推进线 / 中段回报 / 章末钩子
【场景拆分层】场景1-5职责 / 承重 beat / 交接关系
【本章就绪判定】Green / Yellow / Red
```

## 十三、Green / Yellow / Red 就绪判定（新增）

- `Green`：三层齐备，且中段回报、章末钩子、场景拆分都可直接服务正文落稿。
- `Yellow`：主干可写，但仍缺 `1–2` 个非核心项，如场景拆分尚不够细或小钩子位置未钉死。
- `Red`：上游供血答不清、中段回报说不清、唯一职责发虚或场景单元无法支撑正文开写。
