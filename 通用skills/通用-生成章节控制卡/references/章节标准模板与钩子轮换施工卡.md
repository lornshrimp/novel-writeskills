# 章节标准模板与钩子轮换施工卡

> 来源口径：`网文技法.md`（标准章节模板、钩子轮换）。

## 标准章节模板（施工视角）

```text
前200字：钩子（冲突/悬念/异常）
中段A：主线或支线推进
中段B：微爽点（小反转/信息差/小成功）
中段C：持续加压或误判升级
后200字：留钩（未闭合问题/新变量）
```

## 四拍落点（控制卡必填）

- 钩子：读者为何继续读。
- 蓄力：冲突如何被放大。
- 回报：本章值回票价的具体收益。
- 留钩：下章接棒对象与驱动问题。

## 施工层级速判（新增）

- `上游对齐层`：本章承接哪个故事面、位于本卷哪一段、章级身份与主抓手是否清楚。
- `章内四拍层`：钩子、蓄力、回报、留钩是否真的分配到本章，而不是只有一句抽象概括。
- `跨章接棒层`：本章留钩是否可由下一章第一场直接承接，钩子类型是否轮换得当。

默认顺序：先判 `上游对齐层`，再判 `章内四拍层`，最后判 `跨章接棒层`；不要先写钩子类型，再发现本章根本没回报。

## 钩子轮换规则

避免连续 3 章使用同类钩子。推荐轮换：

```text
悬念钩 → 冲突钩 → 异常钩 → 动作钩 → 对话钩 → 悬念钩...
```

## 轮换审计清单

- 本章钩子类型是否与前两章重复。
- 本章钩子是否与章节职责匹配（信息章/对抗章/转折章）。
- 本章留钩是否可由下一章第一场直接承接。

## 失误警报

- 只有钩子没有回报：读者疲劳。
- 只有回报没有留钩：追读断裂。
- 章章同一钩子类型：节奏麻木。

## 最低可执行输出（控制卡内）

```text
本章钩子类型：____
本章中段回报：____
本章留钩对象：____
下章接棒问题：____
```

## 四拍最小工件（新增）

```markdown
【本章主抓手】
【四拍最重项】钩子 / 蓄力 / 回报 / 留钩
【本章中段回报载体】
【本章留钩对象】
【下一章第一场如何接】
```

## Green / Yellow / Red 就绪门禁（新增）

- `Green`：上游对齐、章内四拍、跨章接棒三层都清楚，可直接进入控制卡正文。
- `Yellow`：四拍大致成立，但某一层仍偏虚，例如回报可感度不够或下章接棒问题偏泛。
- `Red`：没有明确中段回报、留钩对象说不清、下一章第一场无法接、或本章钩子与章级职责不匹配。
