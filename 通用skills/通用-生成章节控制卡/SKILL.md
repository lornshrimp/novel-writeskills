# 通用-生成章节控制卡

控制卡不是摘要，而是施工命令。它要回答“这一章怎么写”，而不是“这一章大概写什么”。

## 保真迁移要求（强制）

命中写前控制任务时，除本文件外，**必须继续读取**：

- `references/章节标准模板与钩子轮换施工卡.md`（**必读** — 写前节拍模板与钩子轮换）

- `references/章节控制卡模板.md`
- `references/场景单元拆分法.md`
- `references/中段回报与章末钩子检查表.md`
- `references/爽点递进与300_1000_3000节奏锚点.md`
- `references/九部件预埋与进度节奏微表.md`
- `references/上游供血对齐与回填接口表.md`
 - `references/腾讯专栏动作反应施工增补.md`
 - `references/腾讯专栏情节细节坐标轴与新人禁忌增补.md`
 - `references/腾讯专栏动作反应两对完整结构增补.md`

## 卖点保真要求（强制）

- 每张控制卡都必须说明：本章对主卖点的关系是强化、兑现、转化、保温还是暂时铺垫。
- 若本章不是卖点显性回报章，也必须写清它怎样避免卖点断供，以及下一次卖点回报将在何处接棒。
- 若控制卡无法说明“这一章读者为什么还会继续记得本书卖点”，说明本章准备仍不合格。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，生成控制卡前必须先读取并纳入本章施工判断。
- 若目标平台为起点中文网，或未注明平台但默认按起点口径处理，且当前题材目录下尚无可用竞对主报告，生成控制卡前应优先调用 `通用-分析竞对作品`，至少补齐 **Top10 候选池、`3–5` 本主样本与四层深拆速览**；若任务限制导致本轮无法补齐，必须显眼记录盲区，并降低本章开写就绪判定。
- 不得把这些报告只当成背景资料；控制卡必须显式说明本章准备在哪些维度追平、反超竞对，以及哪些旧坑必须避开。
- 控制卡的默认目标不是“能开写就行”，而是让当前作品在关键维度上**全方位超越竞争对手**；若看不出明确超越路径，应继续补施工而不是草率开写。

## 起点化竞对吸收口径（新增）

当控制卡服务起点中文网，或未标平台而默认按起点基线处理时，默认把竞对吸收拆成以下四层，并翻译成写前施工约束：

- **市场数据层**：用榜单位次、均订 / 首订 / 收订比代理信号、推荐资源位等，反推本章在前 `300 / 1000 / 3000` 字抓力、中段回报、章末续读驱动与题材归类效率上至少该达到什么强度。
- **内容创作层**：重点吸收主样本在章首切入、动作链、信息差、人物抓手、规则边界、回报位与钩子位上的做法，把它翻成控制卡里的唯一职责、四拍分配、场景职责与风险提示。
- **运营策略层**：重点吸收主样本的追读工程、本章说讨论点、更新节奏匹配、卷首 / 卷尾功能位与评论触发设计，把它翻成中段回报与章末留钩施工要求。
- **受众反馈层**：重点吸收本章说 / 书友圈 / 书评区里高频被夸的值点、被骂的拖点、老白读者不耐烦的病灶，提前写进本章控制卡的避坑和加码清单。

起点语境下，默认尽量使用以下术语，而不是写成空泛的“读者会喜欢这一章”：`均订`、`高订`、`首订`、`追订`、`收订比`、`分类月票榜`、`畅销榜`、`新书榜`、`三江`、`本章说 / 段评`、`书友圈`、`老白读者`。拿不到精确值时必须写成代理信号，不得伪造具体数字。

## 常见触发词 / 用户说法速查

- 先给我这章控制卡
- 这一章到底要推进什么
- 帮我拆成几个场景单元
- 这章的中段回报是什么
- 写前作战卡 / 施工卡
- 哪些冷线该在这章回来了
- 这章的章名候选先给我

## 何时使用

- 在正式写章节前，需要把卷纲 / 章纲压缩成可执行的本章控制卡。
- 中段容易塌，需要先明确本章任务、回报、冷线回归和钩子。
- 用户希望先看章名候选、POV、节奏档位、场景单元，再决定是否动笔。

## 本 Skill 的核心任务

每张控制卡至少要写清：

1. 本章**必须改变什么**。
2. 哪条线要推进，哪条线要回归。
3. 本章中段的**明确回报**是什么。
4. 本章的主要场景单元和压力变化。
5. 本章风险：最容易写崩的点是什么。
6. 章末留给下一章的钩子类型与对象。
7. 本章怎样让主卖点继续被看见或被记住。

## 最小核心与问题引擎校验（新增）

- 开卡前必须先写一句“最小核心”：`谁在什么处境下，为了什么目标，正被什么阻力压迫`。
- 若该章属于第一章或卷首章，控制卡必须显式写出“问题引擎”：读者看完本章后心里要留下的具体追问是什么。
- 若控制卡只有信息点、设定点、背景点，没有可被读者追问的问题，视为未达开写条件。
- “问题引擎”必须与本章动作链绑定，不能只写抽象悬念词（如“真相如何”）而没有现场触发物。

## 章节四拍与首屏控制（新增）

- 控制卡默认要写清本章的四拍分配：**钩子、蓄力、爆发 / 回报、留钩**，至少说明哪一拍最重、哪一拍绝不能缺位。
- 若本章承担卷首、卷中转折或关键续读职责，控制卡里必须显式写出前 `300` 字准备让读者看到什么人物抓手、眼前麻烦或反常信号。
- 中段回报不能只写“推进了主线”；必须写成读者可感知的具体收益：关系推进、误判修正、局面反击、信息开口、身份抬升或代价落地。
- 若本章主要是保温章，也必须写明如何维持卖点可见度，避免连续多章只有流程推进没有值回票价的变化。

## 唯一职责、beat、情感弧与就绪判定补强（新增）

- 每张控制卡都必须单列“**本章唯一职责**”与“**不应包含**”：唯一职责需要能用一句话说清本章让读者完成什么认知 / 情绪 / 追问；不应包含则负责声明哪些线、不该在哪些位置抢跑的解释与结论不在本章处理。
- 每个场景默认补齐：`POV`、场景目标、情感转变、参与角色、情感基调、冲突 / 张力来源、信息流、与下一场景的关系、过渡方式、建议字数。
- 每个场景默认拆 `1–4` 个 beat，并标 `承重 / 灵活`；承重 beat 负责本章叙事承重，默认不得删除或无故换序，灵活 beat 可合并、弱化或换实现方式。
- 情感弧必须写清读者情感入口、读者情感出口与弧线模式；若入口出口几乎无变化，本章大概率会读感发平。
- 控制卡尾部默认补一张“钩子链验证”：上章出口钩、本章入口响应、本章出口钩、下章预期响应；上章钩子断裂且无解释，默认降级处理。
- 就绪判定默认分 `Green / Yellow / Red`：`Green`=职责、beat、情感弧、钩子链、节奏权重、字数分配都齐；`Yellow`=缺 `1–2` 个非核心项但 writer 可谨慎推进；`Red`=唯一职责说不清、承重 beat 漏失、钩子链断裂、连续重章无调整方案等，需先回炉。

## 执行顺序

0. 若按起点口径执行且当前题材目录下缺少可用竞对主报告，先调用 `通用-分析竞对作品`，补齐 `Top10 候选池 → 3–5 本主样本 → 四层深拆` 的最小结论集。
1. 读取上游卷纲 / 章纲 / 单元案 / 相关台账。
2. 若按起点口径执行，先吸收竞对主样本四层结论，并写清本章要追平 / 反超的关键项。
3. 先做冷线与债务扫描，再决定本章要承接谁。
4. 先钉死六件事：核心改变、主推进线、回归压力、中段回报、主要风险、章末钩子。
5. 先用“上游供血对齐与回填接口表”钉死本章承接哪个故事面、位于本卷哪一段、属于显性回报章还是保温章。
6. 若本章结构位置发虚，先用“九部件预埋与进度节奏微表”确认本章所在段落与 3–5 个节拍。
7. 按模板输出控制卡：身份、任务、回报、场景、风险、标题候选、钩子。
8. 若控制卡无法说清中段回报，说明本章仍未准备好。
9. 若本章需要拆成 3–5 个较清晰的动作单元，继续按场景单元拆分法展开。
10. 输出结果优先服务正文创作，而不是取代正文创作本身。

## 默认输出口径

- 默认输出一张可直接服务正文创作的控制卡。
- 默认同时检查卷纲、冷线、回报、章末钩子与章名候选。
- 默认同时输出“唯一职责 / 不应包含 / 情感弧 / beat / 钩子链验证 / 就绪判定 / writer 注意事项”。
- 若用户只要极简控制卡，也至少不能删掉“中段回报”和“章末钩子类型”。
- 若任务是长篇连载，默认同时检查前 300 / 1000 / 3000 字的节奏锚点与爽点递进。

## 最低交付

- 能解释这章为什么非写不可。
- 能一句话说清本章唯一职责，并写清本章不该提前处理什么。
- 能指出本章主推进线。
- 能指出本章中段回报。
- 能指出本章与主卖点的关系。
- 能拆出 3–5 个带压力变化的场景单元。
- 能说清本章风险与钩子类型。

## 与其他 Skill / Prompt 的边界

- 本 Skill 是正文创作前的上游准备层。
- 本 Skill 不直接完成正文，也不负责终审评分。
- 连续性热度管理应由 `通用-管理连续性冷热线` 继续承接；两者可联动但不互相替代。
