# 通用-生成章节控制卡

控制卡不是摘要，而是施工命令。它要回答“这一章怎么写”，而不是“这一章大概写什么”。

## 保真迁移要求（强制）

命中写前控制任务时，除本文件外，**必须继续读取**：

- `references/章节控制卡保真规则.md`
- `references/章节控制卡模板.md`
- `references/场景单元拆分法.md`
- `references/中段回报与章末钩子检查表.md`
- `references/爽点递进与300_1000_3000节奏锚点.md`
- `references/九部件预埋与进度节奏微表.md`
- `references/上游供血对齐与回填接口表.md`

## 卖点保真要求（强制）

- 每张控制卡都必须说明：本章对主卖点的关系是强化、兑现、转化、保温还是暂时铺垫。
- 若本章不是卖点显性回报章，也必须写清它怎样避免卖点断供，以及下一次卖点回报将在何处接棒。
- 若控制卡无法说明“这一章读者为什么还会继续记得本书卖点”，说明本章准备仍不合格。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，生成控制卡前必须先读取并纳入本章施工判断。
- 不得把这些报告只当成背景资料；控制卡必须显式说明本章准备在哪些维度追平、反超竞对，以及哪些旧坑必须避开。
- 控制卡的默认目标不是“能开写就行”，而是让当前作品在关键维度上**全方位超越竞争对手**；若看不出明确超越路径，应继续补施工而不是草率开写。

## 常见触发词 / 用户说法速查

- 先给我这章控制卡
- 这一章到底要推进什么
- 帮我拆成几个场景单元
- 这章的中段回报是什么
- 写前作战卡 / 施工卡
- 哪些冷线该在这章回来了
- 这章的章名候选先给我

## 何时使用

- 在正式写章节前，需要把卷纲 / 章纲压缩成可执行的本章控制卡。
- 中段容易塌，需要先明确本章任务、回报、冷线回归和钩子。
- 用户希望先看章名候选、POV、节奏档位、场景单元，再决定是否动笔。

## 本 Skill 的核心任务

每张控制卡至少要写清：

1. 本章**必须改变什么**。
2. 哪条线要推进，哪条线要回归。
3. 本章中段的**明确回报**是什么。
4. 本章的主要场景单元和压力变化。
5. 本章风险：最容易写崩的点是什么。
6. 章末留给下一章的钩子类型与对象。
7. 本章怎样让主卖点继续被看见或被记住。

## 执行顺序

1. 读取上游卷纲 / 章纲 / 单元案 / 相关台账。
2. 先做冷线与债务扫描，再决定本章要承接谁。
3. 先钉死六件事：核心改变、主推进线、回归压力、中段回报、主要风险、章末钩子。
4. 先用“上游供血对齐与回填接口表”钉死本章承接哪个故事面、位于本卷哪一段、属于显性回报章还是保温章。
5. 若本章结构位置发虚，先用“九部件预埋与进度节奏微表”确认本章所在段落与 3–5 个节拍。
6. 按模板输出控制卡：身份、任务、回报、场景、风险、标题候选、钩子。
7. 若控制卡无法说清中段回报，说明本章仍未准备好。
8. 若本章需要拆成 3–5 个较清晰的动作单元，继续按场景单元拆分法展开。
9. 输出结果优先服务正文创作，而不是取代正文创作本身。

## 默认输出口径

- 默认输出一张可直接服务正文创作的控制卡。
- 默认同时检查卷纲、冷线、回报、章末钩子与章名候选。
- 若用户只要极简控制卡，也至少不能删掉“中段回报”和“章末钩子类型”。
- 若任务是长篇连载，默认同时检查前 300 / 1000 / 3000 字的节奏锚点与爽点递进。

## 最低交付

- 能解释这章为什么非写不可。
- 能指出本章主推进线。
- 能指出本章中段回报。
- 能指出本章与主卖点的关系。
- 能拆出 3–5 个带压力变化的场景单元。
- 能说清本章风险与钩子类型。

## 与其他 Skill / Prompt 的边界

- 本 Skill 是正文创作前的上游准备层。
- 本 Skill 不直接完成正文，也不负责终审评分。
- 连续性热度管理应由 `通用-管理连续性冷热线` 继续承接；两者可联动但不互相替代。
