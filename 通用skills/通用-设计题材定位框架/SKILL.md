# 通用-设计题材定位框架

题材定位不是写一句“这本书是什么”，而是先把这本书为什么能成立、为什么能持续、为什么读者会继续追钉死。

## 保真迁移要求（强制）

命中题材定位任务时，除本文件外，**必须继续读取**：

- `references/题材定位保真规则.md`
- `references/平台定位与追读承诺检查表.md`
- `references/现实锚点与代价锚点矩阵.md`
- `references/创新组合写法模板.md`

## 何时使用

- 开书前，先校准作品母体与主平台口径。
- 总纲已写出，但平台口径、现实锚点、代价链、读者承诺仍发虚。
- 需要把“职业切口 / 社会压力 / 机制母体 / 城市空间 / 规则系统”咬成一个统一卖点。

## 本 Skill 的核心任务

在正式设计总纲、卷纲、单元案之前，先回答并写实以下问题：

1. 这本书的**第一眼新鲜点**是什么。
2. 主角为什么必须入局。
3. 主线谜团为什么足以撑长篇。
4. 中后段持续回报靠什么轮换升级。
5. 主平台口径、现实锚点、代价锚点与创新组合是否已经形成闭环。

## 执行顺序

1. 读取作品现有的简介、总纲、相关设定与人物入口文件。
2. 先用保真规则中的“四问校准法”给出定位摘要。
3. 再明确：主平台口径、主角抓手、母体来源、现实锚点、代价锚点、创新组合。
4. 再用平台定位检查表与锚点矩阵检查是否失真。
5. 输出结果时，优先给可直接回写到简介、总纲总则或定位说明的文字。

## 硬规则

- 禁止平台口径含糊，最后变成“每个平台都沾一点”。
- 禁止主角只有头衔，没有方法论抓手。
- 禁止机制 / 超常像万能工具，没有触发条件、代价和边界。
- 禁止现实议题只做主题词，不进入事件链、证据链与人物代价。
