# 通用-设计题材定位框架

题材定位不是写一句“这本书是什么”，而是先把这本书为什么能成立、为什么能持续、为什么读者会继续追钉死。

## 保真迁移要求（强制）

命中题材定位任务时，除本文件外，**必须继续读取**：

- `references/题材定位保真规则.md`
- `references/平台定位与追读承诺检查表.md`
- `references/现实锚点与代价锚点矩阵.md`
- `references/创新组合写法模板.md`
- `references/立意升维与如果会怎样问题链.md`
- `references/腾讯专栏题材库与生活化增补.md`
- `references/腾讯专栏YY缺什么补什么增补.md`
- `references/腾讯专栏YY缺什么补什么增补.md`
- `references/腾讯专栏读者爽感与财色权武增补.md`
 - `references/腾讯专栏文风平台匹配增补.md`
	- `references/腾讯专栏题材背景三类坐标系增补.md`
	- `references/腾讯专栏题材金手指融合方法与核心点增补.md`

## 卖点保真要求（强制）

- 任何作品都必须先钉死一个**突出、鲜明、可快速被编辑和读者识别**的主卖点；副卖点只能服务主卖点，不得抢口径。
- 题材定位必须同时回答：卖点是什么、为何新、为何能持续供血、如何在简介与开篇阶段被迅速看见。
- 若定位摘要不能让陌生人用几句话说清“这本书最值得点开的地方”，视为定位未完成。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，必须先读取并纳入定位校准。
- 不得把这些报告只当成背景资料；必须显式对比主卖点、平台口径、主角抓手、现实锚点、代价锚点与创新组合，说明我方准备在哪些维度追平、反超竞对。
- 定位框架的默认目标不是“做出同类感”，而是让当前作品在关键维度上**全方位超越竞争对手**；若看不出明确超越路径，应判为定位仍未完成。

## 常见触发词 / 用户说法速查

- 先做题材定位
- 这本书的平台口径不清
- 帮我钉死主卖点和主角抓手
- 看看现实锚点和代价链够不够
- 这本书到底该怎么归类

## 何时使用

- 开书前，先校准作品母体与主平台口径。
- 总纲已写出，但平台口径、现实锚点、代价链、读者承诺仍发虚。
- 需要把“职业切口 / 社会压力 / 机制母体 / 城市空间 / 规则系统”咬成一个统一卖点。

## 本 Skill 的核心任务

在正式设计总纲、卷纲、单元案之前，先回答并写实以下问题：

1. 这本书的**主卖点与第一眼新鲜点**是什么。
2. 主角为什么必须入局。
3. 主线谜团为什么足以撑长篇。
4. 中后段持续回报靠什么轮换升级，并继续兑现卖点。
5. 主平台口径、现实锚点、代价锚点与创新组合是否已经形成闭环。

## 执行顺序

1. 读取作品现有的简介、总纲、相关设定与人物入口文件。
2. 先用保真规则中的“四问校准法”给出定位摘要。
3. 再明确：主平台口径、主角抓手、母体来源、现实锚点、代价锚点、创新组合。
4. 再用平台定位检查表与锚点矩阵检查是否失真。
5. 输出结果时，优先给可直接回写到简介、总纲总则或定位说明的文字。

## 默认输出口径

- 默认输出一份可直接回写到简介、总纲总则或定位说明的定位框架文本。
- 默认同时给出主卖点、平台口径、现实锚点与代价锚点。

## 最低交付

- 能说清主卖点、主平台口径与主角抓手。
- 能指出现实锚点、代价锚点与创新组合。
- 能说明为什么这本书值得持续追读。

## 硬规则

- 禁止平台口径含糊，最后变成“每个平台都沾一点”。
- 禁止主角只有头衔，没有方法论抓手。
- 禁止机制 / 超常像万能工具，没有触发条件、代价和边界。
- 禁止现实议题只做主题词，不进入事件链、证据链与人物代价。
- 禁止卖点只停在概念层，不进入简介、开篇、总纲与后续正文。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责题材定位与平台口径钉死。
- 不直接代替总纲、分卷或正文创作。
