# 现实锚点与代价锚点矩阵

## 一、现实锚点矩阵

优先从下列维度中为主线和支线各落至少一项：

| 维度 | 常见写法 | 容易失真点 |
| --- | --- | --- |
| 职业流程 | 排班、取证、问询、交接、外勤、审核 | 只写结果，不写动作 |
| 制度摩擦 | 盖章、封存、流程拖延、权限不够 | 只写“上面不让查”这种空话 |
| 地域空间 | 楼道、旧小区、机房、门岗、医院后区 | 只有地名，没有空间功能 |
| 经济压力 | 债务、房租、补贴、赔偿、调岗 | 只提一嘴，不影响选择 |
| 社会关系 | 家庭、同事、邻里、群聊、物业、学校 | 只做人名关系表，不进冲突 |

## 一点五、来源层速判（新增）

- `现实贴地层`：职业流程、制度摩擦、地域空间、经济压力、社会关系是否真的进入事件链。
- `代价落点层`：身体、职业、关系、名誉、法律 / 制度、心理代价是否有明确落法。
- `升级闭环层`：现实锚点与代价锚点是否会随剧情升级，而不是只在开篇亮一下。

## 二、代价锚点矩阵

| 代价类型 | 常见落法 | 升级方式 |
| --- | --- | --- |
| 身体 | 失眠、旧伤、透支、感官异常 | 从短期不适升级到功能受损 |
| 职业 | 记过、停职、调岗、失去权限 | 从局部受限升级到身份危机 |
| 关系 | 信任裂、误解、连累、隔阂 | 从沉默升级到决裂 |
| 名誉 | 谣言、流言、记录污点 | 从私下议论升级到公开处分 |
| 法律/制度 | 违规风险、程序漏洞、责任追究 | 从灰区试探升级到明确追责 |
| 心理 | 强迫、回避、认知扭曲、触发点增多 | 从隐性代价升级到选择受损 |

## 三、检查口径

- [ ] 主线至少有 2 个现实锚点来源
- [ ] 主线至少有 2 类代价锚点来源
- [ ] 支线不是只承载气氛，也承载现实摩擦或代价升级

## 最小锚点工件（新增）

```markdown
【现实贴地层】主线2个现实锚点 / 支线1个现实锚点
【代价落点层】主线2类代价 / 支线至少1类代价
【升级闭环层】这些锚点后续如何升级或反咬
【当前判定】Green / Yellow / Red
```

## Green / Yellow / Red 锚点判定（新增）

- `Green`：现实、代价、升级三层都清楚，题材不会悬浮成纯设定泡泡。
- `Yellow`：有现实也有代价，但升级闭环仍偏弱，后续可能只剩一次性贴地。
- `Red`：现实锚点只停在名词，代价也不进选择链，定位会迅速失重。
