# 平台定位与追读承诺检查表

## 一、平台主口径检查

- [ ] 已明确“主平台口径”与“可兼容平台口径”
- [ ] 已说明为什么不是另一种主口径
- [ ] 已写清平台读者最敏感的回报类型

## 一点五、来源层速判（新增）

- `平台口径层`：主平台、兼容平台、为什么不是另一种主口径。
- `追读承诺层`：主线真相、能力 / 方法、关系变化、规则 / 代价四类承诺是否齐。
- `失真预警层`：异常推进、主角抓手、机制显影、平台混用是否已被检查。

## 二、追读承诺表（最少 4 条）

按以下格式填写：

- 承诺内容 → 首次兑现节点 → 升级方式 / 新缺口

至少覆盖：

- 主线真相推进
- 主角能力 / 方法升级
- 关系变化 / 团队裂变
- 规则显影 / 代价升级

## 三、失真警报

若出现以下情况，说明定位仍未校准：

- 所有异常都靠“更诡异”推进，没有现实摩擦
- 主角没有具体抓手，只有“天赋”
- 民俗 / 志怪 / 机制只管气氛，不进证据链和代价链
- 平台口径混用：章首想一种，中段像另一种，结尾又回第三种

## 最小平台工件（新增）

```markdown
【平台口径层】主平台 / 兼容平台 / 不能选另一口径的原因
【追读承诺层】4条追读承诺与首次兑现节点
【失真预警层】当前最大混用风险或失真点
【当前判定】Green / Yellow / Red
```

## Green / Yellow / Red 平台判定（新增）

- `Green`：平台口径、追读承诺、失真预警三层都清楚，可直接回写定位说明。
- `Yellow`：口径大方向对，但承诺层或预警层仍偏粗，后续易平台串味。
- `Red`：只知道“想发哪里”，却说不清为什么、读者要拿什么、混用会怎么炸。
