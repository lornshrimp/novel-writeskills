# 书名设计流程与评估维度

## 适用任务

- 为整部小说、分部或分卷设计书名。
- 为同一作品设计多平台书名。
- 重做、重排、重审已有书名方案。
- 从大批候选里收窄 shortlist。

## 默认输入规则

优先从当前会话、当前编辑文件与工作区材料中自动提取；只有确实无法继续时，才追问最小必要信息。

### 必要输入

- 作品名称（若已有）
- 设计层级：`全书 / 分部 / 分卷`
- 目标平台：默认全平台
- 候选数量：默认每个平台 10 个

### 可选参数

- `scope`：`全书 / 分部 / 分卷`
- `platforms`：平台子集
- `countPerPlatform`：每个平台候选数
- `styleBias`：`商业点击 / 口碑质感 / 出版审稿 / 海外传播`
- `outputMode`：`完整方案 / shortlist / 决赛版`
- `overwriteCurrentFile`：是否覆盖当前方案文件
- `withRubric`：是否启用书名评分 Rubric（默认启用）

## 必须先做的上下文读取

在开始设计前，必须先阅读或检索工作区中与书名直接相关的材料；不要拍脑袋起名。

优先顺序：

1. `写作研究/` 中的平台研究文档
2. `小说大纲/小说总大纲.md`
3. `小说大纲/故事多面体大纲.md`
4. `小说大纲/分卷故事面推进表.md`
5. `小说大纲/伏笔与呼应总纲.md`
6. `小说大纲/线索台账.md`、`嫌疑池台账.md`、`时间轴（总）.md`
7. 若当前已有书名方案文件，也必须读它，先复盘旧问题，再重做

如果用户只要求某一卷或某一部分标题，则重点读对应分卷 / 分部材料，不必把整套总纲重复展开。

## 正式产出前必须完成的思考

### 先抓四个核心词

1. 题材标签词
2. 主角职业 / 身份词
3. 危险 / 规则 / 反常词
4. 目标平台偏好的语气词

### 再提炼标题母核

至少完成以下动作：

1. 提炼作品的 3–5 个标题母核。
2. 判断哪些卖点适合做书名，哪些只适合做简介或宣传语。
3. 区分以下母核类型：
   - 世界观核
   - 悬疑现实核
   - 人物伦理核
   - 灾难事件核
4. 为每个平台选择更合适的命名主轴，而不是整表共享一套换词模板。

## 书名判断的核心经验

### 先判断“像不像书名”

每个候选都必须先过这一关：

- 能不能被当成一本长篇总书名？
- 能不能被读者自然念出来、记下来？
- 能不能覆盖全书，而不是只覆盖某一个片段？

如果更像以下任一种，就要降级或淘汰：

- 宣传文案
- 推文标题
- 章节标题
- 爆点导语
- 一句口号

### 一个标题只打一个主轴

- 一个标题只允许有一个主轴，最多一个辅助阴影。
- 过于贪多，通常会变成“什么都写了，但不像书名”。

### 长句标题不是原罪，但要顺、稳、可复述

出现以下情况时，应优先压缩：

- 一口读不顺
- 信息重心失衡
- 结构像广告语
- 像一句故事摘要而不是标题

### 不得新增材料里不存在的核心卖点

- 不得为了起名好听而发明新设定、新人物关系、新世界规则。
- 标题只能围绕现有材料做高密度提炼。

## 常见命名方向

- 职业 / 身份 + 反常
- 规则 / 禁令 / 代价句
- 主角身份 + 危机
- 物件 / 地标 + 悬念
- 灾难事件 / 危机现场
