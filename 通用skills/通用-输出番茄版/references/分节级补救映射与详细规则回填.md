# 通用-输出番茄版｜分节级补救映射与详细规则回填

## 来源与恢复等级

- 恢复等级：B（补救性回填）
- 主要来源：两题材现存薄 Prompt、`pilot-shared-platform-section-mapping.md`、番茄平台研究文件、同类题材长规则模式。

## 0. 平台任务定义与基准继承

- 任务目标：把章节改写成更适合番茄连载阅读与追更节奏的版本。
- 必须同时继承：`创建小说正文.prompt.md`、`正文润色.prompt.md`、`去AI味.prompt.md`。
- 继承时不得削弱：章首抓人、中段回报、章末硬钩子、零新增事实、信息量不缩水。

## 0.2 零新增事实

- 不新增人物、地点、时间点、设定、证据、因果。
- 允许重排、改写、镜头化、动作化、感官化，但不得改坏原事实链。
- 精确数值、时间戳、ID、票据字段、文书字段必须原样保留。
- 叙事人称与视角必须与原文一致。

## 0.3 沉浸式小说体裁

- 正文必须首先是小说，不得写成总结、清单、教程、复盘。
- 文件结构只允许：章标题、正文、`## 作者有话说`。
- 禁止额外小标题、列表、表格、代码块、说明性条列。

## 1. 平台理解

### 1.1 平台机制与读者画像

- 番茄优先看前段抓力、段落顺滑度、章内回报密度与章末追更动机。
- 平台读者更接受直给反馈、快速推进、强反应和更清晰的风险升级。
- 标题倾向偏直给、结果导向、冲突导向，但仍需保留小说感。

### 1.2 平台风格门槛

- 前 150–300 字要尽快给出异常、冲突、威胁、反常或损失。
- 每 500–800 字最好出现一个小抓点：新信息、破绽、危机、反手、误导证伪。
- 中段必须有一次实质回报，不能只靠气氛和解释硬拖。
- 章末应留出更明确的“下一章必须看”的理由。

## 2. 输入与输出

- 输出目录根：`番茄小说/`
- 目录映射：从源路径去掉 `小说正文/` 后，保留 `{分部目录}/{分卷目录}` 映射。
- 文件名：`{chapterNo} {平台新标题}.md`

## 3. 输出内容结构

- 顶部标题：`# 第{chapterIndex}章：{平台新标题}`
- 正文：高抓力、短段落、快节奏、强推进
- `## 作者有话说`：中文，200–300 CJK，可轻煽动追更，但不得出戏

## 4. 关键硬约束

- 中文平台正文 CJK 必须在 3500–6500。
- 不得写成泛中文平台稿；必须有番茄式强推进体感。
- 允许直给，但不能失去基本逻辑闭环。
- 降相似度必须通过更深平台化改写完成，禁止噪声降重。

## 5. 改写前推演问题

1. 开头第一波抓点是什么？
2. 中段哪一处回报最值钱？
3. 章末钩子如何更硬，但不新增事实？
4. 哪些段落需要压缩解释、改成动作或对话？

## 6. 平台执行要点

- 短段优先，尽量减少解释腔。
- 风险、对抗、损失、倒计时、破绽应尽量外显。
- 若原文较慢，应通过重排，把最能抓人的信息前置。
- 作者有话说可以更偏追更召唤，但不许运营腔。

## 7. 输出格式与检查

- 检查标题是否够直给。
- 检查前 300 字是否真抓人。
- 检查中段是否有实质回报。
- 检查章末是否形成明确追更驱动。
