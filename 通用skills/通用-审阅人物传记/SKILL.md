# 通用-审阅人物传记

人物传记要当作作者的裁判源来审，而不是当角色介绍页来夸。

## 继续读取的 references

- `references/人物传记审阅检查清单.md`
- `references/人物传记审阅报告增补.md`
- `references/审阅维度说明.md`
- `references/人物传记审阅报告骨架.md`
- `references/人物传记问题分级与回写建议.md`
- `references/腾讯专栏人物审阅可演出性与辨识度增补.md`

## 完整模板执行要求（强制）

- 产出人物传记审阅报告时，必须以 `references/人物传记审阅报告骨架.md` 作为主骨架，不得退化成“几段人物点评 + 少量建议”的简版报告。
- 审阅人物传记模板时，必须先核查其在原有细分结构基础上是否显式覆盖 **背景、性格、外貌、其它** 四大基础栏位；若缺栏、混栏或任一类无法快速定位，必须记为结构缺口。
- 报告至少要完整覆盖：阅读链路确认、基本信息检查、人物一句话定义、角色功能位、首次出场与前三章留存验证、人物—情境—观点合拍度、可演出性与可写性评估、网文留存与角色辨识度复核、身份与组织归属、动机—代价闭环、能力边界与工具箱、关系网与镜像对手、新颖性与趋势适配、人物标签与成长弧核查、可复用写作素材库、秘密与爆点、心理弧线与索引映射、大纲完整性检验、冷热点与回归压力、问题分级、评分、具体修改方案、冲击 9.2+ 综合评分的优化思路与总体评价 / 审阅结论。
- 若当前任务还加载了题材 Skill，则必须在通用骨架的占位章节中继续补写题材专项段、题材专项评分项与题材化结论重点。
- 报告中必须至少产出 `4` 类可复用写作素材，每类 `3–7` 条；并至少给出 `1` 段可直接贴回正文的登场镜头与 `3` 句可直接复用的台词。

## 网文留存与角色回归压力（强制）

- 审人物传记时，必须把人物视为“读者为什么持续追、为什么记得住、为什么愿意等他/她回归”的连载资产，而不是静态设定页。
- 必须显式判断：一句话记忆点、读者继续跟随理由、冷线回归压力、重要对手镜像关系、关系网事件生成能力是否成立。
- 若核心人物缺乏可识别钉子、离线后没有回归压力，或缺陷始终不进入关键选择，不得轻描淡写判为可后补小问题。

## 网文首秀与追读供血（强制）

- 审人物时必须判断：他 / 她第一次出场是否落在最能体现价值与压力的熟悉情境里，且前三章内已经完成一次有效回报、误读、反击或关系拉扯。
- 必须显式判断：这个人物主要靠什么持续供血（专业碾压、情绪回报、关系张力、危险推进、反差幽默、反击爽点、价值站队）。
- 若人物首秀只有履历说明、没有情境动作，或三章内没有可截图 / 可站队 / 可催更的钩子，不得判为网文高完成度人物。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，审阅前必须先读取并纳入对照。
- 若目标平台为起点中文网，或未注明平台但默认按起点口径处理，且当前题材目录下尚无可用竞对主报告，审阅前应优先调用 `通用-分析竞对作品`，至少补齐 **Top10 候选池、`3–5` 本主样本与四层深拆速览**；若任务限制导致本轮无法补齐，必须显眼记录盲区，并下调本轮审阅上限与置信度。
- 审阅报告不得只给内部自洽结论；必须显式判断当前人物相对竞对在哪些维度已领先、持平或落后，以及最先该补哪条超越路径。
- 若当前人物在竞对已验证的关键强点上明显更弱，必须直判为竞争风险；人物审阅默认按“严酷找差距”口径执行，不得为了安抚作者而把首秀发空、代价缺席或关系网无事件功能轻描淡写带过。
- 竞对维度在人物审阅综合评分中默认按 `25%` 计权；若未完成量化差距卡，或关键维度仍显著落后且没有 `72` 小时补丁，综合评分默认不得给高档。

## 起点化竞对吸收口径（新增）

当人物审阅服务起点中文网，或未标平台而默认按起点基线处理时，默认把竞对对照拆成以下四层，并翻译成审阅判断与补丁建议：

- **市场数据层**：用榜单位次、均订 / 首订 / 收订比代理信号、推荐资源位等，反推这个人物是否真的支撑题材归类、持续付费价值与长线商业竞争位，而不是只在作者视角里“资料很全”。
- **内容创作层**：重点对照主样本在人物抓手、欲望与代价、关系拉扯、身份反差、能力边界、语言指纹、成长弧线与反派动机包装上的成熟度，把差距翻成条目级传记补丁与正文级首秀补丁。
- **运营策略层**：重点对照主样本如何通过角色制造追读工程、本章说讨论点、站队点、关系张力与持续回报，把差距翻成触发器、底线 / 开关、留痕偏好、可复盘细节库与回归压力补丁。
- **受众反馈层**：重点对照本章说 / 书友圈 / 书评区里高频被夸的人物魅力点、被骂的脸谱化病灶、老白读者对降智 / 空心 / 无代价主角的敏感点，把差距翻成避坑与加码建议。

起点语境下，默认尽量使用以下术语，而不是写成空泛的“人物竞争力不足”：`均订`、`高订`、`首订`、`追订`、`收订比`、`分类月票榜`、`畅销榜`、`新书榜`、`三江`、`本章说 / 段评`、`书友圈`、`老白读者`。拿不到精确值时必须写成代理信号，不得伪造具体数字。

## 可演出性与作者交付（强制）

- 审人物传记时，必须把它当作“作者的角色操作手册”来审，不是给读者看的漂亮简介。
- 必须显式判断：三秒立人、行动驱动、缺陷可复用、关系网络可用、与题材机制绑定是否成立，以及人物—情境—观点是否合拍。
- 禁止只给概念评价；必须同时产出条目级补丁、登场镜头示例、可复用台词 / 动作 / 雷点 / 把柄素材。

## 来源层、判定门禁与模板化工件镜像（新增）

- 审人物传记时，默认继续检查它是否已长成“可维护的作者工件”，而不是只有静态人设说明：完整 / 精简模板判断、当前状态、章节行为记录、语言指纹情绪变体、关系总览、可复用素材库，都应纳入审阅范围。
- 问题默认标记来源层：`[人物结构层] / [动态状态层] / [正文抓手层]`。人物结构层负责背景、动机、边界、关系主轴；动态状态层负责当前状态、章节行为记录、阶段变化；正文抓手层负责登场镜头、语言指纹、素材库、可贴回正文的动作与台词。
- 默认补 `Green / Yellow / Red` 入库判定：`Green`=四大基础栏位、动机代价、关系事件、动态状态与素材库齐备；`Yellow`=主干成立但动态字段 / 素材库 / 首秀补丁仍弱；`Red`=栏位缺失、能力万能、代价缺位、关系网无事件功能、首秀无抓力或离线无回归压力。
- 若人物传记无法直接支持“首秀三章怎么写、回归时靠什么重新拉热、缺陷怎样反噬关键选择”，默认不得判为成熟裁判源。

## 报告落盘位置与命名规则（强制）

- 人物传记审阅报告必须落在项目根下的 `审阅意见/人物传记审阅报告/` 子目录中，不得散落在 `人物传记/` 目录或 `审阅意见/` 根目录。
- 人物传记审阅报告文件名必须带日期，日期格式固定为 `YYYY-MM-DD`。
- 推荐完整相对路径写法为 `审阅意见/人物传记审阅报告/人物传记审阅报告_[人物名]_2026-04-15.md`。
- 人物传记本体文件默认不带日期；日期只用于对应审阅报告。

## 常见触发词 / 用户说法速查

- 帮我审这个人物传记
- 看看这个人物是不是立得住
- 这份传记能力边界稳不稳
- 关系网有没有问题
- 给我人物传记审阅报告
- 这个人物是不是太万能了

## 何时使用

- 审阅人物传记、角色档案与人物裁判源。
- 检查身份归属、动机代价、能力边界、关系网与写作抓手是否成立。
- 输出可落库、可回写的人物传记审阅报告。

## 默认执行顺序

0. 若按起点口径执行且当前题材目录下缺少可用竞对主报告，先调用 `通用-分析竞对作品`，补齐 `Top10 候选池 → 3–5 本主样本 → 四层深拆` 的最小结论集。
1. 先确认目标传记文件与输出策略。
2. 若按起点口径执行，先吸收竞对主样本四层结论，并写清本轮审阅重点比对的关键项。
3. 按检查清单做结构审计，先确认背景、性格、外貌、其它四大基础栏位是否齐备。
4. 判断动机代价、能力边界、关系网、读者联结、镜像对手与冷线回归压力是否成立。
5. 输出问题分级与最小回写建议。
6. 生成可落库、可回写的审阅报告。

## 默认输出口径

- 默认输出一份带问题分级、证据依据与最小回写建议的人物传记审阅报告。
- 默认不改传记本体，除非用户明确要求边审边回写。
- 报告必须同时回答“哪里不成立”“最小怎么补”“补完后如何更能写、能追、能记住、首秀更抓人”，不能停留在抽象审美判断。
- 默认同时输出：来源层标记、入库判定、最先要回写的层级与 `72` 小时最小补丁链。
- 若按起点口径执行，报告还必须显式覆盖：`Top10 候选池`、`3–5` 本主样本四层速览、竞对差距量化卡，以及当前人物最先该补的超越路径。

## 最低交付

- 能指出背景、性格、外貌、其它四大基础栏位是否齐全、清楚、可回写，哪些地方属于结构缺口。
- 能指出身份、动机、边界、关系、抓手、弧线与索引中最关键的问题位。
- 能说明哪些问题属于高风险，哪些问题可后续优化。
- 能给出最小可回写补丁，而不是只给抽象评价。
- 能给出至少 `1–3` 处可直接写回传记的条目级补丁、至少 `1` 段可直接贴回正文的登场镜头、至少 `3` 句可直接复用的台词，以及至少 `1` 条首秀 / 前三章回报补丁。

## 复审独立性规则（强制）

- 每一轮人物传记审阅都必须针对当前版本重新独立、从严、全面审一遍，不得因为上一轮指出的问题已修复，就直接按“进步明显”给出高分。
- 上一轮问题修复情况必须核查，但这只是基础项；仍需重新审查身份归属、动机代价、能力边界、关系网、写作抓手、心理弧线与索引映射是否成立。
- 若本轮发现新的 OOC 风险、新的代价缺口、新的关系网失效点或新的索引断裂，必须如实写入本轮报告，不得为了保持“评分持续上涨”而省略。
- 本轮评分只能根据当前版本在本轮完整审阅下的实际质量给出，不得按上一轮分数、已修复问题数量或修改幅度自动抬分。

## 硬规则

- 禁止只评价“人物有没有魅力”
- 若按起点口径执行，禁止跳过 `Top10 候选池 → 3–5 本主样本 → 四层对照` 就直接给高分或高档判断。
- 必须落到证据、结构与回写动作
- 必须说明这个人物为什么值得读者继续跟、为什么在长线连载里不会被忘掉
- 发现核心人物缺代价、能力万能、关系网无事件功能时，至少记为高风险
- 禁止把人物问题写成空泛形容词；必须落成可验证证据、具体条目、可直接回写的动作
- 禁止把人物写成“履历很满但首秀很空”的设定板；若前三章里没有真正让读者想继续跟的动作或回报，必须明确记为失血点

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责人物传记的结构审阅与回写建议，不直接代替人物传记重写。
- 若用户要直接重写或补强人物传记，应优先使用 `通用-设计人物传记`。
