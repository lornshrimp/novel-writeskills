# 通用-审阅人物传记

人物传记要当作作者的裁判源来审，而不是当角色介绍页来夸。

## 继续读取的 references

- `references/人物传记审阅检查清单.md`
- `references/人物传记审阅报告骨架.md`
- `references/人物传记问题分级与回写建议.md`

## 默认执行顺序

1. 先确认目标传记文件与输出策略。
2. 按检查清单做结构审计。
3. 判断动机代价、能力边界、关系网和写作抓手是否成立。
4. 输出问题分级与最小回写建议。
5. 生成可落库、可回写的审阅报告。

## 通用硬规则

- 禁止只评价“人物有没有魅力”
- 必须落到证据、结构与回写动作
- 发现核心人物缺代价、能力万能、关系网无事件功能时，至少记为高风险
