# 通用-设计分卷大纲

这里的“分卷”默认同时覆盖分部层与分卷层；核心任务不是继续往下写，而是把上一级已分配内容真正落地。

## 保真迁移要求（强制）

命中分部 / 分卷层任务时，除本文件外，**必须继续读取**：

- `references/分部与分卷设计规则.md`
- `references/分部模板.md`
- `references/分卷模板与章节工件.md`
- `references/标题规范与章节验收增强.md`
- `references/超长篇分卷节奏与单元闭环.md`
- `references/卷级四段推进与关键点布置表.md`
- `references/故事面承接与卷章落点核对表.md`

其中，`references/分卷模板与章节工件.md` 是分卷层的**通用主模板**：章清单、卷级启动表、爽点布点表、逐章微型剧本（具体章节规划）的共性骨架统一维护在这里；题材层只应补写题材增补项，不得各自平行维护一整套通用模板。
执行时若信息过多，固定读取优先级为：先读 `分部与分卷设计规则.md` 判层，再按层读取 `分部模板.md` 或 `分卷模板与章节工件.md`，最后再用节奏与故事面两张核对表做校准。

## 卖点保真要求（强制）

- 每卷都必须回答：本卷是强化卖点、兑现卖点、转化卖点，还是为下一次卖点爆发做铺垫。
- 章清单必须看得出哪些章承担卖点显性落点，哪些章只做推进；纯推进章不能连续过长导致卖点失踪。
- 若本卷读完仍看不出本书最核心的吸引力如何被持续兑现，视为本卷失职。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，必须先读取并纳入卷级拆解判断。
- 若目标平台为起点中文网，或未注明平台但默认按起点口径处理，且当前题材目录下尚无可用竞对主报告，开始卷纲设计前应优先调用 `通用-分析竞对作品`，至少补齐 **Top10 候选池、`3–5` 本主样本与四层深拆速览**；若任务限制导致本轮无法补齐，必须显眼记录盲区，并降低卷纲成熟度判断。
- 不得把这些报告只当成背景资料；必须显式对比卷内起承转合、前 5 章追读链、章中回报、卷尾续购与跨卷交接，说明本卷准备在哪些维度追平、反超竞对。
- 分卷设计的默认目标不是“章清单能排出来”，而是让当前作品在关键维度上**全方位超越竞争对手**；若看不出明确超越路径，应判为卷纲仍未达标。

## 起点化竞对吸收口径（新增）

当当前卷纲服务起点中文网，或未标平台而默认按起点基线处理时，默认把竞对吸收拆成以下四层，并直接翻译成卷级设计约束：

- **市场数据层**：用榜单位次、均订 / 首订 / 收订比代理信号、推荐资源位等，倒推本卷在题材归类、前 `5` 章抓力、卷尾续购缺口与长线商业竞争位上应该达到的最低强度。
- **内容创作层**：重点吸收主样本在卷首启动、单元案与主线咬合、人物驱动、设定边界、卷中回报、卷尾升级上的做法，翻成章清单、卷级启动表与逐章微型剧本的硬约束。
- **运营策略层**：重点吸收更新节奏、回报密度、章末钩子、本章说讨论点、卷中爆发节点与推荐位适配逻辑，转成章节功能位与卷级节拍设计。
- **受众反馈层**：重点吸收本章说 / 书友圈 / 书评区里高频被夸的值点、被骂的拖点、老白读者不耐烦的病灶，提前写进卷纲的避坑与加码清单。

起点语境下，默认尽量使用以下术语，而不是写成空泛的“市场反馈不错”：`均订`、`高订`、`首订`、`追订`、`收订比`、`分类月票榜`、`畅销榜`、`新书榜`、`三江`、`本章说 / 段评`、`书友圈`、`老白读者`。拿不到精确值时必须写成代理信号，不得伪造具体数字。

## 常见触发词 / 用户说法速查

- 帮我拆分卷大纲
- 这卷的章清单怎么排
- 我需要卷级启动表
- 这卷很多章在推进但不成卷
- 帮我做卷纲和章节微型剧本
- 给我逐章微型剧本
- 帮我把章清单继续展开成具体章节规划
- 这几章要补 `≤3章破口计划`
- 这一章的版本破口 / 官方口径破口怎么写
- 看看这卷怎么承接故事面

## 何时使用

- 设计分部大纲、分卷大纲、卷纲、章清单。
- 把总纲或分部分配内容拆成情节块、故事线、故事点。
- 设计卷级启动表、节奏表、章节微型剧本、逐章具体章节规划与相关工件。

## 与总纲 Skill 的边界

- 本 Skill 负责**战役层 / 战术层**：分部大纲、分卷大纲、章清单、卷级启动表、爽点布点表、逐章微型剧本与执行级工件。
- 若用户要求设计的是全书级卖点、主线谜团 `M0`、重要支线 `X01-XNN`、单元案矩阵 `C01-CNN`、跨卷总盘或终局闭环，应切换到 `通用-设计总大纲`。
- 若用户要求把已完成的逐章微型剧本继续压到单章施工卡、场景单元或章节正文，应继续组合 `通用-执行场景单元`、`通用-创建小说正文` 等下游能力。

## 默认执行顺序

固定优先级：先补参并判层，再做承接清单与模板落位，最后才做卷内节奏、故事面承接与一致性校准。

1. 先判定当前是分部层还是分卷层。
2. 若按起点口径执行，先吸收 `Top10 候选池 → 3–5 本主样本 → 四层深拆` 的核心结论，再列“本层承接清单”。
3. 用设计规则确认上级内容是否全量落地。
4. 若卷纲容易写成“很多章在推进，但看不出卷内起承转合”，先用“卷级四段推进与关键点布置表”校准卷内结构；这里的“四段”默认指卷首启动、卷中升压、卷中爆发 / 回报、卷尾交接四段，并要标出每段关键点。
5. 再用“故事面承接与卷章落点核对表”确认本卷承接了哪些故事面、哪些章负责显性落点、哪些章只做保温。
6. 分部层套用分部模板，分卷层套用分卷模板与章节工件；章清单完成后，必须继续下压到“逐章微型剧本（具体章节规划）”，不能停在只够看目录的概述层。
7. 若后续任务要继续把关键章落到单章施工或正文创作，可再组合调用 `通用-执行场景单元`、`通用-创建小说正文` 等下游能力。
8. 最后检查目标字数来源、章数字数换算、章清单与逐章微型剧本一致性、节奏与钩子设计是否自洽，并确认本卷对主样本是否存在明确追平 / 反超路径。

## 核心硬规则

- 上一级分配给本层的内容必须切实、全量、保质落地。
- 目标字数必须以上一级分配为准，不能本层拍脑袋乱定。
- 章数规划默认按每章约 3000 字换算。
- 章清单是后续章节细化的唯一蓝图。
- 逐章微型剧本（具体章节规划）是分卷执行版与正文之间的直接桥梁；若只给章清单、不下压逐章微型剧本，则仍不算可直接落正文的执行版。
- 逐章微型剧本必须与章清单严格一致；只能细化镜头、对话、动作、证据 / 关键载体、状态变化、台账变化等执行细节，不能自由改写视角、章目标、核心冲突、关键情节 / 转折、关联故事线、直接后果、伏笔 / 呼应与钩子类型。
- 当章节抛出截图、成品版本、官方口径、系统记录、删改痕迹、版本差异等“引线型信息”时，应按需补写 `≤3章破口计划`；若该章承担破口起手或关键推进职责，则该字段视为必填。
- 若本卷采用故事日推进，必须先给出统一的 WHEN 口径（如 `D+0 = YYYY-MM-DD`），后续逐章才能使用 D+1 / D+2 等相对写法。
- 每卷既要能独立讲出阶段故事，也必须继续为主线供血。
- 不得让主卖点连续多章甚至整卷失踪，只剩流程推进。
- 分部大纲、分卷大纲、章清单、章节微型剧本等本体文件默认不带日期；日期命名仅用于对应审阅报告、书评等派生产物。

## 极其快的节奏与极其强烈的冲突——分卷层实施方法（新增）

总纲已定义快节奏与强冲突的方向，分卷层应该把这些原则具体化为章清单、卷级启动表与逐章规划。

### 分卷级快节奏的可落地设计

**1. 卷首"黄金三章"节奏锁定**
- **第 1 章**：抛出本卷核心冲突的**第一刺激点**。不做铺垫，直接把主角扔进麻烦，让读者在前 300 字内看见"有问题"。
- **第 2 章**：主角首次应对，**立刻失败或碰壁**。这章必须让读者看见"原来问题没那么简单"，问题强度升级。
- **第 3 章**：揭露或升级，**留下悬念**。可能是新敌人出现、隐藏信息暴露，或主角发现自己陷入更大陷阱。

三章必须形成"冲突 → 应对失败 → 更深层的冲突显露"的递进，每章字数应该在 3500-5000 字之间，不能出现"第 1-2 章信息交换，第 3 章才开始有冲突"的节奏。

**2. 卷中"防冻结"的信息密度设计**
- 中间段（通常 8-15 章）是最容易显得"拖沓"的位置。
- 每 2-3 章应该有一个**新变量、新敌人或新信息**入场，推动冲突往更复杂的方向演进。
- 禁止出现连续 4 章以上"纯推进"的情况：纯推进是指没有新冲突升级、没有人物关系翻面、没有新线索暴露的章节。
- 卷中应该分布 2-3 个**中等高潮**（次要冲突的爆发点），而不是把所有能量都留给卷末。

**3. 卷末"快速闭环"设计**
- 卷末 3-5 章的节奏应该加速：
  - 倒数第 3 章：问题开始显露真相或陷阱启动
  - 倒数第 2 章：主要对抗、生死攸关的选择
  - 倒数第 1 章（卷末）：闭环本卷主案，同时为下一卷埋坑
  
- 卷末不是"舒缓与休整"，而是"推向更危险的悬崖"。主角在卷尾的处境应该比卷首更糟糕、更被逼迫。

**4. 章清单里的"钩子轮换"设计**
在章清单里，应该标记每章的**主要职责**：
- `钩子`：抛出新悬念、新冲突的章
- `蓄力`：积累压力、信息铺垫的章（但必须有"压力在上升"的体感）
- `爆发`：冲突激化、反击或升级的章
- `回报`：兑现之前的伏笔或悬念、给读者"值回票价"感的章
- `转承`：转场或承接的章（但必须有新的冲突或关系变化作为"转场的代价"）

禁止出现连续多章都是"蓄力"或"转承"的排列；应该看出"钩子→蓄力→爆发→回报"的循环轮换，或至少"钩子→爆发→回报→钩子"的三章一轮。

### 分卷级强冲突的可落地设计

**1. 卷级主冲突的"三维压迫"**
每卷应该设计从三个维度同时压迫主角的冲突：
- **维度 A**（外部压力）：敌人追杀、任务倒计时、环境恶劣等
- **维度 B**（内部困境）：关键人物背叛信任、身份曝光、欲望冲突等
- **维度 C**（理念挑战）：看法被推翻、必须做出违心的选择、价值观破灭等

这三维应该循序递进暴露，而不是一开始全部抛出。例如：
- 卷首：外部压力最凸显（敌人在追）
- 卷中：内部困境浮现（发现搭档隐瞒）
- 卷末：理念挑战爆发（必须在"活着"和"保持原则"之间二选一）

**2. 对手的"多角色立体化"**
不要设计单一的对手；应该设计 3-5 个有不同诉求的对方角色：
- **直接对手**：眼下正在争夺、最紧迫的敌人
- **阻挠者**：不是直接敌人，但利益冲突，可能会反水或背刺
- **"假同盟"**：表面上是盟友，但有隐藏的目的
- **"无辜者"**：卷入了冲突，但本身不是敌人，反而可能成为筹码或破局点

这些角色应该在卷中逐个"立体化"：读者一开始认为"简单的敌友关系"，后来发现每个人都有自己的算盘。

**3. "冲突升级链"的设计与标记**
在卷级启动表或章清单里，应该清晰地标记每章的冲突**比上一章升级了什么**：
```
第 1 章：敌人追杀主角 [威胁强度：5/10]
第 2 章：敌人不只追，还捉了主角的伙伴 [威胁强度：7/10，新变量：伙伴安危]
第 3 章：敌人放出风声："你的真实身份已被曝光" [威胁强度：9/10，新维度：身份风险]
第 4 章：主角发现敌人背后有官方势力支持 [威胁强度：9/10，新敌人：官方]
```

这样做的好处是**避免了"为了升级而升级"的虚假冲突**；每一步升级都是解决前面冲突的过程中自然产生的新麻烦。

**4. 关键人物的"利益冲突"嵌入**
每个重要角色都应该在某个章节与主角产生明确的利益或立场冲突。这个冲突不一定要在当卷解决，但必须在当卷"暴露"：
- 第 5 章：主角发现搭档在和敌人有秘密接触（冲突暴露）
- 第 8 章：搭档被逼交代真相，但这样说会毁灭主角的计划（冲突升级）
- 第 12 章：主角和搭档终于对峙，关系翻面（冲突爆发，但可能需要下一卷来修复）

**5. "代价具体化"的卷级标记**
每卷应该有 2-3 个**不可逆的代价**，在章清单里明确标记出来，这样后续每章的规划都能基于这些代价来设计主角的处境：
- 代价 1：失去了一个关键同盟者（这影响下卷的任务执行能力）
- 代价 2：被曝光了一个隐藏身份（这影响下卷的卧底机制）
- 代价 3：激怒了一个之前不是敌人的势力（这为下卷引入新对手）

### 卷级启动表的快节奏与强冲突维度

卷级启动表应该包含以下字段（新增）：

| 字段 | 说明 | 示例 |
|------|------|------|
| 卷号 | 第 N 卷 | 第 2 卷 |
| 卷主题 | 本卷的故事面主题 | "身份暴露与信任破裂" |
| 主冲突 | 本卷最核心的对抗 | 与官方特务的正面对抗 |
| 三维压迫 | 外部、内部、理念压迫各是什么 | 外：追杀 / 内：搭档背叛 / 理：必须牺牲无辜者 |
| 对手阵容 | 本卷出现的 3-5 个对方角色 | 特务头子、内奸搭档、市长办公室、良心警察 |
| 冲突升级点 | 3-5 个"比上章升级"的关键章节 | 第 5 章（曝光）、第 8 章（背叛）、第 12 章（摊牌） |
| 黄金三章职责 | 卷首 1-3 章的明确职责 | 第 1：引入官方介入 / 第 2：搭档异常行为 / 第 3：身份被问询 |
| 不可逆代价 | 本卷必须付出的代价 | 失去搭档信任、暴露一个秘密身份、激怒官方 |
| 钩子链设计 | 卷内主要悬念与钩子 | 谁出卖了主角 → 搭档为什么背叛 → 是否还有第三方参与 |
| 卷尾悬念 | 卷末为下一卷埋的坑 | 官方特务透露：你的真实身份不只一个，还有第三方一直在追杀你 |

### 逐章微型剧本里的节奏与冲突标记（新增）

每个逐章微型剧本应该包含以下信息，用来保证快节奏与强冲突的落地：

```
## 第 X 章：[章题]

**章目标与冲突**
- 这章必须推进什么：[具体事件或冲突升级]
- 这章的主要对立是：[谁 vs 谁，或者 主角 vs 什么困境]
- 读者最想看的是：[什么问题的答案 / 什么冲突的结果]

**节奏标记**
- 节奏属性：钩子 / 蓄力 / 爆发 / 回报 / 转承（选一个）
- 与上一章的冲突升级点：[上章是 X 问题，本章升级到 Y 问题]
- 场景数与预计字数：[几个场景，3500-5000 字]

**冲突深度**
- 立场对立：[主角的诉求] vs [对方的诉求]
- 隐藏变量：[这章暴露或藏着什么信息]
- 代价与后果：[这章的选择会导致什么不可逆的改变]

**场景拆分**
- 场景 1：[什么冲突] → 职责：抓眼 / 回报 / 升级
- 场景 2：[什么冲突] → 职责：推进 / 对话冲突
- 场景 3：[什么冲突] → 职责：升级 / 钩子

**钩子设计**
- 章末钩子：[为下一章留下什么悬念或威胁]
```

这样的标记可以确保每章都在为"快节奏与强冲突"这个总目标服务。

## 黄金三章与单章四拍（新增）

- 卷首前三章默认必须形成一轮清楚的黄金三章职责：第一章立人物抓手与眼前麻烦，第二章让人物顶上第一轮正面冲突，第三章必须给一次可感知的回报、反击或升级。
- 章清单里应尽量看出单章四拍：**钩子 → 蓄力 → 爆发 / 回报 → 留钩**。不是每章都一样重，但不能长期只有推进、没有兑现。
- 若某几章主要承担保温、转场或铺垫职责，必须写清它们怎样维持卖点可见度，避免读者连续多章只看到流程没有值回票价的变化。
- 卷尾钩子最好同时接住：本卷已兑现的东西、下一卷必须追的缺口、人物关系或身份状态的新变化。
