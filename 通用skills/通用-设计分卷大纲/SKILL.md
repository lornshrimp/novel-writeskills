# 通用-设计分卷大纲

这里的“分卷”默认同时覆盖分部层与分卷层；核心任务不是继续往下写，而是把上一级已分配内容真正落地。

## 保真迁移要求（强制）

命中分部 / 分卷层任务时，除本文件外，**必须继续读取**：

- `references/分部与分卷设计规则.md`
- `references/分部模板.md`
- `references/分卷模板与章节工件.md`
- `references/标题规范与章节验收增强.md`

## 何时使用

- 设计分部大纲、分卷大纲、卷纲、章清单。
- 把总纲或分部分配内容拆成情节块、故事线、故事点。
- 设计卷级启动表、节奏表、章节微型剧本与相关工件。

## 默认执行顺序

1. 先判定当前是分部层还是分卷层。
2. 先列“本层承接清单”，再开始拆结构。
3. 用设计规则确认上级内容是否全量落地。
4. 分部层套用分部模板，分卷层套用分卷模板与章节工件。
5. 最后检查目标字数来源、章数字数换算、节奏与钩子设计是否自洽。

## 核心硬规则

- 上一级分配给本层的内容必须切实、全量、保质落地。
- 目标字数必须以上一级分配为准，不能本层拍脑袋乱定。
- 章数规划默认按每章约 3000 字换算。
- 章清单是后续章节细化的唯一蓝图。
