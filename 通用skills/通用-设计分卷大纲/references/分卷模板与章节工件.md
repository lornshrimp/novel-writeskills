# 分卷模板与章节工件

## 1. 强制口径

- `章清单（Chapter Beat Sheet）` 是唯一蓝图。
- `逐章微型剧本（具体章节规划）` 是分卷执行版与正文之间的直接桥梁；若缺少这一层，当前产物只能算卷纲草案，不算可直接落正文的执行版。
- 逐章微型剧本必须与章清单严格一致，只允许细化镜头、对话、动作、证据 / 关键载体、状态变化、台账变化等执行细节；禁止自由改写视角、章目标、核心冲突、关键情节 / 转折、关联故事线、直接后果、伏笔 / 呼应、钩子类型。
- 分卷章数默认按每章约 `3000` 字换算；若用户指定偏移口径，必须显式写明偏移原因。
- 每章必须明确承接哪项上游任务、自己是显性落点章 / 保温章 / 交接章中的哪一种，并说明本章怎样为下一章创造继续阅读理由。
- 当章节抛出截图、成品版本、官方口径、系统记录、删改痕迹、版本差异等“引线型信息”时，应按需补写 `≤3章破口计划`；若该章承担破口起手章或关键推进章职责，则该字段必填。
- 若本卷采用故事日推进，必须先给出统一的 WHEN 口径（如 `D+0 = YYYY-MM-DD`），后续章节才能使用 `D+1 / D+2` 等相对写法；不能全卷只有相对时间没有绝对起点。
- 需要继续把逐章微型剧本下压为单章施工或章节正文时，优先再组合调用 `通用-执行场景单元`、`通用-创建小说正文` 等下游能力。

## 4. 逐章微型剧本补充说明

- `故事梗概` 不是摘要口号，要写到作者能直接据此开工。
- `本章卖点 / 读者记忆点` 尤其适合卷首章、阶段回报章、卷尾章；可用来承接签约抓点、目录页点击点、传播钉子。
- `镜头分解` 建议 `3–5` 个镜头，确保动作、信息与情绪都能被拍出来，而不是只剩抽象说明。
- `关键对话片段` 建议 `3–6` 句，必须带潜台词，避免只写“二人争执 / 达成一致”。
- `动作节点` 至少 `3` 组微因果链；这是微型剧本能否继续写正文的重要分水岭。
- `关键载体 / 证据清单` 采用宽口径：悬疑 / 职场可写物证、回执、工单、记录；情感 / 成长向可写触发物、关键物件、聊天记录、合照、礼物、现场布置等可回指载体。
- `本章台账 / 索引变化` 是按题材需要启用的增强项；若项目存在线索台账、伏笔台账、关系台账、证据台账，建议在逐章微型剧本里同步更新。
- `≤3章破口计划` 适用于截图、成品版本、官方口径、系统结论、删改痕迹、版本差异等“引线型信息”入场的章节。该字段至少写清：当前章属于哪一期破口计划的起手 / 推进 / 回收章，未来 `≤3` 章必须把哪条引线推进到什么可验证破口。
- 若本章的关键载体仍停留在截图、转述、成品视频、官方口径或系统结论层，通常不应省略 `≤3章破口计划`，否则后续容易失去回收压力。
- `章末钩子类型` 建议至少在 `转折 / 揭露 / 危机` 三类中选一，避免只写抽象情绪，不写下一章触发方式。
- 若关键章节已经具备完整逐章微型剧本，后续可继续调用下游单章施工或正文创作能力，避免章清单与正文之间断层。

## 6. 失败信号

- 情节块、故事线、故事点三层混写。
- 分部只有大意，没有跨卷分配。
- 分卷只有剧情概述，没有章级可写信息。
- 章清单和逐章微型剧本互相打架。
- 只有悬念口号，没有关键载体、证据与代价。
- 抛出了截图、成品版本、官方口径、系统结论或版本差异，却没有 `≤3章破口计划`。
- 写了卷级承接清单，但 `本卷上游任务落地核对表` 仍然找不到具体章节落点。
