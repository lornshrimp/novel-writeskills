# 分部与分卷设计规则

## 一、上一级内容全量落地

- 分部 / 分卷设计首先要确认：上一级已经分配给本层的内容，是否被切实、全量、保质落地。
- 这里的“分配内容”包括但不限于：主线故事、支线故事、故事面、情节块、故事线、单元案任务、伏笔与呼应、回收窗口、人物代价、关系变化、主线拼图、时间轴与台账约束。
- 落地至少满足：
  - 全量承接
  - 结构展开
  - 可写、可拆、可验收
  - 可反向追溯

### 1.1 全量落地的判定标准

- **有明确承接位置**：知道它落在哪个情节块、故事线、章节段落、节拍区间或执行工件。
- **有具体执行形式**：知道它如何作为事件、冲突、证据、关系变化、回报或代价出现。
- **有质量保证**：不能只剩一句口号式提及，必须达到能继续拆卷、拆章、拆场景的可执行程度。
- **无遗漏无悬空**：上级已分配给本部 / 本卷的内容，不得出现“这条以后再说”式悬空，除非上级明确写明后置。

### 1.2 禁止写法

- 只落地主线，不落地已分配支线。
- 只落地事件，不落地伏笔、呼应、证据链、代价链。
- 只写“本卷会推进某故事面”，却不写推进到什么程度、通过什么事件推进。
- 把上级已明确分配给本部 / 本卷的内容，改写成模糊背景说明或挪成“后续可选项”。

## 二、分层承接口径

- 分部大纲：承接总纲层文件组分配给本部的全部内容。
- 无分部小说的分卷：直接承接总纲层分配给本卷的全部内容，口径等同分部大纲。
- 有分部小说的分卷：承接本部分配给本卷的全部内容。

### 2.1 分部层总则

- 分部开工前，必须先列“本部承接清单”，把总纲层文件组分配给本部的内容逐项列出。
- 这里的总纲层文件组，至少包括：`小说总大纲.md`、`故事面/`、`伏笔与回收 / 呼应总纲`、相关总表级时间轴 / 矩阵 / 台账文件。
- 后续所有情节块、故事线、分卷分配，都必须能回指这份承接清单。

### 2.2 分卷层总则

- 分卷开工前，必须先明确“上级分配来源”：
  - 无分部结构：直接对照总纲层文件组，列出分配给本卷的内容。
  - 有分部结构：对照本部分部大纲，列出分配给本卷的情节块任务、故事线任务、伏笔任务、回收任务、人物推进任务。
- 随后必须给出“本卷承接清单”，逐项说明：本卷承接了什么、落在本卷哪个阶段 / 哪组章节 / 哪类回报节点、推进到什么程度、给下一卷留下什么交接点。

## 三、目标字数与章数来源

- 本层目标字数必须以上一级分配为准。
- 本层目标章数必须由上一级分配字数换算得出。
- 默认换算口径：每章约 3000 字。

### 3.1 正确顺序

1. 先确认上一级分配给本部 / 本卷的目标字数。
2. 再确认本层需要承接的主线 / 支线 / 伏笔 / 回收 / 代价任务。
3. 最后按每章约 3000 字换算章数，并据此设计节奏与章清单。

### 3.2 禁止写法

- 不看上一级分配，直接凭感觉写“本部 / 本卷约 XX 万字”。
- 为了凑结构好看，先拍一个卷数或章数，再倒推一个并无来源的目标字数。
- 上级已给定目标字数，本层却擅自扩张或缩减而不说明原因。
- 上一级未给定目标字数，却把暂定推算值伪装成正式分配值。

## 四、三层拆解定义

- 情节块：战役，跨卷承接战略故事面。
- 故事线：战斗，构成情节块的具体情节链条。
- 故事点：场景，最小可写单位，通常对应一章。

### 4.0 文件命名建议

- 分部大纲文件建议采用：`X.部标题 - 详细大纲.md`
- 分卷大纲文件建议采用：`X.Y.卷标题 - 分卷大纲.md`
- 若项目已有稳定命名体系，可沿用既有体系；但至少要保证部 / 卷层级一眼可辨，不与总纲文件或审阅报告混名。

### 4.1 核心概念澄清

- **情节块（Plot Block）** 是承接宏大战略故事面的跨卷叙事单元；一个情节块应有自己的起因、发展、高潮和阶段性结局。
- **故事线（Storyline）** 是构成情节块的具体情节链条；同一情节块下可以并行存在多条故事线，它们相互影响，但不是一个故事的不同标题。
- **故事点（Story Point）** 是构成故事线的最小叙事单位；它通常对应单一时间地点下发生的具体场景或事件。

### 4.2 情节块映射规则

- 一个情节块只能唯一映射一个战略故事面。
- 主线故事面在本部内通常拆成 `2–4` 个情节块。
- 支线故事面在本部内通常只对应 `1` 个情节块。
- 每个情节块必须在至少 `2` 卷中有明确推进，并在分卷分配中标注各卷承担的阶段职责。

## 五、分部层具象化要求

### 5.1 情节块必须写清

- 故事梗概
- 对应战略故事面
- 本部发展目标
- 核心人物与关系变化
- 时间跨度与关键节点
- 阶段性主矛盾
- 包含哪些故事线
- 在本部叙事结构中的功能（开篇布局 / 中期转折 / 高潮收束）
- 它承接了上级分配内容中的哪几项，以及各项在本块内推进到什么程度

### 5.2 每条故事线必须具象化

- 故事梗概
- `WHO / WHAT / WHEN / WHERE / WHY / HOW`
- 关键事件序列
- 冲突发展
- 转折点
- 与其他线的时间关系
- 反向引用
- 分卷分配
- 本线落实的上游任务

## 六、分卷层具象化要求

### 6.1 分卷不是摘要，而是执行蓝图

分卷大纲至少要写清：

- 本卷主题与主冲突。
- 本卷承接清单（上级已分配内容的全量落地表）。
- 本卷上一级分配目标字数、本卷采用目标字数与目标章数（按约 3000 字 / 章计算）。
- 卷内 `1–5` 章连续追读链。
- `5 / 15` 节奏布点。
- 章清单与逐章微型剧本。
- 卷级伏笔推进与证据 / 关键载体回收。
- 爽点布点、代价升级与章末钩子分布。

### 6.1.1 分卷 / 章纲的四种可读性

- **画面可读性**：读者能看见具体空间、动作、物件与场面变化。
- **机制可读性**：读者能理解规则、流程、权限、阻力或关键载体如何运作；不同题材可替换成制度、系统、修炼、关系、职业程序等对应机制。
- **情绪可读性**：读者能感到人物此刻怕什么、赌什么、舍不得什么、为什么现在必须往前。
- **追读可读性**：章末为什么一定要点下一章，必须说得清。

### 6.1.2 章级推进的“三联动”

每章尽量让以下三项至少出现两项，关键章应三项全齐：

- **关键载体 / 证据变化**：新增、校验、污染、误读、抢夺、销毁、复原、回指。
- **代价变化**：主角或相关角色承担了新的身体 / 关系 / 职业 / 名誉 / 法律 / 心理 / 资源损耗。
- **钩子变化**：产生新的问题、新的更坏后果，或旧问题的更危险版本。

如果一章只有氛围和说明，没有这三项中的任意两项，大概率仍停在“能看但不追”的概述层。

### 6.2 逐章微型剧本最低要求

- `WHEN`：至少精确到 `YYYY-MM-DD`。
- `WHERE`：具体地点。
- `WHO`：在场角色与身位关系。
- 核心冲突：人物诉求 vs 阻力。
- 镜头分解：`3–5` 个镜头。
- 关键对话：`3–6` 句，带潜台词。
- 动作节点：至少 `3` 个“起因 → 动作 → 结果”。
- 关键载体 / 证据清单：至少 `2` 项可审计或可回指载体。
- 状态变化：人物 / 关系 / 资源 / 空间至少一项变化。
- 悬念钩子：明确指向下一章的问题句或触发条件。
- 反向引用：对应到分部故事线 / 情节块。

### 6.3 反模板腔 / 反空泛要求

以下句子在分卷 / 章纲里出现时，通常说明还没写到可执行程度：

- “众人都感到不安”
- “线索逐渐浮现”
- “冲突进一步升级”
- “气氛压抑而诡异”
- “人物情感进一步升温 / 关系更复杂了”

推荐改写方式：统一压到 `场景动作 → 异常 / 关键细节 → 关键载体 / 证据 → 人物选择 → 后果 / 钩子`。

## 七、最终验收口径

- 是否写出了本层承接清单。
- 是否能回指上一级分配来源。
- 是否已落到情节块 / 故事线 / 章清单 / 工件。
- 是否不存在遗漏、缩水与悬空。
- 章清单与逐章微型剧本是否严格一致。
