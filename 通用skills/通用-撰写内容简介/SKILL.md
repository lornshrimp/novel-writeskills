# 通用-撰写内容简介

内容简介不是大纲摘要，而是用最短篇幅把“为什么值得点开继续看”讲清楚。

## 继续读取的 references

- `references/内容简介结构模板与压缩规则.md`
- `references/平台简介差异提醒.md`
- `references/简介落盘与版本管理.md`

## 何时使用

- 新书上架前撰写简介。
- 已有简介太像设定说明、太空、太散，需要重写。
- 针对不同平台生成不同长度、不同口气的简介版本。

## 默认执行顺序

1. 先抓主角当前困境与不可回头的入局点。
2. 再写核心冲突与持续追读承诺。
3. 再按平台长度收束语言、增强钩子与可传播句。
4. 再按独立文件规则落盘并处理版本策略。
5. 最后检查有没有把故事说完，或空到像海报标语。

## 最低交付

- 至少 1 个主版简介。
- 若涉及多平台，至少说明长版 / 中版 / 短版的适配差异。
- 必须让读者知道：谁、陷入什么局、为什么还会越陷越深。

## 通用硬规则

- 禁止只有世界观名词堆砌，没有情境与冲突。
- 禁止把全部反转提前剧透完。
- 禁止只说“命运齿轮开始转动”这类空口号。
- 除非用户明确要求只草拟，不得只在聊天里给简介而不落文件。
