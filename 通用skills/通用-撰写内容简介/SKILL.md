# 通用-撰写内容简介

内容简介不是大纲摘要，而是用最短篇幅把“为什么值得点开继续看”讲清楚。

## 常见触发词 / 用户说法速查

- 帮我写内容简介
- 这本书简介太空了
- 改一下平台简介
- 给我一个能上架的简介
- 这简介像设定说明不是故事

## 何时使用

- 新书上架前撰写简介。
- 已有简介太像设定说明、太空、太散，需要重写。
- 针对不同平台生成不同长度、不同口气的简介版本。

## 默认执行顺序

1. 先抓主卖点、主角当前困境与不可回头的入局点。
2. 再写核心冲突与持续追读承诺。
3. 再按平台长度收束语言、增强钩子与可传播句。
4. 再按独立文件规则落盘并处理版本策略。
5. 最后检查有没有把故事说完，或空到像海报标语。

## 简介共性提醒（新增）

- 简介默认优先交付情境与情绪共识，再补设定名词；若读者先看见一堆专有名词却没看见人物困境，视为失败信号。
- 主卖点最好压成“谁正在失去什么、必须做什么、为什么会越陷越深”这一条主钩，而不是平均陈列多个卖点。
- 若作品的持续吸引力建立在成长线、关系线或身份翻转上，简介里至少要让其中一条可被快速感知。
- 前 `1–2` 句默认还要承担 `30 秒归类` 任务：让编辑和读者能迅速判断这是什么书、主角正卡在哪个当下困局、继续看下去能得到什么。
- 若简介读完仍只能感知世界观名词，却感知不到人物处境、当前压力与持续承诺，默认按“设定说明化”处理并回炉。

## 默认输出口径

- 默认输出至少 1 个可直接使用的主版简介。
- 多平台场景下，默认补充长度差异或平台差异说明。

## 落盘目录层级（强制）

- 本 Skill 的默认工作目录固定为 `小说简介/`。
- 系列全书简介、分部简介、分卷简介统一直接落在 `小说简介/` 根目录。
- 章节简介按小说结构分层：若作品有分部，统一落在 `小说简介/第X部/第Y卷/`；若作品无分部，统一落在 `小说简介/第X卷/`。
- 文件名继续服从日期与版本号规则，不得把章节简介散落到 `章节目录/简介/` 一类旁路目录。

## 最低交付

- 至少 1 个主版简介。
- 若涉及多平台，至少说明长版 / 中版 / 短版的适配差异。
- 前 1–2 句就能让编辑和读者快速感知主卖点，而不是读完整段还只看到设定名词。
- 必须让读者知道：谁、陷入什么局、为什么还会越陷越深。

## 硬规则

- 禁止只有世界观名词堆砌，没有情境与冲突。
- 禁止把全部反转提前剧透完。
- 禁止只说“命运齿轮开始转动”这类空口号。
- 禁止简介承诺一个正文里根本接不住的假卖点。
- 除非用户明确要求只草拟，不得只在聊天里给简介而不落文件。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责简介文本本体。
- 不负责书名设计、书评撰写或投稿材料整体组包。
