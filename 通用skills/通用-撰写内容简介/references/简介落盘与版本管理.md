# 简介落盘与版本管理

## 简介必须分层分文件落盘

默认情况下，以下层级都应有各自独立文件：

- 系列全书简介
- 分部 / 分卷简介
- 章节简介

若存在“总库 / 合集 / 多平台草稿库”文件，它们默认只做：

- 索引
- 版本目录
- 链接与使用说明

不再承载某一卷 / 某一章的完整简介正文。

## 默认工作目录与文件名建议

- 默认工作目录固定为 `小说简介/`。
- 系列全书：`小说简介/《目标投放名》- 系列全书内容简介_vYYYY-MM-DD-rN.md`
- 分部简介：`小说简介/第XX部《目标投放名》- 内容简介_vYYYY-MM-DD-rN.md`
- 分卷简介：`小说简介/第XX卷《目标投放名》- 内容简介_vYYYY-MM-DD-rN.md`
- 章节简介：
  - 有分部：`小说简介/第X部/第Y卷/{章节号与标题}_章节简介_vYYYY-MM-DD-rN.md`
  - 无分部：`小说简介/第X卷/{章节号与标题}_章节简介_vYYYY-MM-DD-rN.md`

若项目已有既定命名体系，以既有体系优先。

## 名称口径规则（强制）

- 文件名、版本标题或正文中若需要出现作品名，统一使用当前交付对象的目标投放名。
- 不分部小说：使用小说的目标投放名。
- 分部小说：凡分部简介、分卷简介、章节简介及其平台变体，统一使用对应部的目标投放名；不得用总书名覆盖分部口径。
- 若本次明确交付的是整套作品总简介，则可使用整套作品自身的目标投放名，但必须在该版本内部保持前后一致。

## 覆盖 / 追加策略

- 用户明确要求“覆盖当前简介” → 在对应独立文件中替换旧版本
- 用户明确要求“另起一版 / 保留旧版本” → 追加带日期和版本标签的小节
- 用户未说明时 → 默认保守追加，保留原文并新增“本轮生成版本（YYYY-MM-DD）”

## 对话侧最小回执

至少说明：

- 本轮写入了哪些版本
- 写入到哪个文件
- 是覆盖还是追加
- 是否同步记录了 Rubric / 自检结果

## 只草拟不落盘的例外

只有当用户明确说“只草拟，不动文件”时，才允许不落盘。
