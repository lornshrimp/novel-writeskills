# 平台简介差异提醒

- 连载平台：更强调入局钩子、持续承诺、追读诱因。
- 社媒 / 宣发平台：更强调一句话卖点与传播句。
- 长简介可多放一层主线承诺，短简介优先保住“人 + 局 + 风险”。

## 平台主钩优先级

| 平台 | 主钩优先级 | 最容易写坏的点 |
| --- | --- | --- |
| 起点 | 世界观异常 + 主角行动 | 写成设定说明，冲突太慢 |
| 番茄 | 异常后果 + 生活入口被关 | 卖点过多，像吆喝文案 |
| 百度百家号 | 现实职业 + 禁忌异常 + 明确后果 | 写得太虚 |
| 小红书 | 日常异化 + 情绪代价 | 写成大纲压缩块 |
| B站 | 画面异常 + 反差冲突 | 只有设定，没有镜头 |
| 新浪微博 | 现场感 + 强感官异常 | 太长、太慢、太解释 |
| 豆瓣阅读 | 冷细节 + 体面 / 羞耻裂缝 | 写成爽文宣发语 |
| 知乎盐选 | 强动作异常 + 反咬代价 | 写成纯设定推理 |
| WebNovel | 高概念异常 + 单次行动 | 人名地名设定过多 |
| GoodNovel / MyFiction | 关系秘密 + 危险感 | 过冷、过技术 |

## 可读性底线

- 同一版本保持单一叙述声道
- 每句默认只承载 1 个主信息点
- 每段 2–4 句，至少 1 句短刀式钉子句
- 同一版本最多只保留 1 套倒计时
- 比喻尽量不超过 1 处，且必须能还原成事实镜头

## 平台适配提醒

- 平台适配优先适配“主钩形态”，不是简单换词
- 不要把所有平台都写成同一个底稿删字版
- 目标读者读完后，应能复述“谁遇到了什么异常 / 代价”
