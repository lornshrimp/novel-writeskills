# 通用-提纯多平台母稿

多平台输出之前的母稿提纯，不是先把原文磨成中性白开水，而是先把它修成**足够硬、足够稳、足够可拆、足够可派生**的源稿。

## 保真迁移要求（强制）

命中“多平台母稿提纯 / 平台改写前清母稿”任务时，除本文件外，**必须继续读取**：

- `references/多平台母稿提纯规则.md`
- `references/多平台母稿提纯检查清单.md`

## 何时使用

- 在多平台输出之前，先把一章整理成更干净、更稳定的母稿。
- 当前源稿平台腔太重，不利于后续再改写到其他平台。
- 需要保住事件链、情绪主轴与章末钩子，但去掉只适合单一平台的冗余壳层。
- 你想减少后续多平台改写时的返工，先把不可动的核心钉牢。

## 不适用情形

- 你要直接做平台差异化输出与落盘。
- 你要把母稿写成平台无差别中性稿。
- 你只想单平台终稿精修，而不是做派生前的底稿纯化。

## 常见触发词 / 用户说法速查

- 先把母稿提纯一下
- 这份底稿太像某个平台了
- 多平台前先修源稿
- 帮我把这章变成可派生母稿
- 先把平台壳层去掉但别磨平

## 本 Skill 的核心任务

1. 提炼一份**更适合派生改写**的母稿口径。
2. 保住：
   - 事件链
   - 角色目标 / 阻力 / 代价
   - 关系变化
   - 规则边界与证据关键点
   - 章首抓力、中段回报、章末钩子
3. 去掉或压低：
   - 单一平台过重口癖
   - 重复强调与重复补偿同一信息
   - 过度解释
   - 不利于跨平台迁移的局部腔体与外壳
4. 默认把“这几章 / 前几章 / 上一章 / 下一章 / 本章 / 这章 / 后文会 / 我们会看到”这类章节自指 / 连载导览 / 作者讲评腔视为必须先清掉的作者壳层，否则会被后续平台版本复制放大。

## 研究回灌补强（2026）

- 母稿提纯时必须保住三类锚点：
   1. 开篇抓力锚点（首屏冲突 / 异常 / 风险）；
   2. 人物情绪锚点（可共情的现实痛点与选择压力）；
   3. 章末驱动锚点（下一步必须追读的未决问题）。
- 不得把“提纯”做成“去情绪”：应去平台壳，不去情绪主轴；应去重复解释，不去关键回报。
- 手机端可读性属于母稿质量的一部分：连续大段、说明墙、句群过匀必须先在母稿层拆解，不把负担留给平台改写层。


## 执行顺序

1. 先判断目标是：提纯现有源章，还是提纯已存在母稿。
2. 先分清两层内容：
   - 必须钉死的核心层
   - 后续可分化的平台层
3. 先查事实和证据有没有松动。
4. 再查章首、中段、章末三个关键支点是否成立。
5. 再查人物当章压力是否具体、可感。
6. 最后处理不利于派生的单平台壳层与重复句群。
7. 复检：后续改写到其他平台时，是否更容易拉出差异化版本。

## 默认输出口径

- 默认面向平台派生前的母稿提纯，不直接生成多平台终稿。
- 默认对象是准备进入多平台流程的中文母稿。
- 默认直接回写母稿文件。

## 最低交付

一次合格的多平台母稿提纯，至少要做到：

- 明确哪些内容是后续所有平台都不能丢的核心。
- 让章首、中段、章末三处关键支点成立。
- 保证证据 / 规则关键点、时间、关系、目标与代价稳定。
- 保留足够可分化空间，让后续平台适配能各写各的。

## 硬规则

- 不得把母稿提纯理解成“去平台感、去个性、去锋利”。
- 不得为了后续好改而把事件、代价、规则、证据、钩子磨平。
- 不得提前把母稿改成某个平台专属腔，再拿它反向套别的平台。
- 不得混用平台稿回修母稿，母稿必须作为源头处理。
- 不得把章节自指、下一章预告或作者讲评口吻误当成“连载感”；真正的连载感来自未完成动作、关系余波、新变量与具体风险。
- 不得把文本提纯成无味白底稿。

## 与其他 Skill / Prompt 的边界

- 本 Skill 处理的是**派生前的底稿纯化**，不是最终平台适配。
- 若用户只想精修单一平台正文，应优先用对应的正文润色类 skill。
- 多平台输出与落盘流程应由平台适配 / 输出 SOP 继续承接。
