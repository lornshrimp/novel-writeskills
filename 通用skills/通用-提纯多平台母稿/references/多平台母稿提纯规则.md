# 多平台母稿提纯规则

## 一、目标：做“可派生”的底稿，不做“某平台终稿”

母稿提纯的目标不是把文本润到某个平台最满，而是把它整理成：

- 事件链稳定
- 情绪主轴清楚
- 钩子仍在
- 平台口癖下降
- 后续改写空间变大

## 二、先分清两层东西

### 1. 必须钉死的核心层

这些东西后续所有平台都不能乱：

- 事实口径
- 事件顺序
- 证据链 / 规则关键点
- 时间与编号
- 人物关系与动机
- 本章短线目标
- 中段回报
- 章末钩子根部

### 2. 后续可分化的平台层

这些东西后续允许按平台拉开：

- 章首切法
- 对话密度
- 段落长短
- 关系戏权重
- 日常缓冲比重
- 标题口味
- 作者有话说口吻

## 三、提纯顺序

1. 先查事实和证据有没有松动。
2. 再查章首、中段、章末三个关键支点是否成立。
3. 再查人物的当章压力是否具体、可感。
4. 最后查哪些地方后续可以安全分化，不要先把它们写死。

## 四、优先去掉或压低的东西

### 1. 单一平台腔体太重

比如只适合某个平台读感的过量口号、过密直接刺激或过度讲解的特殊壳层。

### 2. 同一信息反复补偿

为了照顾某一平台节奏而反复提醒同一信息，容易让后续派生卡死。

### 3. 太难迁移的局部修辞

如果某些句子极强依赖当前平台的腔体，提纯时可先压回更稳的表达层。

## 五、提纯不是删骨头

不能因为要提纯就：

- 丢事件
- 丢目标与代价
- 丢关系变化
- 丢规则
- 丢证据
- 丢钩子

提纯只处理外壳和重复，不处理故事骨架。

## 六、与多平台输出的关系

- 母稿提纯：做干净底稿
- 多平台输出：按平台口径二次改写

通常应先提纯，再派生。

## 七、验收口径

一份合格母稿应满足：

- 后续更容易往不同平台拉开口吻差
- 改写时不必先拆一堆重复壳层
- 读者骨架体验不丢：事件、情绪、回报、钩子仍在
- 核心稳、证据 / 规则关键点稳、后续可分化
