# 通用-设计技术设定

把技术设定写成作者能调用的“规则 + 痕迹 + 代价”文档，而不是技术说明书。

若约束较多，固定按三层优先级执行：先确认目标文件、作者侧口径与写回策略，再补规则条款、证据映射、程序摩擦与代价，最后补阶段演进、验证流程与一致性收口。

命中技术设定设计任务时，除本文件外，**必须继续读取**：

- `references/技术设定模板与镜头包.md`
- `references/证据链与程序摩擦规则.md`
- `references/创新性要素库与阶段红线.md`

- `references/结构规范与真实感要求.md`

## 常见触发词 / 用户说法速查

- 帮我做技术设定
- 重写这套规则文档
- 这条手法边界不清
- 给我补证据链和程序摩擦
- 这份设定太像讲义了

## 何时使用

- 新建某个技术设定 / 手法 / 程序链 / 证据载体文档。
- 重写已有规则文档。
- 给设定补“可证据化映射、镜头包、误导与证伪对”。
- 修复设定万能化、讲课化、空概念化问题。

## 不适用情形

- 只做设定审阅，不改稿：优先使用 `通用-审阅技术设定`。
- 用户已明确要求本轮只做概念讨论、且不落作者侧资料文件。

## 默认输出口径

- 默认产物是**作者侧资料文档**：供大纲、正文、台账交叉引用与核查，不是读者向设定科普。
- 只有在用户**明确要求**“先讨论、不写文件”或**明确标注**“本轮不落库”时，才允许只在聊天中讨论；除此之外，一律直接创建或修改真实资料 Markdown 文件，不允许把“只脑暴、不落盘”当成默认路径。
- 当用户打开某个技术设定文件并调用本技能时，必须把生成或修订后的“手法机制 / 程序链条 / 证据载体与字段 / 现实语境与限制 / 在主线与单元事件中的作用”写入目标文件的对应小节。
- 对本轮涉及的每个关键要素，至少补齐：**一句话口径（动作→凭证→后果）+ 误导点与证伪路径 + 成本如何在正文里看得见**。
- 默认优先补齐以下骨架字段：**正式名称 / 别名黑话 / 代号编号规则 / 首次出现 / 关键节点 / 影响范围 / 敏感等级**，避免只剩一个抽象名词顶在文首。
- 默认优先补齐以下调用层字段：**触发条件 / 标准表现或感官反馈序列 / 失效或衰减规则 / 关键参数（固定值 vs 叙事弹性）/ 最小调用流程卡 / 分级代价与程序痕迹 / 不能直接解决的问题**。
- 涉及阶段演进、终局收束或多版本对齐时，默认补一段**本轮修订（YYYY-MM-DD｜对齐版本）**，写清这轮到底修了什么、对齐了哪条上游口径。
- 若用户明确要求“完全重写”，按模板骨架覆盖重写；若只要求补某一块，则在对应小节追加或替换，默认保留其他未涉及部分。
- 若用户要求为新技术概念单独建档，默认在对应资料目录下新建 `要素名.md`，并确保其时间线与长篇演进口径一致。
- 若原文件历史较长且用户要求“保留原文、只补最新版本”，则在文末追加“本轮修订（YYYY-MM-DD）”小节；若用户未说明，默认采用保守补充和纠偏策略，不直接删除原有设定。
- 聊天侧必须简要说明：写入或新建了哪个资料文件、采用覆盖还是追加策略、是否已对齐总纲 / 多面体 / 伏笔台账等上游口径。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，设计技术设定前必须先读取并纳入本轮判断。
- 若目标平台为起点中文网，或未注明平台但默认按起点口径处理，且当前题材目录下尚无可用竞对主报告，开始技术设定设计前应优先调用 `通用-分析竞对作品`，至少补齐 **Top10 候选池、`3–5` 本主样本与四层深拆速览**；若任务限制导致本轮无法补齐，必须显眼记录盲区，并降低本轮技术设定成熟度判断。
- 不得把这些报告只当成背景资料；技术设定文档必须显式说明本轮准备在哪些维度追平、反超竞对，以及哪些主样本暴露出的旧坑必须避开。
- 技术设定的默认目标不是“规则写清楚就行”，而是让当前作品至少在**规则辨识度、边界与代价、证据化能力、正文可拍性、长线供血**这 `5` 个默认关键维度里形成可举证的追平路径，并尽量在其中 `2–3` 项建立明确领先方案。

## 起点化竞对吸收口径（新增）

当技术设定服务起点中文网，或未标平台而默认按起点基线处理时，默认把竞对吸收拆成以下四层，并翻译成设定级设计约束：

- **市场数据层**：用榜单位次、均订 / 首订 / 收订比代理信号、推荐资源位等，反推这套机制 / 手法是否真的支撑题材归类、持续付费价值与长线商业厚度，而不是只在作者视角里“设定很酷”。
- **内容创作层**：重点吸收主样本在规则边界、失败条件、代价显影、证据链、误导与证伪、现实锚点、正文镜头包上的做法，把它翻成当前设定文档的条款、字段、镜头包与调用流程卡。
- **运营策略层**：重点吸收主样本如何让机制持续供血、制造追读工程、讨论点与阶段升级感，把它翻成设定的阶段演进、版本更新、程序摩擦与章节显影策略。
- **受众反馈层**：重点吸收本章说 / 书友圈 / 书评区里高频被夸的规则爽点、被骂的万能化病灶、老白读者对吃书 / 降智 / 失衡外挂的敏感点，提前写进本轮设定的避坑与加码清单。

起点语境下，默认尽量使用以下术语，而不是写成空泛的“设定更强更好看”：`均订`、`高订`、`首订`、`追订`、`收订比`、`分类月票榜`、`畅销榜`、`新书榜`、`三江`、`本章说 / 段评`、`书友圈`、`老白读者`。拿不到精确值时必须写成代理信号，不得伪造具体数字。

## 设计目标

1. 规则必须能拆成条款。
2. 每条关键规则都尽量有痕迹可取。
3. 设定必须有边界、代价、漏洞与反制。
4. 必须给正文提供镜头化写法。
5. 必须能对齐长篇阶段演进，而不是一次性用完。
6. 必须服务作者调用体验，而不是写成只适合阅读的说明书。
7. 高价值设定默认尽量压成“动作 → 载体 / 字段 → 时间 / 版本 → 后果”的调用格式，减少空概念漂浮。

## 默认执行顺序

固定优先级：先定目标文件与作者侧资料口径，再补规则 / 证据 / 代价，最后补阶段演进、验证流程与一致性收口。

0. 若按起点口径执行且当前题材目录下缺少可用竞对主报告，先调用 `通用-分析竞对作品`，补齐 `Top10 候选池 → 3–5 本主样本 → 四层深拆` 的最小结论集。
1. 先确认目标设定文件、修改范围与写回策略。
2. 若按起点口径执行，先吸收竞对主样本四层结论，并写清本轮设定要追平 / 反超的关键项。
3. 再用模板与镜头包、示例格式搭骨架，并锁定作者向资料口径。
4. 再用证据链与程序摩擦规则补边界、代价、误导与证伪路径。
5. 再用创新性要素库、长篇演进与城市语境对齐阶段使用策略；若涉及超常、民俗或能力机制，再补读科学对冲玄学与能力代价设计。
6. 最后用中国语境与一致性检查、结构规范与真实感要求和验证流程收口，并实际写回文件。

## 世界规则分层、事实更新与真相版本补强（新增）

- 当项目设定量较大时，技术设定默认区分：地理 / 地点、世界规则 / 机制、组织 / 势力、时间线、专项技术或手法文档；不要把所有规则挤在一篇大杂烩里。
- 设定文档默认只记录已确认事实、已显影代价、已发生变化和已存在的版本差异，不预写未来章节可能会怎么发展。
- 涉及历史、神话、制度口径时，默认尽量区分“作者知道的真相”与“角色当前知道的版本”；正文只能调用角色可知口径，不能让角色替作者讲底层真相。
- 章节定稿后若正文与旧设定冲突，应先判定是正文写歪还是设定需更新；若最终以正文为准修设定，必须补写本轮修订说明与对齐口径。
- 高价值规则文档默认能快速回答：谁能用、何时触发、留下什么痕迹、付什么代价、最容易露馅的破绽是什么、不能直接解决什么问题。

## 最低交付

- 能写清规则条款、边界、代价、漏洞与反制。
- 能给出证据载体、程序摩擦与镜头化写法。
- 能作为作者侧资料文件直接被总纲、正文与台账调用。

## 硬规则

执行时只按以下三组顺序检查：先守作者侧落盘边界，再守规则与代价边界，最后守安全与阶段演进边界；前一组未过，不进入后一组。

- 先守作者侧落盘边界：
	- 除非用户明确要求“先讨论、不写文件”，否则禁止只在聊天中输出长篇设定而不落库。
	- 高价值设定默认优先给出原始载体、时间戳 / 版本号、操作人 / 权限与回执 / 日志出口，而不是只给概念名词。
	- 高价值设定默认必须至少给出一张**最小调用流程卡**，把“什么条件下能用、能产出什么、不能越过哪条红线”压成作者速查格式。
	- 若按起点口径执行，动手前必须先写清：本轮设定准备追平 / 反超哪类主样本的哪项规则辨识度、代价显影或证据化能力。
- 再守规则与代价边界：
	- 禁止万能设定。
	- 禁止只有术语没有动作链。
	- 禁止只有设定酷炫没有现实摩擦。
	- 每个高价值设定都必须写“最容易露馅的破绽”。
	- 高价值设定必须能回答：它如何在前三章留下痕迹、谁先为它付代价、主角如何第一次验证它。
- 最后守安全与阶段演进边界：
	- 涉及现实高风险可复刻细节时，必须叙事化、留白化、降教学性。
	- 高价值设定默认必须回答：**它不能直接解决什么问题**、**最贵的后果会反咬谁**、**终局时要不要关门 / 降格 / 留余波**。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责技术设定本体的设计与重写。
- 若用户只要体检、评分和回写建议，应优先使用 `通用-审阅技术设定`。
