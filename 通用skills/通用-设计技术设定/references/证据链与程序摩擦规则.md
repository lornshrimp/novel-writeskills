# 证据链与程序摩擦规则

## 证据链基本要求

- 技术设定里的证据链，默认优先压成：**动作 → 载体 / 凭证 → 可验证后果**，避免只剩“后来查到证据”这种作者上帝视角句。
- 每个关键判断都必须能回指到证据、记录或明确的心理动因，避免“作者手动按下真相按钮”。
- 证据链要写出依赖关系：监控 ↔ 门禁 ↔ 通话 ↔ 票据 ↔ 口供 ↔ 物证如何互相支撑或互相打脸。
- 关键证据必须考虑可伪造性、可篡改性与被误读可能，不能默认天然可信。
- 对同一事实的描述，要在时间、地点、人名、编号、因果上保持口径同步。
- 关键证据最好继续回答：**谁留下的、什么时候留下的、哪个编号 / 版本、谁能改、谁能导、谁能卡住下一步验证**。

## 程序摩擦最低要求

- 流程与权限应大体可信，关键环节不能“瞬移”。
- 时效要有边界：取证、鉴定、走访、协查为什么快、为什么慢，都要给理由。
- 人手、经费、关系、舆论压力与风险代价要显性化，不允许后台自动结算。
- 程序摩擦不是装饰，而是冲突发动机：拖延、回执空白、窗口错过、责任甩锅都应服务剧情推进。
- 程序摩擦至少要有一处能在正文里被直接看见：回执空白、导出失败、权限拒绝、签字卡点、窗口超时、现场被提前污染等。

## 组织流程型证据链默认骨架

当故事发生在公司、平台、机构、学校、社区、医院或任何组织环境中，默认优先把证据链拆成以下五段：

1. **原始载体**：最早留下痕迹的东西是什么。
2. **时间戳 / 版本**：它是什么时候留下的，是否有版本变化。
3. **权限节点**：谁能看、谁能改、谁能删、谁能导出。
4. **回执出口**：主角如何把它带出流程，形成下一步可用材料。
5. **后果落地**：这条证据改变了谁的处境、风险或判断。

只写“有证据”通常不够；最好继续回答：是谁留下的、什么时候留下的、谁能动它、谁能卡住验证动作。

## 组织环境下的常用载体

优先考虑这些现实里更常见、也更容易制造程序摩擦的载体：

- OA / 审批流 / 工单 / 工位系统
- 考勤 / 门禁 / VPN / 登录日志 / 版本记录
- 邮件 / 群聊 / 会议纪要 / 培训记录 / 回执
- 合同 / 对账 / 绩效表 / 报销单 / 外包交接单
- 录音 / 截图 / 导出文件 / 录屏 / 操作留痕

这些载体的价值不在“专业名词很多”，而在它们天然带着：时间、责任、权限与可推诿空间。

## 机制 / 程序条款建议

- 一句话口径：动作 → 载体 / 凭证 → 后果
- 核心机制：它如何成立
- 标准显影：第一次被看见时，最先露出来的纸面 / 界面 / 物件 / 环境征候是什么
- 关键矛盾点：它为什么难查 / 难证 / 难对抗
- 链条结构：输入 → 处理 → 输出
- 关键字段 / 参数：哪些必须精确、哪些可模糊叙述
- 关键字段 / 参数最好继续区分：**固定值**（不能漂）与**叙事弹性**（允许在范围内浮动）
- 最小调用流程卡：判断是否可用 → 拉取什么载体 → 谁来验证 → 红线不能越过什么
- 分级程序摩擦与代价：轻度卡点 / 中度拖延 / 重度翻车 / 终局反咬分别怎么显影
- 前置条件：权限 / 关系 / 资源 / 地点 / 心理前提
- 失败后果：一旦没拿到、来晚了、权限卡住了，会失去什么

## 最小证据流程卡（作者速查）

| 步骤 | 动作 | 可见凭证 | 允许产出 | 禁止事项 |
| --- | --- | --- | --- | --- |
| 1 | 判断原始载体是否还在有效窗口内 | 原始文件、原件、门禁记录、日志首条、物件编号 | 确认是否值得追 | 不得把“听说有记录”直接写成有效证据 |
| 2 | 锁时间戳 / 版本 / 编号 | 时间码、版本号、装订痕、签字页、导出目录 | 建立第一层硬钉子 | 不得跳过版本差异直接下结论 |
| 3 | 穿权限节点 | 审批、授权、回执、导出申请、见证人在场 | 让材料进入下一步剧情 | 不得把越权查看写成零代价 |
| 4 | 做交叉验证 | 第二载体、第三方记录、同屏对照、回拨、复核 | 缩小判断空间 | 不得把单一载体直接升级成终局真相 |

## 分级程序摩擦与代价显影

| 级别 | 常见表现 | 会留下的程序痕迹 | 正文最适合怎么写 |
| --- | --- | --- | --- |
| 1 级：轻度卡点 | 回执延后、导出变慢、签字人不在 | 时间空窗、催办记录、口头推诿 | 先写“还没下来”，再写人物情绪起毛边 |
| 2 级：中度拖延 | 权限被卡、材料被退回、窗口错过 | 退件记录、二次申请、补件通知 | 让人物为一张纸多跑一次，体感才会真 |
| 3 级：重度污染 | 记录被改、现场被动过、口径统一化 | 版本差异、删改痕迹、对照表、异常缺页 | 让“太整齐”本身成为警报 |
| 4 级：终局反咬 | 证据链反向伤到主角 / 同伴 / 组织 | 纪律风险、公开追责、舆情质疑、资格受损 | 只能在已有护栏和代价铺垫时写，不能临时加码 |

## 不能直接解决的问题

- 单一载体不能直接替代终局判断，它只能缩小空间、压实方向或逼出下一步验证。
- 能导出不等于能被采信；能看见不等于能带走；能带走不等于能立刻公开。
- 证据链再硬，也不能替代人物动机、关系站位与对手反制的戏剧过程。
- 高价值证据链默认不应“一拿到就全盘结束”；若一条证据能无代价直接通关，通常说明设定已经滑向万能按钮。

## 误导与证伪对

- 若该设定可被用来误导，必须同时写明：误导点是什么、谁会信、为什么会信。
- 对应证伪路径必须具体到：谁来证伪、借助什么、多久能打穿、要付什么代价。
- 误导不能只靠“大家都笨”，而应来自合理的时间窗、流程盲点、载体局限或认知偏差。

## 三问自检

1. 为何现在能拿到这条线索 / 证据？
2. 为何之前拿不到、没注意、或被错误理解？
3. 为何它仍然不能一锤定音，必须保留推理空间与连载张力？

## 安全与伦理边界

- 避免写成可复刻的现实违法教程。
- 敏感环节优先改为叙事化、留白化表达，并显式说明“这里只保留非教学性描述”。
- 若本轮补的是主线证据护栏、终局材料或组织流程版本差异，建议在设定文档文末补 `本轮修订（YYYY-MM-DD｜对齐 Vxx）`，方便追踪哪一轮开始加护栏。

## 一、证据优先级

优先让设定落到这些载体：

- 日志 / 时间戳 / 版本号
- 门禁 / 监控 / 通话 / 工单 / 回执
- 封存链 / 装订痕 / 编号
- 口供 / 记录体（只能做一部分，不可单独定性）

## 二、程序摩擦必须可见

不要只写“手续很麻烦”，而要写：

- 卡在哪一步
- 谁有权限
- 谁在拖延
- 为什么拖
- 主角为了补链要付什么代价

## 三、误导与证伪对

每个重要设定最好都写：

- 最顺手的错误解释是什么
- 为什么大家会先信它
- 用什么痕迹把它打穿

## 四、边界与反制

至少回答：

- 它为什么不能一锤定音
- 对手如何反向利用它
- 主角如何识别被人动过手脚

## 五、常见失误

- 写成高科技神迹
- 规则很多，但没有痕迹
- 有痕迹，但没有获取难度
- 只写设定如何酷，不写它如何害人或逼人选边站
- 只有“截图 / 录音 / 记录”这些结果词，却完全不写获取路径、权限节点与导出代价
- 写了参数，却不区分哪些值绝不能漂、哪些只允许在范围内浮动
- 没有最小证据流程卡，导致作者知道规则却不知道正文第一步怎么落
- 没写“不能直接解决的问题”，导致证据链天然朝“一条证据定乾坤”滑去
