# 证据链与程序摩擦规则

## 证据链基本要求

- 每个关键判断都必须能回指到证据、记录或明确的心理动因，避免“作者手动按下真相按钮”。
- 证据链要写出依赖关系：监控 ↔ 门禁 ↔ 通话 ↔ 票据 ↔ 口供 ↔ 物证如何互相支撑或互相打脸。
- 关键证据必须考虑可伪造性、可篡改性与被误读可能，不能默认天然可信。
- 对同一事实的描述，要在时间、地点、人名、编号、因果上保持口径同步。

## 程序摩擦最低要求

- 流程与权限应大体可信，关键环节不能“瞬移”。
- 时效要有边界：取证、鉴定、走访、协查为什么快、为什么慢，都要给理由。
- 人手、经费、关系、舆论压力与风险代价要显性化，不允许后台自动结算。
- 程序摩擦不是装饰，而是冲突发动机：拖延、回执空白、窗口错过、责任甩锅都应服务剧情推进。

## 机制 / 程序条款建议

- 核心机制：它如何成立
- 关键矛盾点：它为什么难查 / 难证 / 难对抗
- 链条结构：输入 → 处理 → 输出
- 关键字段 / 参数：哪些必须精确、哪些可模糊叙述
- 前置条件：权限 / 关系 / 资源 / 地点 / 心理前提
- 失败后果：一旦没拿到、来晚了、权限卡住了，会失去什么

## 误导与证伪对

- 若该设定可被用来误导，必须同时写明：误导点是什么、谁会信、为什么会信。
- 对应证伪路径必须具体到：谁来证伪、借助什么、多久能打穿、要付什么代价。
- 误导不能只靠“大家都笨”，而应来自合理的时间窗、流程盲点、载体局限或认知偏差。

## 三问自检

1. 为何现在能拿到这条线索 / 证据？
2. 为何之前拿不到、没注意、或被错误理解？
3. 为何它仍然不能一锤定音，必须保留推理空间与连载张力？

## 安全与伦理边界

- 避免写成可复刻的现实违法教程。
- 敏感环节优先改为叙事化、留白化表达，并显式说明“这里只保留非教学性描述”。

## 一、证据优先级

优先让设定落到这些载体：

- 日志 / 时间戳 / 版本号
- 门禁 / 监控 / 通话 / 工单 / 回执
- 封存链 / 装订痕 / 编号
- 口供 / 记录体（只能做一部分，不可单独定性）

## 二、程序摩擦必须可见

不要只写“手续很麻烦”，而要写：

- 卡在哪一步
- 谁有权限
- 谁在拖延
- 为什么拖
- 主角为了补链要付什么代价

## 三、误导与证伪对

每个重要设定最好都写：

- 最顺手的错误解释是什么
- 为什么大家会先信它
- 用什么痕迹把它打穿

## 四、边界与反制

至少回答：

- 它为什么不能一锤定音
- 对手如何反向利用它
- 主角如何识别被人动过手脚

## 五、常见失误

- 写成高科技神迹
- 规则很多，但没有痕迹
- 有痕迹，但没有获取难度
- 只写设定如何酷，不写它如何害人或逼人选边站
