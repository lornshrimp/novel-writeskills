# 长篇演进与城市语境

本节用于把技术设定放进长篇结构里使用，避免“一次性抛完”的资料式设定。

## 与总纲口径对齐

默认应与以下文件保持一致：

- 《小说大纲/作品名 - 小说总大纲.md》
- 《小说大纲/故事多面体大纲.md》
- 《小说大纲/伏笔与呼应总纲.md》

若项目存在红线表、证据表、手法表，以这些文件措辞为准。

## 阶段发展与红线

### 第一阶段《日常裂缝》

- 核心锚点：异常进入日常与第一批可复盘线索。
- 可为：通过走访、物业台账、公开记录拿到“够用但不完美”的线索。
- 红线：不得开篇直接给决定性证据；不得用万能监控覆盖一切；不得一句话解释全部异常。
- 关键术语 / 设施：门禁点位、监控留存期、接警回执、物业工单、快递面单。
- 升级锚点：同一异常至少 3 次换角度出现。

### 第二阶段《单元案升级》

- 核心锚点：生活纠纷升级到刑事风险，程序链条开始显影。
- 可为：允许红鲱鱼与假线索，但必须安排可证伪路径。
- 红线：不得用神谕式直觉直接锁凶；不得用不解释的高科技一键复原；不得把程序写成传送门。
- 关键术语 / 设施：询问笔录、现场勘验、协查函、封存编号、证据袋。
- 升级锚点：每个单元案至少一次“证据反噬”。

### 第三阶段《黑幕逼近》

- 核心锚点：证据从“找到”升级为“守住 / 交换 / 撕裂”。
- 可为：允许多视角拼图、记录体与伪纪实片段，但必须与正文互证。
- 红线：不得堆文件替代戏剧；不得让关键证据只靠作者安排落手；不得无成本解决舆论 / 权力阻力。
- 关键术语 / 设施：举报与反举报、舆情扩散、证人保护、灰产话术、交易地点规矩。
- 升级锚点：卷内至少 3 次证据链断裂 / 被篡改 / 被偷换危机。

### 第四阶段《终局对决》

- 核心锚点：真相靠证据链闭环 + 人物代价揭示，并阶梯式释放。
- 可为：允许终局反转，但必须回指前文伏笔。
- 红线：不得用大段口供自白替代证明；不得临时添加关键设定 / 证据；不得让主角无代价胜利。
- 关键术语 / 设施：对质、庭审 / 听证、关键物证归属、赔偿 / 追责、消失的证人。
- 升级锚点：终局至少三处“同一真相的不同证据侧面”互证。

## 线索 / 手法 / 证据的传承逻辑

- 积累：前一阶段的红线边界，成为后一阶段的对照基线。
- 节点：通过事件触发升级，而不是无因跃迁。
- 扩散：从个案 → 同类案 → 产业链 / 组织规矩 → 系统性黑幕。
- 适应：用制度、市场、人情、伦理四维联动呈现阻力。
- 反向影响：证据链反过来改变人物关系与阵营。

## 长篇演进

设定不能只解决当章问题，还要回答：

- 开篇如何投放异常
- 中盘如何升级为证据战争
- 后盘如何与组织 / 黑幕联动
- 终局如何闭环且不临时加规则

## 城市语境

优先考虑：

- 物业 / 社区 / 网格 / 医院 / 平台公司等机构链路
- 老旧小区、园区、医院、地铁等空间
- 熟人关系、人情与程序摩擦并存
- 群聊、截图、删帖、匿名爆料等舆论生态

## 叙事边界

- 不把设定写成教程
- 不用宏论替代人物处境
- 灵异 / 民俗 / 技术规则必须可复盘且有代价
