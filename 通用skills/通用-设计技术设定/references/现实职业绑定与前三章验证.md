# 现实职业绑定与前三章验证

## 核心口径

- 技术设定最稳的入口，不是设定本身有多怪，而是它和主角职业 / 身份 / 日常流程绑得有多紧。
- 若换掉职业、岗位或现实流程后，设定仍然完全成立，通常说明绑定不够深。
- 职业绑定最好也压成：**职业动作 → 可见载体 / 痕迹 → 现实后果**，而不是只写“他因为专业所以懂”。
- 高价值设定默认要回答：**为什么偏偏是这个人先碰到、先看懂、先被反咬、先付代价**。

## 现实职业绑定最低要求

- **职业接口**：这条设定最先咬住的是哪项岗位职责、日常动作或行业习惯。
- **资源接口**：主角为什么能先接触这个载体 / 场景 / 异常，而别人不能。
- **权限接口**：他能看到什么、不能看到什么、越线会付什么代价。
- **风险接口**：一旦用了这条设定，会最先伤到职业信用、组织位置、人际关系还是身体状态。
- **曝光接口**：它最容易在哪个熟悉流程节点露馅，谁最可能先察觉不对。

## 职业绑定最小卡（作者速查）

| 字段 | 至少要答清 |
| --- | --- |
| 职业 / 身份 | 主角在这个系统里凭什么会先碰到它 |
| 日常流程 | 这条设定最自然嵌进哪个常规动作 |
| 第一触发点 | 第一次出手发生在什么具体场景 |
| 第一痕迹 | 读者最早会看见什么载体 / 征候 |
| 第一代价 | 谁先为它付出身体、关系或程序代价 |
| 第一误判 | 周围人最可能先把它误解成什么 |

## 前三章验证快检

### 第一章

- 是否已经看见设定第一次咬到现实困局，而不是只在说明层介绍。
- 是否已经留下第一处可回指痕迹：动作、回执、异常读数、物件反应、职业反常等至少一种。

### 第二章

- 是否已经看见主角开始用它判断、保命、翻案、试错或探测异常。
- 是否已经出现第一层程序 / 职业摩擦：权限不够、同伴不信、窗口卡点、口径冲突等。

### 第三章

- 是否已经完成一次有效验证、误判、代价落地或小回报。
- 是否已经让读者知道：它**不能直接解决什么**，而不是默认它后面迟早能通关。

## 首秀最小流程卡

1. **先让职业动作出场**：先写他在做什么工作、履行什么职责，而不是先讲设定原理。
2. **再让异常 / 规则咬住动作**：让设定在职业流程里冒出来，而不是从天而降。
3. **立刻留下第一层可见痕迹**：纸面、界面、身体、环境、关系反应至少命中一类。
4. **当场给一处代价或误判**：不要把代价推迟到很后面才兑现。

## 必答问题

- 这个设定最早会在正文里留下什么痕迹？
- 谁会最先因为它付代价？
- 它最容易在哪个流程节点露馅？
- 若它是强设定，它在前三章**不能直接解决**什么问题？
- 如果只允许保留一处首秀钉子，最值得留下的是哪条动作 / 凭证 / 后果链？

## 常见失误

- 职业只是贴皮，换成别的职业设定照样成立。
- 只有“他懂这个行业”，没有具体日常流程与岗位动作。
- 前三章只在解释设定，没有验证、误判、代价或小回报。
- 设定一出场就替主角解决太多事，导致职业绑定反而失真。
- 只有职业术语，没有第一处可见痕迹与第一层程序摩擦。
