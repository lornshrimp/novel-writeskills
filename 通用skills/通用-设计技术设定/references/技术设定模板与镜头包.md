# 技术设定模板与镜头包

## 文档定位

- 技术设定文档默认是**作者侧设定裁判源 / 调用手册**，服务大纲、正文、台账交叉引用，不是读者向设定科普。
- 允许写得顺滑、好看，但优先级始终是：**可用性 > 一致性 > 叙述气质**。
- 交付时优先补齐以下三类核心产物：
   1. **一句话口径**：动作 → 凭证载体 → 可验证后果。
   2. **镜头化落地**：至少 3 条可写镜头包。
   3. **误导与证伪对**：误导点、打穿方式、显性代价。

## 推荐骨架

### 基础信息

- 要素名称：中文正式名称、别名 / 黑话、代号（如有）
- 要素分类：手法 / 程序链 / 证据载体 / 设定规则
- 一句话口径：用“动作 → 凭证载体 → 可验证后果”说清能做什么、不能做什么
- 时间线锚点：首次出现章节 / 案号、关键节点（引入 / 强化 / 反转 / 回收）
- 可获得性：谁能拿到、通过什么途径、需要多久、是否可伪造 / 篡改
- 影响范围：当章 / 单元案 / 跨卷主线；影响哪些人
- 敏感等级：公开 / 受限 / 高敏

### 机制 / 程序

- 核心机制：手法如何成立、程序如何运转、规则如何触发
- 关键矛盾与破绽：为什么难查 / 难证 / 难对抗；最容易露馅的点是什么
- 链条结构：输入 → 处理 → 输出；谁做、在哪做、用什么做、留下什么痕迹
- 关键字段 / 参数：时间戳、编号、点位、签名 / 盖章、对照口径等
- 前置条件：权限 / 关系 / 资源 / 地点 / 心理条件，以及失败后果
- 误导与证伪对：误导点、证伪路径、耗时、代价
- 三问自检：为何现在能拿到？为何之前拿不到 / 没注意？为何不能一锤定音？

### 应用场景

- 主要用途：制造异常 / 锁定嫌疑 / 反转洗白 / 逼近真相 / 制造恐惧
- 呈现方式：对话 / 道具 / 凭证 / 现场勘验 / 走访 / 缺帧监控 / 记录体
- 可写镜头包：至少 3 条，每条都写“场景 / 信息承载物 / 人物反应 / 新缺口或新代价”
- 适用对象：谁会用、谁看得懂、谁会被误导
- 使用限制：现实限制 + 叙事限制
- 成本分析：时间 / 人情 / 金钱 / 风险成本，而且必须在正文里看得见

### 社会影响

- 经济影响：赔付、医疗、房租、黑产获利、赔偿追责等
- 社会影响：社区信任、邻里关系、受害者家属处境、群体恐惧传播
- 治理 / 程序影响：基层治理、执法 / 协查压力、单位内部处理机制、舆情处置
- 文化 / 心理影响：迷信 / 民俗解读、价值观与安全感冲击
- 伦理争议：隐私、二次伤害、媒体边界、程序正义与效率冲突

## 可写镜头包最低要求

- 不能只说“可写成镜头”，必须把信息承载物写实：纸质回执、界面截图、物件磨损、监控缺帧、登记本、环境反应等。
- 镜头包要同时承担**信息推进 + 情绪推进 + 新缺口投放**，避免只有解释没有戏。
- 若一个设定无法拆出 3 个镜头包，通常意味着它还停留在概念层，需要继续补动作链、取证链与代价链。

## 推荐起草顺序

1. 先写一句话口径和使用边界。
2. 再写链条结构、关键字段与破绽。
3. 再把镜头包和误导 / 证伪对补齐。
4. 最后补社会影响、长篇演进与回收接口。

## 镜头包最低要求

每个设定至少给 3 条镜头包：

- 一条偏发现异常
- 一条偏程序摩擦
- 一条偏证据对照 / 反制

镜头包要写“人正在经历什么”，不要写“设定说明”。
