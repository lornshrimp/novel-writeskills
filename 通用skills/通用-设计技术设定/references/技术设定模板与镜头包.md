# 技术设定模板与镜头包

## 1. 文档定位

- 技术设定文档默认是**作者侧设定裁判源 / 调用手册**，服务大纲、正文、台账交叉引用，不是读者向设定科普。
- 允许写得顺滑、好看，但优先级始终是：**可用性 > 一致性 > 叙述气质**。
- 交付时优先补齐以下三类核心产物：
   1. **一句话口径**：动作 → 凭证载体 → 可验证后果。
   2. **镜头化落地**：至少 3 条可写镜头包。
   3. **误导与证伪对**：误导点、打穿方式、显性代价。

## 2. 读者第一感知顺序（默认）

无论题材为何，技术设定默认都应优先让读者 / 执行层先看懂：

1. **它能做什么**
2. **它会害到谁 / 帮到谁**
3. **它的代价和漏洞在哪里**
4. **它在正文里会以什么镜头出现**

原理、背景、来源可以后补；若一上来先讲体系，通常会把设定写成讲义。

## 3. 推荐骨架

### 3.1 基础信息

- 要素名称：中文正式名称、别名 / 黑话、代号（如有）
- 要素分类：手法 / 程序链 / 证据载体 / 设定规则
- 一句话口径：用“动作 → 凭证载体 → 可验证后果”说清能做什么、不能做什么
- 代号 / 编号规则：谁按什么规则给它编号、内部与外部是否有不同叫法
- 时间线锚点：首次出现章节 / 案号、关键节点（引入 / 强化 / 反转 / 回收）
- 可获得性：谁能拿到、通过什么途径、需要多久、是否可伪造 / 篡改
- 影响范围：当章 / 单元案 / 跨卷主线；影响哪些人
- 敏感等级：公开 / 受限 / 高敏

### 3.2 机制 / 程序

- 核心机制：手法如何成立、程序如何运转、规则如何触发
- 触发条件 / 关键条件：最好拆成 `2–4` 个关键要素，明确缺一不可还是允许替代
- 标准表现 / 感官反馈序列：第一次显影时，人物、物件、界面、环境分别会出现什么可见征候
- 衰减 / 失效规则：窗口期、污染条件、覆盖条件、冷却期、危险阈值、越界后果
- 关键矛盾与破绽：为什么难查 / 难证 / 难对抗；最容易露馅的点是什么
- 链条结构：输入 → 处理 → 输出；谁做、在哪做、用什么做、留下什么痕迹
- 关键字段 / 参数：时间戳、编号、点位、签名 / 盖章、对照口径等；优先区分**固定值**与**叙事弹性**
- 最小调用流程卡：把“判断是否能用 → 允许产出什么 → 必须怎么降格验证 → 哪条红线不能越过”压成作者速查表
- 分级代价与程序痕迹：至少区分 `3–4` 个阶段，写清体感表现、程序留痕、最适合正文怎么写
- 前置条件：权限 / 关系 / 资源 / 地点 / 心理条件，以及失败后果
- 误导与证伪对：误导点、证伪路径、耗时、代价
- 三问自检：为何现在能拿到？为何之前拿不到 / 没注意？为何不能一锤定音？

### 3.3 应用场景

- 主要用途：制造异常 / 锁定嫌疑 / 反转洗白 / 逼近真相 / 制造恐惧
- 呈现方式：对话 / 道具 / 凭证 / 现场勘验 / 走访 / 缺帧监控 / 记录体
- 可写镜头包：至少 3 条，每条都写“场景 / 信息承载物 / 人物反应 / 新缺口或新代价”
- 适用对象：谁会用、谁看得懂、谁会被误导
- 使用限制：现实限制 + 叙事限制
- 不能直接解决的问题：至少列 `3` 条，明确它不能替代什么、不能直接给什么终局答案
- 成本分析：时间 / 人情 / 金钱 / 风险成本，而且必须在正文里看得见

### 3.4 写作落地卡片（推荐补充）

若希望设定能直接转进正文，而不是停留在大纲层，建议再补一张轻量落地卡：

- **适用场景**：最适合在哪类场景第一次亮相
- **不适用场景**：哪些场景里硬塞会显得讲解味太重
- **主角抓手**：主角如何借它推进、破局、试探、交易或自救
- **反派 / 对手抓手**：对手如何滥用、误用、伪造或反咬
- **升级按钮**：后续可通过什么方式把它升级成更大的风险、回报或反转
- **风险级别**：低 / 中 / 高；对应会伤到身体、关系、程序还是社会面

升级按钮优先考虑：链条变长、使用门槛提高、可见后果放大、误导更逼真，而不是单纯把数值写大。

### 3.5 社会影响

- 经济影响：赔付、医疗、房租、黑产获利、赔偿追责等
- 社会影响：社区信任、邻里关系、受害者家属处境、群体恐惧传播
- 治理 / 程序影响：基层治理、执法 / 协查压力、单位内部处理机制、舆情处置
- 文化 / 心理影响：迷信 / 民俗解读、价值观与安全感冲击
- 伦理争议：隐私、二次伤害、媒体边界、程序正义与效率冲突

### 3.6 演进 / 修订补丁（推荐补充）

- 发展历程：按“开篇试水 → 中盘升级 → 后盘失控 / 降格 → 终局关门 / 余波”写清递进
- 依赖关系：与其他设定、组织规矩、总纲主线、案件母板如何互相咬合
- 本轮修订：若本次只对齐某个版本、卷、案件或研究结论，文末建议加 `本轮修订（YYYY-MM-DD｜对齐 Vxx）`
- 余波 / 残留：若终局要关门、失能、降格或制度化，提前写清关门后的残留影响，避免终局硬切断

## 4. 可写镜头包最低要求

- 不能只说“可写成镜头”，必须把信息承载物写实：纸质回执、界面截图、物件磨损、监控缺帧、登记本、环境反应等。
- 镜头包要同时承担**信息推进 + 情绪推进 + 新缺口投放**，避免只有解释没有戏。
- 若一个设定无法拆出 3 个镜头包，通常意味着它还停留在概念层，需要继续补动作链、取证链与代价链。
- 三条镜头包最好分别偏向：**功能显影**、**代价显影**、**破绽 / 证伪显影**，避免三条都在做同一种说明。

## 5. 推荐起草顺序

1. 先写一句话口径和使用边界。
2. 再写触发条件、标准表现、失效 / 衰减规则与关键字段。
3. 再补最小调用流程卡、分级代价与程序痕迹，确认作者怎么调用、哪里会翻车。
4. 再补“写作落地卡片”，确认谁会把它真正用进剧情。
5. 再把镜头包、误导 / 证伪对和“不能直接解决的问题”补齐。
6. 最后补社会影响、长篇演进、版本修订与回收接口。

## 7. 镜头包最低要求

每个设定至少给 3 条镜头包：

- 一条偏发现异常
- 一条偏程序摩擦
- 一条偏证据对照 / 反制

镜头包要写“人正在经历什么”，不要写“设定说明”。

## 8. 推荐附加表格（高价值设定优先）

若设定承担跨卷主线、终局限制或强能力机制，建议优先再补以下表格：

- **关键参数表**：参数 / 精确值 / 叙事弹性 / 越线后果
- **最小流程卡**：步骤 / 动作 / 可见凭证 / 允许产出 / 禁止事项
- **分级代价表**：阶段 / 体感表现 / 程序痕迹 / 正文最佳写法

能把这三张表补齐的设定，通常更适合长线连载调用，也更不容易写成“作者知道，但正文落不下去”的半成品。
