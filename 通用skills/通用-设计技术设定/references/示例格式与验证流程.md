# 示例格式与验证流程

## 示例骨架

```markdown
# [要素名称] - 手法/程序/证据/设定资料文档

## 基础信息
- 正式名称 / 别名 / 代号
- 要素分类
- 一句话口径
- 首次出现 / 关键节点 / 影响范围 / 敏感等级

## 机制 / 程序
### 核心机制
### 关键矛盾与破绽
### 链条结构
### 关键字段 / 参数
### 前置条件
### 误导与证伪对

## 应用场景
### 主要用途
### 可写镜头包
### 使用方式
### 适用对象
### 使用限制
### 成本分析

## 社会影响
### 经济影响
### 社会影响
### 治理 / 程序影响
### 文化 / 心理影响
### 伦理争议

## 演进与回收
### 发展历程
### 关键节点
### 未来方向
### 依赖关系

## 在长篇中的作用
### 开篇 / 中盘 / 后盘 / 终局的作用
### 情节支撑
### 主题表达

## 可信度验证
### 现实 / 程序依据
### 可行性分析
### 风险评估
### 一致性检查
```

## 验证流程

1. 初稿完成：先确认骨架、规则条款与一句话口径是否完整。
2. 现实 / 程序验证：检查流程、权限、时效与资源约束是否可信。
3. 证据验证：检查每条关键规则是否有痕迹、误导与证伪路径。
4. 一致性检查：确认与人物、地点、组织规矩、其他设定文件不冲突。
5. 时间线核对：确认首次出现、强化、反转、回收与失效窗口是否匹配卷结构。
6. 社会影响评估：确认经济、社区、治理、心理与伦理后果不是空白。
7. 文学适用性检查：确认资料设定能被写成镜头，而不是只剩说明书口气。
8. 最终定稿：确保文档达到“可复盘、可复用、可连载”标准后再写回。

## 常见返工原因

- 设定很酷，但剧情里用不上
- 有机制，没有痕迹
- 有痕迹，没有获取难度
- 有规则，没有人物成本与社会后果
- 有说明，没有镜头包
