# 章节链路失配审阅表

适用于在章节审阅阶段，不只看正文表面效果，还显式判断：上游供血、控制卡、场景职责、首尾补丁是否彼此咬合。

## 一、先明确：章节问题不总出在正文表层

读感发弱，常见不是因为一句句子不好，而是链路某处失配：

- 故事面供血没真正落到本章
- 卷纲承接写了，但场景没显性承担
- 控制卡写明了回报，场景却没把它落到动作上
- 章首 / 章末补丁在遮问题，而不是解决问题

## 二、审阅时至少查四层链路

### 1. 上游供血层

- 本章是否真承接某条故事面 / 卖点供血线？
- 还是只是沾了主题边？

### 2. 控制卡层

- 本章主抓手、中段回报、章末钩子是否在正文里真实显影？
- 还是只存在于写前计划里？

### 3. 场景职责层

- 第一场是否真在抓眼
- 中段那场是否真在回报
- 最后一场是否真在钩子

### 4. 首尾补丁层

- 开头 / 结尾的强化是否真的增强了章节
- 还是只是把上游设计问题修得更好看

## 二点五、来源层映射

- `供血层`：故事面 / 卖点 / 上游承诺是否真的进入本章
- `控制卡层`：唯一职责、中段回报、情感弧、钩子链是否被正文兑现
- `场景职责层`：第一场、中段场、最后一场是否各自承担了该承担的职责
- `首尾补丁层`：强化动作是在放大有效结构，还是在遮羞

## 三、最小失配审阅表

| 观察到的问题 | 可能失配层 | 优先建议 |
| --- | --- | --- |
| 开头抓人但与本章主线脱节 | 控制卡 / 场景职责 | 回写主推进线与第一场职责 |
| 中段有很多内容但没有值回票价 | 控制卡 / 场景职责 | 回写中段回报落点 |
| 结尾有气氛无驱动 | 场景职责 / 章末强化 | 回写最后一场职责 |
| 卖点只在开头出现一次 | 上游供血 / 控制卡 | 回写本章与卖点关系 |
| 对话自然但谁都不像谁 | 场景职责 / 去AI味后复检 | 回写声音差依据 |
| 通篇都顺，但读完没记忆点 | 上游供血 / 控制卡 / 场景职责 | 回查主抓手显影位 |

## 四、何时把问题判为 P1 而不是 P2

以下情况，默认优先按 P1，而不是“文笔还可优化”：

- 供血线在本章实际失踪
- 中段回报只存在于解释里
- 章末钩子只剩语气，没有动作对象
- 开头 / 结尾补丁在遮羞，上游职责没有真正站住
- 去 AI 味 / 润色后，职责被洗平或磨圆

## 五、报告里建议单列的结论

审阅报告中可额外判断：

- 本章当前问题，主要是正文表层问题，还是链路失配问题？
- 若属链路失配，优先回写层级：
  1. 故事面 / 卖点供血
  2. 控制卡
  3. 场景职责
  4. 开头 / 章末补丁

## 五点五、链路判定门禁

- `Green`：供血层、控制卡层、场景职责层与首尾补丁层基本咬合，补丁在放大而不是遮羞。
- `Yellow`：主链在，但存在一层轻度失配，如中段回报只在控制卡里、还没完全进入正文。
- `Red`：至少两层同时失配，或补丁已经在遮结构问题；此时不宜继续只修文面。

## 六、最小输出模板

```markdown
【链路判定】Green / Yellow / Red
【主要失配层】供血层 / 控制卡层 / 场景职责层 / 首尾补丁层
【先回写哪一层】
【若继续只修文面会发生什么】
```

## 六、一句话收口

**章节审阅不只是判句子和桥段，还要判这章是不是把上游承诺真正写出来了，还是只是靠补丁把问题抹顺了。**
