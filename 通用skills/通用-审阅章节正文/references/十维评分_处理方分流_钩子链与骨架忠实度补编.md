# 十维评分_处理方分流_钩子链与骨架忠实度补编

> 吸收外部 `reviewer-skill`、`review-checklist`、`quality-standards` 与 `review-report-template` 的共性框架，用于把章节审阅稳定到“十维评分 + 处理方分流 + 来源标注 + 钩子链验证 + 骨架忠实度评估”。

## 一、审阅输入最小包

理想情况下至少读取：

- 待审章节正文
- 该章控制卡 / 场景规划
- 骨架稿（如有）
- 上一章结尾或上一章规划（用于钩子链验证）
- 涉及人物的当前状态字段
- 活跃悬念 / 活跃伏笔 / 已知决策背景
- 体裁字段（用于判断审阅侧重）

## 二、十维评分基线

每维 `1–5` 分，总分 `50` 分：

1. 叙事逻辑（核心）
2. 人物一致性（核心）
3. 连贯性
4. 场景规划执行
5. 节奏与张力
6. 对话质量
7. 描写质量
8. 去 AI 味
9. 钩子与悬念
10. 字数与密度

### 分值口径

- `5`：优秀，基本无明显问题
- `4`：良好，有小瑕疵但不伤阅读
- `3`：合格，可读但有明显改进空间
- `2`：存在较大问题，需要改
- `1`：严重不合格，相关段位应重写

## 三、问题来源标注

每个问题都要标：

- `[骨架层]`：规划 / 骨架本身就错了，需回到上游重建
- `[扩展层]`：骨架对了，但正文展开跑偏或缺失
- `[表达层]`：结构基本成立，问题主要在措辞、节奏、去味、标签、描写密度

## 四、处理方分流

### 交给 writer 的问题

- 因果链断裂
- 人物行为与设定冲突
- 时间线 / 空间线错误
- 承重 beat 缺失或顺序乱
- 唯一职责未完成
- 情感弧未执行
- 核心事件被跳过或严重压缩
- 上章钩子未回应且无理由
- 本章结尾没有真实驱动力
- 需要改变“写什么”

### 交给 polisher 的问题

- 去 AI 味
- 对话标签、语气、声音微调
- 描写精炼
- 节奏微调（不改结构）
- 段落长短和句式均匀问题
- 花哨表达、填充短语、同义词转盘
- 需要改变“怎么写”

## 五、十维审阅时的特别抓手

### 维度四：场景规划执行

重点查：

- 规划中的核心事件是否都已呈现
- 场景目标是否完成
- 情感基调是否实现
- 承重 beat 是否全部在场且顺序正确
- 灵活 beat 的合并 / 删减是否有叙事理由
- 本章唯一职责是否完成
- 是否残留大纲语言

### 维度五：节奏与张力

重点查：

- 开头前 `20%` 是否建立吸引力
- 是否有节奏变化而非全章齐平
- 高潮是否有铺垫
- 结尾是否有真实翻页驱动力
- 实际节奏是否匹配规划中的轻 / 中 / 重
- 情感弧入口 / 出口是否成立

### 维度九：钩子与悬念

重点查：

- 前三段内是否有有效抓力
- 结尾是否留下未解问题或未完成动作
- 是否回应了旧悬念
- 钩子类型是否在最近几章重复
- 本章开头是否响应上章出口钩
- 本章出口钩是否可信地拉动下一章

## 六、钩子链验证格式

| 项 | 内容 |
| --- | --- |
| 上章出口钩 | 类型 + 方向 |
| 本章入口响应 | 是 / 否 / 部分；直接 / 间接 / 延迟 |
| 本章出口钩 | 类型 + 方向 |
| 下章预期响应 | 预期下章如何接棒 |
| 钩子类型重复 | 最近 3 章是否重复 |
| 钩子链断裂 | 有 / 无；若有写原因 |

## 七、骨架忠实度评估

至少单列：

- 承重 beat 在场：是 / 否
- beat 顺序保持：是 / 否
- 大纲语言残留：有 / 无
- 结构偏离：是否合理偏离
- 骨架忠实度：高 / 中 / 低

### 自动判重的信号

- 承重 beat 缺失
- 承重 beat 无故换序
- 正文把骨架的核心职责改没了
- 大纲语言残留超过 `2` 处

## 八、就绪判定

### Green

- 总分 `≥35`
- 无单项 `≤2`
- 无红线触发
- 骨架忠实执行成立

### Yellow

- 总分 `25–34`
- 或有单项 `=2`
- 或轻微节奏违规
- 或骨架局部偏离但可修复

### Red

- 总分 `<25`
- 或核心维度 `≤1`
- 或红线触发

### 自动 Red 红线

- 角色知道了不可能知道的信息
- 因果链断裂
- 承重 beat 缺失导致叙事断裂
- 情感弧完全未执行
- 上章钩子完全未回应且无理由
- 大纲语言残留超过 `2` 处

## 九、体裁权重调整口径

权重调整只影响从严程度，不改单维满分：

- 武侠：动作清晰度、侠义一致性更严
- 仙侠：体系一致性、尺度感更严
- 悬疑：信息控制、误导公平性、钩子悬念更严
- 言情：情感真实性、潜台词更严
- 科幻：概念清晰度、一致性更严

## 十、报告最小字段

章节审阅报告至少包含：

- 基本信息
- 总分与结论
- 就绪判定（含判定信号 / 红牌 / 阻塞项）
- 十维评分
- 至少两项优点
- 结构 / 逻辑类问题（writer）
- 表达类问题（polisher）
- 修改任务汇总
- 钩子链验证
- 骨架忠实度评估
- 情感弧检查
- 连贯性备注
- 复审记录（复审时）

## 十一、跨维度复核提醒

有些问题要一次标出多个影响面：

- 角色突然知道不该知道的信息 → 叙事逻辑 + 人物一致性
- 大段无功能描写 → 描写质量 + 节奏与张力 + 字数密度
- 对话信息倾倒 → 对话质量 + 去 AI 味
- 旧悬念遗忘 → 连贯性 + 钩子与悬念

## 红线

- 不得只给读感，不给位置和可执行建议
- 不得不区分 writer / polisher
- 不得不标问题来源层
- 不得因为文章“整体还行”就跳过钩子链和骨架忠实度
- 不得把复审做成“只核旧坑、不看新坑”
