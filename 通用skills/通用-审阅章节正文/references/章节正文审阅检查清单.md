# 章节正文审阅检查清单

## 一、先看抓力，再看文采

章节正文对象的首要判断不是“句子漂不漂亮”，而是：

- 读者能不能被章首拖进去
- 中段有没有明确收获
- 章末有没有下一章动力

## 二、章首抓力

- 前 2–3 段是否至少命中以下两项：
  - 异常 / 冲突触发
  - 情绪锚点
  - 悬念缺口
- 是否避免了背景说明或设定科普压过故事入场

## 三、中段兑现

- 中段是否至少有一次可指认的回报：
  - 新证据落袋
  - 误导被证伪
  - 关系发生变化
  - 规则边界被试出
  - 有效反击成立
- 是否指出：回报类型、落在哪个镜头、是否足够清晰

## 四、章末钩子

- 钩子是否落在具体动作 / 信息 / 危险 / 未完成决定上
- 是否避免了空泛感慨、空问号或假危机

## 五、现实与规则落地

- 文本中是否有可感知的现实锚点或场景支撑
- 规则 / 证据 / 程序 / 代价是否不是只在口头存在
- 现实锚点是否具体到空间、物件、职业流程或日常界面，而不是只剩抽象气氛词

## 六、工程核查

- 本章应落地的卷纲功能点是否都已落地
- 出场人物是否已做传记核查，是否存在身份 / 关系 / 能力边界冲突
- 关键判断是否能回指到证据
- 误导是否可证伪
- 是否出现作者管理编号、具体年份或突然全知解释等出戏红线

## 七、读者端核查

- 每段是否有明确焦点
- 对话是否承担功能
- 是否至少有一处情绪兑现
- 是否至少有一处贴角色、贴时代的记忆点
