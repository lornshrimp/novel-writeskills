# 通用-审阅章节正文

章节正文审阅的第一判断不是“漂不漂亮”，而是“读者有没有被拖进去、有没有拿到回报、下一章值不值得看”。

## 继续读取的 references

- `references/章节数据看板与阶段达标线.md`（**必读** — 读感+数据双轨审阅）
- `references/章节正文审阅检查清单.md`
- `references/章节正文审阅报告骨架.md`
- `references/章节正文审阅完整模板.md`
- `references/问题分级与补丁动作.md`
- `references/章节链路失配审阅表.md`
- `references/腾讯专栏章节审阅追读与节奏红线增补.md`
- `references/腾讯专栏章节审阅快审红线子模板.md`
- `references/腾讯专栏章节审阅手机端可读性子模板.md`
- `references/腾讯专栏章节审阅动作反应闭环增补.md`

## 审前裁判源复核（强制）

- 出具任何章节审阅结论前，必须认真仔细阅读与本章实际出场人物对应的人物传记，以及与本章背景、技术、规则、证据、程序链、世界观有关的技术设定文档。
- 不得只凭读感、题材经验、上一轮审阅报告或控制卡摘要直接下结论；凡涉及人物动机、能力边界、关系状态、设定代价、证据机制的判断，都必须回到作者侧裁判源核对。
- 若同一人物传记或技术设定存在多个版本，必须先确认当前可执行的最新版本；若关键裁判源缺失，必须先列为阻断项，再继续审阅，而不是边猜边打分。
- 审阅报告里必须显式写出本轮核对了哪些人物传记 / 技术设定，以及正文哪些细节与之吻合、哪些冲突、哪些仍待补丁。

## 完整模板执行要求（强制）

- 产出章节正文审阅报告时，必须以 `references/章节正文审阅完整模板.md` 作为主骨架，不得退化成“几段点评 + 少量建议”的简版报告。
- 报告至少要完整覆盖：阅读链路确认、内容概要、字数与结构核查、节奏与钩子、题材专项核查、读者端留存信号、大纲与传记落地核查、问题分级、最小返工处方、前后呼应与主题检查、读者体验评估、记忆点评估、评分、具体修改方案、总体结论。
- “字数与结构核查”不得只凭目测；必须用 `scripts/count-chapter.ps1 -MinCJK 3500 -MaxCJK 6500` 单独核正文。若同文件含 `## 作者有话说`，也必须把它作为独立门禁项单列：中文用 `scripts/count-afterword.ps1 -MinCJK 200 -MaxCJK 300`，英文或英文平台稿用 `scripts/count-afterword-words.ps1 -MinWords 100 -MaxWords 150` 判断。
- 章节审阅报告中的字数字段，必须显式记录脚本输出的 `Len / CJK / BodyCJK / Meets3500 / WithinRange`；`BodyCJK` 是 3500–6500 门禁的唯一依据，`Len` 仅作参考。
- 若报告出现“正文字数：XXXX（去空白统计）”或其它以自定义字符数替代 `BodyCJK` 的写法，视为审阅未完成，必须回炉重跑脚本并刷新报告。
- 若用户要求连同 `## 章节后记` 一起审，至少要核对：位置与标题是否正确、字段是否覆盖本章定位 / 功能点落地核对 / 冲突与悬念 / 爽点登记 / 伏笔与回收 / 设定或程序呈现 / 人物发展 / 证据要素 / 钩子类型 / 数字纪律检查 / 偏离记录 / 后续铺垫 / 读者吸引点 / 优化建议，且这些字段都与正文逐项对应。
- 若当前任务还加载了题材 Skill，则必须在通用完整模板的占位章节中继续补写题材专项段、题材专项评分项与题材化结论重点。
- 若存在上游控制卡、分卷承诺或人物传记约束，报告必须显式指出哪些已兑现、哪些仍悬空，不得只给纯读感结论。

## 章节执行质量基准（新增）

本轮审阅中，以下基准数据为强制门禁：

**字数门禁**：
- 中文正文（脚本 `BodyCJK`）：3500–6500 字范围内，超出范围视为门禁失败。
- 中文作者有话说：200–300 字，超出范围视为门禁失败。
- 脚本输出 `Meets3500` / `WithinRange` 必须同时为 `True`。

**钩子与节奏门禁**：
- 前 200 字钩子：必须存在，类型为冲突 / 悬念 / 异常 / 强情绪 / 动作 / 对话中之一。
- 后 200 字钩子：必须存在，不得是问号或虚空狠话。
- 单章对话占比：25%–55% 范围内（脚本扫描：`grep -c "「" / 总行数`）。
- 中段回报：必须存在，类型为情绪转折 / 处境改变 / 信息释放 / 能力显示 / 关系推进中之一。

**卖点与AI痕迹门禁**：
- 若本章承担卖点回报职责，该卖点必须在本章明确兑现或推进。
- 正文中不得出现以下AI万能句式（每项超过2次视为病灶）：
  - "眸中闪过一丝 X"
  - "XX缓缓说道"
  - "XX心中暗道"
  - "总而言之 / 可以说 / 值得注意的是"
  - "突然 / 忽然 / 猛地"（应通过语序制造意外感，而不是副词）
  - "他非常 X"（应改为动作+生理反应+决策三段式）
- 章节自指（"这几章""前几章""下一章""本章"等）与连载导览腔（"后文会""我们会看到""读者会发现"）视为沉浸事故，命中即为门禁失败。

**规则与证据链门禁**：
- 涉及技术设定、规则机制、证据程序的表述，必须与当前可执行的技术设定裁判源一致。
- 人物的能力边界、代价显现、动机推动，必须与当前人物传记版本相符。

## 追读概率评估（新增）

审阅报告中必须显式给出"本章是否促进读者继续追/继续付费"的判断，维度包括：

| 维度 | 强信号 | 弱信号 | 失败信号 |
|------|--------|--------|----------|
| 情感代入 | 主角处境与读者产生共鸣 | 情感表达正确但冷淡 | 只有事件无人物情绪 |
| 悬念驱动 | 章末留下明确未解答问题 | 有疑问但不紧迫 | 平铺直叙收尾 |
| 爽点兑现 | 至少一次主角主动或有效反制 | 只有被压无反制 | 全章被动无翻身点 |
| 记忆点 | 具体、可讨论、可分享 | 正确但平凡 | 无可驻足的细节 |
| 下章连接 | 下章接棒对象明确存在 | 下章方向暗示不足 | 完全不知下章干什么 |

综合判断：
- `强`：预期留存概率 80%+ 
- `中`：预期留存概率 50-80%
- `弱`：预期留存概率 <50%（需马上返工）
## 首屏与手机端审阅补充（新增）

- 审阅时默认单列检查前 `300` 字：人物抓手、第一压力、异常 / 冲突、继续阅读问题，至少命中两项才算过线。
- 若正文大量依赖抽象总结、作者说理或空情绪，而缺少动作、感官、后果与关系变化，应在报告里明确判为沉浸风险。
- 若段落长期过厚、句群过匀、手机端阅读容易形成视觉疲劳，应在报告里单列为排版 / 阅读摩擦问题，而不是只算文风差异。
- 单章节拍默认还要检查“钩子 → 蓄力 → 回报 → 留钩”是否成立；若长期只有推进没有兑现，应明确写成结构问题。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，审阅前必须先读取并纳入对照。
- 审阅报告不得只给内部自洽结论；必须显式判断当前章节相对竞对在哪些维度已领先、持平或落后，以及最先该补哪条超越路径。
- 若当前章节在竞对已验证的关键强点上明显更弱，必须直判为竞争风险；章节审阅的默认目标是推动当前作品在关键维度上**全方位超越竞争对手**。
- 默认按“严酷找差距”口径执行：重点写清当前章节为什么还赢不过已过审竞对章节，不得为了照顾作者情绪把留存断裂、回报不足或钩子发虚写成“文风问题”。
- 竞对维度在章节审阅综合评分中默认按 `25%` 计权；若未完成量化差距卡，或关键维度仍显著落后且没有 `72` 小时补丁，综合评分默认不得给高档。

## 报告落盘位置与命名规则（强制）

- 章节审阅报告统一落在项目根下的 `审阅意见/` 体系内，不得直接放回章节正文所在目录。
- 若作品存在分部，报告路径必须为 `审阅意见/第X部/第Y卷/章节审阅报告_第X部_第Y卷_第Z章_2026-04-15.md` 这一类“部 → 卷 → 报告文件”的结构。
- 若作品没有分部，报告路径必须为 `审阅意见/第X卷/章节审阅报告_第X卷_第Y章_2026-04-15.md` 这一类“卷 → 报告文件”的结构。
- 写入报告前，必须先从目标章节正文路径、所属分卷大纲或项目目录结构中确认 `第X部 / 第Y卷 / 第Z章` 标识，并先定位到对应子目录后再落盘。
- 不得因为分部或分卷信息暂时缺失，就把报告退回 `审阅意见/` 根目录；若确实无法判断，必须先把“分部 / 分卷定位信息缺失”列为阻断点，补齐后再写文件。
- 章节审阅报告文件名必须带日期，日期格式固定为 `YYYY-MM-DD`。
- 章节正文主文件，以及其中附带的 `## 作者有话说`、`## 章节后记` 等同文件工作流区块默认不带日期；日期只用于对应审阅报告。

## 常见触发词 / 用户说法速查

- 帮我审这章
- 看看这章值不值得追
- 给我这一章的审阅报告
- 这章章首 / 中段 / 章末哪里有问题
- 帮我分级指出补丁位
- 这章到底是文笔问题还是结构问题

## 何时使用

- 审阅章节正文的实际执行质量与追读价值。
- 需要同时判断章首抓力、中段回报、章末钩子、现实落地与链路失配。
- 需要产出可回写的章节审阅报告。

## 不适用情形

- 只做整章重写或精修，不先出审阅结论。
- 只做开头或结尾的局部强化。

## 默认执行顺序

1. 先确认目标章节、上游承接对象，以及本轮必须认真仔细阅读的人物传记 / 技术设定 / 背景设定文档。
2. 先按检查清单查章首 / 中段 / 章末三处功能，并判断主卖点是否被快速感知或持续兑现。
3. 再查现实落地、证据 / 规则 / 视角 / 人物一致性，以及对话里的声音差 / 塑人 / 联结是否成立。
4. 再用“章节链路失配审阅表”判断问题主要落在正文表层，还是故事面 / 控制卡 / 场景职责 / 首尾补丁的失配。
5. 用问题分级与补丁动作输出最小修复建议。
6. 生成可回写、可落库的审阅报告。

## 默认输出口径

- 默认输出一份带读者结论、工程结论、问题分级与补丁位的章节审阅报告。
- 默认不直接改正文，除非用户明确要求边审边回写。
- 若主要问题属于 AI 味，默认还要给出去味病灶一句话总结，并建议应进入的手术强度档位。

## 最低交付

- 能指出章首、中段、章末的关键表现与短板。
- 能说明问题属于正文表层还是链路失配。
- 能给出具体补丁位置与最小回写建议。

## 复审独立性规则（强制）

- 每一轮章节审阅都必须把当前正文版本当作新的交付物重新独立、从严、全面审一遍，不得因为上一轮问题已修复，就默认本轮应给更高分。
- 核查上一轮问题是否修复很重要，但不足以替代本轮对章首抓力、中段兑现、章末钩子、现实落地、证据规则、人物一致性与卖点保真的完整复审。
- 若本轮发现新的出戏点、新的节奏塌陷、新的证据链问题、新的卖点弱化或新的留存风险，必须如实加入本轮报告，不得因“旧坑补上了”而淡化新坑。
- 本轮评分必须只依据当前版本在本轮完整审阅下的真实表现给出，不得按上一轮分数、修复数量、返工次数或作者投入自动加分。

## 硬规则

- 不能只给情绪化好恶评价
- 必须指出具体段位、具体问题、具体补丁位置
- 必须同时给“读者结论”和“工程结论”
- 未认真仔细阅读本章相关人物传记与技术设定 / 背景设定文档前，不得出具“人物成立 / 设定成立 / 规则成立 / 证据成立”这类结论。
- 必须明确说明本章的追读反射弧强弱、微爽点 / 中爽点配置与下章接棒对象是否成立
- 必须判断本章有没有把全书卖点写弱、写泛、写没，并给出具体补丁位
- 必须判断对话是否只在搬信息，还是已经承担人物声音区分、人物塑造与读者跟随理由
- 默认审阅对象是章节正文本体；`## 作者有话说`、`## 章节后记`、润色说明、回写标记等即使与正文放在同一文件，也视为工作流设计，不因同文件共存而单列为“读者出戏”或发布事故。
- 不得把 `## 作者有话说` 并入正文 CJK 计数；正文与作者有话说的字数门禁必须分开记录、分开判断。
- 不得把“去空白统计”“NoWhitespaceLen”“编辑器字符数”等替代口径写成正文字数门禁结论；若需要保留参考信息，必须明确标注“仅参考，不参与门禁判定”。
- 只有当作者侧内容实际闯入正文段落、章标题，或用户明确要求按“给读者直接看到的终稿 / 裸发稿 / 合刊稿”口径严判时，才把它视为发布风险。
- 默认报告中的“是否建议直接放行 / 是否可发布 / 是否建议发布”只针对读者可见层（正文、章引语、`## 作者有话说`）；若用户未明确要求审原始工作文件是否可直接裸发，不得写“整文件不建议直接发布”“文件不可直接发布”之类结论。
- 若用户还要求连同 `## 作者有话说` 或 `## 章节后记` 一起审，再对对应区块单独给出作者侧建议；其中 `## 章节后记` 必须重点检查字段完整性、与正文对应关系，以及是否拿后记偷补正文缺失内容；但不得把“存在这些区块”本身当成正文质量缺陷。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责章节正文审阅与回写建议，不直接代替正文重写。
- 若用户确认问题后要进入改稿，应转入 `通用-正文润色`、`通用-去AI味重写` 或首尾专项强化。
- 当建议转入 `通用-去AI味重写` 时，应尽量说明是皮相拆洗、骨肉重塑还是灵魂重构，不要只给笼统判断。
