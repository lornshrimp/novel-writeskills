# 通用-审阅章节正文

章节正文审阅的第一判断不是“漂不漂亮”，而是“读者有没有被拖进去、有没有拿到回报、下一章值不值得看”。

## 继续读取的 references

- `references/章节正文审阅检查清单.md`
- `references/章节正文审阅报告骨架.md`
- `references/问题分级与补丁动作.md`

## 报告命名规则（强制）

- 章节审阅报告文件名必须带日期，日期格式固定为 `YYYY-MM-DD`。
- 日期应位于文件名末尾、扩展名 `.md` 之前；可写作 `章节审阅报告_第X章_2026-04-15.md`。
- 章节正文主文件，以及其中附带的 `## 作者有话说`、`## 章节后记` 等正文章节本体内容默认不带日期；日期只用于对应审阅报告。

## 默认执行顺序

1. 先确认目标章节与上游承接对象。
2. 先按检查清单查章首 / 中段 / 章末三处功能，并判断主卖点是否被快速感知或持续兑现。
3. 再查现实落地、证据 / 规则 / 视角 / 人物一致性，以及对话里的声音差 / 塑人 / 联结是否成立。
4. 用问题分级与补丁动作输出最小修复建议。
5. 生成可回写、可落库的审阅报告。

## 复审独立性规则（强制）

- 每一轮章节审阅都必须把当前正文版本当作新的交付物重新独立、从严、全面审一遍，不得因为上一轮问题已修复，就默认本轮应给更高分。
- 核查上一轮问题是否修复很重要，但不足以替代本轮对章首抓力、中段兑现、章末钩子、现实落地、证据规则、人物一致性与卖点保真的完整复审。
- 若本轮发现新的出戏点、新的节奏塌陷、新的证据链问题、新的卖点弱化或新的留存风险，必须如实加入本轮报告，不得因“旧坑补上了”而淡化新坑。
- 本轮评分必须只依据当前版本在本轮完整审阅下的真实表现给出，不得按上一轮分数、修复数量、返工次数或作者投入自动加分。

## 通用硬规则

- 不能只给情绪化好恶评价
- 必须指出具体段位、具体问题、具体补丁位置
- 必须同时给“读者结论”和“工程结论”
- 必须判断本章有没有把全书卖点写弱、写泛、写没，并给出具体补丁位
- 必须判断对话是否只在搬信息，还是已经承担人物声音区分、人物塑造与读者跟随理由
- 默认审阅对象首先是读者可见层；若章节文件同时包含 `## 作者有话说`、`## 章节后记`、润色说明或其它作者侧工件，不得机械把“同文件共存”误判为发布级事故。
- 如用户审的是“可直接给读者的终稿 / 裸发稿”，才把作者侧内容混入读者层视为阻断问题；否则应先判断它是否真的混进正文或 `## 作者有话说`。
