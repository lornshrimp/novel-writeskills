# 通用-输出纵横小说版｜平台共性执行细则

## 适用目标

将章节改写成更适合纵横小说网的版本，同时保持：

- 主要信息不丢失
- 中等节奏与克制悬念优先于降相似度
- 输出为纯小说正文结构

## 一、平台共性任务定义

- 面向纵横小说网男频读者（以 80、90 后男性为核心），强调中等节奏、现实锚点、逻辑推进与人性落点。
- 保留事件链、关键对话、关键设定、伏笔与章末悬而未决钩子。
- 不允许新增事件链、证据、设定、人物、地点、时间点或因果。

## 二、平台生态特征

### 读者画像

- 核心群体：80、90 后男性，审美成熟，对"悬浮感"容忍度极低。
- 核心需求："有逻辑、不脑残、能长期追更"，追求与主角同步感知、同步推理的沉浸感。
- 核心厌恶：纯靠噱头无逻辑、节奏失衡（拖沓或仓促）、全知上帝视角破坏推理参与感。

### 内容分发逻辑

- 平台以"读者阅读偏好标签匹配 + 内容长期数据留存"为核心分发机制，而非短期流量爆发。
- 章节完读率与读者评论互动量是权重最高的两个指标。
- 内容稳定（节奏稳定性、风格统一性、数据长期成长性）的作品才能进入正向流量循环。

### 平台主流赛道

- 悬疑类在纵横处于上升期；都市背景悬疑占平台悬疑类作品 60% 以上份额。
- 爆款公式：**现实性锚定 + 强悬念逻辑 + 慢热推进 + 人性落点**。
- 主流子类型：职业流（职业细节 + 社会议题 + 悬疑冲突）、民俗流（民俗文化 + 都市现实 + 悬疑冲突）。

## 三、跨题材共同硬约束

### 1. 零新增事实

- 所有情节与信息必须来自原始章节正文。
- 允许的扩写仅限于等价重述、感官细节丰富化、心理动作细节化。
- 精确数值、时间戳、ID、回执编号、界面字段必须原样保留。
- 叙事人称与视角必须与原文一致。
- 钩子增强只能靠呈现方式，不得新增"下一章事件"。

### 2. 沉浸式小说体裁

- 输出文件只能写成小说正文 + `## 作者有话说`。
- 正文中禁止教程、复盘、清单、方法论、额外小标题、列表、表格、代码块、引用块。
- 证据材料只能以剧情内短插入形式出现，并紧接人物反应与下一步行动。

### 3. 共同门禁

- 正文字数以 CJK 统计，必须在 2000–5000 之间。
- 单章字数参考区间：2000–3000 字（标准中等节奏单章）。
- 必须满足足够比例重写，不得直接复用大量原句与原段落结构。
- 必须去除模板腔、企业行话、说明书式段落。
- 标题必须重写，且同步写入文件名与正文标题。

## 四、纵横平台节奏共性

### 1. 中等节奏定义与内容分配

纵横都市悬疑以"中等节奏"为主流区间：

- **字数基准**：单章 2000–3000 字（区别于番茄等快节奏平台）。
- **内容比例基准**：40% 剧情推进 + 30% 人物互动 + 20% 细节铺垫 + 10% 情绪爆发。
- **单元节奏模型**：3–5 章完成一个"小冲突抛出 → 初步线索释放 → 误导性线索佐证 → 关键转折解析"的完整逻辑闭环。
- **章内节奏**：快慢交替，张弛有度；每个紧张场景后安排适度缓冲，不全程绷紧也不全程平淡。

### 2. 表达风格：接地气、重细节、去华丽化

纵横读者口味：不矫情、重逻辑、求沉浸。

- **叙述语言**：客观、克制、以传递信息为核心；类似"报告式"表达，不直接渲染情绪。
- **核心技巧**：白描为主，少用形容词，多用名词和动词。
  - 不写"水下很恐怖"，写"钩机的长杆在水面投下阴影，铁链碰撞的声音在河面上格外刺耳"。
  - 不写"房间很阴森"，写"阳光挣扎着透过满是灰尘的窗户，洒在杂乱无章的办公桌上"。
- **句式原则**：
  - 紧张快节奏场景（追逐、关键线索、对峙）→ 短句为主、断裂感强、单句成段。
  - 调查、回忆、对话缓冲场景 → 长句与短句结合，相对舒缓。
  - 关键悬念节点 → 场景句单独成段，强化记忆点。
- **词汇优先级**：
  - 优先符合现实场景逻辑的词汇，避免生僻词、晦涩词、网络热词过度使用。
  - 优先能调动读者感官的词汇：嗅觉（腐臭、铁锈味、霉味）、听觉（嘎吱、滴答、沙沙）、触觉（冰凉、粘稠、粗糙）。

### 3. 叙事视角：严格限制性视角

- **首选**：第一人称或限制性第三人称，严格锁定主角感知边界。
- **禁止**：随意突破视角限制、通过上帝视角补充主角感知外的信息。
- **感知锚点规则**：所有场景描写必须有主角感知锚点。
  - 不写"房间里有血腥味"，写"他闻到房间里有一股淡淡的血腥味"。
  - 不写"楼梯上有脚步声"，写"他听到楼梯方向传来一阵轻微的脚步声"。
- **视角切换限制**：短暂切换到其他角色视角时，不能披露主角尚未掌握的核心线索；切换后必须快速回归主角视角。

### 4. 线索铺陈：3:1 原则 + 间隔释放

- **3:1 原则**：每 3 个误导性线索，搭配 1 个关键性线索。
- **间隔释放**：关键线索均匀间隔在单元故事的叙事流程中。
  - 开篇：释放第一个"小线索"，抛出次一级悬念。
  - 前半部分：释放"误导性线索"，将调查方向引向歧途。
  - 中期：释放"关键性线索"，拉回正确推理轨道。
  - 后半部分：释放指向单元核心真相的"关键性线索"。
- 禁止一次性解开所有悬念；章末必须留下"悬而未决"的钩子。

### 5. 章末钩子：悬而未决型

纵横读者需要"解谜爽感 + 继续追更动力"的双重满足：

- 章末不完全解开谜团，留下 1–2 个与主线或下一单元相关的新伏笔。
- 优先使用"在悬念最高点结束"而非"全部交代清楚"。
- 优先钩子类型：新线索出现、证词矛盾显现、关键人物不明举动、调查方向突然翻转。
- **禁止**：完全闭环后才结章；全部真相在一章内解决。

### 6. 氛围营造：现实化、多感官融合、侧面心理刻画

- **场景选择**：优先都市常见但有阴暗面属性的场景（旧写字楼、废弃工厂、老宅、夜晚地铁站、偏僻河岸、冷门案件现场），而非深山老林或偏远古堡。
- **多感官组合**：每个关键场景重点刻画 2–3 种感官，不做"五官俱全"的堆砌。
  - 悬疑感：优先视觉 + 听觉。
  - 恐怖感：优先嗅觉 + 触觉。
  - 紧张感：优先触觉 + 听觉。
- **环境细节悬念化**：强化场景违和感（旧设备 + 新脚印）、运用象征暗示、强化危险信号。
- **心理刻画**：侧面化、细节化，避免直接大段内心独白。
  - 不写"林宇很紧张"，写"他的指尖微微发凉，手里的遗书被捏出了褶皱"。
  - 用手指颤抖、手心冷汗、喉咙滚动、不经意停顿、紧绷身体等具象动作替代直接情绪词。

### 7. 人性落点：每章至少保留一处

纵横读者对"人性"的关注度高于"悬疑诡计"本身：

- 故事的最终指向必须是"人性的复杂"或"社会现实的映射"，而非单纯"凶手是谁"。
- 改写时检查：本章是否有让读者感受到人性温度或社会现实的内容？若原章有，必须保留；若原章弱，可在改写中适度强化（但不得新增事实）。

### 8. 职业 / 行业术语的场景化使用

- 精准性优先：术语必须符合现实职业或行业的真实状态。
- 场景化嵌入：术语不能是生硬的知识点堆砌，必须通过角色行为、场景布置、细节点缀自然流露。
- 超现实元素限制：
  - 比例限制：超自然力量只作为点缀，不能超出阈值而跑偏到玄幻赛道。
  - 逻辑限制：超现实元素必须有明确使用边界与合理代价，不能前后矛盾。
  - 落点限制：案件谜底核心必须在"人"或"社会"身上，而非在"超自然"上。

## 五、平台审核红线（内容硬约束）

- 不涉及政治敏感内容：不攻击国家制度、不损害国家形象、不煽动民族仇恨、不宣扬恐怖主义或宗教极端主义。
- 不宣扬负面价值观：不渲染血腥暴力色情，正面角色价值观符合主流，反派最终必须被绳之以法。
- 不引用现实真实案件、地名或国家机构名称，避免对号入座。
- 章节标题不得为纯符号、emoji 或"标题党"式夸张表述。
- 专业知识（法医学、刑侦学等）表述必须准确。

## 六、改写门禁清单（落盘前自检）

- [ ] 字数在 2000–5000 CJK 之间（目标 2500–4000）。
- [ ] 叙事视角严格锁定，未出现上帝视角补充信息。
- [ ] 所有场景描写有主角感知锚点（"他看到…""他听到…""他闻到…"）。
- [ ] 节奏快慢交替，未全程绷紧或全程平淡。
- [ ] 章末留有悬而未决钩子，未完全闭环。
- [ ] 保留了至少一处人性落点或现实社会映射。
- [ ] 原章关键事件链、证据 / 线索链、人物代价未被洗掉。
- [ ] 表达风格：白描为主，无情绪化形容词堆砌，无华丽辞藻。
- [ ] 感官描写：关键场景有 2–3 种感官融合细节。
- [ ] 心理描写：侧面化动作细节替代直接情绪词。
- [ ] 内容未触碰平台审核红线。
- [ ] 未出现模板腔、企业行话、说明书式段落。

## 统一9步编排母模板（平台优先，题材兜底）

> 本节为统一执行母模板；本文件当前无独立“执行顺序”节，默认按本节执行。

1. 先锁定源稿不可变层：事实、事件链、证据链、角色关系与因果顺序不得改坏。
2. 读取对应题材的 `题材名-输出X版` Skill，确认题材入口与路由关系。
3. 再读取本 Skill 的 `references/平台共性执行细则.md` 与 `references/分节级补救映射与详细规则回填.md`，执行本平台特异强化。
4. 由题材包装层继续读取其题材 references，锁定题材边界、题材禁行项与题材专项补丁（作为兜底约束）。
5. 在“平台优先、题材兜底”前提下执行改写正文，确保平台化表达与源稿保真同时成立。
6. 最后才处理降相似度，不得为了降重牺牲平台读感与题材关键抓手。
7. 执行 POV 契约复核，确保与 `platformPovContract` 一致，不得出现未授权人称漂移。
8. 改写输出完成后，必须立即执行“输出后自检与修订”节的五项自检，不得跳过。
9. 若自检未通过，必须按结论回炉修订并复检，直至“自检通过/自检修订完成”后方可落盘。
