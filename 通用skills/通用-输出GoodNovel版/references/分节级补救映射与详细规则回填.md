# 通用-输出GoodNovel版｜分节级补救映射与详细规则回填

## 来源与恢复等级

- 恢复等级：B（补救性回填）
- 主要来源：两题材薄 Prompt、GoodNovel 研究文件。

## 0. 平台任务定义与基准继承

- 任务目标：把章节改写为更适合 GoodNovel 的英文连载版本。
- 继承创建正文、正文润色、去AI味中的事实与结构硬约束。
- 平台重点：情绪黏性、关系张力、plain global English、持续阅读意愿。

## 1. 平台理解

- GoodNovel 读者对 emotional pull、relationship tension、chapter-end suspense 更敏感。
- 英文需要顺、清、易读，减少过重本土知识负担。
- 标题应更具情绪和关系张力，但不能假装成 romance 脱离原题材。
- 平台原生章节更短、更密，习惯 1500–2500 词级别的快速推进；仓库长章输出时，应在章内做更多清晰的推进节点与局部钩子，而不是一整章只堆一个高潮。
- 读者偏好“限制性视角 + 公平信息释放 + 可讨论的章尾问题”，不适合全知旁白式讲述。

## 关键门槛

- 必须为英文沉浸式小说，零中文/CJK。
- 正文 Len 不得低于 6500。
- 关系张力必须嵌入事件与风险链。
- 非西方文化元素必须先做功能化转译；若保留原词，也要让英文读者在上下文里立即理解它会带来什么后果。
- 情绪必须转化成动作、判断、身体反应或关系变化，不能只停留在抽象感受词。

## 输入与输出路径

- 输出工作目录：`GoodNovel/`
- 章节目录（有分部）：`GoodNovel/第X部/第Y卷/`
- 章节目录（无分部）：`GoodNovel/第X卷/`
- 文件名：`{chapterNo} {平台新标题}.md`

## 平台执行要点

- 让冲突同时推动情节和情绪。
- 章末更强调 emotional cliff 或 relational cliff。
- 作者有话说英文 100–150 words，可轻互动，但不得像社媒贴文。

### 1.1 通用补救映射

- 若原章偏解释堆叠，优先拆成动作、对话和可见发现，让读者跟着角色一步步知道，而不是被一次性告知。
- 若原章偏本土表达，优先把文化词、制度词、民俗词翻成“功能 + 后果 + 风险”，避免硬直译。
- 若原章情绪不足，优先补“角色在此刻失去什么 / 害怕什么 / 不敢说什么 / 还想抓住什么”，让情绪长在 stakes 上。
- 若原章只有强情节没有连载钩子，优先在结尾补入信息遮断、关系未结算点或即将到来的选择压力。

### 1.2 题材共通的GoodNovel底色

- 悬疑线：公平线索 + 限知推进 + emotional cliff。
- 志怪线：西方日常锚点 + 超自然功能清晰 + 关系受规则施压。
- 职场线：全球通用职场痛点 + 情感爽感 + 读者可秒懂的反击链。

## 检查

- 是否具备 GoodNovel 式情绪拉力。
- 是否保住主线事件链。
- 是否英语自然顺滑。
