# 通用-输出出版社版｜分节级补救映射与详细规则回填

## 来源与恢复等级

- 恢复等级：A-（相对可严格回填）
- 主要来源：两题材薄 Prompt、出版社研究文件、同仓库其他题材仍保留的长规则写法。

## 0. 平台任务定义与基准继承

- 任务目标：把章节改写成更接近传统出版社 / 纸媒审读口味的版本。
- 必须继承创建正文、正文润色、去AI味。
- 平台重点：结构稳健、语言克制、逻辑闭环、审读友好、文笔有控制力。

## 0.2 零新增事实

- 所有事件链、证据链、人物链与动机链必须来自原文。
- 允许镜头化、动作化、体裁化，但不得编造新的程序、证据、结论。

## 0.3 沉浸式小说体裁

- 不得写成审读报告、投稿说明、案例分析、技巧讲解。
- 证据、制度、规则必须融入情节，不得写成说明书。
- 文件结构只允许：章标题、正文、`## 作者有话说`。

## 1. 平台理解

### 1.1 平台机制与审读偏好

- 审读更看重逻辑自洽、人物动机、结构完整性与细节可信度。
- 相比网文平台，更厌恶过密感叹、网文口号腔、纯爽点说明书。
- 标题需稳，不宜刻意标题党。

### 1.2 平台风格门槛

- 节奏可以稳，但不能空、不能松。
- 每一段都应推进信息、人物、情绪或风险中的至少一项。
- 专业流程、社会结构、物证材料应体裁化呈现。
- 作者有话说必须极轻，不可写成创作说明或价值总结。
- 出版社取向不是“去刺激”而是“去虚火”：删掉口号腔、连载提示腔、过密感叹和解释腔，但不能删掉有效推进。
- 单章内部仍需局部闭环；系列感、单元感、长线感都必须建立在当章完成度成立的前提上。

## 2. 输入与输出

- 输出工作目录：`出版社/`
- 章节目录（有分部）：`出版社/第X部/第Y卷/`
- 章节目录（无分部）：`出版社/第X卷/`
- 文件名：`{chapterNo} {平台新标题}.md`

## 3. 输出内容结构

- 顶部标题：`# 第{chapterIndex}章：{平台新标题}`
- 正文：克制、稳定、逻辑清楚
- `## 作者有话说`：中文，200–300 CJK，但宜偏短、偏轻

## 4. 关键硬约束

- 中文平台正文 CJK 必须在 3500–6500。
- 不得把“更像稿件”误写成“更像报告”。
- 不得用“更文气”替代结构稳健与证据闭环。
- 不得为了去网文腔而写成无推进的空文本。
- 所有专业、民俗、制度、程序内容都必须有现实承托，不能只留设定名称。
- 主角的成功、判断、成长与反击必须来自可追索的能力、认知或选择，而不是突然开挂。

## 5. 改写前推演问题

1. 本章最关键的逻辑链是什么？
2. 哪些地方应从解释改为动作、物证、制度话术？
3. 哪些网文腔句块必须去掉？
4. 章末如何在克制状态下仍保留翻页感？

## 6. 平台执行要点

- 优先稳结构、稳因果、稳叙述。
- 优先动作化心理、体裁化信息、物证化推进。
- 人物与社会结构应更可信。
- 章末要留余波，但不做夸张宣判。

### 6.1 出版社向通用补救映射

- 若原章网文味过重，先删功能性提醒、硬钩重复、显性爽点解释，再补回现实质地、对象细节与后果链。
- 若原章专业感不足，优先补“可验证的动作与对象”而不是补术语堆：谁看了什么、碰了什么、改了什么、按了什么流程、承担了什么风险。
- 若原章过于抒情，应补回事件驱动点：一个物件、一次判断、一个制度阻力、一次关系错位、一个未完成动作。
- 若原章只有强情节没有余波，应在章末增加人物命运、伦理压力、关系裂痕或认知反噬中的至少一项。

### 6.2 不同题材的共通出版向底色

- 悬疑线看重“现实专业型表达”：限知、证据、程序、职业视角、公平线索。
- 志怪线看重“现实肌理 + 民俗锚点”：地域经验、旧物、仪式、规则代价、科学对冲。
- 职场线看重“现实规则 + 可验证专业能力”：组织流程、权限结构、汇报逻辑、可复用技巧、节制爽感。

## 7. 输出格式与检查

- 检查逻辑链、人物动机、体裁化是否成立。
- 检查是否误写成审读报告腔。
- 检查作者有话说是否足够克制。
