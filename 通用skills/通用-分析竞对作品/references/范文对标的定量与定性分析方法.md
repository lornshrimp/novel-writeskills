# 范文对标的定量与定性分析方法

> 目的：把“感觉判断”改为“可复核证据”。

## 一、定量对标（先做）

### 1) 章节长度

- 指标：均值、中位数、波动范围。
- 结论用途：判断节奏与读时成本。

```bash
for f in 范文/vol-1/*.md; do
  wc -m < "$f"
done
```

### 2) 对话密度

- 指标：对话行占比（建议统计引号行）。
- 结论用途：判断“叙述驱动”还是“对话驱动”。

```bash
for f in 范文/vol-1/*.md; do
  d=$(grep -c "「" "$f")
  t=$(wc -l < "$f")
  awk "BEGIN{printf \"%s %.0f%%\\n\", \"$f\", $d*100/$t}"
done
```

### 3) 钩子覆盖率

- 指标：前 200 字开场钩子率、后 200 字留钩率。
- 结论用途：判断留存驱动强度。

```bash
for f in 范文/vol-1/*.md; do
  head -c 200 "$f" | grep -q "「\|？\|!\|！" && echo "$f 开场有钩"
done
```

### 4) 段落节奏

- 指标：平均段长、长段比例（如 >200 字）。
- 结论用途：判断手机端阅读摩擦。

```bash
awk '!/^$/{sum+=length($0);n++}END{print int(sum/n)}' 范文/vol-1/*.md
```

## 二、定性拆解（再做）

### 1) 章首钩子

检查点：

- 前 50 字是否有人物动作或异常。
- 前 100 字是否出现冲突或风险。
- 前 200 字是否形成“继续阅读问题”。

### 2) 章末钩子

检查点：

- 是否留有未闭合动作 / 新变量 / 新代价。
- 是否避免“总结腔”和“预告腔”。
- 是否自然牵引下一章接棒对象。

### 3) 爽点节拍

检查点：

- 小爽点间隔是否稳定（常见 1-3 章）。
- 中爽点是否在阶段关键处出现。
- 最长铺垫期是否过长导致失温。

### 4) 对话冲突

检查点：

- 是否存在意图对立而非信息问答。
- 是否有潜台词和位差（权限、口径、时间窗）。
- 是否用动作穿插避免纯对白漂浮。

### 5) 信息投放

检查点：

- 关键线索是否“提出→强化→回收”。
- 误导是否可证伪（公平）。
- 终局解释是否可回溯前文证据。

## 三、评分建议（可落表）

```markdown
| 维度 | 分值(10) | 证据 | 结论 |
|------|----------|------|------|
| 开场钩子 | 8 | ch-001前200字 | 强 |
| 章末留钩 | 7 | ch-001末200字 | 中强 |
| 对话冲突 | 6 | ch-003对话段 | 中 |
| 爽点节拍 | 8 | 卷一分布 | 强 |
| 信息投放 | 7 | 主线伏笔链 | 中强 |
```

## 四、迁移原则（防抄袭）

- 学“结构比例、节拍与机制”，不学“情节与句子”。
- 学“问题设计与兑现路径”，不学“人物关系复制”。
- 学“读者体验工程”，不学“文本外观模仿”。

## 五、常见误区

- 只看热度，不看可迁移机制。
- 只做结论，不给证据出处。
- 只报差距，不给修复动作。

## 六、输出最低要求

- 至少 1 张定量表 + 1 组定性结论。
- 每条结论至少绑定 1 个可回溯证据点。
- 最后必须给“当前项目三条优先补丁动作”。
