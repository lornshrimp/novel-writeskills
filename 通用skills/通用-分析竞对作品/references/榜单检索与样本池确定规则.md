# 榜单检索与样本池确定规则

本规则用于把“找同题材最优秀作品”从随手点开几本热门书，收束成**口径明确、可复盘、可解释、可服务四层深拆**的样本池建设动作。

## 一、默认参数与样本层级

- 默认目标平台：`起点中文网`
- 默认榜单规模：`Top10`
- 默认目标：锁定与当前项目**商业竞争位最接近**的强样本，而不是广义意义上的“所有出名作品”
- 默认样本层级：
  1. **TopN 候选池**：用于扫描整个赛道当前谁在打头部位次；
  2. **正式深拆样本池**：默认从候选池筛出 `3–5` 本主样本做重度拆解；
  3. **边界 / 旁证样本**：可保留用于说明赛道边缘打法、不同竞争位打法或作者存量打法，但不能和主样本混层下结论。

> 若用户明确要求 TopN 全量逐本落盘，则可把 TopN 全部纳入正式样本池；否则默认不把所有候选都视为同等权重的主样本。

## 二、起点中文网主样本优先筛选标准

当目标平台为起点中文网时，正式深拆主样本优先满足以下条件中的大多数；若某项数据无法公开稳定获取，应记录替代证据与盲区，而不是默默跳过：

### 2.1 成绩门槛

- 优先选择**连续 3 个月稳居分类月票榜 / 畅销榜 / 分类强榜 Top10** 的头部作品；
- 若能获取订阅数据，优先选择**均订破万、收藏量破 50 万**的作品；
- 若是已完结作品，优先补看**完结推荐票破 100 万**或其他能证明长期粘性的指标；
- 不单看某一次爆榜，要看它是否兼具“短期爆发力 + 长期留存能力”。

### 2.2 题材匹配

- 样本必须与你的当前项目处于**同题材或高度相近的细分赛道**；
- 若题材边界模糊，允许组合两个相近榜单建立候选池，但必须解释主榜与辅榜各自负责什么；
- 不得为了凑知名作品，硬塞进读者承诺、平台预期、主卖点明显不同的伪同题材样本。

### 2.3 更新 / 完结状态

- 优先选择**已完结**或**连载超过 100 万字**的作品，因为它们更适合验证长线结构与读者续航；
- 连载不足 `30` 万字的新书，默认只适合作“新书观察样本”，不应直接代替成熟爆款主样本；
- 若任务明确研究新书打法，可以新书榜样本为主，但必须在报告中显式写出这与成熟作品样本不在同一竞争位。

### 2.4 作者属性

- 优先选择**该作者的第一本头部爆款作品**或最能代表其首次破圈的作品；
- 大神作者的爆款默认要额外拆清楚：哪些优势来自文本本体，哪些来自作者粉丝基础、平台资源倾斜或既有品牌效应；
- 若样本的成功高度依赖影视改编、跨圈热度或作者存量，应降为“边界样本”或“旁证样本”，而不是直接当成主样本。

## 三、榜单口径选择顺序

### 3.1 若用户未指定榜单类型

默认按以下顺序判断：

1. **与当前项目最接近的商业竞争位榜单**
   - 例如：畅销、热销、月票、分类强榜等。
2. **若当前项目明显处于新书策略阶段**
   - 可优先补看新书榜 / 新锐榜 / 近期强势样本。
3. **若题材边界模糊**
   - 允许组合两个相近榜单建立样本池，但必须写清主榜与辅榜的职责。

### 3.2 若用户明确指定榜单类型

- 直接按用户指定执行；
- 但若指定榜单与当前项目阶段明显不匹配，必须在报告中显眼写出这会带来什么偏差。

## 四、搜索与核验要求（强制）

- 必须使用多种可用搜索方式做交叉检索，不得只依赖单一搜索源。
- 至少完成：
  1. **一次主检索**：锁定榜单候选；
  2. **一次交叉检索**：确认榜单与题材标签是否一致；
  3. **一次结果核验**：确认样本没有明显跑错赛道。
- 若官方榜单页难以直接打开，允许通过多组关键词、不同搜索源和多篇外部页面还原样本池；但必须把盲区写清楚。
- 对起点中文网样本，可直接使用 `https://www.qidiantu.com/` 作为补充检索与核验入口：
  - 榜单页可回看推荐位；
  - 首订页可看新书首订与真实粉丝增量；
  - 徽章页可看荣誉徽章口径；
  - 盟主页可看盟主数量；
  - 作品详情 / 数据页可补充推荐票、收藏数、真实粉丝数、月票数、字数等；
  - 但至少仍要配一条官方页或其他外部来源做交叉核验。
- 若官方作品页、目录页或试读页因验证码、反爬、登录门槛或频控受阻，允许转向**更稳定、可核验的替代来源**补足样本信息，例如官方 App 试读、作者或平台公开物料、媒体报道、读者书评、论坛讨论、剧情梳理、百科 / 设定页等；**不要把来源不明、不可稳定核验的全文镜像页或转载聚合页列为核心检索来源。**

## 五、推荐搜索词模板

以下搜索词可按需拼接：

- `起点中文网 {题材} 榜单`
- `起点 {题材} Top10`
- `起点 {题材} 月票榜`
- `起点 {题材} 畅销榜`
- `起点 {题材} 热门作品`
- `{题材} 起点 推荐作品`
- `{作品名} 起点图`
- `site:qidiantu.com {作品名}`
- `{作品名} 起点`
- `{作品名} 目录`
- `{作品名} 书评`
- `{作品名} 评论`
- `{作品名} 读后感`
- `{作品名} 拆解`

> 若题材边界不止一个叫法，必须并行检索多个同义口径，例如“都市悬疑 / 都市惊悚 / 都市怪谈 / 都市异闻”等，并在最终样本池说明里解释取舍。

## 六、样本入池与分层规则

### 6.1 进入 TopN 候选池的最低要求

只有满足以下至少多数条件的作品，才应进入 TopN 候选池：

- 与当前项目属于**同平台或高度相近平台语境**；
- 题材承诺、读者预期、商业竞争位大体一致；
- 能找到足够多的外部证据支持后续深拆；
- 不是只靠作者名气被讨论，而是作品本身确有可分析价值；
- 不是明显跑偏到另一赛道的“伪同题材样本”。

### 6.2 进入正式深拆样本池的额外要求

若要被纳入 `3–5` 本正式深拆主样本，通常还应满足：

- 市场数据层有足够证据证明它不是偶然热度；
- 内容创作层存在清晰、可归纳、可迁移的优势；
- 至少能补齐“榜单 / 数据、目录 / 正文片段、读者反馈”三类核心证据；
- 与当前项目在竞争位上足够接近，拆出来的结论不会严重误导项目方向。

### 6.3 边界样本 / 旁证样本的使用方法

- **边界样本**：看起来同赛道，但卖点偏移、竞争位略有偏差，适合说明边界打法；
- **旁证样本**：知名度高、讨论度高，但不应被当作主对标，只用于补充说明某个维度；
- 这两类样本可以引用，但不得与主样本混成同一层级结论。

## 七、样本排除规则

出现以下情况时，默认不应直接纳入主样本池：

- 平台机制明显不同，导致读者预期完全变形；
- 表面同题材，实则主卖点完全不同；
- 几乎找不到可用外部证据，只能靠模糊印象猜测；
- 作品讨论度很高，但核心优势明显依赖影视改编、作者存量或跨圈热度，而不是文本本体；
- 与当前项目赛道差距过大，把它纳入反而会误导分析维度。

> 若某部作品知名度极高、但赛道并不完全一致，可以放入“旁证样本”或“边界样本”，但不能伪装成主样本。

## 八、榜单与样本池记录格式（建议）

| 字段 | 说明 |
| --- | --- |
| 榜单名称 | 最终采用的榜单口径 |
| 目标平台 | 起点中文网 / 其他 |
| 题材口径 | 例如：都市悬疑 / 异能志怪 |
| TopN 数量 | 如 10 |
| 正式深拆样本数 | 如 3 / 5 |
| 检索时间 | YYYY-MM-DD |
| 主要搜索词 | 本次主要搜索词组合 |
| 主要搜索来源 | 榜单页、作品页、书单页、推荐文等 |
| 交叉核验来源 | 至少 1 类 |
| 主样本名单 | 最终纳入深拆的作品清单 |
| 边界 / 旁证样本 | 仍值得保留但不作为主对标的作品 |
| 排除样本与原因 | 为什么没有纳入 |
| 当前盲区 | 还原样本池时仍存在的不确定点 |

## 九、样本池确定后的最低动作

样本池一旦锁定，接下来至少要做：

1. 为每本主样本建立证据卡；
2. 对每本主样本至少补齐“市场数据、目录 / 正文片段、读者反馈”三类核心证据；
3. 再决定哪些作品是“全维度强标杆”，哪些只是“局部强样本”；
4. 至少完成 `3` 本主样本后，尝试提炼横向共性、差异化路径与市场空白点；
5. 最终为每本主样本单独落盘详细报告，而不是把全部样本挤成一份混合摘要。

## 十、常见误区

- 只因为作品很火，就自动视为最适合当前项目对标的样本。
- 只选自己熟悉或喜欢的书，而不是按竞争位选样本。
- 用不同榜单、不同时间、不同赛道的书拼凑一个看上去很强的名单，却不解释混样口径。
- 没有完成样本池核验就开始逐本下判断。
- 用“大家都说好看”代替“它具体为什么强”。
- 因官方源受阻，就默认转去来源不明、不可稳定核验的全文镜像页硬抓目录、正文与章节清单。
