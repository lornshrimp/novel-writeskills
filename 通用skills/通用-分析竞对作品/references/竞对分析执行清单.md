## 五、单书证据采集最小线（按四层执行）

| 层级 | 最低要求 | 备注 |
| --- | --- | --- |
| 市场数据层 | 榜单 / 作品基础信息至少 `1` 条；如能补数据页更佳 | 需要能支撑“它确实是强样本”的判断 |
| 内容创作层 | 目录 / 卷名 / 正文片段 / 设定中至少 `3` 类证据 | 需要能支撑“它到底强在哪”的判断 |
| 运营策略层 | 更新节奏、互动、资源配合中至少 `1–2` 类证据 | 找不到也要写“找不到” |
| 受众反馈层 | 至少 `2` 类不同来源反馈 | 尽量覆盖正向与负向 |

> 对起点中文网样本，建议额外补查 `https://www.qidiantu.com/`：其作品详情页 / 数据页 / 榜单页 / 首订页 / 徽章页 / 盟主页 / 书单页通常可补充推荐票、收藏数、真实粉丝数、月票数、字数、荣誉徽章、盟主数、书单收录、评论 / 书评入口等信息；已知书籍 ID 时，可优先看 `https://www.qidiantu.com/info/{书籍ID}` 与 `https://www.qidiantu.com/info/{书籍ID}/d`。
>
> 这类页面适合补“热度与反馈数据证据”，不替代官方目录、试读正文与正文片段的核验。
>
> 如果某一层证据客观难找，不允许直接跳过；必须写明“搜过什么、没搜到什么、因此不确定什么”。
>
> 若官方站点因验证码、反爬、登录门槛或频控导致正文 / 目录证据不可稳定获取，允许改用**更稳定、可核验的替代来源**（如官方试读、作者公开物料、媒体报道、书评短引文、论坛讨论、读后感、剧情总结等）；**不要把来源不明、不可稳定核验的全文镜像页或转载聚合页当作补正文的核心证据。**

## 六、单书报告必答问题

每份单书报告至少要回答：

1. 这部作品为什么是强样本，而不是普通热门书。
2. 它的核心竞争优势到底是什么。
3. 它的市场数据层说明了什么：长期稳定性、商业价值、核心流量破口分别是什么。
4. 它的内容创作层是如何在题材定位、开篇、节奏、人物、设定、语言上形成竞争优势的。
5. 它的运营策略层做了什么放大内容效果。
6. 它的受众反馈层最强共鸣点、主要吐槽点与归属感来源分别是什么。
7. 哪些优点值得当前项目学习。
8. 哪些优点只是它的个体条件、时代窗口或作者存量，不应机械照搬。
9. 还有哪些盲区没有证据，不该强下结论。

## 八、证据写作规范

- 所有“强点”判断都要尽量落到具体证据，不写空泛夸奖。
- 所有“可学点”都要写成可执行动作，而不是形容词。
- 所有“不可照搬点”都要写明原因：平台机制、题材差异、作者存量、更新节奏、个人声线、读者预期等。
- 若引用了 `竞对分析/竞对资料/` 中的既有深入研究、章节摘要或资料摘记，要标明它属于工作区补充参考，而不是外部核心证据。
- 尽量区分：
  - **事实**：明确可见的榜单、目录、正文、评论信息；
  - **推断**：基于多条证据形成的判断；
  - **假设**：证据不足但合理怀疑的方向。
