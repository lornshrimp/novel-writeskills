# 事件案件引擎保真规则

## 一、事件 / 案件不是“发生一件怪事”

一个合格的事件 / 案件，至少要同时承担：

- 局部悬疑任务
- 主线供血任务
- 规则显影任务
- 人物代价任务

少一项都容易显得只是热闹。

## 二、标准发动链

推荐按以下顺序设计：

1. **异常信号**：最先被看到 / 听到 / 闻到 / 记录到的反常。
2. **现实摩擦**：为什么这件事一开始没有被当成超常，而是被误判成日常问题。
3. **证据载体**：回执、监控、门禁、病历、群聊、工单、录音、日志、现场物件等。
4. **误导链**：谁的话、哪份记录、哪个旧印象把调查带偏。
5. **可证伪点**：最终靠什么把误导拆掉。
6. **主线拼图**：本案到底给主线或支线添了哪块图。
7. **代价**：谁付出什么，最好带不可逆损耗。
8. **余波**：事件结束后，还留下什么更大的危险或债务。

## 三、四种常见失真

### 1. 只有气氛，没有证据

写得很吓人，但全是感觉，没有可复盘载体。

### 2. 只有拼图，没有戏

全案只是为了交一条线索，人物没压力、没代价、没场面。

### 3. 只有诡计，没有母题

案子能反转，但和这本书的现实母体、规则或主线无关。

### 4. 只有局部完结，没有余波

事件一结束什么都没留下，后续章节像重开新副本。

## 四、案件供血检查

设计完成后，逐条自检：

- [ ] 本案推进了哪块主线拼图
- [ ] 本案引出了哪条新风险
- [ ] 本案让谁付出了更高代价
- [ ] 本案对嫌疑池 / 关系网 / 规则理解造成了什么变化
- [ ] 本案结束后是否留下至少一个新缺口

## 五、读者向执行口径

- 读者在本案开场前 150–300 字内，应该先被什么反常点抓住。
- 读者在案中中段，应该明确拿到哪一类回报：新证据、误导被证伪、规则边界被试出、关系变化、局部反击。
- 读者在案末，应该被什么更危险的问题推向下一章 / 下一案。

## 六、现实锚点要求

- 每个事件都必须附着在读者熟悉的现实界面上。
- 每个事件至少设计 1 个“日常中的微反常”，且该反常必须能够进入证据链。
- 每个事件至少内置 2 条可审计证据要素。

## 七、中段兑现要求

事件 / 案件必须能支撑正文的中段回报，而不是只有章首炸点与章末钩子。因此骨架中要预埋至少一个：

- 新证据落袋
- 误导被证伪
- 规则边界第一次被摸到
- 人物关系发生变化
- 一次有效反击成立

## 八、人物代价与现实摩擦要求

- 动机必须先于诡计，现实摩擦必须先于超常解释。
- 推进不能只靠“有人更坏”，还应体现流程卡点、权限不足、交接失败、窗口关闭、审批拖延、名誉风险等现实压力。
- 代价至少命中以下一项：身体、关系、职业、名誉、法律风险、心理。

## 九、作者侧编号隔离要求

后台编号可以用于对齐，但进入正文可见内容时，必须改写成世界内真实存在的号、页、记录或点位信息。

## 十、最终验收口径

一个可直接进入正文施工的事件 / 案件，至少同时满足：

- 有可见反常，不只是概念异样
- 有现实摩擦，不只是超常解释
- 有证据链，不只是气氛描写
- 有人物代价，不只是拼图推进
- 有主线供血，不只是单元热闹
- 有中段兑现，不只是开头和结尾
- 有余波，不是结案归零
