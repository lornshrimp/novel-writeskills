# 案件余波与主线供血检查表

## 一、案内收束后必须留下什么

- [ ] 一个更大的风险
- [ ] 一笔新的代价
- [ ] 一块新的主线拼图
- [ ] 一处关系变化或信任裂缝

## 一点五、来源层速判（新增）

- `案内收束层`：本案局部到底收了什么、没收什么。
- `供血推进层`：主线拼图、规则理解、关系变化与风险等级是否真的被改写。
- `余波交接层`：更大风险、新代价、新缺口与后续接棒是否明确。

## 二、供血判断

若事件结束后只能回答“这件事很有气氛”，不能回答“主线因此发生了什么变化”，说明供血失败。

## 三、余波的四种常见形态

1. **制度余波**：上面开始掩、压、调档。
2. **人物余波**：有人因此失去职业、关系、名誉或安全感。
3. **规则余波**：主角对机制 / 规则的理解被改写。
4. **主线余波**：旧案 / 旧人 / 旧组织与当前事件并上了线。

## 最小余波工件（新增）

```markdown
【案内收束层】
【供血推进层】
【余波交接层】
【当前判定】Green / Yellow / Red
```

## Green / Yellow / Red 余波判定（新增）

- `Green`：收束、供血、交接三层都清楚，本案不会“结完就归零”。
- `Yellow`：案内收束已在，但供血推进或余波交接仍偏软，需要继续补钉子。
- `Red`：案件只能回答“发生过”，不能回答“主线因此怎样变了”。
