# 通用-审阅分卷大纲

分部 / 分卷审阅的第一问题不是“顺不顺”，而是“上一级已经分配给它的内容，真的落下来了没有”。

## 保真迁移要求（强制）

命中分部 / 分卷层审阅任务时，除本文件外，**必须继续读取**：

- `references/分部与分卷审阅专属检查清单.md`
- `references/分部与分卷审阅报告增补.md`
- `references/分部与分卷审阅完整模板.md`
- `references/腾讯专栏分卷审阅阶段目标与中段回报增补.md`

## 完整模板执行要求（强制）

- 产出分部 / 分卷 / 单元案审阅报告时，必须以 `references/分部与分卷审阅完整模板.md` 作为主骨架，不得退化成“简单点评 + 少量建议”的简版报告。
- 报告至少要完整覆盖：阅读链路确认、概览复核、P0 工程结构与可执行性、题材专项核查、超长篇工程化、节奏与结构规划、系列连贯性、机制递进、时间线、主题承诺、问题分级、评分、具体回写建议、要求对照、执行顺序、总体结论。
- 若当前任务还加载了题材 Skill，则必须在通用完整模板的占位章节中继续补写题材专项段、题材专项评分项与题材化结论重点。
- 若未先写“上游分配落地诊断”就直接评价节奏与钩子，视为流程不合格。

## 卖点保真要求（强制）

- 审阅时必须单独标出本卷卖点落点、卖点弱化段与卖点遗忘风险。
- 必须检查章清单是否让读者在卷内反复感知到这本书的独特吸引力，而不是只剩流程推进。
- 若卖点相关章节过少、过晚或过弱，应明确判为留存风险与签约风险。

## 网文卷级留存与平台节奏（强制）

- 审分部 / 分卷 / 单元案时，必须把每一卷视为“读者是否继续追、是否继续买下一卷”的付费阶段，而不是孤立结构块。
- 必须显式判断：`前 5 章追读链`、`卷中回报密度与新鲜冲突比例`、`钩子类型轮换`、`单元案独立度与主线关联度`、`卷末续购与跨卷交接` 是否成立。
- 若一卷只有章首抓眼和章末硬钩，中段缺回报、卷末缺更贵缺口，即使结构看似完整，也不得判为高可用卷纲。

## planner / reviewer 镜像补强（新增）

- 审分卷时，默认继续往下看“章节微型剧本是否已经长出 planner 风格的施工字段”：每章是否写清唯一职责、不应包含、情感弧、`1–4` 个 beat、钩子链响应与基本就绪判定。
- 若章清单只有标题和事件，不足以让下游直接做控制卡或正文，默认应判为“执行接口层发虚”，而不是只算节奏小瑕疵。
- 审阅报告里的问题默认标记来源层：`[卷层结构层] / [章清单层] / [微型剧本层]`。卷层结构层负责整卷起承转合，章清单层负责功能排布，微型剧本层负责能否直接拆成控制卡和场景施工。
- 默认补一个 `Green / Yellow / Red` 执行判定：`Green`=卷级结构、章功能轮换、微型剧本施工字段齐；`Yellow`=卷骨架可用但部分章节职责 / beat / 钩子链仍虚；`Red`=上游承接缺失、黄金三章失效、卷中回报稀薄、章清单无法直接下沉执行。
- 审分卷不仅要看“写得顺不顺”，还要判断这份卷纲能不能稳定生成控制卡、场景卡、正文与连续性更新；不能生成，说明它还只是半成品。

## 极其快节奏与极其强冲突审阅门禁（新增）

分卷审阅必须把“快节奏 + 强冲突”写成可量化、可判级、可回炉的审阅结论。

### 快节奏分卷门禁

- **黄金三章门禁**：卷首前三章必须形成“麻烦入场 → 正面对抗 → 首次升级/回报”。任一缺失，判 `P1`。
- **卷中防失温门禁**：卷中不得出现连续 `4+` 章等效“无新变量/无回报/无关系变化”的平推段，命中即 `P1`。
- **章功能轮换门禁**：章清单应看出“钩子/蓄力/爆发/回报/转承”轮换；若连续多章同功能堆叠导致节拍单一，判节奏风险。
- **卷末续购门禁**：卷末须留下下一卷必须处理的高价值缺口；若仅总结或抒情收束，判 `P1`。

### 强冲突分卷门禁

- **冲突三维压迫**：本卷至少覆盖外部压力、关系/身份压力、价值选择压力中的两项；长期单维冲突判 `P1`。
- **升级链可追踪**：章清单需能回答“本章比上章升级了什么”；若无升级轨迹，判结构空转。
- **代价落点清晰**：本卷必须有 `2-3` 个可验证的不可逆代价（失信、失位、失资源、失身份窗口等）。缺失判 `P1`。
- **关键关系翻面**：重要关系至少出现一次可感知位移（结盟/裂变/反咬/误解加深/信任修复失败）；长期无关系位移判留存风险。

### 快冲突评分卡（分卷层）

| 维度 | 权重 | 过线标准 |
| --- | --- | --- |
| 卷首启动与前5章追读链 | 20% | 前 5 章持续抬压，不靠单次爆点 |
| 卷中回报密度与新变量 | 25% | 每 2-3 章有可感知升级或回报 |
| 冲突升级链与关系位移 | 25% | 章章可追踪升级，关系不齐平 |
| 卷末续购与跨卷交接 | 15% | 卷末缺口明确且更贵 |
| 卖点显影与兑现节拍 | 15% | 卖点贯穿卷内而非点状出现 |

若“卷中回报密度与新变量”或“冲突升级链与关系位移”任一不达标，综合评分默认不得给高档。

### 报告中必须输出的最小回炉动作（新增）

- 指定本卷最先要改的 `3` 章，并说明每章该补“钩子 / 回报 / 升级 / 关系位移”中的哪一项。
- 指定一条“72 小时补丁链”：优先改哪些章节即可最快抬升追读与续购信号。
- 明确哪些章节应删解释、增动作、补代价，不得只给“节奏再紧凑点”这类抽象建议。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，审阅前必须先读取并纳入对照。
- 若目标平台为起点中文网，或未注明平台但默认按起点口径处理，且当前题材目录下尚无可用竞对主报告，审阅前应优先调用 `通用-分析竞对作品`，至少补齐 **Top10 候选池、`3–5` 本主样本与四层深拆速览**；若任务限制导致本轮无法补齐，必须显眼记录盲区，并下调本轮审阅上限与置信度。
- 审阅报告不得只给内部自洽结论；必须显式判断当前分卷相对竞对在哪些维度已领先、持平或落后，以及最先该补哪条超越路径。
- 若当前分卷在竞对已验证的关键强点上明显更弱，必须直判为竞争风险；分卷审阅的默认目标是推动当前作品在关键维度上**全方位超越竞争对手**。
- 默认按“严酷找差距”口径执行：重点写清当前分卷为什么还赢不过已过审竞对，不得为了照顾作者情绪把卷中塌陷或续购不足轻写成“节奏小瑕疵”。
- 竞对维度在分卷审阅综合评分中默认按 `25%` 计权；若未完成量化差距卡，或关键维度仍显著落后且没有 `72` 小时补丁，综合评分默认不得给高档。

## 起点化竞对吸收口径（新增）

当分卷审阅服务起点中文网，或未标平台而默认按起点基线处理时，默认把竞对对照拆成以下四层，并翻译成评分判断与补丁建议：

- **市场数据层**：用榜单位次、均订 / 首订 / 收订比代理信号、推荐资源位等，反推这卷是否真的支撑题材归类、前 `5` 章追读、卷尾续购与持续付费价值，而不是只在作者视角里“章清单排得很满”。
- **内容创作层**：重点对照主样本在黄金三章、卷中回报、冲突升级链、关系位移、单元案供血与卷尾交接上的成熟度，把差距翻成卷层、章清单层与微型剧本层补丁。
- **运营策略层**：重点对照主样本如何制造卷内讨论点、回报波形、推荐位适配、卷末催更点与跨卷交接，把差距翻成前 `5` 章、卷中、卷末的节拍与钩子补丁。
- **受众反馈层**：重点对照本章说 / 书友圈 / 书评区里高频被夸的值点、被骂的卷中失温病灶、老白读者对平推章和假钩子的敏感点，把差距翻成避坑与加码建议。

起点语境下，默认尽量使用以下术语，而不是写成空泛的“分卷竞争力不足”：`均订`、`高订`、`首订`、`追订`、`收订比`、`分类月票榜`、`畅销榜`、`新书榜`、`三江`、`本章说 / 段评`、`书友圈`、`老白读者`。拿不到精确值时必须写成代理信号，不得伪造具体数字。

## 常见触发词 / 用户说法速查

- 帮我审这卷
- 看看这份分卷大纲有没有问题
- 章清单是不是发塌了
- 这卷回报够不够
- 给我分卷审阅报告
- 看看上一级分配有没有真落地

## 何时使用

- 审阅分部大纲、分卷大纲、章清单、单元案执行版。
- 检查 5/15 节奏、章末钩子、章中回报与可拆章性。
- 检查上游承接关系、单元案供血主线与阶段闭环。

## 报告落盘位置与命名规则（强制）

- 分部大纲、分卷大纲审阅报告必须直接落在项目根下的 `审阅意见/` 目录中，不再为分部或分卷单独建立子目录。
- 若当前对象是本 Skill 覆盖的单元案执行版，也沿用 `审阅意见/` 根目录口径，避免与分层大纲审阅拆成两套位置规则。
- 分部 / 分卷 / 单元案审阅报告文件名必须带日期，日期格式固定为 `YYYY-MM-DD`。
- 文件名可分别写作 `审阅意见/分部审阅报告_[对象名]_2026-04-15.md`、`审阅意见/分卷审阅报告_[对象名]_2026-04-15.md`、`审阅意见/单元案审阅报告_[对象名]_2026-04-15.md`。
- 分部大纲、分卷大纲、章清单、单元案执行版等本体文件默认不带日期；日期只用于对应审阅报告。

## 默认执行顺序

0. 若按起点口径执行且当前题材目录下缺少可用竞对主报告，先调用 `通用-分析竞对作品`，补齐 `Top10 候选池 → 3–5 本主样本 → 四层深拆` 的最小结论集。
1. 先列出本层应承接的上游内容。
2. 再判定当前对象是分部层、分卷层还是单元案层。
3. 若按起点口径执行，先吸收竞对主样本四层结论，并写清本轮审阅重点比对的关键项。
4. 按检查清单核查上游内容是否全量落地。
5. 再评估节奏、钩子、章中回报、单元案闭环与主线供血。
6. 最后按报告增补输出最小可回写补丁。

## 默认输出口径

- 默认输出一份带承接核查、节奏判断、卖点风险与最小回写建议的分卷审阅报告。
- 默认同时输出：来源层标记、执行判定、优先回炉的 `3` 章与一条 `72` 小时补丁链。
- 默认不直接改分卷本体，除非用户明确要求边审边回写。
- 若按起点口径执行，报告还必须显式覆盖：`Top10 候选池`、`3–5` 本主样本四层速览、竞对差距量化卡，以及当前分卷最先该补的超越路径。

## 最低交付

- 能指出上游分配是否全量落地。
- 能指出卷内节奏、章中回报、钩子与主线供血的关键风险位。
- 能给出最小可回写补丁，而不是只给笼统判断。

## 复审独立性规则（强制）

- 每一轮分部 / 分卷 / 单元案审阅都必须对当前版本重新独立、从严、完整复审，不得因为上一轮提出的问题已经修复，就默认本轮可以给高分。
- 核查上一轮问题是否修复是必要步骤，但仍必须重新检查上游承接、节奏、钩子、回报、卖点落点、单元闭环与主线供血，不能只做“对照上一轮问题单”的补丁式复查。
- 若本轮在新版本里发现新的节奏塌陷、卖点弱化、承接遗漏、章节功能失配或其它新问题，必须如实新增到本轮报告里，不得因“整体比上一轮更好”而省略。
- 本轮评分只看当前版本在本轮完整审阅下的真实质量，不得按上一轮分数、修复数量或修改幅度自动折算加分。

## 审阅重点

- 上游分配是否全量落地
- 卖点落地数量、位置与强度是否达标
- 前 5 章追读链与卷中回报是否值回票价
- 5/15 节奏与钩子策略是否成立
- 钩子类型是否疲劳，卷末续购是否成立
- 章中回报与读者留存是否足够
- 单元案是否闭环且供血主线

## 硬规则

- 禁止只评价“这卷顺不顺 / 够不够热闹”。
- 若按起点口径执行，禁止跳过 `Top10 候选池 → 3–5 本主样本 → 四层对照` 就直接给高分或高档判断。
- 必须落到承接、回报、节奏、供血与回写动作。
- 必须说明本卷是否会让读者继续追、卷末是否会让读者继续买下一卷，以及当前节奏更偏哪类平台口径。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责分卷层审阅与回写建议，不直接代替分卷重写。
- 若用户要直接搭建或重写分卷大纲，应优先使用 `通用-设计分卷大纲`。
