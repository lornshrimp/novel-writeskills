# 通用-审阅分卷大纲

分部 / 分卷审阅的第一问题不是“顺不顺”，而是“上一级已经分配给它的内容，真的落下来了没有”。

## 保真迁移要求（强制）

命中分部 / 分卷层审阅任务时，除本文件外，**必须继续读取**：

- `references/分部与分卷审阅专属检查清单.md`
- `references/分部与分卷审阅报告增补.md`
- `references/分部与分卷审阅完整模板.md`
- `references/腾讯专栏分卷审阅阶段目标与中段回报增补.md`

## 完整模板执行要求（强制）

- 产出分部 / 分卷 / 单元案审阅报告时，必须以 `references/分部与分卷审阅完整模板.md` 作为主骨架，不得退化成“简单点评 + 少量建议”的简版报告。
- 报告至少要完整覆盖：阅读链路确认、概览复核、P0 工程结构与可执行性、题材专项核查、超长篇工程化、节奏与结构规划、系列连贯性、机制递进、时间线、主题承诺、问题分级、评分、具体回写建议、要求对照、执行顺序、总体结论。
- 若当前任务还加载了题材 Skill，则必须在通用完整模板的占位章节中继续补写题材专项段、题材专项评分项与题材化结论重点。
- 若未先写“上游分配落地诊断”就直接评价节奏与钩子，视为流程不合格。

## 卖点保真要求（强制）

- 审阅时必须单独标出本卷卖点落点、卖点弱化段与卖点遗忘风险。
- 必须检查章清单是否让读者在卷内反复感知到这本书的独特吸引力，而不是只剩流程推进。
- 若卖点相关章节过少、过晚或过弱，应明确判为留存风险与签约风险。

## 网文卷级留存与平台节奏（强制）

- 审分部 / 分卷 / 单元案时，必须把每一卷视为“读者是否继续追、是否继续买下一卷”的付费阶段，而不是孤立结构块。
- 必须显式判断：`前 5 章追读链`、`卷中回报密度与新鲜冲突比例`、`钩子类型轮换`、`单元案独立度与主线关联度`、`卷末续购与跨卷交接` 是否成立。
- 若一卷只有章首抓眼和章末硬钩，中段缺回报、卷末缺更贵缺口，即使结构看似完整，也不得判为高可用卷纲。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，审阅前必须先读取并纳入对照。
- 审阅报告不得只给内部自洽结论；必须显式判断当前分卷相对竞对在哪些维度已领先、持平或落后，以及最先该补哪条超越路径。
- 若当前分卷在竞对已验证的关键强点上明显更弱，必须直判为竞争风险；分卷审阅的默认目标是推动当前作品在关键维度上**全方位超越竞争对手**。
- 默认按“严酷找差距”口径执行：重点写清当前分卷为什么还赢不过已过审竞对，不得为了照顾作者情绪把卷中塌陷或续购不足轻写成“节奏小瑕疵”。
- 竞对维度在分卷审阅综合评分中默认按 `25%` 计权；若未完成量化差距卡，或关键维度仍显著落后且没有 `72` 小时补丁，综合评分默认不得给高档。

## 常见触发词 / 用户说法速查

- 帮我审这卷
- 看看这份分卷大纲有没有问题
- 章清单是不是发塌了
- 这卷回报够不够
- 给我分卷审阅报告
- 看看上一级分配有没有真落地

## 何时使用

- 审阅分部大纲、分卷大纲、章清单、单元案执行版。
- 检查 5/15 节奏、章末钩子、章中回报与可拆章性。
- 检查上游承接关系、单元案供血主线与阶段闭环。

## 报告落盘位置与命名规则（强制）

- 分部大纲、分卷大纲审阅报告必须直接落在项目根下的 `审阅意见/` 目录中，不再为分部或分卷单独建立子目录。
- 若当前对象是本 Skill 覆盖的单元案执行版，也沿用 `审阅意见/` 根目录口径，避免与分层大纲审阅拆成两套位置规则。
- 分部 / 分卷 / 单元案审阅报告文件名必须带日期，日期格式固定为 `YYYY-MM-DD`。
- 文件名可分别写作 `审阅意见/分部审阅报告_[对象名]_2026-04-15.md`、`审阅意见/分卷审阅报告_[对象名]_2026-04-15.md`、`审阅意见/单元案审阅报告_[对象名]_2026-04-15.md`。
- 分部大纲、分卷大纲、章清单、单元案执行版等本体文件默认不带日期；日期只用于对应审阅报告。

## 默认执行顺序

1. 先列出本层应承接的上游内容。
2. 再判定当前对象是分部层、分卷层还是单元案层。
3. 按检查清单核查上游内容是否全量落地。
4. 再评估节奏、钩子、章中回报、单元案闭环与主线供血。
5. 最后按报告增补输出最小可回写补丁。

## 默认输出口径

- 默认输出一份带承接核查、节奏判断、卖点风险与最小回写建议的分卷审阅报告。
- 默认不直接改分卷本体，除非用户明确要求边审边回写。

## 最低交付

- 能指出上游分配是否全量落地。
- 能指出卷内节奏、章中回报、钩子与主线供血的关键风险位。
- 能给出最小可回写补丁，而不是只给笼统判断。

## 复审独立性规则（强制）

- 每一轮分部 / 分卷 / 单元案审阅都必须对当前版本重新独立、从严、完整复审，不得因为上一轮提出的问题已经修复，就默认本轮可以给高分。
- 核查上一轮问题是否修复是必要步骤，但仍必须重新检查上游承接、节奏、钩子、回报、卖点落点、单元闭环与主线供血，不能只做“对照上一轮问题单”的补丁式复查。
- 若本轮在新版本里发现新的节奏塌陷、卖点弱化、承接遗漏、章节功能失配或其它新问题，必须如实新增到本轮报告里，不得因“整体比上一轮更好”而省略。
- 本轮评分只看当前版本在本轮完整审阅下的真实质量，不得按上一轮分数、修复数量或修改幅度自动折算加分。

## 审阅重点

- 上游分配是否全量落地
- 卖点落地数量、位置与强度是否达标
- 前 5 章追读链与卷中回报是否值回票价
- 5/15 节奏与钩子策略是否成立
- 钩子类型是否疲劳，卷末续购是否成立
- 章中回报与读者留存是否足够
- 单元案是否闭环且供血主线

## 硬规则

- 禁止只评价“这卷顺不顺 / 够不够热闹”。
- 必须落到承接、回报、节奏、供血与回写动作。
- 必须说明本卷是否会让读者继续追、卷末是否会让读者继续买下一卷，以及当前节奏更偏哪类平台口径。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责分卷层审阅与回写建议，不直接代替分卷重写。
- 若用户要直接搭建或重写分卷大纲，应优先使用 `通用-设计分卷大纲`。
