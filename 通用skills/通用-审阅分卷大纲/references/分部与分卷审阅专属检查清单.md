# 分部与分卷审阅专属检查清单

## 一、先判上游分配是否全量落地

- 是否已列出本层应承接的上游内容
- 这些内容是否有明确落点、执行方式与可拆章程度
- 是否存在遗漏、缩水、悬空与只挂名不落实

## 二、分层检查口径

### 分部层

- 是否正确承接总纲层分配
- 是否能指导后续分卷落地

### 分卷层

- 是否把上一级分配内容落到卷概览、章清单、工件与章节规划
- 5/15 节奏是否标清
- 钩子类型是否有分布策略

### 单元案层

- 启动 → 砸实 → 对抗 → 翻案 → 余波 是否完整
- 是否既能阶段闭环，又能给主线供血

## 三、常见严重问题

- 没有 5/15 节奏布点
- 章中没有回报，章末只有硬钩
- 上游已分配内容大量缺失、悬空或被偷换成一句摘要
- 单元案闭环了，但没反哺主线

## 四、网文卷级留存专项核查

- 前 5 章是否形成连续追读链，而不是只靠第一章抓眼
- 卷中是否持续提供真实回报，而不是连续推进已知信息
- 钩子类型是否轮换，是否出现连续疲劳
- 单元案是否既能独立成立，又能给主线供血

## 五、卷末续购与跨卷交接核查

- 卷末是否留下更贵的新缺口，而不是漂亮收尾
- 下一卷第一屏要接的人、物、信息、关系、后果是否已经明确
- 单元案闭环后，主线拼图、人物代价或制度后果是否仍在推动后续

## 六、平台节拍与拆章适配核查

- 当前卷长、章长、钩子密度更偏哪类平台
- 若目标平台与当前卷节奏错配，是否已明确指出错配点
- 是否存在“结构能看，但一拆章就失温”的风险

## 七、章节微型剧本镜像核查

- 章清单是否继续下压到逐章微型剧本，而不是只停在标题 + 事件概述
- 每章是否写清唯一职责与不应包含
- 关键章节是否补出读者情感入口 / 出口、`1–4` 个 beat、承重 / 灵活标记
- 是否存在钩子链验证与基本就绪判定，而不是把成熟度判断交给下游临场猜

## 八、问题来源层与执行判定

- 问题是否区分为：卷层结构层 / 章清单层 / 微型剧本层
- 是否给出 Green / Yellow / Red 执行判定，而不是只有“能写 / 不能写”的空话
- 是否指出最先回炉的 `3` 章，以及每章最该补“钩子 / 回报 / 升级 / 关系位移”中的哪一项
