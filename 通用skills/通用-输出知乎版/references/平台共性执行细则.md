# 通用-输出知乎版｜平台共性执行细则

## 适用目标

将章节改写成更适合知乎盐选 / 专栏 / 连载语境的版本，同时保持：

- 主要信息不丢失
- 逻辑链清楚、证据可追、现实映射明确
- 输出仍然是纯小说正文，而不是议论文或备忘单

## 一、平台共性任务定义

- 强化高概念、现实映射、可验证质感、反直觉但自洽的推进方式。
- 保留关键事件链、人物目标/代价/关系变化、伏笔与章末钩子。
- 不允许新增事件、证据、规则、人物、地点、时间点或因果。
- 知乎系平台更强调“短时间内给出值得讨论的问题”，而不是传统长篇式的慢铺垫。

## 二、跨题材共同硬约束

### 1. 零新增事实

- 所有情节与信息必须来自原始章节正文。
- 允许的扩写只限于动作镜头化、等价重述、微反应补足。
- 数值、时间戳、ID、回执编号、界面字段必须原样保留。
- 叙事人称与视角必须保持一致。
- 章末钩子只能靠排序、留白与句式强化。

### 2. 沉浸式体裁约束

- 正文必须首先是小说，不得写成备忘单、教程、复盘、分点清单或解释稿。
- 文件中只允许 `# 第X章：...` 与 `## 作者有话说` 两种标题行。
- 证据材料必须写成角色正在看到、听到、拿到的东西，不得写成分析栏。

### 3. 共同门禁

- 正文 CJK 必须在 3500–6500 之间。
- 必须高比例重写，不得大量直接复用原句。
- 必须清除模板腔、企业行话、说明书式结构。
- 标题必须重写，长度与气质需符合知乎语境。

## 三、知乎平台节奏共性

### 1. 开头问题意识

- 前几段内必须抛出异常、规则、代价、疑问或冲突。
- 最好让读者感到“这章正在回答一个危险问题”。
- 第一人称或深限知质感优先；若原文不是第一人称，也要尽量保持近距离亲历感。
- 若题材允许，可借知乎社区基因使用“问答体 / 自述体 / 手记体”的入口感，但正文仍必须是小说，不得变成问答帖。
- 开头最好同时给出问题和现实支点：读者先知道“为什么要继续看”，再知道“为什么这件事会出事”。
- 问题感不是标题党，而是本章真正要推进的核心追问。

### 2. 逻辑链与证据感

- 读者应能追上“为什么这样做、看到了什么、接下来为什么这么推”的链条。
- 优先通过回执、时间线、口径差、流程话术、物证细节等建立可验证感。
- 讨论感必须来自剧情中的两难与漏洞，而非作者评注。
- 认知升级优先于能力炫示，重点是主角如何理解更危险的一层规则。
- 平台读者天然愿意参与判断，因此“公平给线索、克制给结论”比“作者直接解释明白”更重要。
- 逻辑链要让读者能回头复盘：看到什么、判断什么、因此怎么做，三步必须清楚。
- 现实细节越可验证，知乎感越强；不要把证据写成抽象氛围。

### 3. 阅读组织

- 采用短句 + 勤分段，保持手机端易读。
- 紧张段更短，思考段允许稍长，但每段必须有推进。
- 禁止大段概念解释与空转思辨。
- 知乎原生强项是短篇 / 中篇集约叙事；若源作品体量极长，平台化时应尽量做出“系列化准长篇”的读感：每章像在推进一个清楚的问题单元，同时又隐约供血更大的主线。
- 思考段也不能只是分析，必须落回人物判断或下一步行动。
- 如果一段只是在抒情或解释，读者会立刻跳出去；要把问题重新绑回现场。

### 4. 标题与作者有话说

- 标题更适合规则句、代价句、问题句、冲突句。
- `## 作者有话说` 可以带一点知乎式提问或现实余味，但不得写成运营话术或教程。
- 作者有话说宜短、克制、留问题，不替正文做结论。

## 四、知乎的表达原则

- 句子要清楚，推理要清楚，情绪要克制但不空。
- 高概念要有现实支点，不要只给抽象观念。
- 讨论感来自两难、漏洞、代价与认知升级，不来自作者点评。
- 如果原章偏网文爽点，知乎化时要把爽点翻译成“为什么这一步成立”。

### 5. 平台生态补充

- 知乎盐选 / 盐言的核心优势不是“超长篇注水追更”，而是“30 分钟内情绪浓度饱和”的强问题型内容。
- 对本仓库这类长线平台派生稿来说，知乎化的关键不是缩成短篇，而是改出：
  - 更强的问题意识
  - 更清楚的现实议题锚点
  - 更明显的章节内闭环
  - 更高的读者参与判断感

## 四、路径与输出结构

- 输出工作目录固定为 `知乎/`，不得包含 `小说正文/`。
- 若作品有分部，章节目录统一为 `知乎/第X部/第Y卷/`；若作品无分部，统一为 `知乎/第X卷/`。
- 输出文件只包含：章标题、正文、`## 作者有话说`。

## 统一9步编排母模板（平台优先，题材兜底）

> 本节为统一执行母模板；若与下方旧版“执行顺序”存在冲突，以本节为准。

1. 先锁定源稿不可变层：事实、事件链、证据链、角色关系与因果顺序不得改坏。
2. 读取对应题材的 `题材名-输出X版` Skill，确认题材入口与路由关系。
3. 再读取本 Skill 的 `references/平台共性执行细则.md` 与 `references/分节级补救映射与详细规则回填.md`，执行本平台特异强化。
4. 由题材包装层继续读取其题材 references，锁定题材边界、题材禁行项与题材专项补丁（作为兜底约束）。
5. 在“平台优先、题材兜底”前提下执行改写正文，确保平台化表达与源稿保真同时成立。
6. 最后才处理降相似度，不得为了降重牺牲平台读感与题材关键抓手。
7. 执行 POV 契约复核，确保与 `platformPovContract` 一致，不得出现未授权人称漂移。
8. 改写输出完成后，必须立即执行“输出后自检与修订”节的五项自检，不得跳过。
9. 若自检未通过，必须按结论回炉修订并复检，直至“自检通过/自检修订完成”后方可落盘。

## 六、迁移期说明

- 本文件承接知乎平台跨题材共性。
- 题材特有的现实议题、民俗规则、调查压力等，仍应继续读取各自题材 references。
- 若源 Prompt 仍处于迁移期，应将其视作细节补充来源，而不是被绕过的旧文件。
