# 通用-输出知乎版｜分节级补救映射与详细规则回填

## 来源与恢复等级

- 恢复等级：B（补救性回填）
- 主要来源：两题材现存薄 Prompt、试点映射台账、知乎盐选研究文件。

## 0. 平台任务定义与基准继承

- 任务目标：把章节改写成更适合知乎盐选 / 知乎读者语境的版本。
- 必须继承：`创建小说正文.prompt.md`、`正文润色.prompt.md`、`去AI味.prompt.md`。
- 继承重点：高概念入口、信息清楚、可验证质感、讨论张力与沉浸式阅读并存。

## 0.2 零新增事实

- 不新增设定、证据、人物、地点、时间点或因果。
- 可通过信息排序、限知视角、细节显影强化参与感。
- 所有精确字段、数值、证据编号必须原样保留。

## 0.3 沉浸式小说体裁

- 不得写成问答总结、观点帖、案例分析、教程或拆书稿。
- 保持纯小说结构：章标题、正文、`## 作者有话说`。

## 1. 平台理解

### 1.1 平台机制与读者画像

- 知乎读者更能接受“高概念切口 + 现实映射 + 可验证细节”。
- 读者期待更强的参与感：愿意跟着角色一起推、一起怀疑、一起做判断。
- 标题可更偏问题感、反差感、概念感，但不能失真。

### 1.2 平台风格门槛

- 开头应尽快给出一个值得继续读的核心问题。
- 逻辑链必须清晰，可让读者复盘。
- 现实细节、程序口径、证据碎片应更可信、更可讨论。
- 允许更强的“我想知道为什么”的阅读牵引。

## 2. 输入与输出

- 输出目录根：`知乎/`
- 目录映射：保持源 `{分部目录}/{分卷目录}` 结构。
- 文件名：`{chapterNo} {平台新标题}.md`

## 3. 输出内容结构

- 顶部标题：`# 第{chapterIndex}章：{平台新标题}`
- 正文：高概念入口 + 现实细节 + 逻辑推进
- `## 作者有话说`：中文，200–300 CJK，可轻提问或轻悬念，不得讲解主题

## 4. 关键硬约束

- 中文平台正文 CJK 必须在 3500–6500。
- 平台化时不得把小说改成“知乎体回答”。
- 平台差异应主要体现在概念入口、逻辑清晰度、参与感和讨论张力上。

## 5. 改写前推演问题

1. 本章最值得知乎读者点开的核心问题是什么？
2. 哪些证据/细节最适合做“可讨论点”？
3. 哪一处逻辑推进最应被放大？
4. 作者有话说是留问题，还是留余味？

## 6. 平台执行要点

- 优先强化“高概念入口 + 现实支点”。
- 让读者感觉自己能提前半步参与判断。
- 尽量减少纯抒情与过度模板化爽点说明。
- 保持限知、公平、可复盘。

## 7. 输出格式与检查

- 检查标题是否具有问题感或概念感。
- 检查正文是否保留逻辑清楚与可讨论性。
- 检查作者有话说是否克制、非说教。
