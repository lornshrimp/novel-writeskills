# 通用-输出知乎版

这是平台共性本体，负责承接“知乎版输出”的跨题材共性规则。

## 常见触发词 / 用户说法速查

- 输出知乎版
- 改成知乎能发的版本
- 做知乎盐选 / 专栏改写
- 帮我把这章改得更有逻辑链和讨论感

## 何时使用

- 把任意题材章节改写成更适合知乎读者阅读与讨论的版本。
- 需要强化逻辑可追、信息结构、观点感与节制钩子。
- 需要为题材包装层提供统一的知乎平台共性骨架。

## 不适用情形

- 只做题材边界补充，不做平台输出本体。
- 只做多平台编排、落盘和终检，而不处理知乎平台风格本身。

## 本层职责

- 维护知乎平台的跨题材共性规则。
- 统一承接知乎版输出的结构约束、标题倾向、讨论感与字数/风格门禁。
- 为 `题材名-输出知乎版` 提供稳定的通用平台骨架。

## 继续读取的 references

- `references/跨题材共性与迁移来源.md`
- `references/平台共性执行细则.md`
- `references/分节级补救映射与详细规则回填.md`
- `references/平台研究补充_知乎盐选都市悬疑.md`

## 默认执行顺序

1. 先读取对应题材的 `题材名-输出知乎版` Skill。
2. 先锁定零新增事实、体裁结构、工作目录、章节层级、字数门禁与标题同步规则。
3. 再执行知乎平台的高概念、现实映射、逻辑链、可验证质感与讨论张力强化。
4. 最后才处理降相似度，不得为了降重写成泛中文平台稿。

## 默认输出口径

- 默认输出一版可直接继续落盘或进入平台终检的知乎派生正文。
- 默认保留原章核心信息、逻辑链、可验证质感与讨论张力，不新增关键事实。

## 最低交付

- 有明确的逻辑可追性、讨论感与信息结构。
- 保住原章核心事件链、判断链与追读驱动。
- 读起来不像泛平台稿，也不像观点文硬套小说。

## 硬规则

- 命中本技能时，必须同时加载对应题材的 `题材名-输出知乎版` Skill。
- 题材特有规则不得回写到本文件中平行维护。
- 修改知乎平台共性规则时，应优先修改本 Skill，而不是多个题材入口。
- 平台派生正文默认落在本 Skill 的工作目录 `知乎/` 下。
- 章节级派生正文目录统一按小说结构决定：若作品有分部，则位于 `知乎/第X部/第Y卷/`；若作品无分部，则位于 `知乎/第X卷/`。
- 平台派生正文文件名默认不带日期，沿用既有章节号与平台标题同步规则；日期只用于配套审阅报告、书评等派生产物。
- 平台稿落盘后必须显式运行 `scripts/count-chapter.ps1 -MinCJK 3500 -MaxCJK 6500` 核正文，并运行 `scripts/count-afterword.ps1 -MinCJK 200 -MaxCJK 300` 核 `## 作者有话说`。
- 在无损迁移完成前，不得删除源 `输出知乎版.prompt.md` 中尚未完成映射的内容。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只负责知乎平台的跨题材共性骨架。
- 题材边界、题材禁行项与题材特有口径，继续由对应 `题材名-输出知乎版` 承接。
