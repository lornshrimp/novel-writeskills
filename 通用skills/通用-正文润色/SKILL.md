# 通用-正文润色

正文润色的目标不是单纯把句子写漂亮，而是把**已有章节**修到“更顺、更准、更抓人、但不失真”。

当题材 Skill 与本 Skill 组合调用时：

- `通用-正文润色` 负责共性流程、默认交付、弱项路由、句级动作与沉浸控制
- 题材 Skill 负责题材边界、项目特例与裁判源补充

## 默认对象

- 已有完整章节草稿
- 优先增强追读感
- 默认直接回写原文件
- 默认维护作者工作文件 / 源 Markdown 结构

## 共性执行顺序

1. 先确认范围、强度与不能动的钉子。
2. 先保信息、事实、因果、视角与结构功能。
3. 先修章首抓力，再修中段回报，再修章末钩子。
4. 再按弱项路由处理对话、动作、心理、肖像、景物、静态或场面。
5. 最后做句级精修、沉浸复检并直接回写。

## 弱项路由规则

- 对话发直、口气混、说明腔重 → 走“语言类”修法
- 动作发虚、推进靠概括 → 走“行动类”修法
- 心理发空、情绪只会直说 → 走“心理类”修法
- 人物脸模糊、神态无抓手 → 走“肖像类”修法
- 环境不立、氛围不入戏 → 走“景物类”修法
- 房间器物证物写死 → 走“静态类”修法
- 多人现场不热不紧、没有焦点 → 走“场面类”修法

这些文件属于共性描写技法库，只用于提炼观察角度、句法动作、感官组织方式与描写抓手，不得直接照抄其中现成段落。

当用户明确要求以下任一高强度口径时，再继续读取 `references/正文润色高级强化技法.md`：

- 按完整版口径精修
- 开头雷击感更强
- 中段更抓人、不发塌
- 章末钩子更致命
- 连载感、追读感、卷内驱动更强

## 默认交付口径

- 若用户说“润这章 / 精修正文”，默认按正式润色并回写处理。
- 若用户明确要求“先示范 / 先看方案 / 不写回文件”，才改为示例模式。
- 完成后应简述本次主要强化了哪几项：开头 / 中段 / 钩子 / 对话 / 动作 / 情绪 / 氛围。

## 通用硬规则

- 不新增关键事实，不删减核心信息。
- 不得把正文润成说明书、散文或空抒情。
- 不得把中段回报和章末钩子润没。
- 不得为了顺口删掉关键证据、编号、时间点、规则边界。
- 不得在不必要时整章推倒重写。
