# 正文润色总裁判与默认交付

## 裁判顺序

1. 不丢信息量，不改事实口径
2. 不破坏时间线、证据链、规则边界与人物关系
3. 先救章首抓力，再救中段回报，再钉硬章末钩子
4. 优先增强代入感，而不是堆辞藻
5. 优先让抽象判断落回镜头，而不是越修越悬空

## 默认目标

- 把已有草稿修到更能追读，而不是更像散文
- 把“信息都在但像说明书”的文本修成“信息都在且像正文”
- 让读者更容易感到推进、回报、危险与下一章动力

## 来源层与边界速判（新增）

- `结构职责层`：章首抓力、中段回报、章末钩子、主卖点、情感弧、承重 beat 是否仍成立。
- `场景兑现层`：动作链、信息显影、关系变化、规则代价、证据载体是否仍清楚。
- `表达提纯层`：句长、段长、感官、动作、停顿、声音差、移动端阅读体验是否提升。

默认顺序：先保 `结构职责层`，再保 `场景兑现层`，最后修 `表达提纯层`；不得反过来为了句面顺滑把前两层磨平。

## 默认交付

- 默认直接回写原文件
- 默认只给修改摘要，不在聊天中整章贴出
- 默认说明：改了哪一章、是整体替换还是局部替换、强化了哪些关键点

## 默认润色回执模板（新增）

```markdown
【本轮主要处理层】结构职责层 / 场景兑现层 / 表达提纯层
【保住的硬钉子】
【本轮强化点】章首 / 中段 / 章末 / 对话 / 动作 / 情绪 / 氛围
【放行判定】Green / Yellow / Red
【超出润色边界需回退 writer 的项】
```

## 默认不做的事

- 不凭空新增关键剧情与新设定
- 不把章节整体改造成另一种文风工程
- 不因为追求“高级感”而把人物声音与现实质地磨平

## Green / Yellow / Red 放行判定（新增）

- `Green`：更顺、更准、更抓人，且结构职责、场景兑现、事实口径都未受伤。
- `Yellow`：表达层改善明显，但局部仍需补一轮场景 / 钩子 / 回报复检。
- `Red`：润色后出现“更顺但更弱”“更漂亮但更不像这章”“更自然但更少事”等假改善，或触碰事实与结构边界。
