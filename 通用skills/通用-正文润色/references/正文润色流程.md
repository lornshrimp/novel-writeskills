# 正文润色流程

## 一、先判断润色等级

- 微调提纯：结构已成立，主要修句式、节奏、AI 味
- 局部重写：某场戏或某段功能明显偏弱
- 整章精修：功能点还在，但全章阅读体验偏弱

## 二、推荐顺序

1. 先看本章承接了什么、必须交付什么
2. 先修章首抓力
3. 再修中段回报
4. 再修章末钩子
5. 最后修句子、节奏、对话与镜头感

## 三、执行时的共性补强动作

- 若章首只有背景说明，优先改成“异常 / 冲突 / 情绪锚点”先入场
- 若中段只有搬运和解释，优先补一次明确回报：证据、证伪、关系变化、规则试错、代价兑现其一
- 若结尾只是总结，优先改成未完成动作、新变量入场或更糟后果
- 若文本太像说明书，优先把信息改成“动作 + 物件 / 界面 / 凭证 + 后果”

## 四、输出规范（共性）

- 默认直接回写原文件，而不是只在聊天里贴整章
- 默认只给修改摘要，不给冗长对照稿
- 应简要说明：更新了哪个文件、整体替换还是局部替换、强化了哪些关键点

## 五、最终复检

- 信息口径是否稳定
- 功能点是否更清晰
- 读者是否更容易感到推进、危险与回报
