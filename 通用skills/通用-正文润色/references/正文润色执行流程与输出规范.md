# 正文润色执行流程与输出规范

本文件沉淀正文润色任务的**共性执行口径**，适用于跨题材的章节精修、局部提纯、局部重写与发布前润色。

## 适用范围

- 已有完整章节草稿的整章精修
- 已有场景、段落的局部提纯
- 在不改核心剧情的前提下做节奏增强
- 与题材 Skill 组合调用时的通用流程底座

## 默认输入

若用户未特别说明，默认视为：

- 已有完整章节草稿
- 允许直接回写原文件
- 优先提升追读感，再提升语句质感
- 先保事实、结构、功能，再做文风处理

## 默认输出

- 直接回写目标文件，而不是只停在聊天建议
- 若用户明确要求“先示范一段 / 先看方案 / 不要回写”，则改为示例输出
- 任务结束时，应简要说明本次主要强化了什么：
  - 章首抓力
  - 中段回报
  - 章末钩子
  - 对话 / 动作 / 情绪 / 氛围 / 节奏

## 标准执行顺序

1. **确认范围与强度**
   - 整章 / 场景 / 段落
   - 轻润 / 中润 / 重润
   - 哪些信息、句子、称谓、编号不能动
2. **校准裁判源**
   - 人物关系
   - 时间顺序
   - 因果链
   - 证据、规则、程序、设定边界
3. **优先修追读链**
   - 章首是否尽快抛出异常、冲突、悬念或压迫
   - 中段是否存在实质推进、破绽、反击、发现、情绪回报
   - 章末是否留下具体钩子，而非空泛收束
4. **再修人物与场景承载**
   - 对话辨识度
   - 动作连续性
   - 心理贴身度
   - 场景气压与空间成立
5. **最后做句级精修**
   - 清理说明腔、模板腔、总结腔
   - 优化长短句节奏与段落呼吸
   - 统一叙事视角与口吻
6. **复检后回写**
   - 核对是否仍保有原章功能点
   - 检查是否出现误改、漏改、过润、跑题

## 整章处理要求

当任务属于整章正文润色时，优先遵守：

- 不要逐段孤立抛光，要按整章节奏看抓力、推进与钩子
- 不要只挑漂亮句，要优先处理最影响追读的 1–3 个核心问题
- 至少增强一项关键读者收益：抓力 / 回报 / 钩子 / 代入 / 张力

## 局部处理要求

当任务只涉及某几个段落时：

- 先判断这些段落在全章中的功能位置
- 不能把局部润顺后，反而与前后文口气脱节
- 若局部问题本质是结构问题，应明确指出，不要硬润

## 文件回写规范

- 默认维护作者工作文件 / 源 Markdown 的现有结构
- 若同文件中同时存在：
  - `章节正文`
  - `## 作者有话说`
  - `## 章节后记`
  - `## 润色说明`
  视为正常源稿结构，不应误判为污染
- 面向平台的净稿剥离，应在后续导出阶段处理，不在此流程中误删作者侧内容

## 交付摘要建议

完成正文润色后，对外说明优先写：

- 润了哪一章 / 哪一段
- 这次主要修了哪些问题
- 重点强化了哪些抓手
- 是否已直接回写原文件

## 红线

- 不得把共性正文润色变成只换同义词的表面修辞
- 不得跳过“追读链”直接堆华丽句子
- 不得为了输出报告而放弃默认回写
- 不得把源稿结构与读者发布稿结构混为一谈
