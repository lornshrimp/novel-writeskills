# 正文润色数值双轨禁忌与沉浸控制

本文件把正文润色中的“功能线”和“质感线”拆成双轨，避免只顾好看却把章节功能润坏。

## 双轨定义

### 功能线

指这章必须成立的硬内容：

- 事实
- 因果
- 证据
- 设定规则
- 时间顺序
- 程序阻力
- 人物关系与动机
- 章首 / 中段 / 章末的结构功能

### 质感线

指读起来的感受：

- 是否顺
- 是否抓人
- 是否有镜头感
- 是否有情绪张力
- 是否有节奏
- 是否去掉了说明腔、模板腔、AI 味

## 核心原则

先稳功能线，再优化质感线。

如果两者冲突：

- 宁可句子朴一点，也不要破坏事实
- 宁可保留一点硬度，也不要把证据链写糊
- 宁可少抒情，也不要把程序、编号、时间工程润没

## 常见禁忌

### 1. 为了顺口，删掉必要信息

错误倾向：

- 把证据字段删掉
- 把真实编号、省略号、时间点、称谓层级润没
- 把关键信息分布打散，导致读者无法拼链

### 2. 为了高级，增加作者腔

错误倾向：

- 无端加议论
- 无端加抽象感悟
- 无端加诗化景物
- 无端给角色“上帝视角”式点评

### 3. 为了去 AI 味，反而破坏清晰度

错误倾向：

- 把原本清楚的说明改得晦涩
- 把必要步骤写得过于隐晦
- 用太多花活替代基本可读性

### 4. 为了紧张，制造假冲突

错误倾向：

- 新增并不存在的误会
- 放大人物反应到失真
- 让角色突然降智，只为制造戏剧性

### 5. 为了节奏，切断逻辑

错误倾向：

- 段落切得很碎，但因果断裂
- 只剩情绪闪回，没有行动承接
- 省略必要过桥句，导致跳帧

## 沉浸控制

沉浸感通常来自“读者忘了作者在场”。

因此要控制：

- 少用作者替读者总结
- 少用“他觉得一切都怎样怎样”这种大包句
- 多让读者从动作、反应、物件、声音里自己感到压力
- 多让情绪通过身体和场景显影，而不是空口命名

## 适合保留的硬度

有些文本不能被润得太软：

- 证据链段落
- 程序推进段落
- 决策、对峙、交代任务段落
- 含编号、时间、术语、步骤的关键段落

这类段落的优化方向是：

- 清楚
- 准确
- 节奏更利落

而不是：

- 更华丽
- 更抒情
- 更散文化

## 自检问题

润完后至少问自己：

1. 这章最重要的功能点还在吗？
2. 有无因为润色而损失证据、规则、程序或时间信息？
3. 钩子是更具体了，还是更空了？
4. 对话是更像人了，还是更像作者在说话？
5. 有无把该硬的地方润虚？

## 一句话提醒

好润色不是“更像文学”，而是“更像这一章本来就该有的最好版本”。
