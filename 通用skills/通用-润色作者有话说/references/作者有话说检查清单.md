# 作者有话说检查清单

## 一、范围确认

- [ ] 只处理 `## 作者有话说`
- [ ] 不误改正文主体
- [ ] 不误改 `## 章节后记`

## 二、读者感

- [ ] 开头是否抓住了一个足够小、足够具体的细节？
- [ ] 口吻是否像在对读者说话，而不是在对同行解释？
- [ ] 是否尽量用贴近读者感受的表达，而不是一股脑说“我怎么设计的”？

## 三、轻量感

- [ ] 是否只围绕一个情绪方向，不东一锤西一棒？
- [ ] 是否有留白，而不是把该说的不该说的都说完？
- [ ] 是否避免了长篇讲道理？

## 四、互动感

- [ ] 结尾是否留下一个自然可留言的小问题或小岔口？
- [ ] 问题是否贴着剧情感受，而不是考试题式设问？

## 五、禁区复检

- [ ] 没有“本章完成了什么”这类复盘话术
- [ ] 没有设定讲义或参数说明
- [ ] 没有作者侧编号与工作流口径
- [ ] 没有直白预告下一章剧情
- [ ] 没有把作者有话说润成鸡汤、营销或自我感动

## 六、读者向成型检查

- [ ] 有余波 / 追问 / 微镜头 / 反讽中的至少一种
- [ ] 读起来像小说章后小剧场，而不是总结报告
- [ ] 与本章情绪或危险感相连，但没有简单重复正文
- [ ] 最后一句最好留一点余味或下一章冲动

## 七、长度检查

- [ ] 中文平台约 200–300 字
- [ ] 英文平台约 100–150 words
