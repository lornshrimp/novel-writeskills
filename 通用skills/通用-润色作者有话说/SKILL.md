# 通用-润色作者有话说

本 Skill 负责从作者侧执行口径润色 `## 作者有话说`：目标是把它修成读者向、小剧场式的轻声交流，而不是创作说明会、设定讲义或章节总结。

## 保真迁移要求（强制）

命中“作者有话说 / 后记前小剧场”任务时，除本文件外，**必须继续读取**：

- `references/作者有话说读者向规则.md`
- `references/作者有话说检查清单.md`
- `references/腾讯专栏作者有话说互动固粉增补.md`

## 来源层、判定门禁与模板化工件镜像（新增）

- 润色作者有话说时，默认先判三层：`余波素材层 / 读者交流层 / 互动固粉层`。
- `余波素材层` 负责本章到底留哪个小细节、小余波、小刺点；`读者交流层` 负责口吻是不是在和读者说话、有没有自然小问题；`互动固粉层` 负责是否形成轻互动、记忆点与作者存在感，而不是作者自我陈述。
- 默认补 `Green / Yellow / Red` 放行判定：`Green`=小细节、轻共鸣、轻追问都成立；`Yellow`=能读但仍像总结或轻复盘；`Red`=创作说明会、设定讲义、直白剧透、鸡汤式自嗨或完全没有读者向交流。
- 若一段作者有话说没有办法用“一句话说清这次和读者交换了什么”，说明它仍停留在作者工作台口吻，不宜直接放行。

## 何时使用

- 章节正文后的 `## 作者有话说` 已有草稿，但读起来像创作说明、章节总结、设定报告或大纲复述。
- 你想把作者有话说修得更像读者向小剧场：更近、更轻、更能引发留言。
- 你想保留一点情绪余波、轻互动、轻悬念，但避免剧透与说教。
- 你要统一整卷作者有话说的口吻，避免时而像编辑手记，时而像作者采访。

## 不适用情形

- 你要写的是 `## 章节后记` 或作者内部复盘。
- 你主要想改正文主体，而不是作者有话说。
- 你要做平台差异化输出整体统筹，而不只是改这段读者向内容。

## 常见触发词 / 用户说法速查

- 帮我润一下作者有话说
- 这段作者有话说太像复盘了
- 把后记前这段改得更像小剧场
- 别那么像创作说明会
- 给我一个更读者向的作者有话说

## 本 Skill 的核心任务

1. 用一个贴着正文的小细节，把读者重新拉回本章余波，而不是把整章再总结一遍。
2. 保留本章情绪、代价、悬念或刺点，并给读者一点贴着本章余波的情绪共鸣，例如后怕、心软、想笑、想骂或替角色憋气中的一种，但不大剧透。
3. 在最后留一个开放式轻追问、轻岔口或轻余味；这里的“轻追问”只指能引发留言或代入的小问题，不要求标准答案，也不直接剧透下一章。

## 执行顺序

固定优先级：先确认只处理 `## 作者有话说` 这一段，再判断当前病灶类型，再锁定小细节、情绪共鸣与轻追问，最后才复检字数门禁。

1. 先确认目标只在 `## 作者有话说` 段，不误伤正文与 `## 章节后记`。
2. 判断当前问题属于哪类：
   - 太像创作说明
   - 太像总结陈词
   - 太满、太硬、太会讲道理
   - 太剧透
3. 判断本章最适合留给读者的是什么：
   - 一个小细节
   - 情绪余波
   - 道德选择
   - 反常细节
   - 下一步危险感
4. 删除或改写以下内容：
   - 创作说明
   - 写法自夸
   - 设定讲义
   - 明示下一章计划
   - 作者侧编号与工作流口径
5. 输出成一段读者向短小剧场：
   - 尽量用贴近读者的口气说话，可用“你”，也可用更轻的第二人称省略口气
   - 最多只留一丢丢创作花絮
   - 必须有一点贴着本章余波的情绪共鸣，例如后怕、心软、想笑、想骂或替角色憋气中的一种
   - 最后留一个轻追问、小岔口或小余味
   - 这里的“轻追问”只指能引发留言或代入的开放式小问题，不要求标准答案，也不直接剧透下一章
6. 复检：是否仍像小说的一部分，而不是作者开会；是否没有把暗线、主题判断或“所有制 / 阶级 / 分配权”这类核心词直接说死。
7. 回写前必须显式运行作者有话说字数门禁脚本：中文用 `scripts/count-afterword.ps1 -MinCJK 200 -MaxCJK 300`；英文用 `scripts/count-afterword-words.ps1 -MinWords 100 -MaxWords 150`。

## 默认输出口径

- 默认只修改 `## 作者有话说` 小节，不改正文，不改 `## 章节后记`。
- 若原文没有 `## 作者有话说`，只有在用户明确要求时才新增。
- 中文平台默认控制在 **200–300 字**；英文平台默认 **100–150 words**。
- 默认尽量从一个具体小细节起笔，并用读者向口气收成一个开放式轻追问；这里的“轻追问”默认指可留言、可代入，但不要求读者回答标准答案的小问题。
- 默认直接回写该段，而不是另外写分析报告。
- 默认同时给出：本轮主要问题层、`Green / Yellow / Red` 判定，以及若仍不过线时该先补哪一层。

## 最低交付

一次合格的作者有话说润色，至少要做到：

- 删除创作术语、结构复盘、大纲说明和显眼剧透。
- 留住一个贴着本章的小细节或小余波，而不是空泛总结。
- 让口吻像在和读者说话，而不是在向同行讲写法。
- 最终只保留“一丢丢创作花絮 + 一点贴着本章余波的情绪共鸣 + 一个开放式轻追问”，而不是扩成小论文。
- 结尾留下一个轻追问、轻岔口或轻余味，能自然引发留言或下一章冲动。

## 硬规则

执行时只按以下三组顺序检查：先守内容禁区，再守口吻边界，最后守格式门禁；前一组未过，不进入后一组。

- 先守内容禁区：
   - 不得把作者有话说写成章节总结。
   - 不得复述“本章完成了什么功能点”。
   - 不得用“本章 / 这一章 / 我想表达的是”领起整段，并把作者有话说写成创作答辩。
   - 不得直接预告下一章具体剧情。
   - 不得堆世界观术语、机制名、编号、参数。
   - 不得把“所有制”“阶级”“分配权”等核心思想或暗线结论直接说死；只能借问题和生活小场景轻轻带到边缘。
- 再守口吻边界：
   - 不得把口吻润成鸡汤文、营销文或编辑荐语。
- 最后守格式门禁：
   - 不得用手感替代字数门禁；回写前必须显式跑作者有话说对应脚本并确认达标。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只管 `## 作者有话说` 这段读者向内容。
- 不负责章节正文整体润色。
- 不负责后记工作字段与作者侧登记。
