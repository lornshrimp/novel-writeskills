# 通用-执行微空间受限场景

微空间不是背景，而是冲突发动器。它的价值不在“小地方”，而在“熟悉、受限、贴脸、后果立即”。

这是**专项子 Skill**，不是独立替代 `通用-执行场景单元` 的平行入口。命中本 Skill 时，应理解为：先用 `通用-执行场景单元` 搭场景骨架，再用本 Skill 给微空间加专项工法。

## 必须先加载的母 Skill

命中微空间 / 受限场景任务时，**必须先加载并使用**：

- `通用-执行场景单元`

其中：

- `通用-执行场景单元` 负责场景目标、阻力、动作链、证据链、状态变化与钩子骨架
- 本 Skill 负责空间常态、微反常升级、动作失手、即时后果、现实针脚与专项复检

其中，以下“场景通用骨架”现已上收至母 Skill，不再由本子 Skill 平行维护通用本体：

- 场景单元施工卡模板
- 场景动作后果速配表
- 场景动作失手升级清单
- 场景去模板腔清单

## 常见触发词 / 用户说法速查

- 楼道这场怎么写
- 电梯场景怎么更吓人
- 医院走廊这段太普通了
- 这个值班室 / 地下车库 / 出租屋怎么做细思极恐
- 把日常空间写异化

## 何时使用

- 任务聚焦于一个**具体空间场景**。
- 需要把现实功能感、压迫感、人物动作与后果升级咬在一起。
- 需要把“日常中的微反常”写成可执行的场景，而不是只停在氛围词上。

## 不适用情形

以下情况不应直接只用本 Skill：

- 任务还停在通用场景搭建阶段，尚未明确具体空间类型
- 场景不是微空间 / 受限空间，而是开放空间、群像大场面或跨空间调度场景
- 需求只是通用的“把场景写成可执行单元”，而不是“把微空间写得贴脸、受限、可复盘”

## 本 Skill 的核心任务

1. 先确定空间常态是什么。
2. 再放入一个足够小但足够反常的细节。
3. 让角色必须做选择，而不是只站着感到害怕。
4. 让反常最终能接回证据链、规则链或后续冲突，而不是只做气氛摆设。

## 执行顺序

1. 先加载 `通用-执行场景单元`，搭好本场目标、阻力、动作链、证据链与钩子骨架。
2. 再读取场景上下文，明确此处的现实功能。
3. 先定义空间常态与第一处微反常。
4. 按升级梯子确认异常不是一步到顶，而是沿 `可忽略 → 可怀疑 → 可确认 → 后果落地` 升级。
5. 增加角色动作和现场反馈，让惊悚通过“动作 → 后果”成立。
6. 若动作失手，也要让失手本身成为升级器。
7. 若需要从零设计整场，先填施工卡模板。
8. 写完后，用执行检查表与去模板腔清单复检。

## 与其他 Skill / Prompt 的边界

- 本 Skill 是场景技法模块，不替代整章控制卡或整章正文 prompt。
- 本 Skill 不替代 `通用-执行场景单元` 的通用场景骨架职责。
- 若场景还需要题材素材库、句库、空间字典与感官工具箱，应由题材包装层继续补充。
