# 微反常升级梯子

## 一、使用目的

微空间最怕两种死法：

- 第一下太轻，像没事发生
- 第一下太重，直接脱离现实

目标是让反常沿着**可相信 → 可怀疑 → 可确认 → 可后果化**的路径升级，而不是一脚踩穿。

## 二、四级升级路径

### L1：可忽略异常

读者和角色第一反应都可能把它当成普通问题。

要求：

- 必须足够小
- 必须贴近日常
- 必须能逼出第一个动作

### L2：可怀疑异常

角色开始怀疑“这不只是普通故障”。

推荐动作：

- 回头再看
- 贴近检查
- 重放 / 对照
- 再按一次
- 去确认来源

### L3：可确认异常

角色已经不能再拿“故障”搪塞自己，必须判断接下来怎么办。

这一层必须给出至少一个：

- 证据载体
- 误判代价
- 新风险入口

### L4：后果落地

异常真正改变局势，不再只是氛围。

常见落点：

- 人物误判了对象或方向
- 关键证据提前入场
- 角色暴露了自己
- 规则边界第一次显影
- 下一场冲突被直接触发

## 三、标准节拍

推荐使用：

`L1 微反常 → 动作确认 → L2 再次冲突 → 动作升级 → L3 异常坐实 → L4 后果落地`

短场景至少走到 `L3`。

## 四、红线

- 不要一上来就显形到脱离现实
- 不要第一处异常就大到没人会忽略
- 不要升级很多次，却没有一次真实后果
- 不要升级只靠形容词，必须靠动作验证失败完成

## 五、验收口径

一场合格的微空间场景，反常升级后应满足：

- 读者能跟上角色的怀疑过程
- 角色每次升级都有动作依据
- 最后至少换来一个可复盘后果
- 压迫不是平白变大，而是一步步失控
