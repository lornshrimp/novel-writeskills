# 微空间场景模式

## 一、常见微空间类型

- 楼道 / 楼梯间
- 电梯 / 电梯前室
- 值班室 / 保安亭 / 前台后区
- 地下车库 / 设备间
- 出租屋 / 合租公共区
- 医院走廊 / 候诊区 / 配药口
- 办公室后区 / 茶水间 / 机房

也可按功能分成四类：

- **通勤过渡空间**：楼道、电梯、车库、地铁通道、连廊
- **居住空间**：出租屋、合租公共区、卫生间门口、厨房、阳台
- **职业空间**：值班室、保安亭、机房、档案室、殡仪后区、检验间
- **半公共空间**：候诊区、配药口、便利店后门、社区公告栏前、物业前台后区

## 二、推荐结构

### 1. 空间常态

先让读者知道这个地方平时是干什么的。

最少要回答：

- 谁通常会来这里
- 这里平时最常见的动作是什么
- 这里平时最应该稳定存在的秩序是什么

### 2. 微反常

不要一上来就大怪相，优先用：

- 灯亮错格
- 门未合严
- 监控角度不对
- 门禁提示音延迟
- 气味不该出现却出现
- 某件物品位置被轻微改动

也可按空间功能选择第一处异常：

- **通勤空间**：灯亮错格、监控角度偏了、出口门回弹异常、鞋印方向不对
- **居住空间**：锅里有热气却没人、拖鞋潮了、冰箱压缩机停了却有嗡鸣、门锁反拧过
- **职业空间**：登记表多出一行字、门禁记录错时、值班交接没对上、工具回位顺序被动过
- **半公共空间**：叫号停在某个数字不动、公告栏纸角潮湿、配药口玻璃有呼吸雾、塑料凳脚印方向反了

### 3. 人物动作

让角色必须：

- 走近 / 伸手 / 回头 / 检查 / 退出 / 停住

没有动作，压迫就只剩氛围词。

若场景要更贴脸，优先选择带验证感的动作：

- 再按一次
- 截图 / 重放 / 对照
- 用手机补光
- 闻一下 / 摸一下 / 试探性挪动
- 改走另一条动线

### 4. 后果

反常必须立刻带来一个后果：

- 误判
- 信息缺口
- 证据入场
- 风险升级

还可继续细分为：

- **流程后果**：错过窗口、交接失败、门禁锁死、线路被截断
- **关系后果**：把同伴当成异常、把异常当成同伴、互信被打断
- **规则后果**：第一次意识到这里有禁行项、禁时项、禁触项

## 三、通用模式模板

### 模式 A：日常工作流被轻微打断

`值班 / 巡楼 / 配药 / 回家 / 取件 → 某个流程环节不对劲 → 角色复核 → 发现异常不是故障 → 本场留下证据或代价`

### 模式 B：熟悉空间里的错误重复

`角色第二次经过同一处 → 发现物件、数字、声响或气味重复得不对 → 角色对照确认 → 异常坐实 → 引出新风险`

### 模式 C：先确认常态，再让常态失灵

`先写这里平时如何稳定运转 → 写第一处轻微偏差 → 角色尝试恢复常态 → 失败反而暴露更大的问题`

### 模式 D：职业流程压住恐惧，再被现实反噬

`主角用专业流程压场 → 暂时拿回掌控 → 新细节证明流程只压住表层 → 本场抬出更深问题`
