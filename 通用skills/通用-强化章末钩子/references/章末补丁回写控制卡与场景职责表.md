# 章末补丁回写控制卡与场景职责表

适用于章末钩子强化后，判断问题到底只在最后一句，还是已经暴露出中段回报、最后一场职责或下章接口设计不稳。

## 一、先明确：章末钩子不总是结尾句问题

很多结尾不够翻页，不是因为最后一句不狠，而是因为上游没钉死：

- 哪一场负责最后升级
- 哪一场负责章末钩子
- 本章中段回报是否先给足
- 下一章到底接什么

这些没定清，结尾再修也容易像“临门补针”。

## 二、什么时候只修章末即可

满足以下大多数情况时，可视为局部补丁：

- 本章已有明确的最后一场职责
- 中段回报已经成立，只是最后落针不够尖
- 下一章接口本来就清楚，只是结尾表现偏虚
- 修最后 150–300 字后，不需要重新分配前面场景任务

## 三、什么时候必须回写上游

出现以下任两项，建议回写控制卡或场景职责：

### 1. 不知道最后一场为什么存在

- 只能说“最后收一下”
- 却说不清它负责钩子、升级还是回响

### 2. 章末必须靠新增感慨才有劲

- 说明本章前面没有把局面推到足够糟

### 3. 结尾不尖，是因为本章中段没给足回报

- 读者中段没拿到东西，结尾很难自然冲高

### 4. 结尾问出了问题，但下一章接棒对象说不清

- 说明不是章末句的问题，而是章节接口没钉死

### 5. 章末要靠偷塞新事实才有翻页感

- 这通常意味着最后升级场职责设计失败

## 四、回写优先级

### 第一优先：回写场景职责

先问：

- 最后一场是否明确承担章末钩子
- 倒数第二场是否承担升级 / 失手 / 反噬
- 最后两场之间有没有足够的移交关系

### 第二优先：回写章节控制卡

若场景职责也不稳，再回写：

- 中段回报
- 章末钩子类型与对象
- 下章接棒对象
- 本章对主卖点的关系

## 四点五、来源层映射（新增）

- `控制卡层`：中段回报、章末钩子类型与对象、下章接棒对象、本章与主卖点关系。
- `最后两场职责层`：倒数第二场升级职责、最后一场钩子职责、两场移交关系。
- `章末补丁层`：最后一句落针、最后一段危险载体、具体节拍与截断位置。

默认回写顺序：先 `最后两场职责层`，再 `控制卡层`，最后才继续钉 `章末补丁层`。

## 五、最小回写表

| 章末暴露的问题 | 优先回写层级 | 该补哪项 |
| --- | --- | --- |
| 结尾只有情绪，没有翻页动作 | 场景职责 | 最后一场职责改为钩子场 |
| 结尾不尖，是因为前面没把局面推坏 | 场景职责 | 倒数第二场升级职责 |
| 下一章承接什么说不清 | 控制卡 | 下章接棒对象 |
| 章末只能靠新信息硬顶 | 控制卡 | 章末钩子类型 / 中段回报 |
| 结尾和本章卖点脱节 | 控制卡 | 本章与主卖点关系 |

## 六、章末强化后的三问复检

1. 最后一场现在能否一句话说清自己在收什么钩子？
2. 这一针之所以尖，是来自本章前面已经铺出的局面，还是纯靠最后一句硬抬？
3. 不改上文，只改结尾，下一章接口还能不能清楚成立？

若第 2 题偏后者，优先回写上游层级，而不是继续加狠话。

## 六点五、Green / Yellow / Red 回写判定（新增）

- `Green`：问题主要在章末补丁层，局部修就够。
- `Yellow`：章末补丁层之外，已轻度暴露最后两场职责或下章接口问题。
- `Red`：控制卡层与最后两场职责层至少一层明显失效，继续只修结尾只会越补越假。

## 六点六、最小输出模板（新增）

```markdown
【章末判定】Green / Yellow / Red
【主要失效层】控制卡层 / 最后两场职责层 / 章末补丁层
【先回写哪一层】
【若继续只修章末会发生什么】
```

## 七、一句话收口

**章末强化只负责“把人推出去”，但如果前面没有足够的升级、回报和接口设计，那就必须回写控制卡与最后两场的职责分配。**
