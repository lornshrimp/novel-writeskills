# 章末钩子强化规则

## 一、结尾的目标不是“收好”，而是“推下一章”

一个有效的章末，不一定要最漂亮，但要让读者自然产生：

- 还有事没完
- 我得知道接下来会怎样
- 刚才这一下比我以为的更危险

## 二、优先落点

优先把结尾落在以下四类之一：

### 1. 新缺口

本章回答了一个问题，但立刻打开一个更尖的问题。

### 2. 新代价

角色刚拿到一点东西，却发现代价更具体、更近。

### 3. 未完成动作

让人物停在“马上要做 / 不得不做 / 来不及不做”的动作上。

### 4. 更危险的新解释

同一信息突然换了一个更坏的解释方向。

## 二点五、来源层速判（新增）

- `升级蓄势层`：倒数第二场是否真的把局面推坏、推深、推到更贵的位置。
- `钩子落针层`：最后 `150–300` 字是否落在新缺口、新代价、未完成动作或新解释上。
- `下章接口层`：下一章接谁、接什么、接哪种压力是否清楚。

默认顺序：先判 `升级蓄势层`，再判 `钩子落针层`，最后判 `下章接口层`；不要只看最后一句狠不狠。

## 三、常见失败写法

- 空感慨
- 主题升华
- 解释句收尾
- 作者替读者总结
- 把危险说虚，不落到具体东西上
- 结尾只是总结
- 有危险感但太空
- 有事，但不够尖

## 四、优先改法

- 用字段、截图、话术、来电、动作、门响、脚步、口径差收尾
- 用短句把最后一下砸实
- 留一个缺口，不把意思说满
- 让新信息带来代价、时限或选择压力
- 让它立刻改变下一章的走法

## 五、推荐公式

1. 本章已有结果刚刚落地。
2. 更糟后果紧跟着出现。
3. 留一个未完成动作或新变量。
4. 收在“下一步必须做”的压力上。

## 五点二、章末补丁最小工件（新增）

```markdown
【本章最后一场职责】
【倒数第二场负责什么升级】
【本章章末钩子类型】
【下一章接棒对象】
【最后一下的具体载体】动作 / 物件 / 口径差 / 来电 / 门响 / 截图 / 时间窗
```

## 六、回响补丁

强钩子之外，章末还可以补一个**回响点**，让读者不只想翻页，还能记住这一章最后的情绪：

- 一句回扣前文的小动作 / 小话语
- 一处更冷、更静、更准的落针
- 一个让“真好”或“糟了”瞬间成立的最后印象

注意：

- 回响不是抒情拖尾。
- 回响要服务下一章驱动，而不是把张力泄掉。

## 七、牺牲与代价

当章末承担重大选择时，优先让读者看见：

- 主角若继续追，得牺牲什么
- 主角若停下，又会失去什么

能让钩子更疼的，往往不是“更大的谜”，而是“更具体的失去”。

## 八、禁止事项

- 不要写“下一章会……”
- 不要明确剧透答案
- 不要凭空加新设定
- 不要把结尾改成后记口气
- 不要空喊危险而没有载体

## 九、验收口径

章末强化完成后，应满足：

- 更具体
- 更危险
- 更想翻页
- 但不自曝答案、不抢掉下一章的入场空间

## 九点五、Green / Yellow / Red 判定门禁（新增）

- `Green`：前面已经铺够，最后一下具体、危险、可接棒，且不是靠新事实硬顶。
- `Yellow`：结尾已有翻页感，但下一章接口或钩子类型仍偏虚，可能需要补一轮回写判断。
- `Red`：只能靠感慨、狠话、偷塞新事实或主题总结收尾；或者倒数第二场根本没把局面推到足够糟。

## 十、三段式标准格式（施工公式）

> 来源：`修订体系.md` §C批次·章末钩子强化。

```text
[最后的动作/对话] + [环境异常/未闭合的信息] + [暗示下一章的方向]
```

示例：

```text
他推开门。
屋里没有人。
但桌上的茶水——还是温的。
```

说明：

- 第一段：落在一个具体动作或台词上，不是总结性句子。
- 第二段：立即给出一个违背预期的细节（异常/矛盾/新信息）。
- 第三段（可选）：让这个异常指向下一章的压力方向，但不说破答案。

## 十一、钩子强度三级评级

> 来源：`范文参照.md` §3.2 章尾钩子分析。

| 强度 | 判断标准 | 举例 |
| --- | --- | --- |
| **弱** | 章尾平淡收束，无未闭合问题 | "他关灯睡了" |
| **中** | 有疑问但紧迫感不强 | "明天还有一件事要处理" |
| **强** | 读者必须知道接下来发生了什么 | "身后的门慢慢打开了" |

审稿时对每一章末尾标注强度，**后 200 字钩子强度目标：≥90% 达到"中"，≥50% 达到"强"**。
