# 章末钩子检查清单

## 一、范围

- [ ] 只处理最后一段到最后约 150–300 字
- [ ] 没误伤上文主体推进

## 二、具体性

- [ ] 结尾是否落到了具体动作、具体后果或具体新变量？
- [ ] 危险是否有可指认载体，而不是空泛形容？

## 三、驱动性

- [ ] 读者是否能明确感觉到下一章必须承接什么？
- [ ] 这一针是否比本章中段更危险或更尖？
- [ ] 读者是否会自然追问“接下来呢”？

## 三点五、旧悬念 / 新悬念测试

- [ ] 本章是否至少回应了一个旧问题、旧债或旧悬念？
- [ ] 本章结尾是否同时打开了一个更贵的新问题，而不是只有抽象危险感？
- [ ] 当前钩子是小悬念 / 中悬念 / 大悬念中的哪一档，对应回收窗口是否合理？

## 四、自然性

- [ ] 钩子是否顺着本章既有信息长出来，而不是硬插进去？
- [ ] 是否保持了人物动机、视角和口径的一致？

## 五、节奏

- [ ] 没有空泛感慨收尾
- [ ] 没有解释性总结收尾
- [ ] 最后一句有落针感，而不是松掉

## 五点五、钩子链节律

- [ ] 最近 3 章钩子类型是否有轮换，而不是连续重复同一招
- [ ] 若本章选择延迟响应上章钩子，是否仍在可接受窗口内（通常不宜超过 3 章）
- [ ] 本章结尾是否把“倒数第二个事件”当作更好的截断点，而不是把所有结果都交代完

## 六、红线

- [ ] 没有直接剧透下一章
- [ ] 没有用“下一章会 / 后面会 / 我们会看到”之类预告腔冒充钩子
- [ ] 没有新增关键事实
- [ ] 没有把后记语气混进正文结尾
- [ ] 没有把控制卡 / 后记里的“下章接棒点”原样翻进正文

## 七、回写判断

- [ ] 若结尾仍不尖，是否已判断问题出在最后一句，还是出在最后一场职责不清？
- [ ] 若章末必须靠新事实才能成立，是否已判断要不要回写控制卡中的中段回报或章末钩子类型？
- [ ] 若下一章接口说不清，是否已判断要不要回写“下章接棒对象”与最后两场职责分配？
