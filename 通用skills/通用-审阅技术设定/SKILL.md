# 通用-审阅技术设定

审阅技术设定，重点不是“懂不懂技术”，而是“能不能进故事、能不能留下痕迹、会不会变万能”。

## 保真迁移要求（强制）

命中技术设定审阅任务时，除本文件外，**必须继续读取**：

- `references/技术设定审阅检查清单.md`
- `references/技术设定审阅报告增补.md`
- `references/审阅维度说明.md`
- `references/设定审阅报告模板.md`
- `references/腾讯专栏技术设定可证据化与边界风险增补.md`

## 完整模板执行要求（强制）

- 产出设定审阅报告时，必须以 `references/设定审阅报告模板.md` 作为主骨架，不得退化成“几条规则点评 + 少量建议”的简版报告。
- 报告至少要完整覆盖：阅读链路确认、基础信息检查、写作落地卡片、设定用途一句话、首次出场与前三章验证、人物—情境—观点合拍度、规则条款清单、现实可行性与程序合理性检验、网文长线与规则演进复核、边界/代价/漏洞、可证据化映射、程序摩擦与现实阻力、误导与证伪窗口、现实痛点与情绪共鸣牵引、跨文档一致性总检、社会面/舆情面/合规预警、与人物/组织对齐、叙事落地建议、问题分级、评分、具体修改方案、冲击 9.2+ 综合评分的优化思路与总体结论。
- 对承担核心能力、主线程序链、关键证据载体或终局关门功能的设定，报告必须额外显式审：**一句话口径是否为动作→凭证→后果、触发条件 / 标准表现 / 失效规则是否完整、关键参数是否区分固定值与叙事弹性、最小调用流程卡是否成立、分级代价与程序痕迹是否可见、是否明确写出“不能直接解决的问题”、是否需要本轮修订 / 终局关门 / 残留余波块**。
- 若当前任务还加载了题材 Skill，则必须在通用骨架的占位章节中继续补写题材专项段、题材专项评分项与题材化结论重点。
- 报告中必须至少产出 `1–3` 张“写作落地卡片”，让作者能把规则直接拷回大纲、章节执行表或正文镜头。

## 网文长线与规则演进（强制）

- 审技术设定时，必须把它视为“能否撑住开篇抓力、中盘升级、终局回收”的长线引擎，而不是一次性炫技说明书。
- 必须显式判断：第一次出手怎么抓人、中盘如何升级、终局会产生什么更贵后果、代价是否能被正文真实感知、是否存在读者可讨论的误导点与证伪窗口。
- 若设定承担强能力或高概念机制，还必须显式判断：它的**最小可调用版本**是什么、第一次越界会留下什么程序痕迹、终局是否必须关门 / 降格 / 留余波。
- 若设定开篇就用尽最强功能、代价只存在文档里、或误导与证伪全靠作者解释，不得判为可用高分设定。

## 网文首秀与读者供血（强制）

- 审技术设定时，必须判断它第一次出手是否能在前三章内咬到现实困局、留下可回指痕迹，并带来一次有效小回报、误判或代价。
- 必须显式判断：它是否绑定了最能凸显价值的代表性人物与熟悉日常情境；设定主要供血类型是什么（智性破局、恐惧打穿、制度反击、关系翻车、传播讨论、反差幽默）。
- 若设定首秀只能靠讲解、和人物 / 场景脱节，或没有可截图 / 可站队 / 可催更的钩子，不得判为网文高完成度设定。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，审阅前必须先读取并纳入对照。
- 审阅报告不得只给内部自洽结论；必须显式判断当前设定相对竞对在哪些维度已领先、持平或落后，以及最先该补哪条超越路径。
- 若当前设定在竞对已验证的关键强点上明显更弱，必须直判为竞争风险；设定审阅默认按“严酷找差距”口径执行，不得为了安抚作者而把规则空转、代价失真或首秀发虚轻描淡写带过。
- 竞对维度在设定审阅综合评分中默认按 `25%` 计权；若未完成量化差距卡，或关键维度仍显著落后且没有 `72` 小时补丁，综合评分默认不得给高档。

## 常见触发词 / 用户说法速查

- 帮我审这份技术设定
- 看看这套规则是不是会万能
- 这份设定能不能进正文
- 证据化和程序摩擦够不够
- 给我设定审阅报告
- 这条规则边界是不是没写清

## 何时使用

- 审阅某个技术设定 / 手法 / 规则文档。
- 检查规则是否能拆条款、是否有边界、代价、漏洞与反制。
- 检查设定是否能落到日志 / 回执 / 编号 / 时间戳等载体。

## 报告落盘位置与命名规则（强制）

- 设定审阅报告必须落在项目根下的 `审阅意见/设定审阅报告/` 子目录中，不得散落在设定源文件目录或 `审阅意见/` 根目录。
- 设定审阅报告文件名必须带日期，日期格式固定为 `YYYY-MM-DD`。
- 推荐完整相对路径写法为 `审阅意见/设定审阅报告/设定审阅报告_[设定名]_2026-04-15.md`。
- 技术设定、手法、程序链、证据载体或规则文档本体默认不带日期；日期只用于对应审阅报告。

## 默认执行顺序

1. 确认目标设定文件与关联剧情位置。
2. 先用检查清单做设定体检。
3. 再用审阅维度说明统一评分重心，并判断长线演进、误导—证伪与社会面后果。
4. 再按报告增补与报告模板输出可回写的审阅报告。

## 默认输出口径

- 默认输出一份带规则体检、风险分级与最小回写建议的设定审阅报告。
- 默认不直接改设定本体，除非用户明确要求边审边回写。
- 报告必须同时给出“作者向依据”和“正文镜头化改写抓手”，前者用于保证自洽，后者用于保证可写；不得只给说明书式建议。
- 报告必须同时回答“哪里失血”“最小怎么补”“补完后为何更抓人、更想追”，而不是只回答设定对不对。

## 最低交付

- 能指出规则条款、边界、代价、漏洞、反制与证据化问题。
- 能指出哪些问题会直接影响正文可写性。
- 能给出最小可回写补丁，而不是只说“设定还不够细”。
- 能产出至少 1 条可直接写回设定文档的规则级改写、至少 1 条可直接写进正文的镜头级改写，以及至少 1 条针对首秀 / 前三章回报的补丁。

## 复审独立性规则（强制）

- 每一轮设定审阅都必须把当前设定版本当作新的完整对象重新审查，不得仅因上一轮问题已修复，就把本轮判为高分或默认通过。
- 检查上一轮问题修复情况只是复审入口，不是复审主体；仍需独立重查规则条款、边界、代价、漏洞、反制、证据化、程序摩擦与镜头化落地。
- 若本轮发现新的万能化风险、新的规则冲突、新的证据链断裂或新的落地困难，必须如实记入本轮报告，不得因为“旧问题修好了”而跳过新问题。
- 本轮评分必须依据当前版本在本轮完整审阅下的实际质量给出，不得按修复率、修改投入或上一轮评分惯性顺延上调。

## 审阅重点

- 规则是否能拆条款
- 是否有边界、代价、漏洞、反制
- 是否能可证据化
- 是否具有程序摩擦和现实阻力
- 是否能写成镜头，而不是只能讲解
- 是否存在足够硬的现实痛点 / 情绪牵引 / 讨论度，而不只是“规则写得很全”
- 是否真的支持长线升级，而不是开篇一亮相就把底牌打完

## 硬规则

- 禁止只评价“设定是否酷 / 是否专业”。
- 必须落到条款、边界、代价、证据化与回写动作。
- 必须说明设定第一次怎样进正文、最危险的万能点在哪里、最贵的后果会落到谁身上。
- 必须说明设定是否已经写清“不能直接解决的问题”、哪些参数绝不能漂、哪些红线一旦越过就该进入终局 / 关门逻辑。
- 必须说明设定前三章为什么值得读者继续跟，不能只给“后面会很厉害”的延期承诺。
- 禁止只给作者侧法条 / 资料依据，不给正文镜头、回执、对话或动作化落地方案。

## 与其他 Skill / Prompt 的边界

- 本 Skill 负责技术设定审阅与回写建议，不直接代替设定重写。
- 若用户要直接设计或补强技术设定，应优先使用 `通用-设计技术设定`。
