# 技术设定审阅报告增补

## 报告里必须写重的部分

### 1. 写作落地卡片

不能只说“设定很完整”，必须至少产出 `1–3` 张作者可直接拿去写场景的落地卡片。每张卡片至少要写清：

- 适用场景
- 触发条件
- 可视化回执
- 对手常用口径
- 主角反制抓手
- 反噬与代价
- 升级按钮

### 2. 叙事最小用例

不能只说“设定很完整”，必须指出：

- 它最适合在哪类场景里出手
- 一次出手会改变什么

### 3. 最危险的万能点

每份报告最好明确指出：

- 这个设定最可能在哪个位置变成万能按钮
- 应该补哪条边界规则堵住

### 4. 证据化是否足够硬

优先评价：

- 有没有硬痕迹
- 痕迹是否容易拿到
- 拿到痕迹是否就能立刻定性（如果能，通常太强）

### 5. 最小回写建议

推荐句式：

- 在【规则条款】补写【失败条件】，防止设定万能化。
- 在【证据映射】增加【痕迹载体】，让设定能被看见。
- 在【程序摩擦】增加【审批 / 权限 / 窗口期】，让世界更可信。
- 把【抽象说明】改写为【镜头包】，让作者能直接调用。

### 6. 代价是否真能被看到

报告里最好明确指出：

- 代价会以什么形式被人物真实承受
- 读者最早会在第几个关键场景里感到这个代价

### 6.1 参数、阈值与最小流程卡

报告里最好明确指出：

- 这份设定有没有把关键参数区分成固定值与叙事弹性
- 是否已经给出最小调用流程卡，还是仍停留在“概念知道、作者不会落”的状态
- 哪个阈值最容易被作者写漂，漂了之后会导致什么逻辑事故

### 7. 长线演进与误导窗口

优先评价：

- 开篇第一次出手有没有抓人
- 中盘升级是不是只加码、不换打法
- 终局后果是否足够贵
- 误导点与证伪点是否都能靠证据和行动完成，而不是靠解释完成
- 若设定需要终局关门 / 降格 / 失能，报告里最好提前判断是否保留了关门后的余波与版本修订痕迹

### 8. 现实痛点 / 情绪共鸣 / 讨论度

每份报告最好明确指出：

- 这个设定压中了什么现实焦虑、身份不安、行业痛点或情绪落差
- 这种压力是否已经被转译为读者能感到的具体场景
- 是否具备可截图、可讨论、可站队的传播钩子

### 9. 社会面 / 舆情面 / 合规预警

每份报告最好明确指出：

- 设定一旦暴露，会先惊动谁、影响谁、反噬谁
- 是否存在平台合规、改编删削或高教学性风险

### 10. 冲击 9.2+ 综合评分的优化思路

在完成基础修复建议后，必须继续回答：

- 若要把这份设定的综合评分继续往 `9.2+` 推，最该优先增强的是哪三处
- 如何同时提升：现实可信度、压迫 / 恐惧 / 悬疑感、反击 / 破局 / 智性爽感
- 至少给出 `1–3` 处可直接写回设定文档的示例级改写文案

### 11. 作者向依据与正文向镜头双交付

报告允许补充作者向依据（流程依据、制度依据、规则依据），但不得停留在“作者知道了就行”。

- 每条关键修复建议都应同时给出：作者侧依据 + 正文可见的回执 / 对话 / 场景动作
- 不得建议把法条、科普、制度依据原样塞进正文；必须转译成镜头、动作、载体与后果

### 12. 首次出场与前三章留存验证

每份报告最好明确指出：

- 这条设定第一次亮相是否已经咬到现实困局 / 熟悉流程 / 日常危机
- 前三章内是否已经出现一次有效验证、误判、反噬或小回报
- 首秀是否留下了可回指痕迹、可截图钩子或可催更的问题

### 12.1 “不能直接解决的问题”是否够硬

每份报告最好明确指出：

- 设定有没有把“不能直接回答什么、不能替代什么、不能越过哪条终局红线”写死
- 若没有，这份设定会最先在哪个环节滑成万能按钮

### 13. 人物—情境—观点合拍度

每份报告最好明确指出：

- 这条设定是否绑定了最合适的代表性人物，而不是谁都能拿来用
- 它是否被放进了最能凸显价值的熟悉场景，而不是靠说明书成立
- 主角 / 对手 / 围观者围绕它的口径与反应，是否符合目标读者最容易代入或争论的情绪站位

### 14. 竞对差距量化卡（必须写重）

报告里不得只写“设定不如竞对成熟”；至少要量化：

- 首秀抓力与前三章回报
- 规则边界与代价显性化
- 可证据化硬度
- 程序摩擦与现实阻力
- 误导与证伪窗口
- 长线递进与终局后果

每项都要写：我方当前分、竞对均值分、分差、关键证据、72 小时补丁、不应照搬点。

### 15. 高分封顶规则（必须写明）

- 若竞对量化卡未完成：综合评分默认不得高于 `7.0/10`
- 若竞对项低于 `7.0/10`：综合评分默认不得高于 `8.4/10`
- 若竞对项低于 `6.0/10`：综合评分默认不得高于 `7.4/10`
- 若首秀抓力、代价显性化、可证据化三项里有 `2` 项以上明显弱于竞对，不得写成“可直接入库”
