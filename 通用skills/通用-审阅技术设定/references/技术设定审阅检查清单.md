# 技术设定审阅检查清单

## 一、用途是否清楚

- 它在剧情里负责什么：冲突 / 证据 / 围猎 / 反围猎 / 氛围 / 时间窗口
- 是否能用 1–2 句说清最小叙事用途

## 二、规则是否清单化

- 是否按触发 / 输入 / 过程 / 输出 / 失败条件写
- 关键规则是否足够具体

## 三、边界与代价

- 不能解决什么
- 使用一次要付出什么
- 是否存在明显万能风险

## 四、漏洞与反制

- 对手怎么利用 / 规避
- 主角如何识别 / 反制

## 五、可证据化

- 每条关键规则是否有对应痕迹
- 是否能落到日志、回执、时间戳、版本号、装订痕、点位号等

## 六、程序摩擦

- 是否有权限、审批、工单、窗口期、责任甩锅等阻力
- 这些阻力是否真的进入冲突

## 七、镜头化程度

- 是否至少给出几个可拍画面
- 是否过度讲课

## P0 / P1 风险

- 规则互相打架
- 没有边界
- 没有痕迹
- 只有解释，没有动作链
- 与人物资源完全不匹配
