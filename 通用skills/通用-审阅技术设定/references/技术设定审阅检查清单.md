# 技术设定审阅检查清单

## 一、阅读链路与对象边界

- 是否已确认写作口径 Prompt、关键研究、总纲 / 分卷 / 单元案、相关人物传记、目标设定文档与关联设定文件
- 若上游缺失，是否明确记录了阻断点、替代依据与由此带来的风险

## 二、基础信息检查

- 设定命名规范性：术语 / 别名 / 代号是否统一
- 一句话口径是否已压成“动作 → 凭证载体 → 可验证后果”，而不是只有概念定义
- 代号 / 编号规则是否清楚，内部与外部叫法是否会打架
- 设定分类准确性：规则 / 程序 / 机制 / 手法 / 能力 / 证据载体是否归类清楚
- 时间线与卷级递进匹配：前期 / 中期 / 后期的强度与使用窗口是否合理
- 首次出现、强化、反转、回收、余波等关键节点是否交代清楚
- 现实可行性 / 世界观自洽：现实题材是否合理，幻想题材是否在已建规则内自洽
- 影响范围设定合理：个人 / 组织 / 单元案 / 城市 / 社会面的波及边界是否明确
- 敏感等级与信息边界是否明确：哪些角色知道、哪些角色只能看到表层版本
- 新颖性与独创性：不是换皮老套路，且有清晰的识别点与讨论点

## 三、写作落地卡片（作者向交付物｜强制）

- 是否至少产出 `1–3` 张可直接拷回大纲 / 章节执行表的落地卡片
- 每张卡片是否都有：适用场景、触发条件、可视化回执、对手常用口径、主角反制抓手、反噬与代价、升级按钮
- 卡片是否真的能直接生成场景，而不是继续写成抽象说明

## 四、设定用途与最小叙事句

- 它在剧情里负责什么：冲突 / 证据 / 围猎 / 反围猎 / 氛围 / 时间窗口 / 升级阀门 / 误导工具
- 是否能用 `1–2` 句说清最小叙事用途

## 五、首次出场与前三章验证

- 第一章是否已经看见设定第一次咬到现实困局、日常危机或熟悉流程
- 第二章是否已经看见人物因此判断、保命、试错、翻案或被反咬
- 第三章是否已经完成一次有效验证、误判、反噬或小回报
- 首秀是否留下可回指痕迹、可传播钩子、可站队口径，而不是只有概念说明

## 六、人物—情境—观点合拍度

- 是否绑定了具有代表性的使用者 / 受害者 / 质疑者，而不是空中掉下一条规则
- 是否把设定放进最能凸显其价值的熟悉日常情境，而不是只能在说明文里成立
- 主角态度、对手口径、围观者反应是否贴近目标读者的现实想法与情绪站位
- 是否能把设定写成“一个人被逼着这么用它”，而不只是“作者知道它怎么运作”

## 七、规则是否清单化

- 是否按触发 / 输入 / 过程 / 输出 / 失败条件写
- 是否至少能拆出关键规则，而不是只有散文式讲解
- 关键规则是否足够具体，不依赖作者临场脑补
- 是否补了标准表现 / 感官反馈序列，让作者知道第一次显影该怎么写
- 是否补了衰减 / 失效 / 污染 / 覆盖条件，而不是永远可用
- 是否给出关键参数表，并区分固定值与叙事弹性
- 是否至少给出一张最小调用流程卡
- 是否给出分级代价 / 程序痕迹表，而不是只说“代价很大”

## 八、现实可行性与程序合理性

- 流程链 / 规则链是否自洽且有阻力
- 权限与资源：谁能做、凭什么能做、越权会怎样
- 证据保全：如何取证、如何转移、哪里会污染、是否可信
- 时间与成本：耗时、金钱、人情、风险是否符合故事与现实 / 世界观逻辑

## 九、边界、代价、漏洞与反制

- 不能解决什么
- 使用一次要付出什么
- 是否存在明显万能风险
- 对手怎么利用 / 规避
- 主角如何识别 / 反制
- 是否明确写出“不能直接解决的问题”，而不是默认读者自己懂它不能开天眼
- 终局是否需要关门 / 降格 / 留残留余波，若需要是否已提前铺垫

## 十、可证据化与口径破口

- 每条关键规则是否有对应痕迹
- 是否能落到日志、回执、时间戳、版本号、装订痕、点位号、能力残留等载体
- 若涉及对外口径，是否指出最容易被打穿的破口

## 十一、程序摩擦与现实阻力

- 是否有权限、审批、工单、窗口期、责任甩锅、设备限制、协同失误等阻力
- 这些阻力是否真的进入冲突，而不是只在设定文档里存在

## 十二、镜头化与作者调用体验

- 是否至少给出几个可拍画面
- 是否给出声音 / 提示 / 纸面 / 界面等具体载体
- 是否过度讲课、镜头嵌入成本过高
- 是否支持规则编号、正文调用、台账回指与跨文档复用

## 十三、长线演进与平台节拍

- 开篇第一次出手、中盘升级与终局后果是否已经区分开
- 是否过早用尽最强功能，导致中段无牌可打
- 设定释放节拍是否与目标平台的回报节拍大体匹配
- 对抗维度是否从局部试探逐步升级到更贵后果
- 若文档使用了多轮补丁，是否保留了“本轮修订（YYYY-MM-DD｜对齐版本）”之类的同步块，方便追踪规则升级与红线变化

## 十四、现实痛点 / 情绪共鸣 / 讨论度

- 这个设定压中了什么现实焦虑、行业痛点、日常不安、身份恐惧或情绪落差
- 这些痛点是否已经被转化为具体场景，而不是只存在说明层
- 是否具备可讨论、可站队、可转述的读者钩子
- 设定主要供血类型是否明确，且首秀三章内已兑现至少 `1` 类回报

## 十五、跨文档一致性总检

- 术语统一：同一概念是否只用一个口径
- 逻辑映射：证据要素 / 规则编号 / 台账锚点能否对齐
- 时间规定：窗口期、冷却期、有效期、升级期是否前后一致
- 与总纲、分卷、人物传记、线索台账、时间轴是否对齐

## 十六、社会面与合规风险

- 一旦设定暴露，社会 / 组织 / 舆情会如何反应
- 是否包含高教学性、可复刻伤害或敏感细节
- 是否已经做了叙事化、留白化、降教学化处理

## P0 / P1 风险

- 规则互相打架
- 没有边界
- 没有痕迹
- 只有解释，没有动作链
- 与人物资源完全不匹配
- 没有现实痛点或情绪牵引，导致读者只觉得“说明很全但没感觉”
- 没有最小调用流程卡，导致作者知道原理却不会落正文
- 没有“不能直接解决的问题”，导致设定天然朝万能按钮滑去
