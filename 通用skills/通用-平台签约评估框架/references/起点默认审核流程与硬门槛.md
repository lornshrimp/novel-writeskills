# 起点默认审核流程与硬门槛

> 适用范围：
>
> - 明确针对起点中文网的签约 / 过稿评估
> - 未注明平台时，作为本项目默认评估基线

## 审核顺序（默认理解）

1. 先过基础门槛：字数、材料、更新稳定性、存稿与投稿路径是否成立。
2. 再看内容质量：开篇抓力、主角抓手、冲突、逻辑、长线供血。
3. 最后看题材创新、市场适配与后续数据潜力。

> 其中“更新稳定性 / 存稿 / 投稿路径”默认只决定**是否适合现在投稿**与作者执行风险高低，不应直接并入小说文本质量高阶判断。

## 常用门槛锚点

### 机器初审风险

- 约 6000 字后，通常会先暴露出：原创性、违规风险、章节字数合规性等基础问题。
- 若作品在原创 / 违规 / 极端不合规章节结构上有明显风险，应优先写成 P0 或至少重大预警。

### 自主申签

- 通用层默认只把“是否达到当前题材 / 平台设定的自申正文门槛”当成必检项；若题材层有更严格数值，题材层覆盖通用层。
- 起点不同题材研究里，自主申签观察带常见落在约 `5` 万–`10` 万字；若本轮题材研究明确给出更严格门槛，不得再拿通用层模糊口径回冲题材层。
- 申请时若已断更或明显更新不稳定，审核优先级通常会下降。

### 编辑内投

- 先区分**形式最低提交线**与**建议成熟内投线**，不要混写。
- 形式最低提交线：约 `6000` 字连续正文 + `300` 字大纲 + 核心人设 / 人设表；这只能说明“材料可提交”，不等于适合直接给出“建议现在内投”。
- 建议成熟内投线：约 `2` 万字开篇正文 + `2000` 字左右大纲；只有当前门槛、首轮回报、首卷承诺与投稿包成熟度都能被验证时，才更适合给“建议现在内投”。
- 直发初审通常可把“满 `6000` 字、约 `3` 章、单章 `2000` 字 ±`200`”视为基础观察点。

### 执行准备

- 默认建议至少 `15` 章存稿；按起点常见 `2000` 字 / 章口径，约等于 `3` 万字级别的连续准备。
- 默认建议可证明日更 4000 字的稳定更新能力。
- 若单章长期明显低于 `1800` 或高于 `2200`，或作者已有长期断更、频繁超 `3` 天停更、完全没有可执行更新计划，应显著下调置信度与建议结论；但这类问题默认归为作者执行风险，不直接上升为作品内容质量高阶缺陷。

### 连载与试水补充理解

- 起点长篇默认要按“持续追读工程”理解，而不是只准备一个强开头。
- 若目标是靠自然连载吃到后续推荐 / 试水机会，通常要把**约 6 万字上下**视为一个现实观察点：到这里还看不见稳定钩子密度与回报节奏，后续放大概率会继续吃力。
- 因此，评估时不只问“首章炸不炸”，还要问：到 3–5 章、10–15 章、约 50 章时，是否都有可预期的还账设计。

## 快审核心看点

- 300 字内是否有抓力。
- 黄金三章是否能完成：立主角抓手、启动第一个挑战 / 对抗、交付一次小高潮或更强悬念升级。
- 前 1000 字是否同时给出一个硬抓手与一个软压力，而不是只给设定说明。
- 前 3–5 章是否至少有一次小回报，前 10–15 章是否至少有一次中回报。
- 前 3 万字是否具备持续追读理由，而不是只有一个首屏噱头。

## 使用方式

- 这份规则更适合作为“先判门槛与快审逻辑”的母锚点。
- 具体题材差异、加减分项与红线，再交给对应题材 Skill 继续细化。
