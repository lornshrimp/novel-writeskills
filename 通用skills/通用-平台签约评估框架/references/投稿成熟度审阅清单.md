# 投稿成熟度审阅清单

适用范围：默认口径（起点 / 未标平台）

用途：在平台签约评估中独立判断“是否建议现在投稿 / 内投”，与内容主分分开记录。

---

## P0 必查项：投递规范

- 邮件标题是否清晰（书名 + 作者 / 项目名）。
- 邮件正文是否结构化（项目概览、题材标签、亮点、正文附件说明）。
- 附件命名是否清晰可检索。
- 是否能让编辑低成本完成首轮判断。

---

## P0 必查项：材料完整度

- 书名、简介、样章、大纲是否齐备。
- 大纲是否覆盖全书骨架（非仅第一卷“待定”）。
- 样章是否支持快审（前 300 / 黄金三章可核验）。
- 书名与简介是否同向承诺同一故事。

---

## P1 建议项：文本与投递一致性

- 简介噱头是否能在正文首章呼应。
- 卖点是否在样章内可见，不是只存在于介绍文案。
- 若存在“文案A、正文B”错配，应下调投稿建议结论与置信度。

---

## 结论输出格式（强制）

- 投稿成熟度结论：`可投 / 补齐后可投 / 暂不建议投`
- 阻断项：逐条列出 P0 未通过项
- 最小补丁顺序：按“先补结构缺件，再补快审可见证据”排序
- 置信度：`高 / 中 / 低`

说明：本清单属于“投稿准备风险”评估，不直接替代内容主分。
