# 平台签约评估维度与权重

## 默认平台基线（起点 / 未标平台）

在起点或未注明平台时，默认按付费阅读平台的严口径判断：

1. **逻辑自洽**：题材、机制、人物动机、推进链条能自圆其说。
2. **现实代入**：主角、处境、目标、回报路径能被读者快速理解并代入。
3. **长线付费价值**：不仅能抓住第一眼，还能支撑持续追读与后续付费。

高分的常见共性，不是“设定听起来花”，而是：

- 编辑能快速归类；
- 卖点与主角 / 机制 / 冲突强绑定，不是随手可替换；
- 开篇到中前期存在持续回报链，而不是只靠一个噱头撑首屏。

通用评估至少覆盖以下 6 维：

1. 题材与平台适配
2. 书名、简介与第一印象
3. 开篇 / 黄金三章或其等价就绪度
4. 人物与爽点机制
5. 长线架构与可持续性
6. 更新执行与投稿准备

## 评估模式

- 正文正式评估：已有样章，按正文实绩打分
- 大纲预评估：无正文样章，按开篇方案和执行就绪度打分

## 推荐权重（默认母版）

| 维度 | 推荐权重 |
| --- | ---: |
| 题材与平台适配 | 20 |
| 书名、简介与第一印象 | 10 |
| 开篇 / 黄金三章或其等价就绪度 | 25–30 |
| 人物与爽点机制 | 15 |
| 长线架构与可持续性 | 15–20 |
| 更新执行与投稿准备 | 10–15 |

> 说明：具体题材包装层可在此基础上微调，但不得删掉任何一个维度。

## 高分三问（建议在评分前先回答）

1. 这部作品能否被一句话讲清“主角是谁、靠什么、要解决什么问题”？
2. 核心卖点是否与主角身份 / 核心资源 / 核心冲突强绑定，而不是换个设定也能成立？
3. 从开篇到中前期，是否已经能看到持续回报、持续升级或持续破局的路径？

## 缺失材料时的处理原则

- 缺前三章 / 前 6000 字：开篇维度不得按正文实绩高打分
- 缺总纲 / 分卷：长线维度必须降权或封顶
- 缺更新计划或存稿信息：执行维度不得给高置信度判断
- 只有一句简介或零散片段：只能做预判版，不得给高概率结论

## 大纲预评估的强制提醒

- 评的是“是否已经设计到位”，不是“正文已经做到”
- 若开篇节点只停留在总纲口号，没有落到章节拍 / 启动表 / 微型剧本，不得高分
- 若上游工程很强但执行面完全空白，不应直接进入“立即投稿”结论
- 若题材、人物、机制三者的绑定仍停留在抽象概念层，也不应给高置信度结论
