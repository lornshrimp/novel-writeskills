# 平台签约评估维度与权重

## 默认平台基线（起点 / 未标平台）

在起点或未注明平台时，默认按付费阅读平台的严口径判断：

### 编辑快审顺序（起点默认骨架）

1. 先看硬门槛是否过线：字数、样章、存稿、更新稳定性、投稿包是否成型。
2. 再看内容质量：首屏抓力、主角抓手、回报链、逻辑、长线供血。
3. 最后看题材创新与市场适配：是否能被快速归类、是否有差异化卖点、是否具备持续付费价值。

> 在起点默认快审里，通常是“情节 / 追读工程 > 人物与机制 > 文字精致度”；更新执行应单列理解为作者执行风险 / 投稿时机问题，而不是正文质量高阶排序项。

默认高分共性，不是“设定听起来很花”，而是：

- 编辑能在首屏快速归类；
- 卖点与主角 / 机制 / 冲突强绑定，不是随手可替换；
- 前三章到前三万字存在持续回报链，而不是只靠一个噱头撑首屏；
- 总纲、分卷与章节执行能证明这本书不是“开头会、后面不会”。

## 证据优先级与交叉核验

平台签约评估默认至少交叉看以下 `8` 层证据；材料越多，置信度才允许越高：

1. **书名 / 简介 / 一句话卖点**：是否能被平台快速归类。
2. **开篇样章**：前 `300 / 500 / 1000 / 3000 / 6000` 字是否成立。
3. **章节正文信号**：章首抓力、中段回报、章末接棒对象是否存在。
4. **人物信号**：主角职业、执念、缺陷、秘密、关系与反派是否立得住。
5. **机制信号**：设定是否有规则、边界、代价、现实锚点与可验证后果。
6. **总纲信号**：主线目标、题材归类、首卷承诺、长线供血本体是否成立。
7. **分卷信号**：分卷规划、前 10 章节拍、第一次有效回报、阶段闭环是否落到执行层。
8. **执行信号**：更新计划、存稿规模、投稿方式与材料成熟度是否可信；其中“材料成熟度”评的是现有材料是否已接近可经 `通用-组包投稿材料` 整理为可投版，而不是要求原始章节文件直接可投。

> 缺失其中任一关键证据，都必须在评分或置信度上反映出来；不得“脑补作者后面会补上”。

## 评估模式

- **总纲闸门预评估**：已有总纲，判断是否适合继续细化分卷。
- **分卷闸门预评估**：已有总纲 + 分卷，判断是否适合继续写章节正文。
- **正文准入评估**：已有开篇正文，判断是否适合继续写作 / 进入正式投稿评估。
- **完整签约评估**：正文、总纲、分卷、人物、机制、执行准备齐备，判断当前签约 / 过稿概率。
- **缺材快筛**：只有书名 / 简介 / 零散片段，只能做低置信度预判。

## 推荐内容权重（默认母版）

| 维度 | 推荐权重 | 核心问题 |
| --- | ---: | --- |
| 题材与平台适配 | 14 | 编辑能不能先把它读对、读清、读进对应赛道 |
| 书名、简介与卖点归类 | 8 | 30 秒内能不能讲清主角、异常 / 卖点、阅读理由 |
| 开篇 / 黄金三章或其等价就绪度 | 18 | 首屏、前三章、前 6000 字会不会把编辑和读者拖进去 |
| 主角、人物关系与读者代入 | 12 | 主角能不能代入，配角 / 反派能不能供血 |
| 付费价值与爽点 / 回报机制 | 10 | 这本书卖的到底是补偿、认同、掌控还是别的持续价值 |
| 设定 / 规则 / 现实锚点可信度 | 10 | 机制是否有边界、代价、现实抓手与自洽性 |
| 章级追读工程 | 8 | 单章是否真的值回时间，是否具备追读与段评触发点 |
| 长线架构、分卷供血与首卷承诺 | 12 | 首卷能否成立，中段会不会失温，终局是不是空的 |

> 说明：以上是**作品内容主分**，合计 `92` 分；题材包装层可在此基础上微调，但不得删掉任何一个内容维度。

## 作者执行风险 / 投稿准备风险（单列，不并入内容主分）

- 默认单列记录：独立更新计划、存稿证明、稳定日更证据、投稿路径准备与组包成熟度。
- 这组信号评的是**作者执行风险与投稿时机**，不是小说文本质量高阶评分项。
- 缺失这些证明时，默认处理为：下调置信度、改判为“补强后投稿”或“可继续准备”，而不是仅凭这一点重压作品内容主分。
- 只有当用户明确要求评估“作者执行可靠性”或“现在是否适合立刻投稿”时，才允许把这组风险提升到建议结论的核心层。

## 分阶段使用方式

### 总纲闸门预评估时

- `开篇 / 黄金三章或其等价就绪度` 改判为：**开篇可落地性预估**
- `章级追读工程` 改判为：**章节拍与回报链设计就绪度**
- `长线架构、分卷供血与首卷承诺` 权重解释上浮，因为当前最重要的是判断这条总纲是否值得细化
- 核心要答的不是“正文已经好不好”，而是：
  1. 题材归类是否清楚；
  2. 主角 / 机制 / 长线是否咬合；
  3. 这条总纲能否自然拆成分卷，而不是越拆越散；
  4. 若现在继续细化，最可能在哪个地方爆掉。

### 分卷闸门预评估时

- `开篇 / 黄金三章或其等价就绪度` 改判为：**前三章设计与前 10 章节拍的起稿就绪度**
- `章级追读工程` 改判为：**章节拍中的章首加压、中段回报、章末接棒是否已被钉死**
- `长线架构、分卷供血与首卷承诺` 与 `付费价值与爽点 / 回报机制` 需要重点看首卷承诺、第一次有效回报、卷末更贵缺口
- 核心要答的不是“正文已经过线没有”，而是：
  1. 是否适合继续写正文；
  2. 哪些节点一旦不先修，正文会被迫回头改上游；
  3. 应停留在分卷层修，还是需要回修总纲。

### 正文准入评估 / 完整签约评估时

- 恢复按正文实绩打分
- 但若同时拿到总纲 / 分卷，必须把正文表现与上游承诺交叉核验
- 核心要答的是：
  1. 正文是否兑现了上游卖点；
  2. 当前问题主要是正文级、分卷级还是总纲级；
  3. 是否已经足以进入正式投稿评估。

## 阶段改判与封顶提醒

- **只有总纲，没有分卷**：`开篇可落地性预估` 与 `章节拍就绪度` 维度都必须封顶，不得高估。
- **有总纲 + 分卷，但没有前 10 章节拍 / 开篇微型剧本**：`开篇就绪度` 维度不得高档。
- **有正文，但没有总纲 / 分卷**：前门可实测，但 `长线架构` 维度必须降权或封顶。
- **缺人物或机制材料，且正文也不足以支撑**：对应维度不得给高档结论。
- **缺更新计划或存稿信息**：`更新执行与投稿准备` 维度不得给高置信度判断。

## 文笔与信息传达如何计入

- 文笔不单独拉成大权重大维度。
- 默认把“文笔是否精准、高效、服务剧情”分摊计入：
  - 书名、简介与卖点归类；
  - 开篇 / 黄金三章；
  - 章级追读工程；
  - 设定 / 规则的说明效率。
- 也就是说：**文笔会影响分数，但不能靠“文笔好”冲淡平台归类、开篇回报、人物空心或长线失温的问题。**

## 起点默认打分锚点（补充）

### 1）题材与平台适配

- 能否被一句话讲清“这是什么题材、给谁看、核心卖点是什么”。
- 是否更像平台主口径项目，而非错配到其他平台 / 其他阅读气质。
- 核心卖点是否是“题材抓手 + 主角抓手 + 当前困局”的组合，而不是抽象概念。

### 2）书名、简介与卖点归类

- 书名是否有类型辨识度、异常点、职业抓手或明确问题感。
- 简介是否在 `300` 字内交代主角、异常 / 卖点、当前困局与继续阅读理由。
- 书名 / 简介 / 正文首屏三者口径是否一致。

### 3）开篇 / 黄金三章或其等价就绪度

- 前 `300` 字：是否出现核心冲突、反常事件或强悬念切口之一。
- 前 `500` 字：编辑是否能看懂主角是谁、现在发生了什么、为什么要继续看。
- 前 `1000` 字：主角抓手 / 异常机制 / 当前目标是否显性化。
- 前 `3000` 字：是否形成“冲突 -> 应对 -> 小回报 / 新危机”的首次闭环。
- 前 `6000` 字：是否仍有升级、反压、反击或更大阴影牵引。

> 在总纲或分卷阶段，上述五条要改判成“能否从总纲 / 分卷 / 章节拍中看见这些节点已被提前设计”。

### 4）主角、人物关系与读者代入

- 主角优先看：真实职业 / 身份、具体欲望、明确缺陷、主动行动能力、可记忆点。
- 配角优先看：功能是否明确、是否有记忆点、是否能提供推进 / 转折 / 主角映衬。
- 反派优先看：动机是否具体、是否非脸谱化、是否足够聪明、是否前置埋伏笔。

### 5）付费价值与爽点 / 回报机制

- 作品至少应有一条明确的付费价值主轴：**补偿 / 认同 / 掌控**。
- 前三章至少应让读者拿到一次有效回报：反击、验证、翻案、破局、赚钱、保命、关系推进等。
- 前三万字应能看见持续回报链，而不只是“后面会很爽”的延期承诺。

### 6）设定 / 规则 / 现实锚点可信度

- 核心机制是否与职业、处境、城市 / 现实流程或主要冲突强绑定。
- 是否有边界、代价、失败条件与现实后果。
- 若设定存在，但长期不进入现实问题与剧情推进，应视为显著降分项。

### 7）章级追读工程

- 章首是否快速点题、入局、抛异常或加压。
- 中段是否至少给出 `1-2` 个微回报或 `1` 个中回报。
- 章末是否有明确接棒对象、下章问题或传播钩子。
- 是否存在可触发段评 / 本章说 / 读者站队的句子、动作、关系或信息点。

> 在分卷阶段，本维默认改判为“章节拍是否已经预埋这些功能位”。

### 8）长线架构、分卷供血与首卷承诺

- 不只看“能不能写长”，还看前 `3` 万字是否有持续钩子密度与阶段闭环。
- 首卷是否有明确承诺：主线目标、阶段回报、第一次大高潮、卷末更贵缺口。
- 分卷是否能回流主线，终局是否与开篇承诺同向，而非后期另起炉灶。

### 9）更新执行与投稿准备

- 是否已达到目标投稿路径最低门槛（自主申签 / 编辑内投 / 仅适合预评估）。
- 是否接近或达到默认的 `15` 章级存稿建议。
- 是否能证明日更 `4000` 字的稳定更新能力。
- 投稿材料是否已经接近“能递到编辑案头”的完整度。
- 若章节文件内仍保留作者有话说 / 章节后记，只能说明投稿前仍需经 `通用-组包投稿材料` 导出或清洗可投正文；不得把这种项目内默认文件组织方式本身写成“投稿正文形态不合格”。
- 若无独立更新计划、无存稿证明、无稳定日更证据，默认应写成**作者执行风险 / 投稿准备风险**，主要影响“是否建议现在投稿”与置信度，不应上升为作品内容质量高阶扣分项。

## 当前层必问四题

1. 这一层的核心材料是否已经足以支撑继续推进，而不是把风险往后拖？
2. 若现在继续往下写，最可能造成大返工的是总纲、分卷还是正文？
3. 当前最小修复闭环是否足以把风险压回可控区？
4. 与同题材榜单前 10 相比，这一层最明显的落后项是什么？

## 缺失材料时的处理原则

- 缺书名 / 简介：`书名、简介与卖点归类` 维度不得高分。
- 缺前三章 / 前 6000 字：`开篇` 与 `章级追读工程` 两维不得按正文实绩高打分。
- 缺总纲：不得高置信度判断长线与终局承诺。
- 缺分卷 / 前 10 章节拍：不得高置信度判断首卷承诺是否真的能落到正文。
- 缺人物材料且正文信息也不足：`人物代入` 维度必须降权或封顶。
- 缺机制 / 设定说明且正文无法证明规则：`设定可信度` 维度必须下调。
- 缺更新计划或存稿信息：`执行与投稿准备` 维度不得给高置信度判断。
- 只有一句简介或零散片段：只能做预判版，不得给高概率结论。
- 若尚未达到投稿路径最低硬门槛，只能判“可继续准备”或“仍处预评估”，不得冒充已具备正式投稿成熟度。
