# 概率分段与一票否决规则

## 先判一票否决

以下问题一旦成立，原则上应直接写明“当前不建议投稿”或给极低概率结论：

- 题材不可归类
- 开篇无有效冲突 / 无主角抓手 / 无首轮回报
- 存在明显违规风险
- 长线结构明显不足
- 平台明显错配

## 起点默认硬门槛导致的封顶 / 改判

以下问题不一定都等于立即否决，但会显著改变建议结论、置信度或概率上限：

- 自主申签目标下，正文总字数明显未达 5 万字。
- 编辑内投目标下，未准备至少 2 万字开篇正文 + 2000 字左右大纲。
- 存稿明显不足 15 章或缺乏可信的补足计划。
- 无法证明能稳定日更 4000 字，或已有明显断更 / 超 3 天不稳定迹象。
- 仅有概念、简介或零散片段，却想直接进入“建议立刻投稿”结论。

### 建议封顶口径

- 投稿路径不满足最低门槛：默认不得进入“建议现在投稿”。
- 缺 15 章级别存稿或可信更新计划：执行维度封顶，且综合概率应明显下调。
- 更新稳定性无法证明：置信度不得高。

## 大纲预评估时的改判原则

若本轮属于大纲预评估：

- `题材不可归类`、`明显违规风险`、`长线结构不足` 仍可直接否决
- 开篇类问题要改判为“设计级预警”而非直接按正文失败判死
- 但若前门关键节点在章节拍 / 启动表 / 微型剧本中仍未落实，综合分也不应进入高档
- 若大纲显示“值得继续写”，也只说明具备潜力，不等于已经满足投稿硬门槛

## 常见封顶逻辑

- 题材清楚，但现实抓手或职业抓手偏虚 → 可封顶
- 开篇基本过线，但 3000 字内无首次小闭环 → 可封顶
- 有总纲，但阶段目标、分卷钩子、升级路径不清 → 可封顶
- 更新计划、存稿、投稿包明显不足 → 可封顶

## 概率输出规则

- 概率必须给区间，不给玄学精确值
- 必须给置信度
- 必须给动作结论：立即投稿 / 补强后投稿 / 先直发观察 / 暂缓 / 建议换平台
- 默认按真实低签约率口径保守输出，高分只代表相对优势，不代表高绝对通过率

## 研究材料中的成功率分层如何使用

- 若研究材料给出了“高 / 中 / 低成功率”条件分层，默认把它视为**成熟度锚点**，而不是直接照搬为最终概率输出。
- 在本项目里，最终概率仍服从“真实低签约率口径”的保守区间。
- 也就是说：
  - 研究中的“高成功率条件”= 可以支持进入本项目里的相对高档；
  - 不等于可以直接写成极高绝对签约率。
