# 概率分段与一票否决规则

## 先判一票否决

以下问题一旦成立，原则上应直接写明“当前不建议投稿”或给极低概率结论：

- 题材不可归类
- 卖点无法用一句话说清，书名 / 简介 / 首屏三者口径长期打架
- 开篇无有效冲突 / 无主角抓手 / 无首轮回报
- 前 1000 字主角仍未成为阅读重心，或异常 / 机制仍未显性
- 前 3000 字没有第一次有效回报，且看不到回报设计留位
- 存在明显违规风险
- 人物、机制、题材三者长期松脱：主角与核心卖点不绑定，换个设定也能成立
- 只有气氛、没有读者可获得的持续付费价值（补偿 / 认同 / 掌控均显著偏弱）
- 长线结构明显不足
- 平台明显错配

## 分阶段概率字段写法（强制）

### 总纲闸门预评估的封顶

- 主概率字段默认写成：**推进分卷大纲的放行概率**
- 若用户还想知道最终签约前景，可额外补写：**最终签约上限预估（低置信度）**
- 不得把总纲阶段的判断冒充成“正文已经过线”或“现在就能投稿”

### 分卷闸门预评估的封顶

- 主概率字段默认写成：**推进章节正文的放行概率**
- 可额外补写：**最终签约上限预估（低置信度）**
- 不得把分卷阶段的判断冒充成“当前签约概率已经稳定成立”

### 正文准入评估的封顶

- 主概率字段默认写成：**继续写作 / 进入正式投稿评估的放行概率**
- 若投稿门槛已基本齐备，可同时补写：**当前签约概率区间**
- 若上游材料明显不足，签约概率区间必须降置信度，并把长线判断封顶

### 完整签约评估

- 主概率字段默认写成：**当前签约 / 过稿概率**

## 当前层放行档位

- **70%–100%**：允许推进下一层；仍要写清最大的返工风险
- **55%–69%**：有条件放行；默认先补 1 轮最小修复，再推进下一层
- **35%–54%**：暂不放行；停留当前层继续修
- **0%–34%**：不放行；若问题扎根上游，应明确回退到总纲或分卷层

> 说明：这里评的是“当前层是否值得继续往下推进”，不是最终绝对签约率。

## 起点默认硬门槛导致的封顶 / 改判

以下问题不一定都等于立即否决，也不应直接上升为作品内容高阶缺陷；它们主要改变建议结论、置信度、投稿时机或完整评估可进入性：

- 自主申签目标下，当前题材 / 平台设定的正文门槛尚未达线。
- 编辑内投目标下，起点内投统一提交线（至少 `30000` 字连续正文 + `内投版总大纲（最低 5000 字）` + `内投版第 1 卷卷纲` + `内投版简介`）尚未达线。
- 即使达到统一提交线，但模板符合性、前门效率、长线供血、执行准备或竞对差距仍明显不足，默认也不得给“建议现在内投”。
- 存稿明显不足 `15` 章（按起点常见 `2000` 字 / 章口径约 `3` 万字）或缺乏可信的补足计划。
- 无法证明能稳定日更 4000 字，或已有明显断更 / 超 3 天不稳定迹象。
- 仅有概念、简介或零散片段，却想直接进入“建议立刻投稿”结论。

### 额外封顶提醒

- 缺书名 / 简介或二者明显失效：默认不得进入高置信度高档判断。
- 缺前 3 章 / 前 6000 字：开篇与章级追读相关维度默认封顶。
- 缺总纲 / 分卷 / 前 10 章节拍：长线维度默认封顶。
- 缺人物 / 机制材料且正文也不足以支撑：人物或机制维度默认封顶。

## 分阶段封顶规则

### 总纲闸门预评估

- 只有总纲、没有分卷：`推进分卷大纲的放行概率` 默认上限 **70%**，置信度不得高于“中”。
- 总纲存在，但主角 / 机制 / 长线三者有一条明显发虚：默认上限 **54%**。
- 只有抽象 premise，没有可交易的一句话卖点：默认上限 **35%**。
- 若要补写“最终签约上限预估”，在**没有分卷和开篇执行材料**时，默认不应高于 **12%–15%**。

### 分卷闸门预评估

- 总纲 + 分卷齐备，但没有前 10 章节拍 / 卷级启动表 / 开篇微型剧本：`推进章节正文的放行概率` 默认上限 **60%**。
- 总纲 + 分卷 + 章节拍齐备，但看不见前三章第一次有效回报设计：默认上限 **54%**。
- 首卷承诺、阶段目标、卷末更贵缺口三者缺任一：默认不得进入“允许直接推进”档。
- 若要补写“最终签约上限预估”，在**尚无正文样章**时，默认不应高于 **18%**。

### 正文准入评估

- 只有正文、没有总纲 / 分卷：可以评前门，但长线与最终签约概率必须封顶。
- 有正文但前 3000 字仍看不见首次有效回报，或看得见需要回改总纲 / 分卷的证据：`继续写作 / 进入正式投稿评估的放行概率` 默认不得进入高档。
- 有正文但未达投稿路径最低门槛：不得输出“建议现在投稿”。

### 建议封顶口径

- 投稿路径不满足最低门槛：默认不得进入“建议现在投稿”。
- 缺 15 章级别存稿或可信更新计划：应单列为执行风险，投稿建议不得写成“建议现在投稿”。
- 更新稳定性无法证明：置信度不得高，但不得仅凭这一点把作品内容主判断直接压成低档。
- 卖点只能写成抽象主题句，无法落成“主角 + 机制 / 异常 + 当前问题”：综合概率默认不得进高档。
- 只有开篇炸点，没有前三万字阶段闭环或首卷承诺：长线维度不得高，综合概率也应明显下调。

## 阶段预评估时的改判原则

若本轮属于总纲闸门预评估或分卷闸门预评估：

- `题材不可归类`、`明显违规风险`、`长线结构不足` 仍可直接否决
- 开篇类问题要改判为“设计级预警”而非直接按正文失败判死
- 章级追读工程与首轮回报缺失，也应优先改判为“执行层未落实”而非直接当成正文失败
- 但若前门关键节点在章节拍 / 启动表 / 微型剧本中仍未落实，综合分也不应进入高档
- 若大纲显示“值得继续写”，也只说明具备潜力，不等于已经满足投稿硬门槛
- 若本轮已能明确判断“继续往下写只会放大返工成本”，必须直接写“不放行下一层”，而不是用“先写写看”回避判断

## 回修级别改判原则

- 问题扎根于题材归类、主卖点、主角驱动、长线供血本体 -> 默认回修到**总纲层**
- 问题主要扎根于首卷承诺、阶段目标、前 10 章节拍、第一次有效回报设计 -> 默认回修到**分卷层**
- 问题主要扎根于节奏、信息投放、场景写法、段落推进、章末钩子 -> 默认判为**正文级返修**

除非证据已经清楚显示“上游设计本身错了”，不得把正文执行问题直接升级成“总纲重写”。

## 常见封顶逻辑

- 题材清楚，但现实抓手或职业抓手偏虚 → 可封顶
- 开篇基本过线，但 3000 字内无首次小闭环 → 可封顶
- 书名 / 简介有点子，但第一屏无法把卖点交易清楚 → 可封顶
- 主角和核心机制都单独看不错，但两者之间缺真实绑定 → 可封顶
- 有总纲，但阶段目标、分卷钩子、升级路径不清 → 可封顶
- 更新计划、存稿、投稿包明显不足 → 可单列为执行风险，并改判投稿建议 / 投稿时机

## 概率输出规则

- 概率必须给区间，不给玄学精确值
- 必须给置信度
- 必须给动作结论：立即投稿 / 补强后投稿 / 先直发观察 / 暂缓 / 建议换平台
- 在非完整评估阶段，必须额外给出：当前层是否放行下一层
- 默认按真实低签约率口径保守输出，高分只代表相对优势，不代表高绝对通过率
- 若快审模拟在 `30 秒 / 3 分钟 / 3 万字` 任一层明显断裂，概率应优先服从断裂层级，而不是被其它长板硬拉高

## 竞对维度联动封顶（强制）

- 竞对对照维度默认按 `25` 分计入内容主分；该维度不达标时，概率必须触发封顶。
- 若竞对维度 `<15/25`：当前层主概率区间默认上限 **54%**。
- 若竞对维度 `<12/25`：当前层主概率区间默认上限 **34%**，并应直接给出“暂不放行下一层”或“不建议现在投稿”倾向结论。
- 若未完成 Top 10 逐本深搜或缺少“分差 + 补丁动作 + 不应照搬点”量化卡：竞对维度默认不得高于 `10/25`，并按上限 **54%** 封顶。
- 只有当竞对维度 `>=20/25` 且无一票否决项时，当前层主概率才允许进入高档区间。

## 研究材料中的成功率分层如何使用

- 若研究材料给出了“高 / 中 / 低成功率”条件分层，默认把它视为**成熟度锚点**，而不是直接照搬为最终概率输出。
- 在本项目里，最终概率仍服从“真实低签约率口径”的保守区间。
- 也就是说：
  - 研究中的“高成功率条件”= 可以支持进入本项目里的相对高档；
  - 不等于可以直接写成极高绝对签约率。
