# 评估模式与落盘规则

## 默认总原则

- 平台签约评估默认按**分阶段准入评估**执行，不是只做“材料缺失提醒”。
- 材料停在哪一层，就先判断这一层是否值得放行到下一层：
  1. 总纲 -> 是否值得继续细化分卷；
  2. 分卷 -> 是否值得继续写章节正文；
  3. 正文 -> 是否值得继续推进投稿 / 完整签约评估。
- 在总纲与分卷阶段，主概率字段默认评的是**当前层放行概率**，不是“正文已经过线”或“现在就能投稿”的概率。
- 若用户还想知道最终签约前景，可以额外补写**最终签约上限预估**；但它只能作为低置信度上限，不得替代当前层放行判断。

## 支持的评估模式

### 总纲闸门预评估

至少应具备：

- 作品名称 / 一句话卖点 / 简介
- 总纲

若同时具备以下材料，应全部纳入，不得只看总纲：

- 主角设定
- 机制 / 规则设定
- 世界观或现实锚点说明
- 更新 / 投稿目标说明

本模式必须回答：

- 当前总纲是否适合推进分卷大纲；
- 若现在继续往下写，最可能在哪个结构点爆雷；
- 需要先补哪几刀，才能降低后续大返工风险。

### 分卷闸门预评估

至少应具备：

- 总纲
- 分卷大纲

最好同时具备以下之一：

- 前 10 章节拍
- 开篇微型剧本
- 卷级启动表
- 前 3 章设计稿

本模式必须回答：

- 当前分卷是否适合推进章节正文；
- 首卷承诺、前 10 章节拍、第一次有效回报是否已经被钉死；
- 若现在直接写正文，最可能逼着作者回头改的是总纲、分卷还是开篇执行。

### 正文准入评估

至少应具备：

- 前 500 字
- 推荐前三章或前 6000 字

若存在总纲 / 分卷 / 人物 / 机制材料，必须一并交叉核验。

本模式必须回答：

- 当前正文是否适合继续推进；
- 是否已经适合进入正式投稿评估；
- 若正文暴露问题，问题究竟属于总纲级、分卷级还是正文执行级。

### 完整签约评估

当书名 / 简介、正文、人物、机制、总纲 / 分卷、更新执行与投稿准备能形成交叉核验时，进入完整评估。

本模式的主概率字段才默认指向**当前签约 / 过稿概率**。

### 缺材快筛

当只有书名 / 简介 / 零散片段时，只能执行低置信度快筛。

本模式仍要给出**当前层是否建议继续补下一层材料**，但不得冒充完整评估。

## 开始前最小材料清单

1. 作品名称 / 一句话卖点 / 300 字内简介
2. 当前所在层的核心材料：总纲 / 分卷 / 前 10 章节拍 / 正文样章
3. 主角：职业、核心欲望、核心缺陷、人设标签
4. 机制：触发方式、边界、代价
5. 长线：总纲 / 分卷 / 前 10 章节拍 / 卷级启动表
6. 连载执行：更新计划、存稿规模、投稿方式
7. 若按起点口径评估：起点同题材榜单 Top 10 检索记录（检索工具 / 关键词 / 榜单名 / 检索时间 / 对照样本名单）

## 有材料时默认追加核验的证据层

只要材料存在，评估时默认继续检查：

1. 书名 / 简介 / 一句话卖点
2. 前 500 字、前三章、前 6000 字
3. 人物传记 / 角色设定
4. 技术设定 / 规则设定 / 世界观机制
5. 总纲
6. 分卷 / 前 10 章节拍 / 卷级启动表 / 开篇微型剧本
7. 更新计划、存稿、投稿路径与包材完备度

若其中任一层缺失：

- 必须写明缺失项
- 必须下调置信度
- 必须在对应维度封顶或降权
- 不得假装已有交叉证据
- 不得把“缺下游材料”写成不回答当前层准入问题的理由

## 上游返工定位规则

### 总纲级问题

出现以下情况时，默认应判为总纲级问题：

- 题材归类、主卖点、主角抓手、长线承诺本身就不成立；
- 换个分卷写法也救不回核心 premise；
- 分卷与正文反复暴露“根题不成立、核心驱动不成立、长线供血不足”。

### 分卷级问题

出现以下情况时，默认应判为分卷级问题：

- 总纲方向基本成立，但首卷承诺、阶段目标、前 10 章节拍、第一次有效回报没有落到可执行层；
- 总纲能用，但分卷拆解明显不适合直接起稿；
- 正文暴露出的主要问题是“卷纲没把前门钉死”，而不是总纲 premise 错了。

### 正文级问题

出现以下情况时，默认应判为正文级问题：

- 总纲与分卷都成立，但正文开篇节奏、信息投放、人物出场、钩子编排没有执行好；
- 同一结构设计在提纲里是对的，只是正文写法拖垮了体验；
- 现有问题主要能通过重写章节正文解决，而不需要回退更上游。

### 回修级别写法

- 只需改正文 -> 写明“正文级返修”
- 需改分卷但总纲可保留 -> 写明“回修到分卷层”
- 需改总纲 -> 写明“回修到总纲层”

除非证据已经足够扎实，不得轻易把正文问题直接升级成“整套总纲都得推倒重来”。

## 起点同题材榜单检索硬要求

- 只要本轮评估落到起点中文网，或未注明平台而默认按起点口径处理，就必须在正文判断前先完成**起点同题材榜单 Top 10 检索**。
- 必须优先组合使用当前环境中可用、效果较好的多种网络搜索 / 抓取工具交叉核验，不得只依赖单一搜索源或只凭旧记忆列样本。
- 默认至少记录：使用了哪些搜索工具 / 来源、检索关键词、锁定的榜单名称、检索时间、Top 10 样本名单、无法核实的盲区。
- 只锁定榜单而不继续逐本研究，视为**样本池建立完成但竞品研究未完成**。
- 对 Top 10 的每一部作品，默认都要形成一张简短证据卡，至少回答：`1）这本书正文 / 试读里最有效的抓手是什么`，`2）读者书评 / 讨论里反复提到的吸引点或争议点是什么`，`3）研究 / 拆解文章里总结出的可验证优势或结构特征是什么`，`4）它相对当前作品最能压制签约竞争力的差距点是什么`。
- 正文 / 试读证据属于每本必读项；若公开条件限制无法读到完整正文，也必须尽量读取试读页、公开章节、摘录或可靠转述，并标注限制。
- 书评 / 讨论与研究 / 拆解材料默认都应优先搜索；若某本书确实难以找到其一，必须记录检索关键词、检索来源与失败结果。
- 逐本深搜完成后，必须再总结榜单前 10 的共性强项与共性门槛，用来反推当前作品为什么还不够格。
- 若起点官方作品页、目录页或试读页因反爬、验证码、登录门槛或频控无法稳定读取，允许改用**可核验的第三方相关内容**补证，包括目录整理、试读摘要、论坛讨论、书评摘录、百科资料、剧情梳理、推荐文与拆解稿。
- 但第三方来源必须优先满足：**来源清楚、可交叉核验、非明显侵权**；不得把盗版站、未授权全文站或明显侵权镜像站当作正文 / 目录核心证据源。若官方与合法第三方来源都不足，就必须写成判断盲区。
- 若当前工作目录、项目目录或题材目录下已经存在 `竞对分析/`，默认应读取其中与本轮目标作品、同题材样本或相近竞争位相关的历史文件，作为已有研究缓存。
- 既有 `竞对分析/` 文件只能辅助复用样本、结论与问题清单，不能替代本轮新的榜单检索、逐本深搜与关键证据复核。
- 若榜单页无法直接访问，也必须通过多组关键词、多来源搜索尽最大努力重建同题材 Top 10 样本；但不能因为访问不便就跳过这一步。
- 同题材对照默认以**严格找差距**为主：重点写当前作品为什么比不过榜单样本、在哪些核心维度明显落后、差距是否足以影响签约结论。

## 起点默认投稿路径门槛（补充）

### 自主申签

- 通用层默认只检查“是否达到当前题材 / 平台定义的自申正文门槛”；若题材层有更严格研究结论，题材层覆盖。
- 若题材层没有更具体口径，才把约 `5` 万–`10` 万字视为常见观察带；不要把这个宽区间误写成单一刚性数字。
- 申请时最好仍保持持续更新状态。

### 编辑内投

- 先区分**形式最低提交线**与**建议成熟内投线**：
- 形式最低提交线：约 `6000` 字连续正文 + `300` 字大纲 + 核心人设 / 人设表。
- 若只达到形式最低提交线，最多说明“可提交 / 可做预备版”，不等于“建议现在内投”。
- 建议成熟内投线：约 `2` 万字开篇正文 + `2000` 字左右大纲；若前 `2` 万字仍无法提供追读动力，即使形式上达线，也不应高估通过率。

### 初审执行准备

- 默认建议准备至少 `15` 章存稿；按起点常见 `2000` 字 / 章口径，约等于 `3` 万字级别的连续准备。
- 默认建议能证明日更 `4000` 字的稳定更新能力
- 单章若长期明显偏离 `2000` 字 ±`200` 的连载口径，或连续超 `3` 天停更，也应单列为执行风险
- 若上述执行准备明显不足，应改判为“可继续准备”而非“建议现在投稿”

若缺失关键材料：

- 必须写明缺失项
- 必须下调置信度
- 不得假装已有证据
- 不得跳过投稿路径门槛核验

## 报告落盘规则

- 报告必须落盘到：`审阅意见/起点评估/`
- 建议主报告路径：`起点签约成功概率评估报告_【评估层级】_【书名】_YYYY-MM-DD.md`
- `【评估层级】` 只能使用：`总纲闸门预评估` / `分卷闸门预评估` / `正文准入评估` / `完整签约评估` / `缺材快筛`
- 若已有**同一作品、同一评估层级**主报告：默认覆盖重写
- 不同评估层级必须使用不同文件名并并列保留，不得跨层覆盖
- 若不存在：必须新建

## 报告开头必须写明

- 评估模式
- 当前评估层级
- 当前层目标
- 当前层是否放行下一层
- 当前层主概率字段到底在评什么
- 若补写最终签约上限预估，必须注明“低置信度上限”
- 目标作品
- 核心样章 / 开篇依据
- 主要大纲依据
- 关键人物 / 关键机制依据（如存在）
- 外部榜单检索依据与同题材 Top 10 取样口径
- 目录纠偏（如适用）
- 输出策略

## 对话侧最小回执

至少说明：

- 本轮读取了哪些核心材料
- 本轮处于哪个评估层级
- 本轮是否放行下一层，若不放行停留在哪一层
- 本轮使用了哪些搜索工具、锁定了哪个同题材榜单、对照样本是谁
- 本轮是否完成了 Top 10 逐本深搜；若有缺口，缺的是哪几本、哪类材料
- 本轮是否使用了第三方相关内容补证；若使用，官方源为什么受阻、第三方来源可信度如何
- 本轮是否读取了既有 `竞对分析/` 文件；若读取，哪些文件仍有效、哪些结论被本轮新证据修正
- 报告写入 / 覆盖到了哪个路径

## 完成底线

- 没有概率区间，不算完成
- 没有说明本轮主概率到底在评什么，不算完成
- 没有当前层放行 / 不放行结论，不算完成
- 没有写明回修层级与最小修复闭环，不算完成
- 没有一票否决 / 封顶检查，不算完成
- 没有明确投稿建议或阶段动作建议，不算完成
- 没有说明本轮主要判断依据与判断盲区，不算完成
- 没有起点同题材榜单 Top 10 检索、逐本深搜与从严对照，不算完成
