# 通用-设计总大纲

总大纲默认不是单一文件，而是一组互相联动的战略层文件。

## 总纲包对象边界（强制）

- `小说总大纲.md` 默认承载小说的主线故事，是主线推进、阶段里程碑与终局收束的基准文档。
- `故事面/` 下各文件默认主要承载支线故事；它们不是主线总纲的注释页、摘要页或拆分副本，而是各自能成立、能持续供血的完整故事文件。
- 设计总纲包时，必须显式设计主线与支线的相互交织、相互联系、相互推动关系，而不是把二者做成互不相干的平行目录。
- `小说总大纲.md` 与 `故事面/` 共享人物、事件、线索、阶段节点通常属于正常联动，不得因此把这种双轨设计误判为重复维护。
- 只有当多份文件在没有新增职责差异的前提下机械重复同一信息，或彼此口径冲突、互相打架时，才应视为冗余维护并要求合并或重写。

## 保真迁移要求（强制）

命中总纲层任务时，除本文件外，**必须继续读取**：

- `通用skills/写作研究/起点模板/总纲模板/README.md`
- `references/总纲设计骨架与对象定义.md`
- `references/总纲模板与验收.md`
- `references/大纲公共裁判规则与统一编号体系.md`
- `references/超长篇卷宗式架构与阶段目标.md`
- `references/四部分结构与关键转折点进度表.md`
- `references/九部件规划与六问总纲质检表.md`

## 总纲模板选择与组合要求（强制）

- 设计或重写总大纲时，必须先从 `通用skills/写作研究/起点模板/总纲模板/README.md` 中选择最贴近当前项目的总纲模板，不得跳过模板库，直接重造高度同质的结构。
- 选主模板前，优先查阅 `README.md` 中的 **“不同题材优先选哪个主模板（速查表）”**；先按作品主承诺选主模板，再用辅助模板补结构痛点。
- 每个项目只能确定 `1` 个主模板，并可再叠加 `0–2` 个辅助模板。
- 主模板优先承接题材承诺、主供血方式与基础结构；辅助模板只负责补结构痛点，如明暗线、地图切换、势力关系、单元案、副本或系统任务。
- 若多个模板都像主模板，只能保留最贴近当前作品主承诺的 `1` 个为主模板，其余降为辅助模板或直接放弃。
- 若没有找到更合适的模板，主模板默认回退到 `通用skills/写作研究/起点模板/总纲模板/九线写作总纲.md`。
- “模板组合”不是把两个模板机械拼接；必须以本 Skill 既有的总纲包骨架为主框架，再把所选模板的关键字段、裁判问题与结构补丁吸收到对应章节。
- 生成总纲时，必须在 `小说总大纲.md` 顶部显式写出 `## 模板组合声明`，至少包含：主模板、辅助模板、是否触发默认九线回退、选择原因、各模板负责的结构职责，以及哪些模板要求已被吸收入主体。

## 卖点保真要求（强制）

- 总纲必须给出主卖点的落地路线：开篇快速露出、卷级强化、中段轮换回报、终局级回收。
- `M0 / X / C` 设计必须说明哪些主线 / 支线 / 单元案直接承载卖点，哪些只做铺垫或转场。
- 若总纲无法回答“编辑先看简介和开篇、读者追到中段以后，为什么还会持续记得这本书的卖点”，则未达标。
- 总纲还必须回答“为什么这本书现在值得写”：现实锚点是什么、类型抓手是什么、读者持续追读承诺是什么。
- 总纲必须显式设计开篇钩子链、卷内钩子链与长线钩子链；若只有“每卷有高潮”的空话，不算达标。

## 成长线与情绪回报要求（新增）

- 总纲必须写清主角成长线：人物起点、核心短板、阶段性认知变化、终局级自我完成各是什么。
- 若作品存在关键关系线、搭档线、亲缘线、师徒线或对抗线，总纲必须明确它们如何与主线相互推动，而不是挂在故事外面当装饰。
- 开篇到前三章的设计，默认至少要保证：`前 300 字有抓手`、`前三章有一轮完整的钩子 / 蓄力 / 爆发或升级`、`读者能感知人物为什么非走下去不可`。
- 总纲里的情绪价值应写成行动、选择、代价与兑现链，而不是只写“很燃 / 很虐 / 很治愈 / 很爽”。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，必须先读取并纳入总纲设计判断。
- 不得把这些报告只当成背景资料；必须显式对比开篇抓力、卷级供血、人物付费价值、机制兑现、长线承诺与终局闭环，说明当前总纲准备在哪些点上追平、反超竞对。
- 总纲设计的默认目标不是“结构完整就行”，而是让当前作品在关键维度上**全方位超越竞争对手**；若看不出明确超越路径，应判为总纲仍未达标。

## 常见触发词 / 用户说法速查

- 帮我搭总大纲
- 重写这本书的全书总纲
- 先把 M0 / X / C 钉死
- 我需要跨卷推进和终局闭环
- 这份总纲现在太散了
- 帮我做总纲级验收

## 何时使用

- 设计或重写全书总纲、总设定框架、战略层文件组。
- 规划主线谜团 `M0`、重要支线 `X01-XNN`、单元案矩阵 `C01-CNN`。
- 设计跨卷推进、终局闭环、长期连载结构、主支线配比与主支线交织关系。

## 总纲包定义（默认口径）

至少包括：

- `小说总大纲.md`（主线故事基准）
- `故事面/`（以支线为主的完整故事文件组）
- `伏笔与回收 / 呼应总纲`
- `故事多面体大纲`
- 以及相关总表级时间轴 / 矩阵 / 台账文件

## 默认执行顺序

1. 先从 `通用skills/写作研究/起点模板/总纲模板/README.md` 选择 `1` 个主模板与 `0–2` 个辅助模板；若没有更合适模板，主模板默认回退到 `九线写作总纲.md`。
2. 再判定本次是单文件修改还是总纲层联动修改。
3. 再明确总纲包中哪些文件需要同步更新，并先写清 `小说总大纲.md` 的主线职责、`故事面/` 的支线职责与二者交接接口。
4. 先在 `小说总大纲.md` 顶部写出 `## 模板组合声明`，说明主模板、辅助模板、默认回退、选择原因与各模板职责分工。
5. 再写一句话主卖点、`M0`、`X`、`C` 三层结构与卖点落地路线，并把所选模板的关键要求吸收到主体章节。
6. 再补跨卷推进、伏笔回收、终局闭环与下游拆分接口。
7. 若总纲显得“事件很多但顺序职责发虚”，先用“九部件规划与六问总纲质检表”回校结构责任。
8. 最后按模板与验收清单自检。

## 最低交付

- 模板组合声明：主模板、辅助模板、是否触发默认九线回退、选择原因、各模板职责。
- 明确本次涉及哪些总纲层文件及其联动关系。
- 明确 `小说总大纲.md` 与 `故事面/` 的职责边界，以及主支线如何相互推动。
- 说明主模板与辅助模板的关键要求已经吸收入总纲主体，而不是只在声明区挂名。
- 全书一句话卖点与差异化点。
- 主卖点在开篇、各卷与终局的主要落点。
- 主线谜团 `M0` 的谜面、真相、代价。
- 重要支线 `X01+` 与单元案矩阵 `C01+`。
- 跨卷推进轨迹、伏笔与回收策略、总纲级验收清单。

## 硬规则

- 禁止把总纲写成世界观说明书或主题散文。
- 禁止只有设定，没有事件链、证据后果或代价。
- 禁止跳过总纲模板选择步骤，直接重造与模板库高度重合的结构。
- 禁止同时并列多个主模板；只能有 `1` 个主模板和 `0–2` 个辅助模板。
- 禁止只在总纲顶部挂模板名，却没有把模板要求真正落入主体结构。
- 找不到更合适模板时，禁止跳过 `九线写作总纲.md` 这个默认基线。
- 禁止不经过总纲闭合就直接跳到分卷细拆。
- 禁止把 `故事面/` 写成 `小说总大纲.md` 的摘要副本，也禁止把 `小说总大纲.md` 写成故事面目录的机械汇总。
- 100 万字以上项目必须显式回答：卷群怎么分、每卷供血什么、每卷卷尾留下什么新缺口。
- 禁止卖点只写在一句话简介里，不进入卷级推进、单元案与终局闭环。
- 总纲包本体文件默认不带日期；日期命名仅用于对应审阅报告、书评等派生产物，不用于总纲主文件及其配套总表。
- 禁止因为主支线共享人物、线索、阶段节点就回避双轨设计；该类重合本来就应该为主支线联动服务。
- 禁止把“都市焦虑 / 冲突升级 / 多线交织 / 主题深刻”这类抽象话当成故事；关键节点必须尽量写成“谁做了什么 → 留下什么可回指物 → 引发什么可验证后果”。
