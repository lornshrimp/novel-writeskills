# 通用-设计总大纲

总大纲默认不是单一文件，而是一组互相联动的战略层文件。

## 总纲包对象边界（强制）

- `小说总大纲.md` 默认承载小说的主线故事，是主线推进、阶段里程碑与终局收束的基准文档。
- `故事面/` 下各文件默认主要承载支线故事；它们不是主线总纲的注释页、摘要页或拆分副本，而是各自能成立、能持续供血的完整故事文件。
- 设计总纲包时，必须显式设计主线与支线的相互交织、相互联系、相互推动关系，而不是把二者做成互不相干的平行目录。
- `小说总大纲.md` 与 `故事面/` 共享人物、事件、线索、阶段节点通常属于正常联动，不得因此把这种双轨设计误判为重复维护。
- 只有当多份文件在没有新增职责差异的前提下机械重复同一信息，或彼此口径冲突、互相打架时，才应视为冗余维护并要求合并或重写。

## 总纲模板选择与组合要求（强制）

- 设计或重写总大纲时，必须先从 `通用skills/写作研究/起点模板/总纲模板/README.md` 中选择最贴近当前项目的总纲模板，不得跳过模板库，直接重造高度同质的结构。
- 选主模板前，优先查阅 `README.md` 中的 **“不同题材优先选哪个主模板（速查表）”**；先按作品主承诺选主模板，再用辅助模板补结构痛点。
- 每个项目只能确定 `1` 个主模板，并可再叠加 `0–2` 个辅助模板。
- 主模板优先承接题材承诺、主供血方式与基础结构；辅助模板只负责补结构痛点，如明暗线、地图切换、势力关系、单元案、副本或系统任务。
- 若多个模板都像主模板，只能保留最贴近当前作品主承诺的 `1` 个为主模板，其余降为辅助模板或直接放弃。
- 若没有找到更合适的模板，主模板默认回退到 `通用skills/写作研究/起点模板/总纲模板/九线写作总纲.md`。
- “模板组合”不是把两个模板机械拼接；必须以本 Skill 既有的总纲包骨架为主框架，再把所选模板的关键字段、裁判问题与结构补丁吸收到对应章节。
- 生成总纲时，必须在 `小说总大纲.md` 顶部显式写出 `## 模板组合声明`，至少包含：主模板、辅助模板、是否触发默认九线回退、选择原因、各模板负责的结构职责，以及哪些模板要求已被吸收入主体。

## 卖点保真要求（强制）

- 总纲必须给出主卖点的落地路线：开篇快速露出、卷级强化、中段轮换回报、终局级回收。
- `M0 / X / C` 设计必须说明哪些主线 / 支线 / 单元案直接承载卖点，哪些只做铺垫或转场。
- 若总纲无法回答“编辑先看简介和开篇、读者追到中段以后，为什么还会持续记得这本书的卖点”，则未达标。
- 总纲还必须回答“为什么这本书现在值得写”：现实锚点是什么、类型抓手是什么、读者持续追读承诺是什么。
- 总纲必须显式设计开篇钩子链、卷内钩子链与长线钩子链；若只有“每卷有高潮”的空话，不算达标。

## 成长线与情绪回报要求（新增）

- 总纲必须写清主角成长线：人物起点、核心短板、阶段性认知变化、终局级自我完成各是什么。
- 若作品存在关键关系线、搭档线、亲缘线、师徒线或对抗线，总纲必须明确它们如何与主线相互推动，而不是挂在故事外面当装饰。
- 开篇到前三章的设计，默认至少要保证：`前 300 字有抓手`、`前三章有一轮完整的钩子 / 蓄力 / 爆发或升级`、`读者能感知人物为什么非走下去不可`。
- 总纲里的情绪价值应写成行动、选择、代价与兑现链，而不是只写“很燃 / 很虐 / 很治愈 / 很爽”。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，必须先读取并纳入总纲设计判断。
- 不得把这些报告只当成背景资料；必须显式对比开篇抓力、卷级供血、人物付费价值、机制兑现、长线承诺与终局闭环，说明当前总纲准备在哪些点上追平、反超竞对。
- 总纲设计的默认目标不是“结构完整就行”，而是让当前作品在关键维度上**全方位超越竞争对手**；若看不出明确超越路径，应判为总纲仍未达标。

## 新手必读：大纲与锚点的关系（强制）

**最常见的新手误区**：一上来就想写"完美大纲"，结果被庞大世界观吓退，反而卡壳。正确的入口是：

- 先找一个让你心跳加速的**锚点**：可以是一个人物、一个场景、一个冲突，任何能勾住你的点。这个锚点就是小说的魂。
- 再用**3要素框架**搭骨架：① 主角核心欲望（要具体，不是"想变强"而是"想在30岁前买房给父母"）② 不可逾越的障碍 ③ 不可逆的改变。
- 沿用**万能故事框架**：某人 → 渴望 → 遇阻 → 改变。这个结构能打开任何类型故事的大门，无论是科幻、现实、奇幻还是悬疑。
- 先写一句“最小核心”，再扩总纲：`谁在什么处境下，为了什么目标，正被什么阻力压迫`。没有这句话，总纲容易空转。
- 总纲必须提前定义“首章问题引擎”：读者看完第一章后，心里最想追问的那个具体问题是什么。
- **大纲是参考，不是束缚**。写着写着人物会活、剧情会跑偏，这是正常现象，千万别死磕大纲。
- **先完成，再完美**。再完美的大纲框架也不如一个烂初稿更能推进故事；最核心的秘诀是"先落地，再打磨"。

## 何时使用

- 设计或重写全书总纲、总设定框架、战略层文件组。

- 设计跨卷推进、终局闭环、长期连载结构、主支线配比与主支线交织关系。

## 总纲包定义（默认口径）

至少包括：

- `小说总大纲.md`（主线故事基准）
- `故事面/`（以支线为主的完整故事文件组）
- `伏笔与回收 / 呼应总纲`
- `故事多面体大纲`
- 以及相关总表级时间轴 / 矩阵 / 台账文件

## 默认执行顺序

1. 先从 `通用skills/写作研究/起点模板/总纲模板/README.md` 选择 `1` 个主模板与 `0–2` 个辅助模板；若没有更合适模板，主模板默认回退到 `九线写作总纲.md`。
2. 再判定本次是单文件修改还是总纲层联动修改。
3. 再明确总纲包中哪些文件需要同步更新，并先写清 `小说总大纲.md` 的主线职责、`故事面/` 的支线职责与二者交接接口。
4. 先在 `小说总大纲.md` 顶部写出 `## 模板组合声明`，说明主模板、辅助模板、默认回退、选择原因与各模板职责分工。
5. 再写一句话主卖点、`M0`、`X`、`C` 三层结构与卖点落地路线，并把所选模板的关键要求吸收到主体章节。
6. 再补跨卷推进、伏笔回收、终局闭环与下游拆分接口。
7. 若总纲显得“事件很多但顺序职责发虚”，先用“九部件规划与六问总纲质检表”回校结构责任。
8. 最后按模板与验收清单自检。

## 极其快的节奏与极其强烈的冲突——总纲层实施方法（新增）

快节奏和强冲突必须从总纲层开始设计，而不是在正文时临时抱佛脚。以下是可落地的实施规则。

### 快节奏的总纲层保证

**1. 冲突频率与密度设计**
- 总纲必须展示：**每卷至少 3-5 个递进式的中层冲突**，而不是一条主线缓缓推进。
- 各卷冲突应该从不同维度压迫主角：第一卷是外部压力（被追杀、任务失败），第二卷转为内部困境与关系翻面（信任崩塌、身份暴露），第三卷才是理念碰撞（世界观破灭）。
- **冲突不能并行**：不同维度的冲突应该是递进而非平行，即解决第一个冲突会直接触发第二个更大冲突；不能让读者看到 3 个冲突同时存在却互不相干。

**2. 卷级节奏设计（卷首-卷中-卷末三段式）**
- **卷首 5-8 章**：快速抛出本卷主冲突，控制在前 3 章内完成"问题暴露 → 主角首次应对 → 初步失败或意外升级"。必须在第 3-4 章时已经让读者感知到"原来问题比我想的大得多"。
- **卷中 8-12 章**：持续升级压力，每 2-3 章补充一次新信息、新变量或新对手，维持信息密度，防止"中段失温"。不能出现 4 章以上纯推进、没有新冲突或新发现的段落。
- **卷末 3-5 章**：快速闭环本卷主案件或问题，同时为下一卷挖出更深的洞。卷尾不是休整，而是把角色推到比卷首还要危险的位置。

**3. 事件密度与时间压缩**
- 避免"描写生活"型的总纲；每个场景、每条支线都应该服务于推进主线冲突。
- 总纲里应该明确标记"哪些事件可以快速推过，哪些必须用场景承载"。快速推过的是纯信息、纯转场；必须承载的是有冲突、有人物对立、有代价的事件。
- **时间跨度设定**：短期节奏（一周内发生的多个冲突升级）比长期空白更能制造紧迫感。若必要有时间跳跃，应该只在"问题已被解决"的段落跳跃，不能在"问题正在激化"时跳。

**4. 钩子链与追读驱动**
- 总纲必须从卷级、章级、场景级三层设计"钩子链"：每卷卷末留下悬念，每章章末留下驱动，每场场末留下冲突升级信号。
- 钩子不能只是"不知道下一步"的空白，必须是"知道有坏事要发生"的具体威胁：新敌人出现、关键人物信息暴露、契约条款启动等。

### 强冲突的总纲层保证

**1. 多维度冲突矩阵设计**
- **立场冲突**：主角与对手的目标、价值观、身份定位发生根本对立，不是简单的"你要抓我、我要逃"。
  - 示例：警察认为规则值得维护，黑帮头子认为规则是幌子；两人的正义定义完全相反，导致无法协商。
  
- **利益冲突**：两方都想要有限资源（权力、金钱、线索、时间），无法共存满足。
  - 示例：两个律师争一个关键证人，每个律师都能让证人指证对方的对手，这样不能简单选择。
  
- **信息不对等冲突**：一方掌握对方不知道的关键信息，以此制造误解、陷阱或权力压制。
  - 示例：妻子知道丈夫的秘密身份，但丈夫不知道妻子在调查他；每次对话都暗藏刀子。
  
- **身份冲突**：主角在不同环境下扮演相互矛盾的身份，这些身份的要求相互抵触。
  - 示例：卧底既要保持黑帮身份的"忠诚"，又要向警察提供信息；每个选择都是背叛。

**2. 冲突升级的三段式结构（总纲版）**
- **第一阶段**：试探与误解。双方都在试图控制局面，信息不完整，导致彼此都误判对方意图。
- **第二阶段**：直面与摊牌。关键信息暴露，双方开始明白"对方不是我以为的那样"；这时冲突会激化。
- **第三阶段**：选择与代价。没有绝对赢家，所有选择都有代价；主角必须在"坏"与"更坏"之间二选一。

**3. 代价的具体化与可验证性**
- 总纲里不能只写"主角承受了很大代价"；必须写清代价是什么、对后续故事的具体影响。
  - ❌ 错误："主角失去了信任"
  - ✅ 正确："主角失去了搭档的信任，导致下一卷必须独自应对敌人的 3 线夹击，使得逃出生天的概率从 60% 降到 20%"
  
- 代价必须是**不可逆的**或**需要长期补偿的**：杀死一个人不能活过来；失去的信任需要整卷来修复。

**4. 对手与敌对者的多层化**
- 不能只有一个终极大敌，应该设计"近敌 - 中敌 - 远敌 - 自我敌对"四层：
  - **近敌**：眼下正在争斗、最紧迫的对手（通常是卷级对手）
  - **中敌**：躲在背后推波助澜的幕后黑手（跨卷对手）
  - **远敌**：更宏观的压力或制度性敌对（终局对手）
  - **自我敌对**：主角内心的矛盾、欲望与理想的撕裂（心理层冲突）
  
- 这四层应该逐卷递进显露，而不是一开始全部抛出。

**5. 关键人物关系的冲突织入**
- 主角身边每个重要角色都应该与主角有某种**潜在冲突**或**利益冲突**，即便表面上是同盟。
  - 示例：搭档表面上忠诚，但暗中在和另一股势力接触；这种"假忠诚"比直接敌对更刺激。
  
- 关键关系线应该在某些卷点达到"必须摊牌"的程度，强制主角做出带有代价的选择。

**6. 主卖点与冲突的绑定**
- 主卖点应该等同于"主角最深的冲突根源"。
  - 示例：若卖点是"冤狱反杀"，那么核心冲突应该是"司法系统的不完美 vs 个人正义的执念"，而不仅仅是"逃狱动作"。
  
- 冲突的每一次升级都应该深化主卖点的含义或复杂性，而不是偏离。

### 快节奏与强冲突的协同设计

这两个维度应该互相支撑：
- **快节奏下的强冲突**：不能因为节奏快就让冲突变浅；应该是"冲突升级得很快"而不是"冲突强度浅但频繁"。
- **强冲突中的快推进**：每解决一个表层冲突，都应该立刻揭露下一层更深的冲突，形成"剥洋葱式"的递进。

### 总纲验收清单（快节奏与强冲突维度）

[ ] 各卷都有明确的主冲突与 3-5 个递进式的中层冲突吗？
[ ] 冲突是递进而非并行的吗（解决一个直接触发下一个）？
[ ] 卷首 3-4 章内能否让读者感知到"问题比想象更大"？
[ ] 卷中是否避免了 4+ 章纯推进无新冲突的段落？
[ ] 每卷的冲突都从不同维度压迫主角吗（外部→内部→理念）？
[ ] 钩子链是从卷级→章级→场景级递进设计的吗？
[ ] 对手是否设计了"近敌-中敌-远敌-自我敌对"四层？
[ ] 代价是否具体化与可验证（不是空洞的"失去"，而是"失去导致……"）？
[ ] 主角身边关键人物是否都有潜在冲突或利益冲突？
[ ] 主卖点是否等同于主角最深的冲突根源？

## 硬规则

- 禁止把总纲写成世界观说明书或主题散文。
- 禁止只有设定，没有事件链、证据后果或代价。
- 禁止跳过总纲模板选择步骤，直接重造与模板库高度重合的结构。
- 禁止同时并列多个主模板；只能有 `1` 个主模板和 `0–2` 个辅助模板。
- 禁止只在总纲顶部挂模板名，却没有把模板要求真正落入主体结构。
- 找不到更合适模板时，禁止跳过 `九线写作总纲.md` 这个默认基线。
- 禁止不经过总纲闭合就直接跳到分卷细拆。
- 禁止把 `故事面/` 写成 `小说总大纲.md` 的摘要副本，也禁止把 `小说总大纲.md` 写成故事面目录的机械汇总。
- 100 万字以上项目必须显式回答：卷群怎么分、每卷供血什么、每卷卷尾留下什么新缺口。
- 禁止卖点只写在一句话简介里，不进入卷级推进、单元案与终局闭环。
- 总纲包本体文件默认不带日期；日期命名仅用于对应审阅报告、书评等派生产物，不用于总纲主文件及其配套总表。
- 禁止因为主支线共享人物、线索、阶段节点就回避双轨设计；该类重合本来就应该为主支线联动服务。
- 禁止把“都市焦虑 / 冲突升级 / 多线交织 / 主题深刻”这类抽象话当成故事；关键节点必须尽量写成“谁做了什么 → 留下什么可回指物 → 引发什么可验证后果”。
