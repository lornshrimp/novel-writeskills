# 总纲设计骨架与对象定义

## 1. 对象先行

- 总大纲默认指总纲层文件组，而不是单一主文件。
- `小说总大纲.md` 默认承载主线故事，是主线推进、阶段里程碑与终局收束的基准文档。
- `故事面/` 默认承载以支线为主的完整故事文件组；它们不是主线总纲的注释页或拆分副本。
- 设计时必须先回答：主线与各条支线怎样相互交织、相互联系、相互推动；共享人物、事件、线索、阶段节点通常属于正常联动，不应直接判为重复维护。
- 设计前必须先判定：
  - 本次是改单文件；
  - 还是改总纲包联动；
  - 哪些文件需要同步更新。
- 总纲层默认至少包含：总纲主文件、故事面目录、伏笔与回收总纲、故事多面体以及相关总表级时间轴 / 矩阵 / 台账；若本次只改其中一处，也要先判断会不会造成总纲层口径漂移。

## 1.3 来源层速判（新增）

- `总纲包边界层`：总纲主文件、故事面、伏笔 / 时间轴 / 矩阵 / 台账之间的职责与联动是否清楚。
- `战略供血层`：一句话卖点、`M0 / X / C`、主支线交织、跨卷推进与终局闭环是否真的形成长期供血。
- `维护规划层`：模板组合声明、章节规划表、TODO、节奏地图、决策日志与后续拆卷接口是否已经显式留出。

默认顺序：先判 `总纲包边界层`，再判 `战略供血层`，最后补 `维护规划层`；不要先堆节奏表，再发现主支线边界还是糊的。

### 1.1 主线总纲 vs 故事面边界（强制）

- `小说总大纲.md` 要优先写清主线故事：主线谜面、主线推进、阶段里程碑、终局闭环、主线代价。
- `故事面/` 要分别写清各条支线故事：各自的冲突发动机、供血方式、阶段出场、与主线的交接接口。
- 若一条故事面不能独立讲成一个完整故事，只能算概念标签，不能进入 `故事面/` 主骨架。
- 若 `故事面/` 只是逐条复述主线节点而没有新增支线职责，也是不合格设计，必须补出它自己的胜负、代价与推进责任。

### 1.2 总纲模板选择与组合（强制）

- 设计总纲前，必须先读取 `通用skills/写作研究/起点模板/总纲模板/README.md`，为当前项目选择最贴近的总纲模板。
- 每次只允许确定 `1` 个主模板，并可再叠加 `0–2` 个辅助模板。
- 主模板优先解决题材承诺、主供血方式与基础结构；辅助模板优先解决明暗线、地图、势力、单元案、副本、系统任务等结构痛点。
- 若没有找到更合适的模板，主模板默认回退到 `通用skills/写作研究/起点模板/总纲模板/九线写作总纲.md`。
- 模板组合不是把多个模板硬拼成一个新模板，而是保留本 Skill 的总纲包主框架，再把选中模板的关键字段与裁判问题吸收到对应章节。
- 生成的 `小说总大纲.md` 顶部必须带 `## 模板组合声明`，说明主模板、辅助模板、是否触发默认回退、选择原因与各模板职责。

## 2. 战略层骨架

总纲至少要写清：

1. 模板组合声明
2. 一句话卖点 / 差异化点
3. 主线谜团 `M0`
4. 重要支线 `X01-XNN`
5. 单元案矩阵 `C01-CNN`
6. 主支线配比与跨卷推进
7. 终局闭环
8. 弹性节点与不可动摇硬约束

## 2.3 总纲最小工件（新增）

```markdown
【总纲包边界层】主文件 / 故事面 / 台账各自职责
【战略供血层】一句话卖点 / M0 / X / C / 终局闭环
【维护规划层】模板组合声明 / 章节规划表 / 节奏地图 / 决策日志
【当前判定】Green / Yellow / Red
```

### 2.1 LOCK 骨架补丁

总纲层不要只写“题材、主题、卷数”，还要先把主情节的 LOCK 骨架钉死：

- **L（主角）**：谁是读者必须一路跟住的人，他为什么值得被跟。
- **O（目标）**：他到底想拿到什么、守住什么、逃离什么。
- **C（冲突）**：谁 / 什么力量会持续阻止他。
- **K（冲击结尾）**：最终结局准备把主角打到什么位置，读者合书时该留下什么感受。

若总纲层说不清这四件事，后续分卷和章节通常只会越写越散。

### 2.2 扰乱事件与两扇门

总纲层至少要明确 3 个结构节点：

1. **扰乱事件**：什么先打乱主角原有生活，让读者知道故事要开始了。
2. **第一扇门**：什么不可逆事件把主角真正踢进主冲突，使他不能再回到原样生活。
3. **第二扇门**：什么重大危机 / 线索 / 失败 / 新解释把故事快速送往终局。

注意：

- 扰乱事件不等于第一扇门。
- 扰乱事件负责“让故事动起来”。
- 第一扇门负责“让主角再也不能装作没事”。
- 第二扇门负责“让终局对撞不可回避”。

### 2.3 附着元素

主线还应写清主角为何不能中途退出，否则读者会问“他为什么不直接走人”。

常见附着元素：

- 职责
- 道德义务
- 生死压力
- 地点限制
- 偏执 / 旧债 / 私人执念

若主线冲突缺少附着元素，总纲层即使题材概念很强，落到长篇执行也容易发软。

## 3. 战略故事面要求

每个战略故事面必须是一个完整故事，而不是主题概念。应包含：

- 故事梗概
- WHO / WHAT / WHEN / WHERE / WHY / HOW
- 跨阶段发展轨迹
- 与 `小说总大纲.md` 的交接接口：它怎样推动主线，又怎样被主线反推
- 体量匹配：每个战略故事面通常应能支撑约 10–15 万字、20–25 章的内容体量。
- 详写深度：每个战略故事面应尽量用 800–1200 字讲清，不得只留一句概念标签。

### 3.1 故事梗概与六要素的最低展开

- 故事梗概：必须能用一段通俗人话把“发生了什么、怎么发展、结局怎样”讲清楚。
- WHO：具体人物、身份、动机、关键对手与受影响群体。
- WHAT：核心事件链、冲突类型与结果影响。
- WHEN：起点、关键时间节点、持续跨度。
- WHERE：主要场景、环境变化、空间对情节的影响。
- WHY：人物动机、历史背景、利益驱动。
- HOW：达成手段、策略变化、程序 / 技术路径。

### 3.2 故事 vs 主题的强制澄清

- 错误写法：只写“都市焦虑、结构性暴力、主题深刻、多线交织”之类概念词，而没有具体人物与事件。
- 正确写法：先能讲出一个完整故事梗概，再拆出 5W1H 六要素。
- 任何战略故事面若无法独立讲成一个故事，就不能进入总纲主骨架。

## 4. 单元案矩阵要求

每个 `Cxx` 至少交代：

- 给 `M0` 供血什么拼图
- 提供什么证据 / 规则 / 关系变化
- 带来什么代价或余波
- 若某个 `Cxx` 无法明确回答“它给 `M0` 输送了哪块拼图 / 哪条规则边界 / 哪次代价升级”，则视为结构性冗余，需要重做。

## 5. 下游拆分接口

总纲必须告诉后续分部 / 分卷：

- 每个战略故事面会拆成哪些阶段
- 每个阶段预计落在哪些卷群
- 哪些信息允许后置，哪些必须提前埋设

## 6. 2026 补强硬约束

### 6.1 为什么这本书现在值得写

总纲必须额外写清：

- 现实锚点：它抓住了哪一种当代城市焦虑或社会摩擦。
- 类型抓手：它靠什么与同题材区分（职业 / 规则 / 物件 / 社会议题 / 民俗 / 科技伦理）。
- 读者承诺：读者持续追更是为了哪种快感（智性满足 / 惊悚日常化 / 情感撕裂 / 规则解密 / 反转代价）。

### 6.2 钩子链总设计

- 开篇钩子链：前 3 章如何做到“回答上一问 + 新增更危险一问”。
- 卷内钩子链：每卷前 5 章怎样形成连续追读。
- 长线钩子链：每卷结尾留下哪种更大的系统性缺口，让读者必须进入下一卷。

### 6.3 人物缺陷—秘密—执念三角

对主角、第一反派、关键同盟、关键背叛者，至少要写清：

- 缺陷会在什么场合反噬。
- 秘密暴露后会改写什么关系或案件判断。
- 执念如何逼人物不断做出高风险选择。

### 6.4 创新矩阵与反模板审计

- 建议为主线与前 6 个单元案建立创新矩阵：现实锚点、类型融合点、关键证据载体、反常机制、角色代价。
- 总纲中的关键节点要尽量写成“谁做了什么 → 留下什么可回指物 → 引发什么可验证后果 / 代价”。
- “氛围压抑”“人物成长”“冲突升级”“主题深刻”这类抽象空话若没有对应事件与证据载体，一律视为空转表达。

## 7. 最终验收口径

- 是否已明确主模板、辅助模板、默认九线回退状态与各模板职责
- 模板要求是否已吸收入总纲主体，而不是只在声明区挂名
- 能否直接指导后续分卷拆解
- `M0 / X / C` 是否互相支撑
- `小说总大纲.md` 与 `故事面/` 是否职责清楚、互相推动，而不是互相抢位或机械重复
- 跨卷推进是否成立
- 终局闭环是否提前布设
- 是否已经回答：一句话卖点是什么、`M0` 的谜面 / 真相 / 代价是什么、每个 `Xxx` 与 `Cxx` 具体供血什么、哪些伏笔何时回收以及余波是什么

## 7.5 Green / Yellow / Red 总纲判定（新增）

- `Green`：总纲包边界清楚、战略供血成立、维护规划接口齐备，可稳定下压到分卷层。
- `Yellow`：主干已有，但至少一层仍偏虚，例如模板组合留了口号、节奏地图未落地，或故事面职责仍有轻度重叠。
- `Red`：边界不清、卖点供血说不清、`M0 / X / C` 打架、终局闭环靠新设定救场，或根本没有维护接口。

## 8. 起点主发布渠道的总纲补丁（默认生效）

### 8.1 赛道适配声明（强制）

- 总纲必须声明本书所属“主赛道 + 子赛道 + 融合点”（如：都市悬疑 + 民俗职业 + 现实程序）。
- 若题材为热门但高度同质，必须再给 1 条不可替代差异点（职业切口 / 规则机制 / 证据载体）。

### 8.2 付费追读价值校验（强制）

- 在总纲层写清“读者为什么愿意连续追读”：至少包含智性满足、情绪兑现、关系代价中的两项。
- 每卷都要有卷级中回报；禁止只铺主线不给阶段兑现。

### 8.3 起点签约导向硬约束

- 黄金三章闭环在总纲中必须前置列出：冲突触发、抓手出现、首回报落地。
- 主角动机必须现实可感，避免“只想变强”这类空泛目标。
- 反派与对抗者必须具备现实动机，不得全员脸谱化。

### 8.4 更新可执行性约束

- 总纲结构需支持稳定日更，不得依赖“每逢大章才推进”的断续设计。
- 单元案 / 支线切分应便于 2000±200 字章节节拍拆分。

### 8.5 起点同质化规避

- 热门题材总纲必须给出“同题材差异化一句话”（不可用抽象词替代）。
- 默认避免“只换皮不换机制”的设定：每条主线至少有 1 个不可替代的规则抓手。
