# 总纲设计骨架与对象定义

## 一、对象先行

- 总大纲默认指总纲层文件组，而不是单一主文件。
- 设计前必须先判定：
  - 本次是改单文件；
  - 还是改总纲包联动；
  - 哪些文件需要同步更新。

## 二、战略层骨架

总纲至少要写清：

1. 一句话卖点 / 差异化点
2. 主线谜团 `M0`
3. 重要支线 `X01-XNN`
4. 单元案矩阵 `C01-CNN`
5. 主支线配比与跨卷推进
6. 终局闭环
7. 弹性节点与不可动摇硬约束

## 三、战略故事面要求

每个战略故事面必须是一个完整故事，而不是主题概念。应包含：

- 故事梗概
- WHO / WHAT / WHEN / WHERE / WHY / HOW
- 跨阶段发展轨迹

## 四、单元案矩阵要求

每个 `Cxx` 至少交代：

- 给 `M0` 供血什么拼图
- 提供什么证据 / 规则 / 关系变化
- 带来什么代价或余波

## 五、下游拆分接口

总纲必须告诉后续分部 / 分卷：

- 每个战略故事面会拆成哪些阶段
- 每个阶段预计落在哪些卷群
- 哪些信息允许后置，哪些必须提前埋设

## 六、最终验收口径

- 能否直接指导后续分卷拆解
- `M0 / X / C` 是否互相支撑
- 跨卷推进是否成立
- 终局闭环是否提前布设
