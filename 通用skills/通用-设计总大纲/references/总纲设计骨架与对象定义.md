# 总纲设计骨架与对象定义

## 一、对象先行

- 总大纲默认指总纲层文件组，而不是单一主文件。
- 设计前必须先判定：
  - 本次是改单文件；
  - 还是改总纲包联动；
  - 哪些文件需要同步更新。
- 总纲层默认至少包含：总纲主文件、故事面目录、伏笔与回收总纲、故事多面体以及相关总表级时间轴 / 矩阵 / 台账；若本次只改其中一处，也要先判断会不会造成总纲层口径漂移。

## 二、战略层骨架

总纲至少要写清：

1. 一句话卖点 / 差异化点
2. 主线谜团 `M0`
3. 重要支线 `X01-XNN`
4. 单元案矩阵 `C01-CNN`
5. 主支线配比与跨卷推进
6. 终局闭环
7. 弹性节点与不可动摇硬约束

### LOCK 骨架补丁

总纲层不要只写“题材、主题、卷数”，还要先把主情节的 LOCK 骨架钉死：

- **L（主角）**：谁是读者必须一路跟住的人，他为什么值得被跟。
- **O（目标）**：他到底想拿到什么、守住什么、逃离什么。
- **C（冲突）**：谁 / 什么力量会持续阻止他。
- **K（冲击结尾）**：最终结局准备把主角打到什么位置，读者合书时该留下什么感受。

若总纲层说不清这四件事，后续分卷和章节通常只会越写越散。

### 扰乱事件与两扇门

总纲层至少要明确 3 个结构节点：

1. **扰乱事件**：什么先打乱主角原有生活，让读者知道故事要开始了。
2. **第一扇门**：什么不可逆事件把主角真正踢进主冲突，使他不能再回到原样生活。
3. **第二扇门**：什么重大危机 / 线索 / 失败 / 新解释把故事快速送往终局。

注意：

- 扰乱事件不等于第一扇门。
- 扰乱事件负责“让故事动起来”。
- 第一扇门负责“让主角再也不能装作没事”。
- 第二扇门负责“让终局对撞不可回避”。

### 附着元素

主线还应写清主角为何不能中途退出，否则读者会问“他为什么不直接走人”。

常见附着元素：

- 职责
- 道德义务
- 生死压力
- 地点限制
- 偏执 / 旧债 / 私人执念

若主线冲突缺少附着元素，总纲层即使题材概念很强，落到长篇执行也容易发软。

## 三、战略故事面要求

每个战略故事面必须是一个完整故事，而不是主题概念。应包含：

- 故事梗概
- WHO / WHAT / WHEN / WHERE / WHY / HOW
- 跨阶段发展轨迹
- 体量匹配：每个战略故事面通常应能支撑约 10–15 万字、20–25 章的内容体量。
- 详写深度：每个战略故事面应尽量用 800–1200 字讲清，不得只留一句概念标签。

### 故事梗概与六要素的最低展开

- 故事梗概：必须能用一段通俗人话把“发生了什么、怎么发展、结局怎样”讲清楚。
- WHO：具体人物、身份、动机、关键对手与受影响群体。
- WHAT：核心事件链、冲突类型与结果影响。
- WHEN：起点、关键时间节点、持续跨度。
- WHERE：主要场景、环境变化、空间对情节的影响。
- WHY：人物动机、历史背景、利益驱动。
- HOW：达成手段、策略变化、程序 / 技术路径。

### 故事 vs 主题的强制澄清

- 错误写法：只写“都市焦虑、结构性暴力、主题深刻、多线交织”之类概念词，而没有具体人物与事件。
- 正确写法：先能讲出一个完整故事梗概，再拆出 5W1H 六要素。
- 任何战略故事面若无法独立讲成一个故事，就不能进入总纲主骨架。

## 四、单元案矩阵要求

每个 `Cxx` 至少交代：

- 给 `M0` 供血什么拼图
- 提供什么证据 / 规则 / 关系变化
- 带来什么代价或余波
- 若某个 `Cxx` 无法明确回答“它给 `M0` 输送了哪块拼图 / 哪条规则边界 / 哪次代价升级”，则视为结构性冗余，需要重做。

## 五、下游拆分接口

总纲必须告诉后续分部 / 分卷：

- 每个战略故事面会拆成哪些阶段
- 每个阶段预计落在哪些卷群
- 哪些信息允许后置，哪些必须提前埋设

## 六、2026 补强硬约束

### 为什么这本书现在值得写

总纲必须额外写清：

- 现实锚点：它抓住了哪一种当代城市焦虑或社会摩擦。
- 类型抓手：它靠什么与同题材区分（职业 / 规则 / 物件 / 社会议题 / 民俗 / 科技伦理）。
- 读者承诺：读者持续追更是为了哪种快感（智性满足 / 惊悚日常化 / 情感撕裂 / 规则解密 / 反转代价）。

### 钩子链总设计

- 开篇钩子链：前 3 章如何做到“回答上一问 + 新增更危险一问”。
- 卷内钩子链：每卷前 5 章怎样形成连续追读。
- 长线钩子链：每卷结尾留下哪种更大的系统性缺口，让读者必须进入下一卷。

### 人物缺陷—秘密—执念三角

对主角、第一反派、关键同盟、关键背叛者，至少要写清：

- 缺陷会在什么场合反噬。
- 秘密暴露后会改写什么关系或案件判断。
- 执念如何逼人物不断做出高风险选择。

### 创新矩阵与反模板审计

- 建议为主线与前 6 个单元案建立创新矩阵：现实锚点、类型融合点、关键证据载体、反常机制、角色代价。
- 总纲中的关键节点要尽量写成“谁做了什么 → 留下什么可回指物 → 引发什么可验证后果 / 代价”。
- “氛围压抑”“人物成长”“冲突升级”“主题深刻”这类抽象空话若没有对应事件与证据载体，一律视为空转表达。

## 七、最终验收口径

- 能否直接指导后续分卷拆解
- `M0 / X / C` 是否互相支撑
- 跨卷推进是否成立
- 终局闭环是否提前布设
- 是否已经回答：一句话卖点是什么、`M0` 的谜面 / 真相 / 代价是什么、每个 `Xxx` 与 `Cxx` 具体供血什么、哪些伏笔何时回收以及余波是什么
