## P0 规则：六步开书框架

### 核心原则

网文是有套路的——本质是满足读者的情感需求点。开书前应先理清步骤，而非写一步是一步。

### 六步骤

```text
步骤一：选择题材方向
步骤二：明确写法（研究同方向火书爽点）
步骤三：确定主角身份
步骤四：设计剧情（先写综述，再写细纲）
步骤五：寻求编辑意见（写一两万字发给编辑看）
步骤六：发布
```

### 步骤一：题材方向

- 主流方向（已被验证的题材）优先于自创背景
- 题材判断维度："你这个文的元素是什么"是开书第一问
- 热门题材方向参考（2020年）：职业向（文娱/医生/商人/体育）、同人向（日漫/历史背景/万界流/西游封神洪荒）、热点元素（签到打卡/幕后/脑补等）

### 步骤二：明确写法

- 找同方向几篇火书，对比罗列剧情方向和主要爽点
- 不要只拿一本参考，要多本对比找规律

### 步骤三：主角身份

**必须满足的核心原则**：

1. 主角要有具体的生活状态（主角现在的处境和生活细节）
2. 主角不能是"遗世独立"的位置——越孤立，越难展开剧情
3. 开局身份尽量与读者平等，甚至低于读者出发点（代入感来源）

**各品类开局身份参考**：

- 医生文：实习医生起点
- 演员文：龙套/不出名的演员起点
- 历史文（大明）：刚登基的正德皇帝（有特色且面临困局）
- 游戏文：替补选手/被贬低的高手

### 步骤四：剧情设计（推荐先写综述再写细纲）

**综述写法（前6万字综述）**：
先以段落形式梳理主要剧情走向、第一个高潮节奏点、金手指出现时机。

**后写细纲**（前几个剧情的章节级梗概）。

### 标准开头节奏：

```text
主角低姿态（读者期待建立）→ 金手指同时出现 → 读者期待拉满 → 主角出手 → 周围人震惊 → 主角收获掌声
```

## P0 规则：串糖葫芦节奏写法

### 核心原则

**一个剧情一个高潮节奏点**，像串糖葫芦一样连续推进。

这种写法：
- 节奏简单，不容易崩
- 适合大部分节奏文
- 维持读者的持续阅读驱动力

### 糖葫芦节奏模型

```text
第一个剧情高潮 → 新剧情展开 → 第二个高潮 → 新剧情展开 → 第三个高潮 → ...
```

每个高潮节奏点可以是：
- 主角装逼打脸
- 重要信息揭露
- 危机解除
- 关系升级
- 技能展示

---

## 总大纲设计时的增补检查项

- [ ] 开书前是否确认了具体题材方向和同方向的火书爽点？
- [ ] 主角是否有具体的生活细节和代入感（不孤立于世界）？
- [ ] 全书主要阶段是否按"串糖葫芦"方式规划了高潮节奏点？
- [ ] 每个阶段是否都有一个清晰可感知的高潮/爽点？
- [ ] 是否在写正文之前先完成了综述+细纲的两层规划？
