# 模板化大纲_节奏地图_决策日志补编

> 吸收外部 `outline-template`、`planner-skill` 与其节奏 / 决策体系，用于把总纲从“故事概述”补强为“带章节规划表、TODO、节奏地图、悬念线、决策日志入口的战略包”。

## 一、总纲最小字段

总纲至少应明确：

- 题材
- 预计章节数
- 目标字数范围
- 核心冲突
- 主题
- 风格调性

## 一点五、来源层速判（新增）

- `模板规划层`：题材、体量、核心冲突、主题、模板组合与章节规划表入口。
- `节奏地图层`：重 / 中 / 轻权重、疲劳风险、悬念线与阶段波形。
- `决策日志层`：结构改动、悬念重排、章节拆并、偏离初稿结构的原因与触发重审条件。

默认顺序：先补 `模板规划层`，再补 `节奏地图层`，最后补 `决策日志层`；不要先记决策，再发现连章节规划表都没开。

## 二、全书悬念线

建议单列：

- 主线悬念
- 支线悬念
- 终极揭秘

要求：

- 主线问题要能贯穿全书
- 支线问题要能阶段性供血
- 终极揭秘不是末章首次发明的新信息

## 三、结构划分

外部模板默认是三幕，但本仓库可转译到现有 `M0 / X / C / 故事面` 体系中：

- 第一幕：建置
- 第二幕：对抗
- 第三幕：解决

关键不是幕数，而是：

- 每一段完成什么
- 结束于什么重大转折
- 是否为后续卷群留出继续升级的空间

## 四、章节规划表

总纲层建议预先建立章节表，至少包括：

|章节|标题|核心事件|节奏权重|悬念钩子|状态|
|---|---|---|---|---|---|
|ch01|暂无|暂无|重 / 中 / 轻|暂无|待创作|

### 衍生规则

- 节奏权重至少区分 `重 / 中 / 轻`
- 禁止连续 `3` 章或以上为 `重`
- 状态字段应能支撑后续从 `待创作 → 进行中 → 已完成` 的推进

## 五、TODO 与章节摘要

### TODO 清单建议三栏

- 待创作
- 进行中
- 已完成

### 章节摘要

每章定稿后追加 `2–3` 句话：

- 核心事件
- 角色变化
- 悬念推进

不要只写“这一章发生了很多事”。

## 六、字数统计与节奏地图

建议总纲层维护：

- 已完成章节数
- 累计字数
- 完成进度
- 可视化节奏图
- 当前节奏模式识别
- 疲劳风险
- 建议调整区间

### 常见节奏模式

- 波浪式：重-轻-重-轻-重
- 三明治：轻-中-重-中-轻
- 爬坡式：轻-轻-中-中-重
- 冲刺与休息：重-轻-轻-重-轻

## 七、决策日志入口

总纲层应预留决策日志机制，用于记录：

- 章节拆分 / 合并原因
- 悬念线设计理由
- 节奏权重分配理由
- 主线 / 支线位置调整理由
- 偏离初稿结构的原因

## 七点五、模板化总纲工件（新增）

```markdown
【模板规划层】题材 / 体量 / 冲突 / 模板组合 / 章节规划表
【节奏地图层】重中轻分配 / 悬念线 / 疲劳风险
【决策日志层】最近关键决策 / 触发重审条件
【当前判定】Green / Yellow / Red
```

## 八、主题生成原则

主题应从人物故事自然长出，而不是先写一句主题再逼人物去演：

- 先有人物经历
- 再有性格底色
- 再由选择和后果长出主题

检验方法：

- 若删掉主题句，故事是否仍成立
- 重大选择是否有内在动机支撑
- 是否存在“为了表达主题而让人物做不符合自己的事”

## 九、总纲层红线

- 不得只有设定和概念，没有章节规划表
- 不得只有主线摘要，没有 TODO / 摘要 / 节奏地图入口
- 不得连着排三章重章却没有疲劳意识
- 不得把主题写成高压口号，再让人物做工具人

## 十、Green / Yellow / Red 模板化判定（新增）

- `Green`：三层都有，且能直接支撑后续总纲维护与拆卷。
- `Yellow`：有模板和节奏感，但决策日志或规划入口仍虚，后续维护风险较高。
- `Red`：只有故事概述，没有模板化规划、节奏地图或决策日志意识，后续很难稳定续写。
