# 通用-输出豆瓣版

这是平台共性本体，负责承接“豆瓣版输出”的跨题材共性规则。

## 常见触发词 / 用户说法速查

- 输出豆瓣版
- 改成豆瓣能发的版本
- 做豆瓣阅读平台改写
- 帮我把这章改得更有文学质感一点

## 何时使用

- 把任意题材章节改写成更适合豆瓣阅读语境的版本。
- 需要强化现实质感、文学性、细节耐读性与克制表达。
- 需要为题材包装层提供统一的豆瓣平台共性骨架。

## 不适用情形

- 只做题材边界补充，不做平台输出本体。
- 只做多平台编排、落盘和终检，而不处理豆瓣平台风格本身。

## 本层职责

- 维护豆瓣平台的跨题材共性规则。
- 统一承接豆瓣版输出的结构约束、标题倾向、审美表达与风格门禁。
- 为 `题材名-输出豆瓣版` 提供稳定的通用平台骨架。

## 继续读取的 references

- `references/跨题材共性与迁移来源.md`
- `references/平台共性执行细则.md`
- `references/分节级补救映射与详细规则回填.md`
- `references/平台研究补充_豆瓣都市悬疑.md`

## 默认执行顺序

1. 先读取对应题材的 `题材名-输出豆瓣版` Skill。
2. 先锁定零新增事实、体裁结构、工作目录、章节层级、字数门禁与标题同步规则。
3. 再执行豆瓣平台的文学质感、现实纹理、留白与克制钩子强化。
4. 最后才处理降相似度，不得为了降重破坏豆瓣气质。

## 默认输出口径

- 默认输出一版可直接继续落盘或进入平台终检的豆瓣派生正文。
- 默认保留原章核心信息、事件链、情绪余韵与克制钩子，不新增关键事实。

## 最低交付

- 有明确的文学质感、现实纹理与克制表达。
- 保住原章核心事件链、人物感受与读者可跟随性。
- 读起来不像泛文艺稿，也不像只换词不换平台感的轻改稿。

## 硬规则

- 命中本技能时，必须同时加载对应题材的 `题材名-输出豆瓣版` Skill。
- 题材特有规则不得回写到本文件中平行维护。
- 修改豆瓣平台共性规则时，应优先修改本 Skill，而不是多个题材入口。
- 平台派生正文默认落在本 Skill 的工作目录 `豆瓣/` 下。
- 章节级派生正文目录统一按小说结构决定：若作品有分部，则位于 `豆瓣/第X部/第Y卷/`；若作品无分部，则位于 `豆瓣/第X卷/`。
- 平台派生正文文件名默认不带日期，沿用既有章节号与平台标题同步规则；日期只用于配套审阅报告、书评等派生产物。
- 平台稿落盘后必须显式运行 `scripts/count-chapter.ps1 -MinCJK 3500 -MaxCJK 6500` 核正文，并运行 `scripts/count-afterword.ps1 -MinCJK 200 -MaxCJK 300` 核 `## 作者有话说`。
- 在无损迁移完成前，不得删除源 `输出豆瓣版.prompt.md` 中尚未完成映射的内容。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只负责豆瓣平台的跨题材共性骨架。
- 题材边界、题材禁行项与题材特有口径，继续由对应 `题材名-输出豆瓣版` 承接。
