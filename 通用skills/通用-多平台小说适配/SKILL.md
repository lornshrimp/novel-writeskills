# 通用-多平台小说适配

多平台适配不是改几个词，而是换一套“读者心智 + 节奏分配 + 承载方式”。

## 保真迁移要求（强制）

命中多平台适配任务时，除本文件外，**必须继续读取**：

- `references/平台差异矩阵.md`
- `references/多平台改写前推演清单.md`
- `references/平台风格总则与撞车修复.md`

## 何时使用

- 要把同一章 / 同一片段适配到多个平台。
- 需要先做改写前推演，而不是直接写一份中性底稿再轻改分发。
- 发现多个平台版本开始撞车，需要修复平台感。

## 与入口 Prompt 的分工

- 多平台输出类 prompt 继续负责编排、落盘、门禁与相似度质检。
- 各平台子 prompt 继续负责本平台硬约束。
- 本 Skill 只负责**跨平台适配方法库**：差异矩阵、推演清单、拉开策略、撞车修复顺序。

## 执行顺序

1. 先读取平台差异矩阵，确认目标平台的核心气质。
2. 再用改写前推演清单，明确本次最该拉开的差异点与压相似度位置。
3. 再读取平台风格总则与撞车修复，确认风格优先级与修复顺序。
4. 明确“本平台最该比其他平台更像什么”。
5. 再进入正文改写与终检。

## 硬规则

- 不允许先写一份中性底稿再轻改分发。
- 压相似度必须服从平台风格，而不是牺牲平台风格。
- 中文平台之间必须明显可区分。
- 英文平台之间也必须明显可区分。

## 最低交付

- 明确本平台最关键的 3 条风格约束。
- 明确本平台最该拉开的 2 个差异点。
- 明确本次在哪些表达位上压相似度。
- 能回答“遮住标题后，读者还能不能认出平台风格”。
