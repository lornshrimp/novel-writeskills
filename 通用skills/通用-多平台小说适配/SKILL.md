# 通用-多平台小说适配

多平台适配不是改几个词，而是换一套“读者心智 + 节奏分配 + 承载方式”。

## 保真迁移要求（强制）

命中多平台适配任务时，除本文件外，**必须继续读取**：

- `references/平台差异矩阵.md`
- `references/多平台改写前推演清单.md`
- `references/平台风格总则与撞车修复.md`
- `references/平台节奏与付费机制补充.md`
- `references/人称视角双向转换适配卡.md`（按需必读 — 支持第一/第三人称双向转换）

## 卖点保真要求（强制）

- 多平台适配允许调整卖点的呈现方式、优先级与镜头承载，但不允许把主卖点适配没了。
- 每个平台版本都必须回答：主卖点怎样被该平台读者更快看见，以及怎样在该平台语境里持续成立。
- 若适配后只剩平台腔调差异，却认不出同一本书的核心吸引力，视为适配失败。

## 母稿前置要求（新增）

- 若源稿仍带明显单平台口癖、事件链不稳、情绪主轴发散或关键钩子尚未钉牢，必须先调用 `通用-提纯多平台母稿`，再进入多平台适配。
- 多平台适配负责“怎么拉开版本”，不负责替松散底稿兜底；若底稿本身不稳，直接分流只会把问题复制到多个平台。
- `通用-提纯多平台母稿` 也属于 AI 生成链路；若 AI 在母稿提纯前就不可用，则本 Skill 与母稿提纯都不得启动，必须直接写入 `blocked_ai_unavailable` 并停止。

## 事故复盘回写（2026-05-11）

- 已验证：在 AI 能力受限时改用脚本批量改写正文，会直接导致平台风格崩坏与模板污染扩散。
- 因此本 Skill 明确要求：AI 不可用时只允许阻塞与日志回执，不允许“先脚本改写再门禁补救”的路径。

## 常见触发词 / 用户说法速查

- 先做多平台适配策略
- 这章上不同平台该怎么拉开
- 帮我做改写前推演
- 这些平台版本开始撞车了
- 别直接轻改，先告诉我平台差异

## 何时使用

- 要把同一章 / 同一片段适配到多个平台。
- 需要先做改写前推演，而不是直接写一份中性底稿再轻改分发。
- 发现多个平台版本开始撞车，需要修复平台感。

## 与其他 Skill / Prompt 的边界

- 多平台输出类 prompt 继续负责编排、落盘、门禁与相似度质检。
- 各平台子 prompt 继续负责本平台硬约束。
- 本 Skill 只负责**跨平台适配方法库**：差异矩阵、推演清单、拉开策略、撞车修复顺序。

## 执行顺序

1. 先读取平台差异矩阵，确认目标平台的核心气质。
2. 再用改写前推演清单，明确本次最该拉开的差异点与压相似度位置。
3. 再读取平台风格总则与撞车修复，确认风格优先级与修复顺序。
4. 若平台策略要求切换视角人称（第一/第三人称），先按“人称视角双向转换适配卡”确定转换方向与保真约束。
5. 明确“本平台在语气、结构或读者预期上，最该比其他平台更强化什么具体特征”。
6. 再进入正文改写与终检。

## 硬规则

- 风格顺序：不允许先写一份中性底稿再轻改分发；压相似度必须服从平台风格，而不是牺牲平台风格。
- 平台区分：中文平台之间必须明显可区分；英文平台之间也必须明显可区分；不允许为了压相似度或贴平台腔而模糊主卖点、削弱卖点落地。
- 人称与研究：人称转换必须可逆且保真，既可 `第一人称→第三人称`，也可 `第三人称→第一人称`；若最终选择偏离“平台-人称默认建议表”，必须在推演清单记录偏离原因；平台研究若明确指向单一平台，应只在矩阵与对应平台链路中使用，不得偷渡回默认未指平台资产。
- 污染禁行：不允许把“抽象自述 + 动词模板 + 口径名词”句群当作平台差异化策略；这类句群属于污染文本，命中即失败。适配前后都必须保证污染短语族清零：`对照项落在`、`先把同一句话拆碎`、`先把顺滑的解释拆开`、`只求能追溯`、`只求能复核`、`只求对得上` 等不得出现在平台正文。
- 脚本禁行：不允许使用 `scripts/append_cn_unique_monologue.ps1`、`scripts/append_cn_unique_narration.ps1`、`scripts/append_cn_unique_thirdperson.ps1`、`scripts/rephrase_cn_body.ps1`、`scripts/rephrase_en_body.ps1`、`scripts/cn_lexicon_profile_transform.ps1` 作为适配主路径。
- AI 阻塞：只要 AI 在母稿提纯或正文适配任一阶段不可用，都必须停止正文适配并写入 `blocked_ai_unavailable`；禁止任何脚本正文改写兜底，避免继续浪费 token 与返工成本。

## 最低交付

- 明确本平台最关键的 3 条风格约束。
- 明确本平台最该拉开的 2 个差异点。
- 明确本次在哪些表达位上压相似度。
- 能回答“遮住标题后，读者还能不能认出平台风格”。
