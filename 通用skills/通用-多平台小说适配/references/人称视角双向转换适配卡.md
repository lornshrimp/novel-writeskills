# 人称视角双向转换适配卡

> 适用：平台分流前的视角策略决策，支持 `第一人称 ↔ 第三人称` 双向转换。

## 何时切换人称

- 当目标平台更偏“纪实贴脸、观点压强”时，可优先第一人称。
- 当目标平台更偏“信息组织、群像承载、叙事稳定”时，可优先第三人称。
- 若同一批次多平台分发，允许不同平台采用不同人称，但必须保持事件链与因果链一致。

## 双向转换总规则

- 必须可逆：`1→3` 与 `3→1` 都可执行。
- 必须保真：不丢信息、不改因果、不偷换立场。
- 必须控歧义：人物首次出现补关系锚，代词链可回指。

## 路径 A：第一人称 → 第三人称

1. 主语替换：`我` 改为角色名/他（按段位选择）。
2. 首现关系锚：首次出现“关系+姓名”同框。
3. 吐槽拆洗：括号吐槽改为动作/对话/体感反应。
4. 代词复检：多人同场时回补名字锚点，避免“他/她”混淆。

## 路径 B：第三人称 → 第一人称

1. 视角锚定：先确定唯一“我”是谁，本章中不漂移。
2. 信息裁剪：删除“我不可能知道”的全知信息。
3. 认知重写：将客观旁白改写为“我感到/我判断/我误判”。
4. 关系回补：把原三人称中的关系信息改成“我与他/她”的可感关系。

## 平台适配提示（简版）

- 偏观点讨论与经验叙述的平台，可优先第一人称强化代入。
- 偏连载工程与群像推进的平台，可优先第三人称保证信息稳定。
- 英文平台切第一人称时，优先短句与低歧义指代；切第三人称时，优先清晰因果与场景锚点。

## 完成判定

- 人称切换后，读者不因指代混乱出戏。
- 关键事件、证据、关系与章末钩子保持一致。
- 遮住平台名后，文本仍能体现目标平台气质，而非“仅改了代词”。
