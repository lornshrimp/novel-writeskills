# 平台差异矩阵

## 中文平台

### 起点中文网

- 核心：长线追读、章级回报工程、类型辨识度
- 更像：从前 300 / 500 / 1000 / 3000 字一路稳稳抓读者的长篇连载稿
- 偏好：主角抓手清、冲突快入场、3–5 章有小回报、10–15 章有阶段兑现
- 不要像：慢热议题论文、只靠氛围支撑的文学稿、首章炸后长期不还账

### 番茄

- 核心：快反馈、强钩子、可读性优先
- 更像：情绪与动作先行、手机端一路往下滑很顺的追读稿
- 偏好：短段落、强对话、即时情绪、马上见效的冲突
- 不要像：过冷、过散、过文学、解释长于推进

### 知乎

- 核心：高概念、逻辑锋利、现实映射
- 更像：一个危险问题正在被主角拆开回答
- 偏好：问题感、观点感、知识型反击、第一人称或强纪实感
- 不要像：纯爽文直给、纯煽情、只有情绪没有判断链

### 豆瓣

- 核心：现实质感、留白、人性复杂度
- 更像：现实流程、关系余味与人物处境一起推进的长篇稿
- 偏好：真实程序、双线成长、节奏克制但不断供
- 不要像：标题党、喊口号、廉价反转堆砌

### 微信订阅号

- 核心：碎片阅读、密钩子、可转发点
- 更像：手机端 3 秒能抓痛点、朋友圈环境里也愿意继续点开的连载稿
- 偏好：社交传播点、聊天式语感、强标题区、短节拍承接
- 不要像：长段冷叙、慢热说明、需要长时间预热才出戏

### 出版社

- 核心：职业真实感、稳节奏、审读友好
- 更像：结构完整、语言克制、专业细节可信的成熟稿件
- 偏好：逻辑自洽、现实纹理、适度留白、可持续阅读的章法
- 不要像：网文腔过重、爽点直白过量、情节只会靠硬拧

## 英文平台

### WebNovel

- 核心：跨文化易读、节奏清楚、因果链清晰
- 更像：短句、直白、全球年轻读者能迅速跟上的 serial draft
- 偏好：通用表达、低文化门槛、节拍清楚、信息不拐弯
- 不要像：大量中文语境内梗、复杂留白、因果跳跃

### GoodNovel

- 核心：情绪张力、关系推拉、cliffhanger
- 更像：章节结尾总能把读者往下一章推一把的情绪型稿件
- 偏好：情绪高压、关系失衡、电影化瞬间、结尾悬挂感
- 不要像：过冷、过远、只剩机制没有情绪代价

### My Fiction

- 核心：关系黏性、互动感、持续追更意愿
- 更像：人物关系先牵住读者，职业线或事件线负责催化关系升级
- 偏好：关系变化、误会 / 信任 / 背叛、持续互动感
- 不要像：只有动作没有情绪、只有设定没有人物站位

## 通用判定问题

- 遮住平台名后，还能看出它是哪一类稿子吗？
- 该平台版本最先被看见的，是它的“平台心智”，还是一份中性底稿？
- 如果平台名一遮就分不清，说明平台化不够。
