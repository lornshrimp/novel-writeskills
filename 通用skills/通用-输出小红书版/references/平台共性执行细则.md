# 通用-输出小红书版｜平台共性执行细则

## 适用目标

将章节改写成更适合小红书语境的版本，同时保持：主要信息不丢失、体感与画面感更强、输出仍为纯小说正文。

## 平台共性任务定义

- 强调即时情绪、体感、画面感与分享冲动。
- 保留关键事件链、人物情绪、风险推进与章末钩子。
- 不允许新增事件、证据、人物、地点、时间点或因果。

## 共同硬约束

- 正文必须是小说，不得写成笔记、攻略、心得或清单。
- 输出文件只允许：章标题、正文、`## 作者有话说`。
- 正文字数以 CJK 计，必须在 3500–6500 之间。

## 小红书平台节奏共性

- 开头尽量更快给出体感异常、情绪抓点或危险画面。
- 章内表达更利于移动端快速进入状态。
- 可强化可分享的感官瞬间，但不得牺牲情节逻辑。
- 短句口语化、有限视角与“执念 + 痛点 + 秘密”的人物共情结构，优先于大设定讲解。
- 优先用视觉、听觉、嗅觉、触觉等高密度日常细节制造“熟悉的恐惧”，少用抽象解释。
- 允许轻喜松弛或自我吐槽来缓冲压力，但必须服务角色关系与追读，不得写成碎片分享体。
- 如果章节本身异常点已经足够强，小红书化优先放大可记忆的感官瞬间与人物反应，不是单纯加重情绪形容词。
- 小红书读感更偏“被瞬间勾住”：一句话、一个动作、一个味道就要让读者想继续往下看。
- 分享感来自“我记住这个画面了”，不是来自文案化金句。
- 即使更强调体感，也不能把小说改成笔记口吻；人物动作和事件推进始终要在前。

## 小红书的表达原则

- 体感描写要具体，最好能落到皮肤、呼吸、味道、声音、温度。
- 口语化可以有，但要像人物在场，不像博主写笔记。
- 若一段只是在抒情，要立刻补一个动作或一个外部变化。
- 轻松弛只做调味，不做主菜；主菜仍是人物处境和局势变化。

## 执行顺序

1. 先加载对应题材 Skill。
2. 先锁定零新增事实、体裁结构、路径与字数门禁。
3. 再处理小红书平台的体感、画面感、即时情绪与分享抓力。
4. 最后才处理降相似度，不得为了降重写成种草笔记腔。

## 四、常见偏差修正

- 若原稿太抽象，优先补具体器物、气味、触感与动作。
- 若原稿太像笔记，删除“我真的”“太绝了”“必须看”等博主话术。
- 若原稿情绪很强但画面弱，先补镜头感，再补情绪词。
