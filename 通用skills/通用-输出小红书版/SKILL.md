# 通用-输出小红书版

这是平台共性本体，负责承接“小红书版输出”的跨题材共性规则。

## 常见触发词 / 用户说法速查

- 输出小红书版
- 改成小红书能发的版本
- 做小红书平台改写
- 帮我把这章改得更有体感和分享感

## 何时使用

- 把任意题材章节改写成更适合小红书传播语境的版本。
- 需要强化体感、画面感、即时情绪反馈与分享冲动。
- 需要为 `题材名-输出小红书版` 提供统一的平台骨架。

## 不适用情形

- 只做题材边界补充，不做平台输出本体。
- 只做多平台编排、落盘和终检，而不处理小红书平台风格本身。

## 本层职责

- 维护小红书平台的跨题材共性规则。
- 统一承接小红书版输出的结构约束、标题倾向、体感表达与风格门禁。
- 为后续新增题材保留直接复用入口。

## 继续读取的 references

- `references/跨题材共性与迁移来源.md`
- `references/平台共性执行细则.md`
- `references/分节级补救映射与详细规则回填.md`

## 默认执行顺序

1. 先读取对应题材的 `题材名-输出小红书版` Skill。
2. 先锁定零新增事实、体裁结构、工作目录、章节层级、字数门禁与标题同步规则。
3. 再执行小红书平台的体感、画面感、即时情绪与分享抓力强化。
4. 最后才处理降相似度，不得为了降重写成种草笔记腔。

## 默认输出口径

- 默认输出一版可直接继续落盘或进入平台终检的小红书派生正文。
- 默认保留原章核心信息、情绪主轴与关键动作，不新增关键事实。

## 最低交付

- 有明确的体感、画面感与即时情绪抓力。
- 保住原章核心事件链与分享触发点。
- 读起来不像种草笔记套壳，也不像泛平台轻改稿。

## 硬规则

- 命中本技能时，必须同时加载对应题材的 `题材名-输出小红书版` Skill。
- 题材特有规则不得回写到本文件中平行维护。
- 平台派生正文默认落在本 Skill 的工作目录 `小红书/` 下。
- 章节级派生正文目录统一按小说结构决定：若作品有分部，则位于 `小红书/第X部/第Y卷/`；若作品无分部，则位于 `小红书/第X卷/`。
- 平台派生正文文件名默认不带日期，沿用既有章节号与平台标题同步规则；日期只用于配套审阅报告、书评等派生产物。
- 平台稿落盘后必须显式运行 `scripts/count-chapter.ps1 -MinCJK 3500 -MaxCJK 6500` 核正文，并运行 `scripts/count-afterword.ps1 -MinCJK 200 -MaxCJK 300` 核 `## 作者有话说`。
- 在无损迁移完成前，不得删除源 `输出小红书版.prompt.md` 中尚未完成映射的内容。

## 与其他 Skill / Prompt 的边界

- 本 Skill 只负责小红书平台的跨题材共性骨架。
- 题材边界、题材禁行项与题材特有口径，继续由对应 `题材名-输出小红书版` 承接。
