# 通用-大纲审阅优化闭环

大纲闭环不是只审一遍，而是把“问题发现 → 结构修补 → 复审验证”真正跑到过线。

## 继续读取的 references

命中本 Skill 时，除本文件外，**必须继续读取**：

- `references/大纲审阅优化闭环总流程.md`
- `references/审阅报告复用与9点2门槛.md`

## 何时使用

- 用户要求把某个总纲 / 分卷 / 单元案审到过线。
- 需要统一处理审阅报告复用、回炉循环与评分门槛。
- 需要一个可落盘、可追轮次的提纲优化 SOP。

## 不适用情形

- 只做一次性审阅、不改稿：优先使用 `通用-审阅总大纲` 或 `通用-审阅分卷大纲`。
- 只新建大纲、不跑审阅闭环：优先使用 `通用-设计总大纲` 或 `通用-设计分卷大纲`。

## 常见触发词 / 用户说法速查

- 把这份大纲审到过线
- 一直回炉到 9.2+
- 用同一份报告循环复审
- 这份总纲 / 分卷反复改到过线
- 给我一个大纲优化闭环

## 默认组合调用顺序

0. 若按起点口径执行、且当前题材目录下缺少可用竞对主报告，先调用 `通用-分析竞对作品`，补齐 `Top10 候选池 → 3–5 本主样本 → 四层深拆` 的最小结论集。
1. 按对象类型进入对应审阅 Skill
2. 读取综合评分
3. 若未达标，进入对应设计 Skill 改稿
4. 覆盖更新同一份主报告
5. 循环，直到连续两轮独立审阅评分在保留两位小数后均严格大于 9.20（首次过线后须再追加一轮改稿 + 独立复审，第二轮也过线方可收口）

## 主报告命名规则（强制）

- 闭环绑定的主审阅报告文件名必须带日期，日期格式固定为 `YYYY-MM-DD`。
- 闭环内默认持续覆盖同一份已绑定的带日期主报告；轮次信息写入报告正文，不得改成无日期文件名，也不得平行分叉出另一份无日期主报告。
- 被优化的总纲 / 分部 / 分卷 / 单元案本体文件默认不带日期；日期只用于主审阅报告。

## 题材竞对分析接入要求（强制）

- 若当前任务服务某个题材，且该题材目录下的 `竞对分析/` 已存在与当前项目、当前平台或当前赛道相关的竞对分析报告，闭环开始前必须先读取并纳入本轮目标。
- 若目标平台为起点中文网，或未注明平台但默认按起点口径处理，且当前题材目录下尚无可用竞对主报告，闭环开始前应优先调用 `通用-分析竞对作品`，至少补齐 **Top10 候选池、`3–5` 本主样本与四层深拆速览**；若任务限制导致本轮无法补齐，必须显眼写出盲区，并下调本轮闭环上限。
- 不得把这些报告只当成背景资料；每一轮审阅、改稿与复审都必须显式判断当前提纲相对竞对的领先点、持平点、落后点与补丁方向。
- 大纲闭环的默认目标不是“分数大于 9.20 就收口”，而是推动当前作品在关键维度上**全方位超越竞争对手**；若看不出明确超越路径，不得把闭环误判为完成。
- 每一轮审阅、改稿与复审都必须先判断竞对维度，再进入综合评分：先写清领先点 / 持平点 / 落后点与补丁方向，再判定竞对维度是否达标。
- 竞对维度默认按 `25%` 计入综合评分；若竞对量化卡仍显示关键维度显著落后，则本轮竞对维度直接判为未达标，且本轮综合评分不得判为严格大于 `9.20`。

## 起点化竞对吸收口径（新增）

当闭环对象服务起点中文网，或未标平台而默认按起点基线处理时，每一轮审阅、改稿与复审都必须把竞对结论拆成以下四层，转译为大纲级补丁动作：

- **市场数据层**：看榜单位次、均订 / 首订 / 收订比代理信号、推荐资源位与长文商业厚度，反推当前提纲的题材归类效率、首卷交易效率与长线商业竞争位是否成立。
- **内容创作层**：重点吸收主样本在开篇承诺、人物驱动、设定边界、单元案供血、首卷承诺与长线升级上的做法，把它翻成总纲 / 分卷的结构补丁，而不是停留在“这本书设定强”。
- **运营策略层**：重点吸收主样本的追读工程、更新节奏匹配、卷中回报密度、卷尾续购缺口与推荐位适配逻辑，倒推当前提纲在卷级推进与章节功能位上的缺口。
- **受众反馈层**：重点吸收本章说 / 书友圈 / 书评区的高频夸点、吐槽点、老白读者敏感点与争议点，提前写进提纲的避坑和加码策略。

起点语境下，默认尽量使用以下术语，而不是把竞对结论写成空泛的“热度高 / 反馈好”：

- 数据层：`均订`、`高订`、`首订`、`追订`、`收订比`、`粉丝值`、`盟主数`、`荣誉徽章`；拿不到精确值时，必须写成代理信号，不得伪造数值。
- 榜单 / 资源层：`分类月票榜`、`畅销榜`、`新书榜`、`三江`、`强推`、`首页封推`、`智能推荐`。
- 读者层：`本章说 / 段评`、`书友圈`、`书评区`、`长评贴`、`老白读者`。

## 极其快节奏与极其强冲突闭环门禁（新增）

闭环不只要“分数达标”，还必须确保快节奏与强冲突能力真实达标，而非靠文案润色抬分。

### 闭环阶段强制检查

- **审阅轮**：必须单列“快节奏门禁结论 + 强冲突门禁结论 + 失败段位清单”。
- **改稿轮**：必须逐条对应门禁失败段位，写明已执行的补丁动作，不得泛写“整体优化节奏”。
- **复审轮**：必须复核“是否真实消除病灶”而不是“文本更顺了”。若只是语言顺滑但冲突结构未变，视为未修复。

### 快冲突闭环过线条件（强制）

要进入“连续两轮 > 9.20”判定，按以下顺序逐层判断；前一项未过线时，本轮不得进入收口判定：

1. 总纲层：启动速度、中段供血、冲突升级闭环、代价可验证全部过线。
2. 分卷层先检查前 5 章追读链是否过线。
3. 分卷层再检查卷中回报密度与升级链可追踪是否过线。
4. 分卷层最后检查卷末续购缺口是否过线。
5. 若闭环对象包含样章/正文，再单独检查正文层：首屏抓力、章中回报、代价显影、章末驱动必须全部过线。

任一层关键门禁未过，即使综合分 `> 9.20` 也不得收口。

### 反作弊规则（新增）

- 禁止通过“降低审阅口径”换取连续高分。
- 禁止通过“删掉冲突段落”让文本看起来更顺来冒充过线。
- 禁止通过“增加解释性句子”伪装节奏清晰；解释增多但冲突密度下降，一律降级。
- 禁止以“与上一轮相比有进步”替代“当前版本是否达标”的独立判断。

### 回炉优先级策略（新增）

每轮闭环默认按以下优先级回炉：

1. 先修 `P0/P1` 的快冲突病灶（启动慢、冲突空、代价虚、章末弱）。
2. 再修中段失温与关系位移不足。
3. 最后才修文风层与表达层问题。

若优先级顺序被颠倒（先润句子再修冲突），闭环效率通常会显著下降，需在报告里标注流程偏差。

## 强制要求

- 审阅阶段不得被裸审替代；改稿阶段不得被裸改替代。
- 审阅报告必须作为同一任务的主报告持续覆盖更新，不得平行分叉。
- `9.20` 不算通过；首次 `> 9.20` 也不允许直接收口，必须追加一轮改稿与独立复审，连续两轮均在保留两位小数时严格大于 `9.20` 才允许收口。
- 修改必须能回溯到上一轮报告中的明确问题与建议。
- 每一轮复审都必须调用对应审阅 Skill 对当前版本做一次独立、从严、全面的重新审阅；不得把“上一轮问题已修复”当成直接抬分或直接过线的依据。
- 复审时既要检查上一轮问题是否修复，也必须重新开放式查找本轮新增问题；一旦发现新问题，必须如实写入主报告并据此重新评分。
- 主报告中的最新评分只能根据当前版本在本轮完整复审下的实际质量给出，不得按修复比例、轮次推进或上一轮分数惯性自动上涨。

## 完成检查

- 是否已读取 2 份 references
- 是否已确认对象类型
- 是否已绑定一份主审阅报告并持续覆盖
- 是否已记录轮次、评分与核心问题变化
