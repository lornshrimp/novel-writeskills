# 大纲审阅优化闭环总流程

## 输入要求

开始前必须补齐：

1. 目标提纲文件路径
2. 对象类型：总大纲 / 分部 / 分卷 / 单元案
3. 本轮目标：综合评分严格大于 9.2

若用户未完整提供：

- 应优先根据文件名、目录结构与上下文推断对象类型
- 可同步推断提纲编号或名称，供主报告匹配使用
- 除非路径不可解析、文件缺失或存在真正阻断错误，否则不得因为命名细节停机追问

## 执行顺序

1. 识别任务对象
2. 定位或创建主审阅报告
3. 执行首轮审阅
4. 读取综合评分
5. 若未过线，则按报告修改提纲
6. 复审并覆盖更新报告
7. 循环直到评分 > 9.2

## 首轮审阅要求

- 首轮必须先产出一份完整版审阅报告并写入主报告路径
- 不得先在聊天里给口头建议、稍后再决定是否写报告
- 报告中必须包含一行可直接解析的评分字段：`- 综合评分：X.X/10`
- 报告中必须能明确读出阻断点 / 风险、具体修改建议，以及“冲击 9.2+ 综合评分的优化思路”

## 对象路由

- 总纲层：走总纲审阅 / 设计链
- 分部 / 分卷 / 单元案层：走分卷审阅 / 设计链

## 审阅阶段强制契约

- 必须产出完整审阅报告
- 报告中必须包含：阻断点 / 风险、具体修改建议、冲击 9.2+ 的优化思路、综合评分
- 必须能从报告里明确读出“为什么不过线 / 为什么已过线”

## 修改阶段强制契约

- 必须直接写回目标提纲文件
- 不得只在聊天里给建议版
- 修改必须覆盖上一轮核心扣分项
- 修改必须同时吸收最新审阅报告中的问题、修改建议与“冲击 9.2+ 综合评分的优化思路”
- 不得引入新的重大口径冲突

## 停止条件

- 只有最新综合评分 **严格大于 9.2** 才允许结束
- `9.2` 不算通过，仍必须继续修改并复审
