## 第一章 提纲设计：互联网裁员维权爽文的情节架构与人物关系

### 1.1 核心逻辑：爽点前置与情绪锚定

番茄读者的核心需求是 “即时爽感 + 情绪宣泄”，因此提纲需遵循 “绝境触发→智斗升级→反杀闭环” 的三幕结构，将传统维权的 “慢节奏取证” 转化为 “高密度爽点”。核心逻辑是：**先把主角按在裁员不公的谷底（3-5 章内），再让其凭借 “法律 + 技术” 的双重金手指，完成从个人维权到行业反杀的升级**。

### 1.2 三幕式长篇结构设计

#### 第一幕：绝境触发（第 1-10 章）

**核心任务**：在 10 章内完成 “主角人设确立 + 核心冲突爆发 + 金手指觉醒” 三大目标，让读者迅速代入主角的愤怒与绝境。



* **场景设计**：聚焦互联网大厂真实裁员场景 —— 如刚出哺乳期被 PIP、加班记录被系统自动删除、补偿金按最低工资计算、工位被强搬至茶水间。

* **爽点预埋**：主角需在绝境中触发金手指（如自学法律的过目不忘能力、程序员的日志恢复技术），完成首次小反击（如录下 HR 违规对话、恢复部分加班数据）。

* **目标**：让读者产生 “主角的委屈我懂，主角的反击我爽” 的代入感。

#### 第二幕：智斗升级（第 11-100 章）

**核心任务**：将维权流程拆解为 “证据收集→谈判博弈→仲裁交锋” 三个单元，每个单元设置 3-5 个爽点，实现 “小爽→中爽→大爽” 的梯度升级。



| 单元   | 核心情节                                    | 爽点设计                                                       |
| ---- | --------------------------------------- | ---------------------------------------------------------- |
| 证据收集 | 主角利用金手指获取核心证据（如恢复被删除的加班日志、获取公司未备案裁员的证据） | 技术取证的细节爽（如程序员用 Python 脚本恢复服务器日志）、规则拆解的智斗爽（如发现公司 PIP 无培训流程） |
| 谈判博弈 | 主角与 HR / 法务对线，用法律条文打脸对方                 | 话术碾压的爽（如当场指出 “经济性裁员需备案”）、反套路的爽（如用公司的考勤制度反证加班事实）            |
| 仲裁交锋 | 主角在仲裁庭提交铁证，反驳公司的虚假证据                    | 证据链闭环的爽（如提交录音 + 流水 + 日志的完整证据）、反派吃瘪的爽（如 HR 在庭上无言以对）         |

#### 第三幕：反杀闭环（第 101-200 章）

**核心任务**：完成从个人胜利到行业影响的升华，满足读者的终极情绪宣泄需求。



* **集体维权**：主角联合被裁同事，发起集体仲裁，让公司面临巨额赔偿。

* **行业曝光**：主角将公司的违规操作发布至行业论坛，引发监管介入。

* **反派下场**：主导裁员的 HR / 高管被公司追责、身败名裂。

* **主角成长**：主角从 “被动维权者” 转变为 “职场维权博主 / 律师助理”，实现自我价值。

### 1.3 人物关系网络：爽文逻辑下的盟友与反派

番茄爽文的人物关系需简单清晰，核心是 “主角→反派→盟友” 的三角对抗，避免复杂的群像戏。



| 角色类型 | 人设细节                                                                                         | 爽点作用                            |
| ---- | -------------------------------------------------------------------------------------------- | ------------------------------- |
| 主角   | 30 岁左右互联网从业者（如程序员 / 运营），冷静理性，有 “法律 + 技术” 双重金手指（如自学法律 3 年 + 全栈开发能力），有明确的软肋（如刚出哺乳期的孩子 / 重病的母亲） | 让读者代入，同时支撑智斗爽点                  |
| 反派   | HR（如 “微笑屠夫” 型，表面和善实则冷酷）、部门总监（如 “甩锅型”，把项目失败推给主角）、公司高管（如 “利益至上型”，主导违规裁员）                       | 制造冲突，让读者产生愤怒感，为后续打脸做铺垫          |
| 盟友   | 被裁同事（如刚毕业的应届生，提供公司内部八卦）、律师朋友（如公益律师，提供专业支持）、技术大牛（如前同事，提供技术取证帮助）                               | 为主角提供资源，推动情节发展，避免主角 “单打独斗” 的悬浮感 |

## 第二章 叙事风格：写实基础上的爽文重构

### 2.1 核心风格：写实 + 爽文滤镜 + 轻悬疑

番茄读者喜欢 “真实感 + 爽感” 的结合，需平衡职场写实与爽文逻辑，避免过于悬浮或过于枯燥。



* **写实基础**：植入互联网大厂真实元素 —— 如 OKR 考核、活水计划、PIP 流程、企业 VPN 考勤、免费加班餐的隐性压榨，让场景有代入感。

* **爽文滤镜**：强化主角的金手指，但金手指需服务于智斗逻辑 —— 如 “过目不忘” 的金手指只能记住法律条文，不能凭空变出证据；“技术取证” 的金手指需符合真实技术逻辑（如用 Git 日志恢复代码提交记录）。

* **轻悬疑感**：设置伏笔 —— 如 HR 电脑里的加密裁员名单、公司未备案的裁员文件，让读者有追更欲。

### 2.2 视角选择：第一人称限知视角

番茄 90% 以上的爆款爽文采用第一人称 “我” 叙事，原因是：



* **代入感强**：让读者直接体验主角的愤怒、冷静与爽感。

* **信息差优势**：通过主角的视角限制，隐藏部分信息（如反派的真实目的），制造悬念。

* **情绪传递直接**：第一人称的心理描写更易传递主角的情绪，让读者共情。

### 2.3 语言风格：浅白直白 + 画面感 + 情绪外露

番茄读者的阅读场景多为碎片化（地铁、睡前），语言需避免华丽辞藻，直接传递情绪。



* **浅白直白**：用口语化的语言叙述，避免专业术语的堆砌 —— 如不说 “经济性裁员需提前三十日通知工会”，而是说 “按法律，裁员得提前一个月跟工会打招呼，你们没做，这就是违法”。

* **画面感**：用具体场景替代抽象描述 —— 如不说 “HR 很冷酷”，而是说 “HR 把辞退通知推过来时，指尖还在刷着奢侈品代购群，连眼神都没抬一下”。

* **情绪外露**：直接描写主角的情绪 —— 如 “我攥着鼠标的指节发白，连呼吸都带着血腥味，原来我三年的加班，在他们眼里就是一张可以随意丢弃的废纸”。

## 第三章 章内写作：爽文节奏的精准把控

### 3.1 章内结构公式：钩子→冲突→爽点→悬念

番茄每章字数需控制在 1500-2000 字（适配碎片化阅读），结构需严格遵循 “钩子→冲突→爽点→悬念” 的公式，让读者每章都有收获。



| 结构节点 | 字数占比           | 内容要求                                              |
| ---- | -------------- | ------------------------------------------------- |
| 钩子   | 10%（150-200 字） | 承接上文，抛出新冲突 —— 如 “我刚把加班日志同步到加密云盘，HR 就带着保安站在了我工位前”。 |
| 冲突   | 40%（600-800 字） | 描写主角与反派的对抗 —— 如 HR 抢夺电脑、主角据理力争。                   |
| 爽点   | 30%（450-600 字） | 主角完成小反击 —— 如主角用法律条文打脸 HR、恢复被删除的证据。                |
| 悬念   | 20%（300-400 字） | 留下断章悬念 —— 如 “HR 冷笑一声：‘你以为删了聊天记录就没事了？’”。           |

### 3.2 冲突升级的技巧：层次感与具体性

冲突是爽文的核心，需避免 “为冲突而冲突”，要体现层次感与具体性。



* **层次感**：从 “个人冲突” 升级为 “规则冲突”—— 如从 “HR 克扣补偿金” 升级为 “公司未备案裁员”，让冲突更有深度。

* **具体性**：用互联网专属的细节强化冲突 —— 如 “HR 把我的绩效从 A 改成 C，理由是‘OKR 未达标’，但我明明完成了 95% 的目标，只是因为没给总监的小舅子背锅”。

### 3.3 爽点设计的核心：“预期违背” 与 “情绪释放”

爽文的核心是 “预期违背”—— 让读者以为主角会吃亏，结果主角反杀。爽点设计需遵循以下原则：



* **智斗爽而非武斗爽**：让主角用法律 / 技术取胜，而非武力 —— 如 “我没跟 HR 吵架，只是把《劳动合同法》第 87 条的截图发在了公司大群里，HR 的脸瞬间白了”。

* **细节爽而非宏观爽**：用具体的细节体现爽感 —— 如 “我把恢复的加班日志打印出来，每一页都标着时间戳和项目名称，HR 翻到第 12 页时，手开始发抖”。

* **情绪释放需铺垫**：先压抑主角的情绪，再释放 —— 如 “我忍了三天的委屈，在仲裁庭上把证据甩出来时，眼泪终于掉了下来，但这不是难过，是解气”。

* **承接上文**：用一句话总结上一章的结果 —— 如 “上一章：我把加班日志甩在 HR 脸上，HR 灰溜溜地走了”。
