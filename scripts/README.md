# scripts/（写作期维护脚本说明）

## 1.2.6 模板句清理与复测

当 `1.2.6` 多平台稿出现“台灯/那点灰/那束光/封条/文件袋”等高重复模板句时，可执行：

`pwsh -NoProfile -File ./scripts/repair_1.2.6_pad_cleanup.ps1`

该脚本会：

- 对中文平台稿（番茄/知乎/豆瓣/出版社/微信订阅号）执行 `reduce_pad_like_lines.ps1` 清理；
- 复跑平台门禁、质量扫描、相似度终检；
- 输出仅指标汇总，不输出正文。

> 定位：本目录脚本仅用于**写作期维护 / 一致性审计 / 抽检**。
>
> 原则：脚本命令与输出**不写入正文**；正文只写“动作/载体/回执/编号/见证”。

## 卷02：《空帧旧案》爽点表抽检

对应大纲文件：`小说大纲/第02卷《空帧旧案》- 分卷大纲.md`

### 1）快速抽检（人读输出）

运行：

- `python scripts/audit_vol2_outline.py "小说大纲/第02卷《空帧旧案》- 分卷大纲.md"`

### 2）结构化抽检（供落库/留痕）

目的：把抽检结果落入 `SOP执行日志/`，便于回溯“哪次修改导致什么指标变化”。

运行：

- `python scripts/audit_vol2_outline.py "小说大纲/第02卷《空帧旧案》- 分卷大纲.md" --json`

留痕建议：

- 每次修改章清单或任一张爽点表后，都跑一次抽检。
- 将输出粘贴到 `SOP执行日志/` 下新文件，建议文件名格式：

  - `SOP执行日志/卷02_爽点表抽检_YYYY-MM-DD_HHMM.md`

- 记录三件事即可：

  1) 修改点（改了哪几章/哪几列/哪一段护栏）
  2) 四项统计（tables / empty_keywords / keyword_repeat_windows / common_action_templates）
  3) 是否通过（通过/不通过 + 回写动作）
