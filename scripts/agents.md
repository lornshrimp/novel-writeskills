# scripts 目录使用说明

## 目录用途

本目录仅用于存放可复用的脚本，所有脚本应具备以下特性：

- **通用性**：脚本应能够在多个项目或场景中复用。
- **独立性**：脚本应尽量避免依赖特定的上下文或临时文件。
- **可维护性**：脚本应具备清晰的注释和文档，便于后续维护。

## 禁止事项

以下内容禁止存放于本目录：

- **过程性脚本**：仅用于单次运行或特定任务的脚本。
- **临时文件**：如调试过程中生成的中间文件或日志。
- **与脚本无关的文件**：如配置文件、数据文件等。

## 示例

符合要求的脚本示例：

- `count-chapter.ps1`：统计章节数量的通用脚本。
- `normalize_cn_body_paragraphs.ps1`：用于规范化中文段落的脚本。

## 维护约定

- 所有新增脚本需经过代码审查，确保符合上述要求。
- 若需存放过程性脚本，请移至其他目录（如 `temp/` 或 `process/`）。
