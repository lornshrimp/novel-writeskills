# Lorn.NovelWriteSkills

一个面向**长篇网文 / 小说创作工作流**的 AI 写作资产库。它不是“随手堆提示词”的仓库，而是一套围绕 **题材设计 → 大纲搭建 → 章节创作 → 审阅润色 → 多平台改写 → 质量门禁 → 分发落盘** 搭起来的可复用写作系统。

这个项目主要使用 Markdown 形式组织：

- **通用 Skill 能力本体**
- **题材包装层 Skill**
- **执行型 Prompt / SOP**
- **局部 Instructions**
- **写作期维护脚本**

如果你希望把 AI 从“陪聊工具”变成“真正能落盘、能复用、能审计的小说生产搭子”，这个仓库就是朝这个方向搭出来的。它不只是会写，还强调：**结构化、可迁移、可终检、可持续迭代**。

## 这个项目适合谁

- 想用 AI 协助写网文、长篇连载、悬疑、科幻、女频或都市题材的人
- 想把零散提示词整理成**长期可维护资产**的人
- 想把“生成正文”升级为“生成 + 审阅 + 门禁 + 分发”的完整流程的人
- 使用 VS Code / GitHub Copilot 自定义 Prompt、Skill、Instruction 工作流的人
- 希望为不同平台准备不同版本，而不是一稿硬拷 8 份的人

## 它和普通提示词仓库有什么不同

很多提示词仓库的终点，是“一段 Prompt 文本”。

这个项目更像一个**写作工作台**：

1. **分层明确**
   - `通用skills/`：沉淀跨题材复用的能力本体
   - `题材目录/.github/skills/`：题材包装层、默认入口、题材边界
   - `题材目录/.github/prompts/`：题材任务入口、SOP、兼容入口
   - `scripts/`：写作期质检、相似度、格式化、平台门禁脚本

2. **不只“会写”，还“会收口”**
   - 章节创作有闭环
   - 审阅与改写有闭环
   - 多平台输出有日志、门禁、相似度质检与摘要收口

3. **题材与平台双层建模**
   - 先保住题材底盘，再做平台适配
   - 平台版本之间强调风格差异，而不是只换标题或改几句话

4. **强调可复用资产，而不是一次性聊天结果**
   - 研究文件会继续沉淀进 Skill
   - Prompt 负责入口
   - Skill 负责能力本体与规则承载
   - 脚本负责执行期校验

一句话概括：

> 这是一个把“AI 写小说”从灵感型操作，推进到**工程化写作流程**的仓库。

这里的目标不是“同一稿复制分发”，而是：

- 再跑字数 / 文本质量 / 相似度 / 反注水等门禁

## 你能怎么使用它

### 用法一：把它当成个人 / 团队写作资产库

你可以把自己的：

- 题材研究
- 平台研究
- 常用 Prompt
- 题材 Skill
- 质量门禁脚本

都按这里的分层方式组织起来，让仓库越写越“会写”。

### 用法二：在 VS Code + GitHub Copilot 工作流里直接使用

这个仓库采用了 `.github/prompts/`、`.github/skills/`、`.github/instructions/` 这样的结构，非常适合在 VS Code 的 GitHub Copilot 自定义工作流中使用。

典型方式包括：

- 用题材 Prompt 作为任务入口
- 用题材 Skill 作为题材路由与边界补充
- 用通用 Skill 承接跨题材共性能力
- 用脚本做写后门禁与抽检

### 用法三：只把它当成“小说工程模板”来参考

即使你不用完整的 Copilot 自定义工作流，也仍然可以把这个仓库当作：

- 小说项目目录模板
- 写作 SOP 参考模板
- 多平台输出策略库
- 写作质检脚本集合

来借鉴和二次改造。

## 推荐上手顺序

如果你第一次接触这个仓库，推荐按这个顺序看：

1. 先看 `通用skills/README.md`
2. 再看一个题材目录（例如 `都市悬疑/` 或 `异能志怪/`）
3. 再看该题材下的 `.github/prompts/` 与 `.github/skills/`
4. 最后看 `scripts/` 里的质检与门禁脚本

如果你想从“生成正文”直接入手，优先关注：

- `通用-创建小说正文`
- `通用-章节创作闭环`
- `通用-生成章节控制卡`

如果你想从“多平台输出”入手，优先关注：

- `通用-提纯多平台母稿`
- `通用-多平台小说适配`
- `通用-多平台输出编排`
- 各题材目录下对应的 `输出*版` Skill / Prompt

## 项目设计原则

这个仓库的很多结构，都是围绕下面几条原则搭起来的：

- **先抽象共性，再补题材个性**
- **先沉淀 Skill，再复制使用**
- **先平台化，再降相似度**
- **先写成资产，再写成结果**
- **尽量让流程可落盘、可恢复、可检查**

换句话说，这个仓库追求的不是“写一次爽一下”，而是：

> 让 AI 写作能力像工程资产一样不断复利。
