# 上传指定章节到 GoodNovel 并保存

用于把你指定的 1 章或多章，上传到 GoodNovel 对应作品下，并按后台要求保存。

## 何时使用

- 你已经把待上传章节放入 `GoodNovel/` 目录。
- 你已经在 `GoodNovel/README.md` 写好账号、密码、GoodNovel 书籍Id、书名信息。
- 你希望按固定流程自动完成：进入后台 → 手动登录确认 → 判断新建/修改模式 → 填写章节标题 → 粘贴正文 → 保存 → 继续下一章。

## 不适用情形

- 你只想做文本润色，不需要打开网站上传。
- 你没有提供 GoodNovel 账号或书籍Id。
- 目标不是“保存章节”，而是只做内容校对。

## 输入前置要求（强制）

### 1) 章节文件目录

- 待上传文件必须放在 `GoodNovel/` 下。
- 你会明确指定要上传哪些章节（可单章，可多章）。
- 章节标题**不以文件名直接照抄为准**，必须从章节文件内容中提取并规范化后填写到页面标题框。

### 1.1) 章节标题提取与组装规则（强制）

- 必须从章节文件中提取：绝对章号、章节标题、正文。
- GoodNovel 页面只有**一个章节标题输入框**，最终填入值必须是完整字符串：`Chapter {绝对章号} {章节标题}`。
- `Chapter`、`绝对章号` 与 `标题` 之间必须保留 **1 个空格**，例如：`Chapter 1 天降奇缘`。
- 若章节文件标题行本身已经是完整格式（如 `Chapter 12 Night Visitor`），且章号一致，则可直接使用。
- 若提取到的标题行只有标题文本（如 `夜半来客`），则必须自动组装成 `Chapter {绝对章号} 夜半来客`。
- 不得把文件名中的平台后缀（如 `_GoodNovel版`）带入标题输入框。

### 1.2) 绝对章号规则（强制）

- 该平台**不分卷**，章节号必须使用“绝对章号”口径（参考 `分发-七猫小说`）。
- 绝对章号按上传顺序与既有记录综合判定：
  - 若仅有单卷结构，则绝对章号可直接使用该章在源稿中的顺序号；
  - 若源稿来自多卷/多部整合，则绝对章号必须是跨卷累加后的结果。
- 绝对章号必须是阿拉伯数字，不得使用中文数字。
- 若 `分发记录.md` 或本地卷章台账存在历史记录，则必须优先依据历史记录续接绝对章号，避免重复编号。

### 2) 元数据来源

- 必须读取 `GoodNovel/README.md`，至少提取以下信息：
  - GoodNovel 作者后台用户名
  - GoodNovel 作者后台密码
  - GoodNovel 书籍Id
  - 小说名称（小说书名，用于当前上传目标）

### 3) 分发记录来源（强制）

- 必须读取 `GoodNovel/分发记录.md`：
  - 若文件不存在，先创建后再执行分发。
  - 该平台无可稳定获取章节ID字段，改用“章节唯一键 + 后台列表标题/状态快照”判定该章是“新建模式”还是“修改模式”。

### 4) 书名判定规则（强制）

- 若小说无分部：小说书名使用该平台目录 `README.md` 中的“小说名称（小说书名）”。
- 若小说有分部（多部）：小说书名使用该章节所在分部的分部名，不得误用总书名。

### 4.1) 书籍Id判定规则（强制）

- 若小说无分部：书籍Id使用该平台目录 `README.md` 中与“小说名称（小说书名）”成对维护的书籍Id。
- 若小说有分部（多部）：书籍Id使用该章节所在分部名对应成对维护的书籍Id。
- 禁止把总书名对应书籍Id用于分部章节，或把分部书籍Id用于非对应分部章节。

### 5) `分发记录.md` 模板（首次创建直接使用）

当 `GoodNovel/分发记录.md` 不存在时，必须按以下模板创建：

```markdown
# GoodNovel 分发记录

> 用途：该平台无可稳定提取章节ID，记录“章节唯一键 -> 后台标题/状态快照”，用于判断新建与修改并做可追溯校验。

## 字段说明

- `章节唯一键`：固定格式 `GoodNovel书籍ID|绝对章号|章名`
- `GoodNovel书籍ID`：对应 `GoodNovel/README.md` 中书籍Id
- `后台章节标题`：保存后在左侧章节列表可见的标题文本
- `后台状态`：保存后在左侧章节列表可见状态（如 Draft/Published）
- `最近操作`：`新建` 或 `修改`
- `最近保存入口URL`：最近一次保存后所在章节页URL（用于审计，不作为章节ID）
- `更新时间`：`YYYY-MM-DD HH:mm:ss`
- `备注`：可选

## 记录表

|章节唯一键|GoodNovel书籍ID|后台章节标题|后台状态|最近操作|最近保存入口URL|更新时间|备注|
|:--|:--|:--|:--|:--|:--|:--|:--|
```

## 执行流程（强制按序）

### 步骤 1：读取本地输入

1. 读取并解析 `GoodNovel/README.md`。
2. 确认已获得：账号、密码、目标书籍Id（按“书籍Id判定规则”选取）、目标小说名。
3. 读取 `GoodNovel/分发记录.md`，建立“章节唯一键 -> 后台标题/状态快照”映射。
4. 读取你指定的章节文件，提取每章：绝对章号、章节标题、正文。
5. 对每章生成“页面标题输入框最终值”：`Chapter {绝对章号} {章节标题}`。

### 步骤 2：进入作者后台

1. **必须新开一个浏览器标签页**，并在该新标签页访问：
   - `https://www.goodnovel.com/uc/stories`
2. 该新标签页仅用于本次 GoodNovel 上传流程，不复用你当前正在进行其他任务的标签页。
3. 若出现登录态失效或未登录：
   - 先进入登录流程，但**必须等待用户手动完成登录操作**。
   - 在用户明确告诉你“登录完成”之前，禁止继续任何后续步骤。
   - 期间不要尝试代填密码、不要伪造登录成功状态。
4. 登录完成后，再继续后续编辑流程。
5. 若编辑页出现新手引导浮层、提示遮挡或确认弹窗：
   - 先关闭、跳过或收起，确保编辑器可正常操作后再继续。

### 步骤 3：进入目标书籍章节页（新建/修改二选一）

1. 在访问前，先查询 `GoodNovel/分发记录.md` 中该章节唯一键是否已有历史记录。
2. 若该章节 **没有** 历史记录（新建模式），访问：
   - `https://www.goodnovel.com/create_chapter/{书籍Id}`
3. 若该章节 **已有** 历史记录（修改模式），访问：
   - `https://www.goodnovel.com/create_chapter/{书籍ID}?chapterId=0`
4. 修改模式下进入后，必须在左侧章节列表中定位并点击目标章节：
   - 先按章节标题匹配；
   - 再用章节状态（列表中可见状态）做二次确认；
   - 若同名章节冲突，优先选择与记录中“后台状态”最接近的一条并人工确认。

### 步骤 4：校验当前作品信息

1. 检查编辑页当前作品是否匹配 `README.md` 中的目标书名。
2. 若页面存在书籍信息或左上角作品标识，则进一步校验目标书名。
3. 若书名不匹配：
   - 立即停止当前章上传。
   - 返回错误说明（书名不一致）。
   - 禁止继续盲传到错误作品。

### 步骤 5：填写标题输入框

1. 找到章节标题输入框。
2. 将其内容设置为完整字符串：`Chapter {绝对章号} {章节标题}`。
3. 输入前应先全选并清空旧值，避免残留“未命名章节”“草稿标题”或历史标题。
4. 再次核对：
   - 使用的是阿拉伯数字章号。
   - `Chapter`、章号与标题之间只有一个空格。
   - 未混入文件名后缀、书名、平台名等无关信息。

### 步骤 6：填写正文

1. 在正文编辑区域输入该章节正文。
2. 必须确保正文来自你指定的章节文件，不得错章串章。
3. 若编辑器为富文本或类富文本区域，输入后需确认可见正文确实已落入编辑区，而不是只进入了错误焦点元素。

### 步骤 7：作者有话说处理

- 该平台没有“作者有话说”输入位置，**本步骤固定跳过**。
- 不得为此虚构字段、不得在正文中拼接“作者有话说”替代。

### 步骤 8：保存并记录上传状态

1. 点击“SAVE”。
2. 等待页面出现保存成功的反馈或等价成功状态。
3. 保存成功后，返回/观察左侧章节列表，确认目标章节行可见且标题与状态可识别。
4. 将“章节唯一键 + 后台章节标题 + 后台状态 + 最近保存入口URL + 更新时间”写回 `GoodNovel/分发记录.md`：
   - 若记录不存在：新增一条；
   - 若记录已存在：更新后台标题/状态、最近操作、最近保存入口URL和更新时间。
5. 该平台无可稳定获取章节ID时，**不得伪造章节ID字段**，只记录可观察证据。

> 章节唯一键生成规则（强制）：
> - 使用字段顺序：`GoodNovel书籍ID|绝对章号|章名`
> - 其中：
>   - `GoodNovel书籍ID` 来自 `GoodNovel/README.md`
>   - `绝对章号` 必须是阿拉伯数字，且按不分卷口径持续累加
>   - `章名` 为章节标题
> - 该规则用于在无章节ID时仍维持稳定的一章一键映射。

6. 若保存成功但未能在左侧列表定位目标章节（或状态不可读）：
   - 本章状态必须标记为“部分成功（已保存，列表快照未确认）”；
   - 立即向用户明确报告阻塞点；
   - 禁止把该章静默算作完全成功。

### 步骤 9：多章循环

- 对每个指定章节，严格重复步骤 3～8，直到全部章节完成。
- 每章开始前都必须先查一次 `分发记录.md` 决定新建/修改模式，不得沿用上一章模式。

## 关键边界与硬规则

- 必须只上传用户明确指定的章节，不得擅自扩展范围。
- 步骤 2 必须先进入 `https://www.goodnovel.com/uc/stories`，并在需要登录时**等待用户手动完成登录**。
- 每章进入步骤 3 前，必须先查询 `GoodNovel/分发记录.md`。
- 新建模式必须使用 `https://www.goodnovel.com/create_chapter/{书籍Id}`。
- 修改模式必须先访问 `https://www.goodnovel.com/create_chapter/{书籍ID}?chapterId=0`，再在左侧章节列表点击目标章节。
- 左侧章节列表定位必须使用“标题 + 状态”双因子确认，避免误点同名章节。
- 章节标题输入框必须填写完整字符串 `Chapter {绝对章号} {章节标题}`，且中间保留一个空格。
- 章节号必须是阿拉伯数字，不得写中文数字。
- 该平台不分卷，章节编号必须使用绝对章号口径，不做卷内相对编号。
- 该平台没有“作者有话说”输入处，必须忽略，不得伪造替代位置。
- 必须点击“SAVE”完成保存。
- 全程没有可稳定提取章节ID的地方，记录方案必须采用“章节唯一键 + 列表标题/状态快照 + 最近保存入口URL”。
- 账号密码仅用于当次登录，不写入产物文件、不外泄。

## 失败与回退处理

- 登录未完成：必须停在登录等待状态，直到用户明确反馈“登录完成”。
- 页面元素缺失（如按钮改版、列表未出现）：记录当前失败步骤并停止本章，继续尝试下一章前需先确认页面可操作。
- 书名不匹配：直接停止该书上传流程，要求先修正 `GoodNovel/README.md` 或目标书籍Id。
- 保存失败：本章标记失败，重试后仍失败则保留失败记录并继续下一章（除非用户要求遇错即停）。
- 列表定位失败：本章标记“部分成功（已保存，无法确认列表快照）”，并要求人工复核，禁止静默成功。
- `GoodNovel/分发记录.md` 写入失败：本章虽然可能已保存，但流程状态必须标记为“部分失败（记录未落盘）”，并立即提示人工补录，禁止静默成功。

## 实操经验沉淀（基于已知页面信息与分发逻辑）

- `www.goodnovel.com` 登录步骤必须留给用户手动完成；这是硬边界。
- 修改模式不是直接用章节ID跳转，而是先用 `chapterId=0` 打开编辑入口，再在左侧章节列表点目标章节。
- 左侧列表会显示章节标题和状态，二者一起用来定位目标章，比只看标题更稳。
- 该平台没有“作者有话说”入口，别在正文里“强行拼贴”。
- 无章节ID并不等于无台账：通过“章节唯一键 + 标题/状态快照 + 最近保存入口URL”一样能实现可追溯增量更新。

## 最小可执行检查表（10秒版）

> 用法：每次分发前按顺序快速过一遍；有任意一项不满足，就先修正再上传。

1. **输入是否齐全**
   - 已明确本次只上传哪些章节（不扩范围）。
   - `GoodNovel/README.md` 可读到用户名、密码、书籍Id、书名。

2. **目标是否正确**
   - 当前编辑页书名与 `README.md` 一致。
   - 本章模式已判定：无记录走新建（`/create_chapter/{书籍Id}`），有记录走修改（`/create_chapter/{书籍ID}?chapterId=0` + 左侧列表点章）。

3. **关键输入是否到位**
   - 标题框是完整格式：`Chapter {绝对章号} {章节标题}`（Chapter + 阿拉伯数字 + 一个空格）。
   - 正文已确认写入编辑器可见区（不是误填到别处）。
   - 已确认本平台无作者有话说入口并按规则跳过。

4. **保存与回写是否闭环**
   - 已点击“SAVE”并确认成功。
   - 已在左侧列表确认目标章节的标题与状态。
   - 已回写 `GoodNovel/分发记录.md`（标题/状态快照 + 最近保存入口URL）。

5. **失败是否明示**
   - 若登录未完成、保存失败、列表未确认、记录未写入：必须标记失败/部分成功并说明阻塞点。

## 完成检查清单

- 已读取 `GoodNovel/README.md` 并提取必需字段。
- 已按指定章节列表逐章执行上传。
- 已在每章执行前查询 `GoodNovel/分发记录.md` 并正确选择新建/修改流程。
- 每章均完成：标题输入框、正文、点击 SAVE。
- 每章均完成绝对章号校验（不分卷口径）。
- 每章均已记录或更新到 `GoodNovel/分发记录.md`（含标题/状态快照）。
- 每章上传前均完成目标作品信息校验。
- 已输出每章结果（成功 / 失败 / 部分成功及原因）。
