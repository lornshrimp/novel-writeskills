# 上传指定章节小说到番茄小说网并保存为草稿

用于把你指定的 1 章或多章，上传到番茄小说网对应作品下，并全部保存为草稿。

## 何时使用

- 你已经把待上传章节放入 `番茄小说/` 目录。
- 你已经在 `番茄小说/README.md` 写好账号、密码、番茄书籍 ID、书名、卷名信息。
- 你希望按固定流程自动完成：登录 → 填章号与章名 → 粘贴正文 → 添加作者有话说 → 存草稿。

## 不适用情形

- 你只想做文本润色，不需要打开网站上传。
- 你没有提供番茄账号或书籍 ID。
- 目标不是“保存草稿”，而是“立即发布”。

## 输入前置要求（强制）

### 1) 章节文件目录

- 待上传文件必须放在 `番茄小说/` 下。
- 你会明确指定要上传哪些章节（可单章，可多章）。
- 章节标题**不以文件名为准**，必须从章节文件正文中提取。

### 1.1) 章节标题提取规则（强制）

- 章节标题来源于章节文件内容，通常是第 1 行的 Markdown 标题行（以 `# ` 开头）。
- 该标题行有时会带章节编号，有时只有标题文本；两种都合法。
- 填写到番茄“章节标题”输入框时，应使用提取后的标题文本，不得直接使用文件名。

### 2) 元数据来源

- 必须读取 `番茄小说/README.md`，至少提取以下信息：
  - 番茄小说网用户名
  - 番茄小说网密码
  - 小说番茄网 ID（纯数字，如 `7605776437705985086`）
  - 小说书名（用于当前上传目标）
  - 每一卷卷名

### 3) 分发记录来源（强制）

- 必须读取 `番茄小说/分发记录.md`：
   - 若文件不存在，先创建后再执行分发。
   - 用于查询“章节 -> 章节Id”映射，判定该章是“新建模式”还是“修改模式”。

### 4) 书名判定规则（强制）

- 若小说无分部：使用作品书名。
- 若小说有分部（多部）：必须使用 `README.md` 中当前这“一部”的名字作为番茄站内书名，不得误用总书名。

### 5) `分发记录.md` 模板（首次创建直接使用）

当 `番茄小说/分发记录.md` 不存在时，必须按以下模板创建：

```markdown
# 番茄小说分发记录

> 用途：记录“章节唯一键 -> 番茄章节Id”映射，供分发流程判断“新建模式/修改模式”。

## 字段说明

- `章节唯一键`：固定格式 `番茄书籍ID|分部名|卷号|章号|章名`（无分部时分部名填 `NA`）
- `番茄书籍ID`：对应 `番茄小说/README.md` 中书籍ID
- `章节Id`：番茄发布页 URL 中 `publish/` 后的数字
- `最近操作`：`新建` 或 `修改`
- `更新时间`：`YYYY-MM-DD HH:mm:ss`
- `备注`：可选

## 记录表

|章节唯一键|番茄书籍ID|章节Id|最近操作|更新时间|备注|
|:--|--:|--:|:--|:--|:--|
```

## 执行流程（强制按序）

### 步骤 1：读取本地输入

1. 读取并解析 `番茄小说/README.md`。
2. 确认已获得：账号、密码、番茄书籍 ID、目标书名、目标卷名。
3. 读取 `番茄小说/分发记录.md`，建立“章节唯一键 -> 章节Id”映射（建议唯一键：`番茄书籍ID|分部名|卷号|章号|章名`；无分部时分部名固定为`NA`）。
4. 读取你指定的章节文件，提取每章：章节号（阿拉伯数字）、章节标题、正文、作者有话说。
   - 其中章节标题必须按“章节标题提取规则（强制）”从文件内容提取，不得用文件名替代。

### 步骤 2：进入作者后台

1. **必须新开一个浏览器标签页**，并在该新标签页访问：
   - `https://fanqienovel.com/main/writer/book-manage`
2. 该新标签页仅用于本次番茄上传流程，不复用你当前正在进行其他任务的标签页。
3. 若出现登录态失效或未登录：
   - 必须切换到“密码登录”
   - 使用 `README.md` 中的用户名和密码登录
4. 若密码登录后页面提示“为保证账号安全，请使用手机验证码登录”或同类提示：
   - 立即进入“手机验证码登录”分支
   - 获取并输入用户实时验证码后再继续
   - 未取得验证码前不得伪造成功态、不得跳过登录校验

### 步骤 3：进入目标书籍章节页（新建/修改二选一）

1. 在访问前，先查询 `番茄小说/分发记录.md` 中该章节是否已有 `章节Id`。
2. 若该章节 **没有** 章节Id（新建模式），访问：
   - `https://fanqienovel.com/main/writer/{番茄网id号}/publish/?enter_from=newchapter`
3. 若该章节 **已有** 章节Id（修改模式），访问：
   - `https://fanqienovel.com/main/writer/{番茄网id号}/publish/{章节Id号}`
4. 页面加载后，记录当前 URL；从 `publish/` 后到 `?` 前（若无 `?` 则到路径结尾）的纯数字即该章节 `章节Id`。
5. `{番茄网id号}` 必须来自 `README.md`，且是数字串。

### 步骤 4：校验左上角书名与卷名

1. 检查页面左上角显示是否匹配：
   - `“{小说名} 第X卷：{卷名}”`
2. 示例：`归墟 第一卷：门禁不认我`
3. 若不匹配：
   - 立即停止当前章上传
   - 返回错误说明（书名/卷名不一致）
   - 禁止继续盲传到错误作品

### 步骤 5：填写章节标题区

目标区域格式为：`第_章 未命名草稿`

1. 将 `_` 替换为阿拉伯数字章节号（如 `5`）。
2. 将 `未命名草稿` 替换为实际章节标题（来源于文件内容，通常为第1行 `# ` 标题行），不得使用文件名。

### 步骤 6：填写正文

1. 在正文文本框输入该章节正文。
2. 必须确保正文来自你指定的章节文件，不得错章串章。

### 步骤 7：添加作者有话说

1. 点击“作者有话说”。
2. 点击“+添加”。
3. 选择“添加图文”。
4. 在出现的输入框中**先全选清空旧内容**，再输入该章“作者有话说”内容，避免与历史文本拼接。
5. 输入后必须检查计数器（常见为 `X/300`）：
   - 若超限，按“保钩子优先”压缩到平台限制内再保存。
   - 若未超限，保持原文语义并保存。
6. 点击有话说区域内“保存/完成”（若页面存在该按钮），确认有话说已落盘。

### 步骤 8：保存草稿

1. 点击右上角“存草稿”。
2. 等待页面反馈已保存成功（或等价成功提示）。
3. 保存成功后，将“章节唯一键 + 章节Id + 更新时间”写回 `番茄小说/分发记录.md`：
   - 若记录不存在：新增一条。
   - 若记录已存在：仅更新 `章节Id`（如变化）和更新时间。

> 章节唯一键生成规则（强制）：
> - 使用字段顺序：`番茄书籍ID|分部名|卷号|章号|章名`
> - 其中：
>   - `番茄书籍ID` 来自 `README.md`
>   - `分部名` 使用当前分部书名；无分部填 `NA`
>   - `卷号`、`章号` 必须是阿拉伯数字
> - 该规则用于彻底避免多部小说下“同卷同章号”冲突。

### 步骤 9：多章循环

- 对每个指定章节，严格重复步骤 3～8，直到全部章节完成。
- 每章开始前都必须先查一次 `分发记录.md` 决定新建/修改模式，不得沿用上一章模式。

## 关键边界与硬规则

- 必须只上传用户明确指定的章节，不得擅自扩展范围。
- 步骤 2 必须使用**新标签页**进入作者后台，禁止占用用户正在使用的原标签页。
- 每章进入步骤 3 前，必须先查询 `番茄小说/分发记录.md`。
- `publish/{章节Id}` 与 `publish/?enter_from=newchapter` 两种地址必须按记录自动分流，禁止混用。
- 每章“存草稿”成功后，必须把最新 `章节Id` 回写到 `番茄小说/分发记录.md`。
- 每章都必须经过“书名/卷名匹配检查”后再写入内容。
- 章节标题必须来自章节文件内容（通常第1行 `# ` 标题行），不得以文件名充当标题。
- 章节号必须是阿拉伯数字，不得写中文数字。
- “作者有话说”必须通过“作者有话说 → +添加 → 添加图文”路径录入。
- 作者有话说录入前必须先清空编辑框，再粘贴本章文本。
- 作者有话说必须满足平台字数限制（常见 300 字）后再保存。
- 最终动作是“存草稿”，不是发布。
- 账号密码仅用于当次登录，不写入产物文件、不外泄。

## 失败与回退处理

- 登录失败：提示账号/密码错误并停止，不继续后续章节。
- 密码登录被拦截为验证码登录：标记为“待验证码”，向用户索取验证码后继续；若用户未提供则暂停流程并回报阻塞点。
- 页面元素缺失（如按钮改版）：记录当前失败步骤并停止本章，继续尝试下一章前需先确认页面可操作。
- 书名/卷名不匹配：直接停止该书上传流程，要求先修正 `README.md` 或目标链接。
- 存草稿失败：本章标记失败，重试后仍失败则保留失败记录并继续下一章（除非用户要求遇错即停）。
- 作者有话说保存失败：本章先不发布，重试“编辑有话说 → 保存有话说 → 存草稿”链路；仍失败则记录原因并继续下一章（除非用户要求遇错即停）。
- `分发记录.md` 写入失败：本章虽然可能已存草稿，但流程状态必须标记为“部分失败（记录未落盘）”，并立即提示人工补录，禁止静默成功。

## 实操经验沉淀（基于真实上传）

- 先校验左上角书名+卷名，再填任何正文内容；顺序不能反。
- 新建页在首次进入后，URL 常会变成 `.../publish/{章节Id}?`，该 `章节Id` 必须提取并回写到分发记录。
- 正文粘贴后要观察顶部“已保存/保存中”与正文字数是否增长，避免“看起来贴上了实际未落盘”。
- 作者有话说编辑器常与正文编辑器并存，必须确认焦点在有话说编辑区再输入。
- 每章结束以“保存成功”提示为唯一完成信号；没有成功提示就不算完成。

## 完成检查清单

- 已读取 `番茄小说/README.md` 并提取必需字段。
- 已按指定章节列表逐章执行上传。
- 已在每章执行前查询 `番茄小说/分发记录.md` 并正确选择新建/修改 URL。
- 每章均完成：章号、章名、正文、作者有话说、存草稿。
- 每章均已记录或更新到 `番茄小说/分发记录.md`（含章节与章节Id映射）。
- 每章上传前均完成“左上角书名+卷名”匹配校验。
- 已输出每章结果（成功/失败及原因）。
